# Q037. Как увеличить читаемость больших чисел?

> **Источник:** [11. es.md](../11.%20es.md) · **Тема:** ECMAScript

---

## Короткий ответ

ES2021 ввёл **числовые разделители** (numeric separators) — знак подчёркивания `_` внутри числовых литералов для улучшения читаемости. `1_000_000` эквивалентно `1000000`. Это исключительно синтаксический сахар: `_` игнорируется движком, не влияет на значение.

---

## Развёрнутый ответ

### Суть и определение

Большие числа в коде трудночитаемы:
```javascript
const MILLION = 1000000;     // сколько нулей? надо считать
const BILLION = 1000000000;  // 9 нулей?
```

Числовые разделители решают эту проблему визуально, не меняя семантики:
```javascript
const MILLION = 1_000_000;      // очевидно
const BILLION = 1_000_000_000;  // считать не нужно
```

### Как это работает

`_` может быть вставлен между **любыми двумя цифрами** числового литерала. Ограничения:
- Нельзя в начале или конце числа: `_100`, `100_` → SyntaxError
- Нельзя два подряд: `1__000` → SyntaxError
- Нельзя сразу после `0x`, `0b`, `0o`: `0x_FF` → SyntaxError
- Нельзя рядом с `.`: `1_.0`, `1._0` → SyntaxError

### Практика и применение

```javascript
// Целые числа
const trillion = 1_000_000_000_000;

// Дробные числа
const pi = 3.141_592_653_589;

// Шестнадцатеричные
const rgb = 0xFF_FF_FF;         // #FFFFFF
const maxUint32 = 0xFF_FF_FF_FF; // 4294967295

// Двоичные
const flags = 0b1010_0001_1000;

// Восьмеричные
const perms = 0o755_644;

// BigInt
const safeBigInt = 9_007_199_254_740_991n;

// Временные интервалы (миллисекунды)
const ONE_MINUTE = 60_000;
const ONE_HOUR   = 3_600_000;
const ONE_DAY    = 86_400_000;

// Финансовые значения
const MAX_TRANSACTION = 10_000_000; // 10 млн
const TAX_RATE = 0.235_7;           // 23.57%

// Группировка по 4 для hex/binary
const color = 0xDEAD_BEEF;
const mask  = 0b1111_0000_1111_0000;
```

**Взаимодействие с функциями:**
```javascript
// Number() и parseInt() не понимают _
Number('1_000');     // NaN — строки с _ не парсятся!
parseInt('1_000');   // 1 — парсит до первого нечислового символа

// Правильно: хранить как число, форматировать через Intl
const amount = 1_000_000;
new Intl.NumberFormat('ru-RU').format(amount); // '1 000 000'
new Intl.NumberFormat('en-US').format(amount); // '1,000,000'
```

### Важные нюансы и краеугольные камни

- **Только в числовых литералах**: в строках `'1_000'` `_` — обычный символ.
- **`Number('1_000') === NaN`** — строку с разделителем нельзя парсить через `Number()` или `parseInt()` ожидаемым образом.
- **Нет ограничений на группировку**: `1_0_0_0` — валидно (хотя и странно).
- **Поддержка транспиляции**: TypeScript 2.7+ и Babel поддерживают numeric separators.
- **Для форматирования чисел** на экране — `Intl.NumberFormat`, не `_` в литерале.

### Примеры

```javascript
// Constants для читаемости
const CONSTANTS = {
  MAX_FILE_SIZE: 10_485_760,     // 10 MB в байтах
  REQUEST_TIMEOUT: 30_000,       // 30 секунд
  MAX_RETRIES: 3,
  RATE_LIMIT: 1_000,             // запросов в час
  DB_POOL_SIZE: 10,
};

// Байты памяти
const KB = 1_024;
const MB = 1_024 * KB;
const GB = 1_024 * MB;

// IPv4 в числовом виде
const localhost = 127_000_000_001; // не стандарт, но читаемо

// Криптографические константы
const RSA_KEY_BITS = 4_096;
const PRIME = 2_147_483_647; // 2^31 - 1 (простое число Мерсенна)

// Сравнение читаемости
// До ES2021:
const maxSafeInt = 9007199254740991;
// С ES2021:
const maxSafeIntReadable = 9_007_199_254_740_991; // Number.MAX_SAFE_INTEGER
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Влияет ли `_` на значение числа?** — Нет, чисто синтаксический сахар.
- **Что вернёт `Number('1_000')`?** — `NaN`: строки с `_` не парсятся стандартными методами.
- **Где нельзя ставить `_`?** — В начале/конце, два подряд, рядом с `.` или префиксом (`0x`).
- **Как форматировать числа для UI?** — `Intl.NumberFormat` с нужной locale, а не `_`.

### Красные флаги (чего не говорить)

- «`_` в числе — это JavaScript оператор» — нет, это часть синтаксиса числового литерала.
- «Строку `'1_000'` можно конвертировать в `1000` через `Number()`» — нет, результат `NaN`.
- «Числовые разделители появились в ES2015» — нет, в ES2021.

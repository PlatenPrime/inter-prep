# Q014. Что такое `BigInt` в ES2020 и для чего он используется?

> **Источник:** [11. es.md](../11.%20es.md) · **Тема:** ECMAScript

---

## Короткий ответ

`BigInt` — примитивный тип ES2020 для работы с целыми числами произвольной длины, выходящими за пределы `Number.MAX_SAFE_INTEGER` (2⁵³ − 1). Создаётся через суффикс `n` (`42n`, `9007199254740993n`) или `BigInt(value)`. Не взаимозаменяем с `Number` — смешивать нельзя без явного преобразования.

---

## Развёрнутый ответ

### Суть и определение

JavaScript использует 64-битный формат IEEE 754 для чисел. Максимальное целое, точно представляемое в `Number` — `2⁵³ − 1 = 9007199254740991` (`Number.MAX_SAFE_INTEGER`). Большие числа теряют точность:

```javascript
9007199254740992 === 9007199254740993; // true — потеря точности!
```

`BigInt` решает эту проблему для целых чисел (не дробных).

### Как это работает

`BigInt` реализован в движке как отдельный тип с собственной арифметикой произвольной точности (arbitrary precision). Хранится не в 64 битах, а динамически — сколько нужно.

**Ключевые характеристики:**
- `typeof 42n === 'bigint'`
- Не может хранить дробные части: `5n / 2n === 2n` (деление целочисленное)
- Операции со смешанными типами (`Number + BigInt`) → `TypeError`
- Нельзя использовать с `Math.*` методами
- Сравнение с Number через `==` работает, через `===` — нет

### Практика и применение

**Когда использовать BigInt:**
- Работа с большими ID (Twitter Snowflake IDs, БД с 64-битными PK)
- Криптография (большие простые числа, RSA)
- Финансовые расчёты с целыми числами (например, суммы в копейках/центах)
- Временные метки в микросекундах/наносекундах

```javascript
// Twitter Snowflake ID — не умещается в Number
const tweetId = 1234567890123456789n;
tweetId.toString(); // '1234567890123456789'

// Факториал больших чисел
function factorial(n) {
  return n <= 1n ? 1n : n * factorial(n - 1n);
}
factorial(50n); // 30414093201713378043612608166979581188299763898377856000000000000n

// Финансы: деньги в центах как BigInt
const price = 999999999999999n; // центы
const tax = price / 100n * 8n;  // 8% налог, без дробей
```

### Важные нюансы и краеугольные камни

- **Смешивать Number и BigInt нельзя**: `1n + 1` → `TypeError`. Нужно явное приведение: `BigInt(1) + 1n`.
- **Деление — целочисленное**: `7n / 2n === 3n`, остаток через `%`: `7n % 2n === 1n`.
- **`JSON.stringify(42n)`** → `TypeError`: BigInt не сериализуется в JSON по умолчанию. Нужен кастомный `replacer` или `.toString()`.
- **`Math.*` не работает с BigInt**: `Math.max(1n, 2n)` → `TypeError`.
- **Сравнение**: `1n == 1` → `true` (нестрогое), `1n === 1` → `false` (разные типы).
- **`Number(bigint)`** теряет точность для больших значений — та же проблема, что и раньше.
- **Performance**: BigInt медленнее Number из-за динамического размера.

### Примеры

```javascript
// Создание
const a = 42n;
const b = BigInt(42);
const c = BigInt('9007199254740993');

// Арифметика
console.log(10n + 3n);  // 13n
console.log(10n - 3n);  // 7n
console.log(10n * 3n);  // 30n
console.log(10n / 3n);  // 3n — целочисленное!
console.log(10n % 3n);  // 1n
console.log(2n ** 64n); // 18446744073709551616n

// Сравнение
console.log(2n < 3);      // true — нестрогое сравнение работает
console.log(2n === 2);    // false — разные типы
console.log(2n == 2);     // true

// Конвертация
console.log(Number(42n)); // 42
console.log(String(42n)); // '42'
console.log(+42n);        // TypeError — унарный плюс не работает

// JSON с кастомным replacer
const data = { id: 123456789012345678901234n, name: 'Alice' };
JSON.stringify(data, (key, value) =>
  typeof value === 'bigint' ? value.toString() : value
);
// '{"id":"123456789012345678901234","name":"Alice"}'

// Проверка потери точности в Number
const big = 9007199254740993n;
console.log(Number(big) === Number(big + 1n)); // true — потеря точности!
console.log(big === big + 1n);                 // false — BigInt точен
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему нельзя смешивать Number и BigInt?** — Разные внутренние представления; неявное преобразование могло бы вызывать трудноотловимые баги.
- **Как сериализовать BigInt в JSON?** — Кастомный `replacer` в `JSON.stringify` или переопределить `BigInt.prototype.toJSON`.
- **Когда BigInt НЕ стоит использовать?** — Дробная математика, взаимодействие с Web API (аргументы — Number), производительно-критичный код.
- **Что такое `Number.MAX_SAFE_INTEGER`?** — `2⁵³ − 1 = 9007199254740991`; числа больше этого теряют точность в Number.

### Красные флаги (чего не говорить)

- «BigInt работает с дробями» — нет, только целые; деление — целочисленное.
- «Можно складывать BigInt и Number без приведения» — TypeError.
- «BigInt — это просто большое число типа Number» — отдельный примитивный тип (`typeof 1n === 'bigint'`).

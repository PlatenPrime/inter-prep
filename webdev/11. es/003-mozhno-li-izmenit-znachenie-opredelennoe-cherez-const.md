# Q003. Можно ли изменить значение определённое через `const`?

> **Источник:** [11. es.md](../11.%20es.md) · **Тема:** ECMAScript

---

## Короткий ответ

`const` запрещает **переназначение** переменной (изменение самой ссылки), но не делает значение иммутабельным. Если `const` хранит объект или массив, их содержимое можно изменять: добавлять/удалять свойства, мутировать элементы. Для полной заморозки используется `Object.freeze()`, но даже она поверхностная — вложенные объекты остаются мутабельными.

---

## Развёрнутый ответ

### Суть и определение

`const` фиксирует **связь** между именем переменной и ячейкой памяти (ссылкой). Для примитивов (число, строка, булево) это означает неизменность значения. Для объектов и массивов — только то, что переменная не может указывать на другой объект/массив, но само содержимое объекта остаётся мутабельным.

### Как это работает

Когда JS-движок видит `const x = value`, он:
1. Создаёт привязку имени `x` в текущем лексическом окружении.
2. Помечает привязку как `[[Mutable]] = false`.
3. Любая попытка присвоить новое значение (`x = ...`) бросает `TypeError` в строгом режиме (и молча игнорируется в нестрогом — хотя `const` всегда строгий).

Операция **мутации** (`x.name = 'Bob'`) не трогает ссылку в `const`, она обращается к объекту по уже зафиксированному адресу — движок не запрещает это.

### Практика и применение

```javascript
const config = { timeout: 3000, retries: 3 };
config.timeout = 5000;      // ОК — мутируем объект
config.newProp = 'added';   // ОК — добавляем свойство
config = { timeout: 9999 }; // TypeError — переназначение запрещено
```

**Типичный паттерн в продакшене:** при работе с Redux/Zustand state объекты часто объявляются через `const`, и мутация всё равно недопустима — но это обеспечивается не `const`, а соглашением/иммутабельными обновлениями.

**Для реальной иммутабельности:**

```javascript
const frozen = Object.freeze({ name: 'Alice', address: { city: 'NY' } });
frozen.name = 'Bob';          // молча проигнорируется (в строгом режиме — TypeError)
frozen.address.city = 'LA';   // ОК — вложенный объект НЕ заморожен!
```

Для глубокой заморозки нужна рекурсия или `structuredClone` + `Object.freeze`.

### Важные нюансы и краеугольные камни

- **`const` с примитивом** — действительно константа: `const PI = 3.14; PI = 3` → `TypeError`.
- **`const` с объектом** — только ссылка константна. Это ловушка для новичков.
- **`Object.freeze()` поверхностная** — только первый уровень. Глубоко вложенные объекты мутируются.
- **В циклах `for...of`** `const` работает нормально, потому что каждая итерация создаёт новую привязку:
  ```javascript
  for (const item of [1, 2, 3]) {
    console.log(item); // 1, 2, 3 — каждый раз новая const-привязка
  }
  ```
- **В классическом `for`** `const` не подходит для счётчика: `for (const i = 0; i < 3; i++)` → `TypeError` при `i++`.

### Примеры

```javascript
// Примитив — истинная константа
const MAX = 100;
// MAX = 200; // TypeError

// Объект — мутабельное содержимое
const user = { name: 'Alice', roles: ['admin'] };
user.name = 'Bob';         // ОК
user.roles.push('editor'); // ОК
// user = { name: 'Charlie' }; // TypeError

// Массив — то же правило
const arr = [1, 2, 3];
arr.push(4);    // ОК: [1, 2, 3, 4]
arr[0] = 99;    // ОК: [99, 2, 3, 4]
// arr = [];    // TypeError

// Object.freeze — поверхностная заморозка
const settings = Object.freeze({ level: 1, nested: { debug: false } });
settings.level = 2;        // игнорируется (TypeError в strict mode)
settings.nested.debug = true; // ОК — вложенный объект не заморожен!
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Как сделать по-настоящему иммутабельный объект?** — Рекурсивный `Object.freeze` или библиотеки типа `Immer`, `immutable.js`.
- **Как `const` в `for...of` работает без ошибки?** — Каждая итерация создаёт новое лексическое окружение с новой `const`-привязкой.
- **Какая разница между `Object.freeze()` и `Object.seal()`?** — `freeze` запрещает изменение и добавление свойств; `seal` позволяет изменять существующие, но не добавлять новые.
- **Влияет ли `const` на производительность?** — Минимально; движок может применять некоторые оптимизации, зная, что ссылка не меняется.

### Красные флаги (чего не говорить)

- «`const`-объект нельзя изменить» — можно, его содержимое мутируется свободно.
- «Для константных объектов надо использовать `Object.freeze`» — это отдельная концепция; `freeze` ломает возможность добавления свойств на рантайме, что не всегда нужно.
- «`const` в `for...of` вызовет ошибку» — нет, каждая итерация — новая привязка.

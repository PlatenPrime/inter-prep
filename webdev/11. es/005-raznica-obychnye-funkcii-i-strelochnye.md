# Q005. Разница между обычными функциями и стрелочными?

> **Источник:** [11. es.md](../11.%20es.md) · **Тема:** ECMAScript

---

## Короткий ответ

Стрелочные функции — более компактный синтаксис с рядом семантических отличий от обычных. Главное: у стрелочных нет собственного `this` (берут из внешнего лексического окружения), нет `arguments`, нет `prototype`, их нельзя использовать как конструкторы (`new`). Обычные функции имеют собственный `this`, определяемый в момент вызова, и поддерживают полный набор возможностей.

---

## Развёрнутый ответ

### Суть и определение

**Обычная функция** (function declaration / function expression) создаёт собственный контекст выполнения с `this`, `arguments`, и может быть конструктором.

**Стрелочная функция** (arrow function, ES2015) — лексическая функция: наследует `this`, `arguments` и `super` из окружающего кода. Проектировалась для замены колбэков, где прежде приходилось использовать `var self = this` или `.bind(this)`.

### Как это работает

**`this` в обычной функции** определяется в момент **вызова** (dynamic `this`):
- `obj.method()` → `this === obj`
- `func()` → `this === undefined` (strict) или `globalThis` (non-strict)
- `new Func()` → `this === новый объект`

**`this` в стрелочной функции** определяется в момент **создания** (lexical `this`):
- Берётся из ближайшего окружающего не-стрелочного контекста
- `.call()`, `.apply()`, `.bind()` не меняют `this` стрелочной функции

### Практика и применение

**Правило применения:**
- Стрелочные — для колбэков, обработчиков событий внутри классов/объектов, функционального стиля (`map`, `filter`, `reduce`)
- Обычные — для методов объектов, конструкторов, функций, требующих динамического `this`, генераторов

**Частый паттерн до ES6 (и его решение):**
```javascript
// До ES6: this терялся в колбэке
function Timer() {
  this.seconds = 0;
  setInterval(function() {
    this.seconds++; // this !== Timer instance — баг
  }, 1000);
}

// ES6: стрелочная функция сохраняет this
function Timer() {
  this.seconds = 0;
  setInterval(() => {
    this.seconds++; // this === Timer instance — ок
  }, 1000);
}
```

### Важные нюансы и краеугольные камни

- **`arguments` недоступен** в стрелочных: используйте rest-параметры `(...args)`.
- **Нельзя вызвать через `new`**: `const f = () => {}; new f()` → `TypeError`.
- **Нет `prototype`**: `(() => {}).prototype === undefined`.
- **Нет `super`** внутри стрелочной функции как собственного: берётся из внешнего контекста.
- **Методы объекта со стрелочной функцией** теряют доступ к объекту через `this`:
  ```javascript
  const obj = {
    value: 42,
    getValue: () => this.value, // this = внешний (undefined/globalThis), не obj!
  };
  obj.getValue(); // undefined
  ```
- **Стрелочные функции как class fields** — удобный способ привязать метод:
  ```javascript
  class MyComponent extends React.Component {
    handleClick = () => { this.setState(...) }; // this всегда экземпляр
  }
  ```

### Примеры

```javascript
// Обычная vs стрелочная: this
const obj = {
  name: 'Alice',
  greetRegular: function() { return `Hello, ${this.name}`; },
  greetArrow: () => `Hello, ${this?.name}`, // this — внешний
};

obj.greetRegular(); // 'Hello, Alice'
obj.greetArrow();   // 'Hello, undefined' — this не равен obj

// arguments vs rest
function sum() {
  return [...arguments].reduce((a, b) => a + b, 0);
}
const sumArrow = (...args) => args.reduce((a, b) => a + b, 0);

sum(1, 2, 3);      // 6
sumArrow(1, 2, 3); // 6

// Стрелочная не-конструктор
const Foo = () => {};
// new Foo(); // TypeError: Foo is not a constructor

// Краткий синтаксис (implicit return)
const double = x => x * 2;
const getUser = id => ({ id, name: 'Alice' }); // объект обернуть в скобки!
```

## Сравнение

| Критерий | Обычная функция | Стрелочная функция |
|----------|-----------------|--------------------|
| `this` | Динамический (зависит от вызова) | Лексический (из внешнего контекста) |
| `arguments` | Есть | Нет (использовать rest `...args`) |
| `prototype` | Есть | Нет |
| `new` (конструктор) | Да | Нет (TypeError) |
| `super` | Собственный | Из внешнего контекста |
| Генераторы (`function*`) | Да | Нет |
| Синтаксис | Многословный | Компактный |
| Подходит для методов объекта | Да | Нет (если нужен `this`) |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему стрелочные нельзя использовать как методы объекта?** — `this` захватывается при создании (внешний контекст), а не при вызове, поэтому `this` не равен объекту.
- **Что произойдёт при вызове `.bind()` на стрелочной функции?** — Ничего: `bind` вернёт обёртку, но `this` не изменится — он лексически зафиксирован.
- **Можно ли создать стрелочную функцию-генератор?** — Нет, стрелочные функции не поддерживают `yield`.
- **Как передать `arguments` в стрелочную функцию?** — Использовать rest: `(...args) => args`, или захватить `arguments` из внешней обычной функции.

### Красные флаги (чего не говорить)

- «Стрелочные функции — это просто короткий синтаксис» — различия в `this`, `arguments`, `prototype` принципиальные.
- «`.bind(this)` изменит `this` стрелочной функции» — нет, `this` лексически зафиксирован.
- «Всегда использую стрелочные» — методы объектов и конструкторы должны быть обычными функциями.

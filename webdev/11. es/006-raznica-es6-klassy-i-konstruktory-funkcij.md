# Q006. Разница между ES6-классами и конструкторами функций?

> **Источник:** [11. es.md](../11.%20es.md) · **Тема:** ECMAScript

---

## Короткий ответ

ES6-классы — это синтаксический сахар над прототипным наследованием: под капотом используются те же прототипы и функции-конструкторы. Однако классы имеют ряд отличий: вызов без `new` запрещён, методы класса не перечисляемы, наследование через `extends`/`super` синтаксически понятнее, поддерживаются приватные поля (`#`), статические блоки, `class` попадает в TDZ.

---

## Развёрнутый ответ

### Суть и определение

**Функция-конструктор** — обычная функция, вызываемая через `new`. Методы добавляются вручную в `prototype`. Была единственным способом ООП до ES2015.

**Класс (ES2015+)** — декларативный синтаксис для того же прототипного наследования, с дополнительными возможностями: строгий режим по умолчанию, встроенный `extends`, статические методы/поля, приватные поля.

### Как это работает

Оба подхода создают функцию-конструктор и используют `[[Prototype]]` для наследования. Разница — в деталях:

| Аспект | Функция-конструктор | ES6 класс |
|--------|---------------------|-----------|
| Вызов без `new` | Разрешён (опасно!) | `TypeError` |
| Методы в prototype | Перечисляемые (`enumerable: true`) | Неперечисляемые |
| Хоистинг | Да (function declaration) | TDZ (как `let`) |
| `strict mode` | Опционально | Всегда |
| `extends` | Ручная настройка `__proto__` | Встроенный синтаксис |
| Приватные поля | Нет (соглашение `_`) | `#field` — настоящие приватные |
| Статические поля | Нет встроенного | `static` keyword |

### Практика и применение

В современном коде используют классы для:
- Компонентов фреймворков (React class components, Angular services)
- Кастомных ошибок (`class AppError extends Error`)
- DAO/Repository паттернов в Node.js
- Сложных структур данных

Конструкторы-функции встречаются в старом коде (до ES2015) и в некоторых functional-style паттернах.

### Важные нюансы и краеугольные камни

- **Классы не хоистятся** — нельзя использовать до объявления (TDZ).
- **`typeof MyClass === 'function'`** — классы это функции под капотом.
- **`extends` корректно настраивает цепочку прототипов**, в том числе для встроенных (`class MyArray extends Array`). С функциями-конструкторами унаследоваться от `Array`, `Map` и т.д. правильно очень сложно.
- **`super()` обязателен** в конструкторе дочернего класса до обращения к `this`.
- **Приватные поля `#`** — не просто соглашение, а жёсткое ограничение движка. Доступ снаружи класса → `SyntaxError`.

### Примеры

```javascript
// Функция-конструктор (ES5-style)
function Animal(name) {
  this.name = name;
}
Animal.prototype.speak = function() {
  return `${this.name} makes a noise.`;
};
Animal.prototype.speak.enumerable; // true — метод перечисляемый!

function Dog(name) {
  Animal.call(this, name);
}
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;
Dog.prototype.bark = function() {
  return `${this.name} barks.`;
};

// ES6 Class (то же самое, но чище)
class AnimalClass {
  constructor(name) {
    this.name = name;
  }

  speak() { // неперечисляемый метод
    return `${this.name} makes a noise.`;
  }
}

class DogClass extends AnimalClass {
  bark() {
    return `${this.name} barks.`;
  }
}

// Приватные поля
class BankAccount {
  #balance = 0; // настоящий приватный

  deposit(amount) { this.#balance += amount; }
  get balance()   { return this.#balance; }
}
const acc = new BankAccount();
acc.deposit(100);
console.log(acc.balance);  // 100
// acc.#balance;            // SyntaxError

// Вызов без new
function Foo() { this.x = 1; }
Foo(); // молча добавит x в globalThis — баг!

class Bar { constructor() { this.x = 1; } }
Bar(); // TypeError: Class constructor Bar cannot be invoked without 'new'
```

## Сравнение

| Критерий | Функция-конструктор | ES6 класс |
|----------|---------------------|-----------|
| Синтаксис | Многословный | Декларативный |
| Вызов без `new` | Работает (опасно) | `TypeError` |
| Перечисляемые методы | Да | Нет |
| Хоистинг | Да (function decl.) | TDZ |
| Наследование | Ручная настройка | `extends`/`super` |
| Приватные поля | Нет (`_` — соглашение) | `#field` |
| `strict mode` | Опционально | Всегда |
| Расширение встроенных | Проблематично | Поддерживается |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Является ли класс функцией?** — Да: `typeof class Foo {} === 'function'`.
- **Почему методы класса неперечисляемые?** — Это лучшее поведение по умолчанию: `for...in` не итерирует методы из прототипа.
- **Что делает `super()` в конструкторе?** — Вызывает конструктор родителя и создаёт объект для `this` дочернего класса.
- **Можно ли унаследоваться от встроенного `Error`?** — Да, через `class MyError extends Error`; с функцией-конструктором это корректно сделать трудно.

### Красные флаги (чего не говорить)

- «Классы — это новый механизм наследования» — нет, это синтаксический сахар над прототипами.
- «Функции-конструкторы лучше, потому что это настоящий JS» — оба подхода одинаково «настоящие».
- «Приватные поля — просто соглашение как `_name`» — нет, `#` даёт настоящую инкапсуляцию на уровне движка.

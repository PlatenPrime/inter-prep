# Q002. Разница между `let`, `const` и `var`?

> **Источник:** [11. es.md](../11.%20es.md) · **Тема:** ECMAScript

---

## Короткий ответ

`var` — устаревший способ объявления с функциональной областью видимости и хоистингом до `undefined`. `let` — блочная область видимости, нет хоистинга в рабочее состояние (есть TDZ), можно переназначать. `const` — то же, что `let`, но без возможности переназначить переменную. Современный код использует `const` по умолчанию, `let` — когда нужна мутация, `var` — только в легаси.

---

## Развёрнутый ответ

### Суть и определение

Три ключевых слова для объявления переменных отличаются по трём параметрам: **область видимости (scope)**, **хоистинг (hoisting)** и **возможность переназначения/переобъявления**.

### Как это работает

**`var`:**
- Scope: функция (или глобальный контекст, если вне функции)
- Хоистинг: поднимается на верх функции/скрипта, инициализируется `undefined`
- Можно переназначать и переобъявлять в той же области видимости

**`let`:**
- Scope: блок `{ }` (if, for, while, просто блок)
- Хоистинг: поднимается в блок, но **не инициализируется** → Temporal Dead Zone (TDZ)
- Можно переназначать, нельзя переобъявлять в той же области

**`const`:**
- Scope: блок `{ }`
- Хоистинг: аналогично `let`, TDZ
- Нельзя ни переназначать, ни переобъявлять

### Практика и применение

- **Правило команды:** `const` → `let` → `var` (в порядке убывания приоритета)
- `const` для примитивов и для ссылок на объекты/массивы, которые не меняются (содержимое объекта менять можно)
- `let` в циклах, счётчиках, флагах
- `var` только при поддержке очень старого кода или специфических кейсах (`var` виден до объявления — иногда используется намеренно в легаси)

### Важные нюансы и краеугольные камни

- **`var` в цикле** — классический баг: все итерации замыкаются на одну переменную. `let` создаёт новую переменную для каждой итерации.
- **`const` не означает иммутабельность**: ссылка неизменна, но объект/массив мутировать можно. Для глубокой заморозки — `Object.freeze`.
- **Переобъявление `var`** в одном скоупе молча перезапишет значение — источник трудноуловимых багов.
- **Глобальный `var`** добавляется в `window` (в браузере), `let` и `const` — нет.

### Примеры

```javascript
// var: функциональный скоуп, хоистинг
function testVar() {
  console.log(x); // undefined (hoisted)
  var x = 5;
  console.log(x); // 5
}

// var в цикле — баг
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i)); // 3, 3, 3 — все видят одну i
}

// let в цикле — правильно
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i)); // 0, 1, 2 — каждая итерация своя i
}

// const: нельзя переназначить, но содержимое объекта менять можно
const user = { name: 'Alice' };
user.name = 'Bob'; // ОК
user = {};         // TypeError: Assignment to constant variable

// let: TDZ
{
  // console.log(y); // ReferenceError: Cannot access 'y' before initialization
  let y = 10;
  console.log(y); // 10
}
```

## Сравнение

| Критерий | `var` | `let` | `const` |
|----------|-------|-------|---------|
| Область видимости | Функция / глобальная | Блок | Блок |
| Хоистинг | Да, инициализируется `undefined` | Да, но TDZ | Да, но TDZ |
| Переназначение | Да | Да | Нет |
| Переобъявление | Да | Нет | Нет |
| Попадает в `window` | Да (глобальный) | Нет | Нет |
| Введён в | ES1 | ES2015 | ES2015 |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Что такое TDZ?** — Temporal Dead Zone: зона между началом блока и строкой объявления `let`/`const`, где обращение к переменной бросает `ReferenceError`.
- **Почему `var` в цикле с `setTimeout` даёт неправильный результат?** — Замыкание захватывает ссылку, а не значение; `var` функционального скоупа — одна переменная для всех итераций.
- **Можно ли заморозить `const`-объект?** — Переменную заморозить нельзя (уже нельзя переназначить), но содержимое объекта — через `Object.freeze()`.
- **Есть ли разница между `let` в модуле и `var` на верхнем уровне скрипта?** — `var` станет свойством `globalThis`, `let` — нет.

### Красные флаги (чего не говорить)

- «`const` делает объект неизменяемым» — нет, только запрещает переназначение самой переменной.
- «`let` и `const` не хоистятся» — хоистятся, но попадают в TDZ, а не инициализируются.
- «Использую `var`, это то же самое» — в современном коде `var` — признак легаси или непонимания скоупинга.

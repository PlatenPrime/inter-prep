# Q001. Что такое TypeScript?

> **Источник:** [14. ts.md](../14.%20ts.md) · **Тема:** TypeScript

---

## Короткий ответ

**TypeScript** — это строго типизированный надмножество JavaScript, разработанное Microsoft, которое компилируется в чистый JavaScript. Он добавляет статическую типизацию, интерфейсы, generics и другие возможности, которые позволяют выявлять ошибки на этапе компиляции, а не в рантайме. Любой валидный JavaScript является валидным TypeScript — миграция инкрементальна.

---

## Развёрнутый ответ

### Суть и определение

TypeScript (TS) — это язык программирования с открытым исходным кодом, создан Microsoft в 2012 году (Anders Hejlsberg). Технически — **надмножество (superset) JavaScript**: любой JS-файл является валидным TS-файлом. TS-компилятор (`tsc`) транспилирует `.ts` → `.js`, удаляя все типовые аннотации.

Ключевая ценность: **статический анализ типов до запуска кода**. Компилятор знает форму данных и может поймать несоответствия типов, опечатки в именах свойств, неправильные вызовы функций — всё это до того, как код попадёт в браузер или на сервер.

### Как это работает

```
Исходный .ts файл
       ↓
  TypeScript Compiler (tsc)
  - парсинг + AST
  - проверка типов (type checking)
  - emit: удаление типов → .js
       ↓
  Валидный JavaScript
  (ES3 / ES5 / ES2015+ — настраивается через tsconfig.json)
```

Конфигурация проекта — `tsconfig.json`. Ключевые опции: `target` (версия выходного JS), `strict` (набор строгих проверок), `module` (формат модулей: CommonJS, ESM и др.).

TypeScript не меняет рантайм-поведение JS — он работает исключительно на этапе разработки/сборки. В рантайме выполняется обычный JS.

### Практика и применение

- **Крупные фронтенд-приложения** (React, Angular, Vue 3) — тип компонентов, пропсов, хуков
- **Node.js бэкенд** (Express, NestJS, Fastify) — типизация роутов, middleware, сервисов
- **Библиотеки и SDK** — публикация с `.d.ts` типами для потребителей
- **Монорепозитории** — общие типы в shared-пакете

Без TypeScript в больших командах рефакторинг опасен: переименование поля объекта вручную не отловишь — TS найдёт все места автоматически.

### Важные нюансы и краеугольные камни

- **TypeScript ≠ серебряная пуля**: ошибки типов не становятся рантайм-ошибками, если разработчик использует `any` или `as unknown as X`
- Компилятор **не замедляет рантайм** — всё происходит на этапе сборки
- Версии TS **не следуют SemVer** строго: минорные релизы могут сломать существующий код новыми проверками
- `strict: true` рекомендуется с нуля; включение позже в legacy-проекте требует больших усилий
- Типы стираются в рантайме — нельзя делать `instanceof MyInterface` (только классы и примитивы)

### Примеры

```typescript
// Без TypeScript — ошибка обнаружится только в рантайме
function greet(user) {
  return "Hello, " + user.nme; // опечатка: nme вместо name
}

// С TypeScript — ошибка на этапе компиляции
interface User {
  name: string;
  age: number;
}

function greet(user: User): string {
  return "Hello, " + user.nme; // TS Error: Property 'nme' does not exist on type 'User'
}
```

```typescript
// tsconfig.json — базовая строгая конфигурация
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "outDir": "./dist"
  }
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Чем TypeScript отличается от JavaScript?** — статическая типизация, компиляция, дополнительные возможности языка (generics, enums, decorators)
- **Что такое `strict` режим?** — набор флагов: `strictNullChecks`, `noImplicitAny`, `strictFunctionTypes` и др.; включается одним флагом
- **Что происходит с типами в рантайме?** — стираются при компиляции, в рантайме их нет (type erasure)
- **Можно ли использовать TS без компилятора?** — с 2025 года Node.js 22+ поддерживает запуск `.ts` через встроенный strip types, но без проверки типов
- **Что такое Declaration Files (`.d.ts`)?** — файлы только с типами, без рантайм-кода; используются для типизации JS-библиотек

### Красные флаги (чего не говорить)

- «TypeScript — это другой язык, несовместимый с JavaScript» — это надмножество, JS валиден в TS
- «TypeScript ловит все ошибки» — только ошибки типов; логические ошибки, сетевые сбои, рантайм-исключения — вне его зоны
- «TypeScript замедляет приложение» — компиляция происходит в dev/build, в рантайме выполняется чистый JS
- «Нужно переписать весь проект сразу» — миграция инкрементальна, файлы JS и TS могут сосуществовать

# Q007. Разница между элементом и компонентом?

> **Источник:** [15. react.md](../15.%20react.md) · **Тема:** React

---

## Короткий ответ

**React-элемент** — это простой иммутабельный объект (plain object), описывающий узел дерева UI: `{ type, props, key, ref }`. **Компонент** — это функция или класс, которая принимает props и **возвращает** элемент (или дерево элементов). Элемент — это *что отрендерить*, компонент — это *как это создать*.

---

## Развёрнутый ответ

### Суть и определение

**React-элемент** — минимальная единица описания UI в памяти. Создаётся через JSX или `React.createElement()`. Это обычный JavaScript-объект, не DOM-узел:

```js
{
  type: 'button',       // строка (DOM-тег) или функция/класс (компонент)
  props: { onClick: fn, children: 'Нажми' },
  key: null,
  ref: null
}
```

**React-компонент** — функция (или класс), принимающая объект props и возвращающая React-элемент(ы) или `null`. Компонент — это *шаблон* или *фабрика* элементов. Каждый раз при рендере React вызывает компонент и получает новый элемент.

### Как это работает

```
Компонент (функция) → вызывается React → возвращает элемент → React строит Virtual DOM
```

Когда вы пишете `<Button label="Нажми" />`, это JSX для:
```js
React.createElement(Button, { label: 'Нажми' })
```
React видит, что `Button` — функция, вызывает её с `{ label: 'Нажми' }` и получает элемент:
```js
React.createElement('button', { className: 'btn' }, 'Нажми')
```

Элементы **иммутабельны** — нельзя изменить props или children после создания. Для обновления создаётся новый элемент.

### Практика и применение

- **Элементы** появляются как возвращаемые значения компонентов, как children других компонентов, как ключи в списках
- **Компоненты** — строительные блоки приложения; каждый отвечает за свою зону UI
- `React.isValidElement(obj)` — проверяет, является ли объект React-элементом (полезно в HOC и утилитах)
- `React.cloneElement(element, newProps)` — создаёт копию элемента с изменёнными props (паттерн в библиотеках типа Radix UI)

### Важные нюансы и краеугольные камни

- Компонент с **заглавной буквы** — это компонент (`<MyButton />`); с **маленькой** — HTML-тег (`<button />`)
- Элемент не монтируется в DOM сам по себе — это только описание; монтирует ReactDOM/React Native
- Один компонент может возвращать несколько элементов через Fragment: `<><A /><B /></>`
- Компонент может возвращать `null` — ничего не рендерится, но компонент остаётся в дереве

### Примеры

```tsx
// ЭЛЕМЕНТ — простой объект
const element = <h1 className="title">Привет</h1>;
// Под капотом: { type: 'h1', props: { className: 'title', children: 'Привет' } }

// КОМПОНЕНТ — функция, возвращающая элементы
interface TitleProps {
  text: string;
}
const Title: React.FC<TitleProps> = ({ text }) => (
  <h1 className="title">{text}</h1>
);

// Компонент возвращает элемент при каждом вызове
const el1 = <Title text="Привет" />;
// React вызовет Title({ text: 'Привет' }) → получит <h1> элемент

// React.isValidElement
console.log(React.isValidElement(<div />));   // true
console.log(React.isValidElement('строка')); // false
console.log(React.isValidElement(Title));    // false — это компонент, не элемент
```

## Сравнение

| Критерий | Элемент (Element) | Компонент (Component) |
|----------|-------------------|-----------------------|
| Что это | Plain object | Функция или класс |
| Создание | JSX / `createElement` | Объявление функции/класса |
| Мутабельность | Иммутабельный | Может иметь state |
| Вызывает React | Нет | Да (вызывает функцию) |
| Возвращает | Описание узла | React-элементы / null |
| Переиспользование | Нет | Да |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Что возвращает `React.createElement`?** — объект-элемент, а не DOM-узел
- **Почему элементы иммутабельны?** — для простоты сравнения в diff-алгоритме; нельзя «частично» обновить элемент
- **Что такое React-инстанс компонента?** — внутреннее состояние, сохраняемое React между рендерами; функциональные компоненты не имеют публичного инстанса

### Красные флаги (чего не говорить)

- «Элемент — это DOM-узел» — элемент это JS-объект в памяти, DOM обновляется отдельно через reconciliation
- «Компонент и элемент — одно и то же» — компонент создаёт элементы, но это разные сущности

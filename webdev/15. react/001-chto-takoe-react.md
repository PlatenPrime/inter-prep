# Q001. Что такое React?

> **Источник:** [15. react.md](../15.%20react.md) · **Тема:** React

---

## Короткий ответ

**React** — это JavaScript-библиотека для построения пользовательских интерфейсов, разработанная Meta (Facebook). Она работает по принципу декларативного описания UI через компоненты и использует Virtual DOM для эффективного обновления реального DOM. React отвечает только за слой представления (View) в архитектуре MVC, поэтому его часто комбинируют с другими инструментами для управления состоянием и маршрутизацией.

## Развёрнутый ответ

### Суть и определение

React — это **UI-библиотека** (не фреймворк), впервые выпущенная Facebook в 2013 году. Ключевая идея: UI строится из независимых, переиспользуемых **компонентов**, каждый из которых описывает, как должна выглядеть его часть экрана. React сам решает, когда и как обновить DOM, основываясь на изменениях данных.

Основные концепции:
- **Компонентная архитектура** — UI = дерево компонентов
- **Декларативность** — описываем *что* отрендерить, а не *как* изменить DOM
- **Однонаправленный поток данных** — данные текут от родителя к потомку через props
- **Virtual DOM** — React поддерживает легковесную копию DOM и минимизирует реальные обновления

### Как это работает

1. Разработчик описывает UI через JSX (или `React.createElement`)
2. React строит Virtual DOM — дерево объектов в памяти
3. При изменении state/props React создаёт новый Virtual DOM
4. Алгоритм **reconciliation** (diffing) сравнивает старый и новый Virtual DOM
5. В реальный DOM вносятся только **минимально необходимые изменения** (patching)

### Практика и применение

- **SPA (Single Page Applications)** — основной сценарий: Gmail-подобные приложения с динамическим обновлением UI
- **SSR через Next.js** — серверный рендеринг для SEO и производительности
- **React Native** — мобильные приложения, переиспользующие тот же компонентный подход
- **Micro-frontends** — каждая зона страницы — независимый React-модуль

Без единой компонентной модели команды создавали бы несовместимые UI-части; React задаёт общий язык.

### Важные нюансы и краеугольные камни

- React — **только библиотека**, не full-stack решение: роутинг, стейт-менеджмент, HTTP-запросы — отдельные инструменты
- Версии 16+ используют **Fiber** — переработанный движок рендеринга с поддержкой прерываемых обновлений
- React 18 ввёл **Concurrent Mode**: рендеринг может быть прерван и возобновлён, что устраняет «заморозки» UI
- React **не реактивен** в классическом понимании (Signals/RxJS) — он инициирует ре-рендер через setState/хуки

### Примеры

```tsx
// Минимальный React-компонент
interface GreetingProps {
  name: string;
}

const Greeting: React.FC<GreetingProps> = ({ name }) => {
  return <h1>Привет, {name}!</h1>;
};

// Точка входа
import { createRoot } from 'react-dom/client';
const root = createRoot(document.getElementById('root')!);
root.render(<Greeting name="Мир" />);
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Чем React отличается от Angular и Vue?** — React — библиотека (только View), Angular — полноценный фреймворк; Vue — прогрессивный фреймворк с реактивной системой
- **Почему React использует Virtual DOM?** — прямые операции с DOM дорогие; VDOM позволяет батчить и минимизировать изменения
- **Что нового в React 18?** — Concurrent Mode, `useTransition`, `useDeferredValue`, стриминговый SSR, автоматическое батчинг
- **Является ли React реактивным фреймворком?** — нет в классическом смысле; он pull-based (опрашивает при setState), а не push-based
- **Что такое React Fiber?** — переработанный движок рендеринга, поддерживающий прерываемые/приоритизированные обновления

### Красные флаги (чего не говорить)

- «React — это фреймворк» — это библиотека; Angular и Next.js — фреймворки
- «React обновляет DOM напрямую» — React работает через Virtual DOM и патчит только изменившиеся узлы
- «React — только для веба» — React Native, React Three Fiber, Ink (CLI) расширяют сферу применения

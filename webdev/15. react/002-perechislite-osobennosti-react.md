# Q002. Перечислите особенности React?

> **Источник:** [15. react.md](../15.%20react.md) · **Тема:** React

---

## Короткий ответ

React выделяется несколькими ключевыми особенностями: компонентная архитектура, декларативное описание UI, однонаправленный поток данных и Virtual DOM. Дополнительно — JSX как синтаксический сахар поверх `createElement`, богатая экосистема и возможность рендеринга на сервере и в нативных приложениях.

---

## Развёрнутый ответ

### Суть и определение

Особенности React — это архитектурные и технические решения, которые отличают его от других подходов к построению UI и делают его масштабируемым для крупных приложений.

### Как это работает

**1. Компонентная архитектура**
UI разбивается на самодостаточные единицы — компоненты. Каждый компонент управляет собственным состоянием и рендерингом. Компоненты можно вкладывать, переиспользовать, тестировать независимо.

**2. Декларативность**
Описываете *что* должно быть на экране при данных данных, а не *как* изменить DOM шаг за шагом. React сам определяет минимальный набор DOM-операций.

**3. Virtual DOM**
React хранит легковесное дерево объектов в памяти. При изменении данных строится новый Virtual DOM, алгоритм diffing находит разницу, и в реальный DOM вносятся только нужные изменения — это дешевле, чем полный перерендер.

**4. Однонаправленный поток данных (One-way data binding)**
Данные передаются от родителя к потомку через props. Это упрощает отладку: понятно, откуда пришли данные и что могло измениться.

**5. JSX**
Синтаксическое расширение JavaScript, позволяющее писать разметку в стиле HTML прямо в JS/TS файлах. Компилируется в `React.createElement` вызовы.

**6. Хуки (Hooks)**
С React 16.8 функциональные компоненты получили доступ к состоянию, побочным эффектам, контексту и другим возможностям без классов.

**7. Экосистема и гибкость**
React — только слой View. Можно выбирать инструменты под задачу: Redux/Zustand/Jotai для стейта, React Router / TanStack Router для навигации, Next.js / Remix для SSR.

**8. React Native**
Тот же компонентный подход для мобильных приложений iOS/Android без WebView.

### Практика и применение

- **Компонентность** — ускоряет разработку: кнопки, формы, модалки переиспользуются в нескольких местах
- **Virtual DOM** — критично для частых обновлений (чаты, дашборды с real-time данными)
- **One-way data flow** — уменьшает число багов типа «откуда взялось это значение» в больших командах

### Важные нюансы и краеугольные камни

- Virtual DOM — не «бесплатный»: создание и diff объектов тоже стоит времени. Для очень простых UI прямой DOM может быть быстрее
- Однонаправленность данных иногда воспринимается как ограничение, но именно она делает state predictable
- JSX — не обязателен; можно использовать `React.createElement` напрямую, но это многословно
- React 18 Concurrent Mode меняет модель рендеринга: обновления можно прерывать и приоритизировать

### Примеры

```tsx
// Декларативность: описываем состояние, а не манипуляции
const [isOpen, setIsOpen] = useState(false);

return (
  <div>
    <button onClick={() => setIsOpen(prev => !prev)}>
      {isOpen ? 'Закрыть' : 'Открыть'}
    </button>
    {isOpen && <Modal />}
  </div>
);

// Компонентность: переиспользуемый Button
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary';
}
const Button: React.FC<ButtonProps> = ({ variant = 'primary', children, ...rest }) => (
  <button className={`btn btn-${variant}`} {...rest}>{children}</button>
);
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Чем Virtual DOM отличается от Shadow DOM?** — VDOM — оптимизация React в памяти; Shadow DOM — браузерная инкапсуляция стилей/разметки
- **Почему однонаправленный поток лучше двунаправленного?** — predictability, проще отлаживать, меньше неожиданных обновлений
- **В чём недостатки JSX?** — смешение логики и разметки, требует знания трансформаций, непривычно для новичков

### Красные флаги (чего не говорить)

- «Virtual DOM всегда быстрее реального» — не всегда; для статичного контента он только добавляет overhead
- «JSX — это HTML» — JSX компилируется в JS-вызовы; атрибуты camelCase, `className` вместо `class`

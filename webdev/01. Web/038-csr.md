# Q038. Что такое CSR?

> **Источник:** [01. Web.md](../01.%20Web.md) · **Тема:** Web Technologies

---

## Короткий ответ

**CSR (Client-Side Rendering)** — HTML и контент страницы **собираются в браузере**: сервер отдаёт минимальный shell (часто пустой `<div id="root">`) и JS-бандл, данные подгружаются через API, React/Vue/Angular рендерят UI на клиенте. Типично для SPA после загрузки. Плюсы — интерактивность и богатый UX после старта; минусы — медленный **первый экран**, SEO и слабые устройства без SSR/SSG.

---

## Развёрнутый ответ

### Суть и определение

CSR — противопоставление **SSR/SSG**, где HTML приходит готовым. Архитектура: API backend (REST/GraphQL) + frontend как отдельное приложение (Vite, CRA, SPA на CDN).

Рендеринг = выполнение JS + virtual DOM / реактивность + mount в DOM.

### Как это работает

1. Браузер запрашивает `index.html` — почти пустой body.
2. Загружает JS (часто большой main chunk).
3. Приложение `fetch` данные, строит дерево компонентов, `createRoot().render()`.
4. Роутинг через History API — без reload.

**Гидратация** не нужна в чистом CSR (нет server HTML); она нужна при гибриде CSR+SSR.

### Практика и применение

- **Админки, дашборды за логином** — SEO не важен, важна интерактивность.
- **Внутренние инструменты** — быстрая разработка, один деплой фронта на S3/CloudFront.
- **Публичный маркетинг** — чистый CSR ухудшает LCP и индексацию без prerender/SSR.

Без оптимизации пользователь видит белый экран до выполнения JS — плохой FCP/LCP.

### Важные нюансы и краеугольные камни

- **SEO:** боты выполняют JS, но с задержкой; критичный контент лучше в HTML.
- **Безопасность:** секреты только на сервере; в бандл попадает всё из `import`.
- Code splitting, lazy routes — обязательны для размера бандла.
- **Time to Interactive** vs **First Contentful Paint** — разные метрики.
- CSR + плохой API = waterfall: HTML → JS → API → render.

### Примеры

```html
<!-- Типичный CSR shell -->
<!DOCTYPE html>
<html>
  <body>
    <div id="root"></div>
    <script type="module" src="/assets/main.js"></script>
  </body>
</html>
```

```javascript
// React CSR entry
import { createRoot } from 'react-dom/client';
import { App } from './App';

createRoot(document.getElementById('root')).render(<App />);
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **CSR vs SSR для лендинга?** — SSR/SSG для SEO и LCP.
- **Как улучшить CSR без SSR?** — prerender, skeleton, code split, prefetch API.
- **Что такое гидратация?** — при SSR, не при чистом CSR.
- **Vite dev vs prod CSR?** — HMR vs optimized chunks.
- **Микрофронтенды на CSR?** — Module Federation, общий shell.

### Красные флаги (чего не говорить)

- «CSR всегда быстрее» — первый визит обычно медленнее SSR.
- «Google не индексирует CSR» — упрощение; важна скорость и качество.
- Один огромный bundle без lazy loading.

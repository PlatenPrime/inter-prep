# Q009. Что такое `HTTP/3`? Особенности `HTTP/3`?

> **Источник:** [01. Web.md](../01.%20Web.md) · **Тема:** Web Technologies

---

## Короткий ответ

**HTTP/3** — версия HTTP, работающая поверх **QUIC** (транспорт на **UDP**), а не TCP. Сохраняет семантику HTTP (методы, заголовки, статусы), но переносит мультиплексирование и шифрование в QUIC: **TLS 1.3 встроен**, отдельный handshake для TLS+TCP не нужен, **head-of-line blocking на уровне транспорта** для отдельных потоков устранён. Быстрее на нестабильных сетях (мобильный интернет), **0-RTT** resumption с осторожностью к replay.

---

## Развёрнутый ответ

### Суть и определение

RFC 9114. HTTP/3 = HTTP framing + QUIC. ALPN: `h3`. Развивается IETF QUIC (не путать только с Google QUIC прошлого).

### Как это работает

1. **UDP** datagrams между клиентом и сервером.
2. **QUIC connection** — connection id, миграция при смене IP (Wi‑Fi → LTE) без разрыва.
3. **Encryption by default** — TLS 1.3, нет cleartext HTTP/3 в браузерах.
4. **Streams в QUIC** — независимые byte flows; потеря пакета в одном stream не блокирует другие (в отличие от TCP в HTTP/2).
5. **QPACK** — сжатие заголовков (аналог HPACK, но с учётом out-of-order delivery — отдельные control streams).
6. **Connection establishment** — 1-RTT (иногда 0-RTT для повторных визитов).

### Практика и применение

- Cloudflare, Google, Facebook CDN — HTTP/3 для значимой доли трафика.
- Браузеры: Chrome, Firefox, Safari — fallback на h2/h1 если UDP заблокирован (корп. firewall).
- **Alt-Svc** заголовок или HTTPS DNS SVCB — объявление `h3=":443"`.
- Мониторинг UDP:443 на firewall — иначе silent fallback.

### Важные нюансы и краеугольные камни

- **UDP blocked** — нет h3, нужен graceful degradation.
- **0-RTT replay** — не идемпотентные запросы нельзя безопасно повторять на 0-RTT.
- **CPU** на edge — QUIC userspace vs kernel TCP; на высоких RPS настройка важна.
- **Load balancing** — connection id вместо только 4-tuple для sticky.
- Отладка сложнее классического TCP dump без QUIC-aware tools.

### Примеры

```http
# Сервер объявляет поддержку HTTP/3
HTTP/1.1 200 OK
Alt-Svc: h3=":443"; ma=2592000
```

```javascript
// Клиент — прозрачно; fetch не выбирает версию вручную
const res = await fetch('https://example.com/api/data');
// DevTools → Protocol: h3 (если UDP/QUIC доступен)
```

```bash
curl --http3-only https://cloudflare-quic.com -I
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **HTTP/2 vs HTTP/3 — где выигрыш?** — lossy networks, connection migration.
- **Почему UDP?** — userspace transport, гибкость, не наследует TCP HOL.
- **Что такое QPACK vs HPACK?** — динамическая таблица на отдельных streams.
- **Безопасен ли 0-RTT?** — только идемпотентные операции.
- **Как узнать h3 в проде?** — DevTools, `curl --http3`, метрики CDN.

### Красные флаги (чего не говорить)

- «HTTP/3 — это WebSocket» — разные протоколы.
- «UDP значит ненадёжно — HTTP/3 не для продакшена» — QUIC добавляет надёжность поверх UDP.
- Утверждать, что **везде** h3 быстрее h2 — на чистом fiber разница мала.

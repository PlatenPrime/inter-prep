# Q044. Что такое Progressive Web Application (PWA)?

> **Источник:** [01. Web.md](../01.%20Web.md) · **Тема:** Web Technologies

---

## Короткий ответ

**PWA (Progressive Web App)** — веб-приложение с **нативным UX**: работает по HTTPS, **устанавливается** на домашний экран (`manifest.json`), может работать **офлайн** и в фоне через **Service Worker**, поддерживает **push-уведомления** (где разрешено). «Прогрессивное» — базовый сайт доступен всем, расширенные возможности добавляются при поддержке браузера. Не отдельный магазин приложений, а улучшенный веб.

---

## Развёрнутый ответ

### Суть и определение

Критерии (Google / web.dev): reliable (offline), fast, engaging (install, push). Технологии: **Web App Manifest**, **Service Worker**, **HTTPS**, опционально Background Sync, Badging API.

Не framework — паттерн доставки и UX поверх React/Vue/любого стека.

### Как это работает

1. **manifest.json:** `name`, `icons`, `start_url`, `display: standalone`, `theme_color`.
2. **SW:** precache app shell, runtime caching API.
3. **Install prompt:** `beforeinstallprompt` (Chromium); iOS — «Add to Home Screen» вручную.
4. **Push:** подписка через Push API + сервер (VAPID).

Lighthouse аудит PWA — чеклист для собеседования.

### Практика и применение

- **Ритейл, новости, полевые приложения** — офлайн-каталог, повторные визиты без store friction.
- **B2B tools** — иконка на рабочем столе без публикации в App Store.
- **Без PWA:** нет офлайн, нет install, зависимость от вкладки браузера.

Ограничения iOS: лимиты storage, background, push (развивалось — проверять актуально).

### Важные нюансы и краеугольные камни

- PWA **не заменяет** нативные API (Bluetooth полный, некоторые sensors).
- Обновления через SW — риск stale cache (версионирование).
- Store policies: TWA (Trusted Web Activity) для Play Store.
- Запрос push при первом визите — отпугивает; timing UX.
- «Установлено» ≠ «офлайн готово» без продуманного caching.

### Примеры

```json
{
  "name": "Inter Prep",
  "short_name": "Prep",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#ffffff",
  "theme_color": "#0f172a",
  "icons": [{ "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }]
}
```

```html
<link rel="manifest" href="/manifest.json">
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Минимальные требования installability?** — manifest, SW, HTTPS, icons.
- **PWA vs native?** — reach, update, store cut, API gaps.
- **Workbox роль?** — стратегии кэша, precache manifest.
- **iOS PWA ограничения?** — storage, push, полноэкранный режим.
- **Как тестировать offline?** — DevTools, Lighthouse.

### Красные флаги (чего не говорить)

- «PWA = обязательно React».
- «Работает везде как native» без оговорок платформ.
- SW без HTTPS на production.

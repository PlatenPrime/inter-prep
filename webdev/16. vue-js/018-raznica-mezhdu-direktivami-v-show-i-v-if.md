# Q018. Разница между директивами `v-show` и `v-if`?

> **Источник:** [16. vue-js.md](../16.%20vue-js.md) · **Тема:** Vue.js

---

## Короткий ответ

`v-if` полностью добавляет/удаляет элемент из DOM, запуская lifecycle hooks при каждом переключении — дорого при частом использовании, но экономит ресурсы при `false`. `v-show` всегда оставляет элемент в DOM, только переключает `display: none` — дёшево при переключении, но всегда рендерится. Правило выбора: `v-show` для частых переключений, `v-if` для редких или условий, которые часто остаются `false`.

---

## Развёрнутый ответ

### Суть и определение

`v-if` — «истинное» условное рендеринг: элемент физически существует в DOM только при `true`.  
`v-show` — CSS-управление видимостью: элемент всегда в DOM, просто скрыт через `display: none`.

### Как это работает

```vue
<template>
  <!-- v-if: при false — элемента нет в DOM вообще -->
  <ExpensiveChart v-if="showChart" />

  <!-- v-show: при false — display: none, но элемент присутствует -->
  <ExpensiveChart v-show="showChart" />
</template>
```

**`v-if` под капотом** — компилируется в тернарный оператор в рендер-функции:
```javascript
// Скомпилированный код
createCommentVNode("v-if", true) // при false — comment placeholder
// или
createVNode(ExpensiveChart)       // при true
```

**`v-show` под капотом** — добавляет директиву с CSS-манипуляцией:
```javascript
createVNode('div', {
  directives: [{ name: 'show', value: isVisible }]
})
// Директива v-show: el.style.display = value ? '' : 'none'
```

### Практика и применение

**Кейс 1: Табы — `v-show` лучше**
```vue
<template>
  <!-- Тяжёлые компоненты инициализируются один раз -->
  <TabContent v-show="activeTab === 'overview'" />
  <TabContent v-show="activeTab === 'stats'" />
  <TabContent v-show="activeTab === 'settings'" />
</template>
```

**Кейс 2: Модальное окно — `v-if` лучше**
```vue
<template>
  <!-- Модалка открывается редко, инициализация несущественна -->
  <ConfirmModal v-if="isModalOpen" @close="isModalOpen = false" />
</template>
```

**Кейс 3: Права доступа — только `v-if`**
```vue
<template>
  <!-- v-show НЕЛЬЗЯ — элемент остаётся в DOM, пользователь может увидеть через DevTools -->
  <AdminPanel v-if="user.isAdmin" />
  <DeleteButton v-if="canDelete" />
</template>
```

### Важные нюансы и краеугольные камни

- **Безопасность**: `v-show` **не скрывает** контент от инспекции DOM — при любом секретном контенте использовать только `v-if`
- **`v-show` не поддерживает `<template>`** — только реальные HTML-элементы и компоненты
- **Transitions**: оба работают с `<Transition>`, но `v-if` даёт более контролируемый enter/leave; с `v-show` нужен `css: false` для некоторых анимаций
- **SSR**: `v-if="false"` не рендерит HTML на сервере; `v-show="false"` рендерит со style hidden — важно для SEO и гидрации

## Сравнение

| Критерий | `v-if` | `v-show` |
|----------|--------|----------|
| DOM при `false` | Отсутствует | Присутствует (hidden) |
| Начальный рендер при `false` | Нет | Да |
| Стоимость переключения | Высокая (mount/unmount) | Низкая (CSS) |
| Lifecycle hooks | Каждый раз | Один раз при монтировании |
| Поддержка `<template>` | Да | Нет |
| Безопасность скрытия | Да | Нет |
| Лучше для | Редкое переключение, security | Частое переключение |
| `<Transition>` анимации | Полная поддержка | Поддержка с нюансами |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему `v-show` небезопасен для скрытия секретных данных?** — элемент остаётся в DOM с `display: none`, любой пользователь может увидеть его через DevTools
- **Что происходит с дочерними компонентами при `v-if="false"`?** — они размонтируются, их `onUnmounted` вызывается, состояние теряется
- **Как сохранить состояние при `v-if`?** — использовать `<KeepAlive>` или перенести состояние в store/composable вне компонента
- **`v-show` vs `visibility: hidden` vs `opacity: 0`?** — `v-show` использует `display: none` (элемент не занимает место); `visibility: hidden` (место занимает); `opacity: 0` (место занимает, события работают)

### Красные флаги (чего не говорить)

- «Используй `v-show` для скрытия секретного UI» — элемент остаётся в DOM, это уязвимость
- «`v-if` всегда хуже `v-show` по производительности» — при частом условии `false` (почти никогда не показывается) `v-if` экономит ресурсы, не рендеря ничего

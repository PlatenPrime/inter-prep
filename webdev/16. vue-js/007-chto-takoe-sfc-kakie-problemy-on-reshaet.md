# Q007. Что такое SFC? Какие проблемы он решает?

> **Источник:** [16. vue-js.md](../16.%20vue-js.md) · **Тема:** Vue.js

---

## Короткий ответ

**SFC (Single File Component)** — это `.vue`-файл, который объединяет шаблон (`<template>`), логику (`<script>`) и стили (`<style>`) одного компонента в едином файле. SFC решает проблему фрагментации кода компонента по трём разным файлам, обеспечивает изолированные стили через `scoped`, позволяет компилятору применять оптимизации к шаблону и упрощает поиск кода при работе с компонентом.

---

## Развёрнутый ответ

### Суть и определение

SFC — это формат файла, специфичный для Vue.js, обрабатываемый компилятором (`vue-loader` для webpack или `@vitejs/plugin-vue` для Vite). Итоговый `.vue`-файл компилируется в обычный JavaScript-модуль.

Структура `.vue`-файла:

```vue
<template>
  <!-- HTML-шаблон компонента -->
</template>

<script setup lang="ts">
  // Логика компонента
</script>

<style scoped>
  /* CSS-стили, изолированные в пределах компонента */
</style>
```

### Как это работает

**Компиляция шаблона**
Блок `<template>` компилируется в `render`-функцию. Компилятор анализирует шаблон статически и применяет оптимизации:
- **Static hoisting** — неизменяемые узлы поднимаются за пределы функции рендера и создаются один раз
- **Patch flags** — узлам присваиваются флаги, указывающие, что может измениться (props, children, текст), чтобы пропускать ненужные сравнения

**Изолированные стили**
`<style scoped>` добавляет уникальный data-атрибут к каждому элементу компонента и к CSS-селекторам:
```html
<!-- Компилируется в -->
<div data-v-7ba5bd90 class="card">...</div>
```
```css
/* CSS становится */
.card[data-v-7ba5bd90] { ... }
```

**`<script setup>`**
Синтаксический сахар над Composition API: переменные и функции, объявленные в `<script setup>`, автоматически доступны в шаблоне без явного `return {}`.

### Практика и применение

- **IDE-поддержка**: расширение Volar (Vue Language Features) для VS Code обеспечивает автодополнение, навигацию, рефакторинг внутри `.vue`-файла
- **CSS Modules** в SFC: `<style module>` предоставляет объект `$style` с хэшированными именами классов
- **Кастомные блоки**: можно добавлять произвольные секции, например `<docs>` для документации компонента
- **`<style lang="scss">`**: поддержка препроцессоров CSS через указание атрибута `lang`

```vue
<script setup lang="ts">
import { ref } from 'vue'

const isVisible = ref(false)
</script>

<template>
  <div class="card" @click="isVisible = !isVisible">
    <slot />
    <p v-if="isVisible" class="hint">Дополнительная информация</p>
  </div>
</template>

<style scoped lang="scss">
.card {
  padding: 16px;
  border-radius: 8px;

  .hint {
    color: gray;
    font-size: 0.875rem;
  }
}
</style>
```

### Важные нюансы и краеугольные камни

- `<style scoped>` **не изолирует** стили от дочерних компонентов полностью — использовать `:deep(.child-class)` для проникновения внутрь
- **`v-bind()` в CSS**: Vue 3 позволяет использовать реактивные переменные напрямую в CSS: `color: v-bind(color)`
- SFC-файлы без сборщика не работают — нужен `vue-loader` или `@vitejs/plugin-vue`
- **Отсутствие глобальных стилей**: для глобальных правил нужен `<style>` (без `scoped`) или отдельный CSS-файл

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Как работает `<style scoped>` под капотом?** — data-атрибуты + трансформация CSS-селекторов компилятором
- **Что такое `:deep()` в scoped стилях?** — позволяет применить стиль к дочерним компонентам, пробивая изоляцию scoped
- **Чем `<script setup>` лучше обычного `<script>`?** — меньше бойлерплейта, нет return, лучшая производительность (компилятор оптимизирует), лучший вывод типов
- **Можно ли использовать несколько `<style>` в одном файле?** — да, можно сочетать scoped и non-scoped стили

### Красные флаги (чего не говорить)

- «SFC — это просто соглашение об именовании» — SFC-файлы требуют компиляции и обрабатываются специальным образом
- «`scoped` полностью изолирует стили» — стили проникают в дочерние компоненты через `:deep()`, что иногда неожиданно

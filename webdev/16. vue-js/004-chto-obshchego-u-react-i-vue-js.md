# Q004. Что общего у React и Vue.js?

> **Источник:** [16. vue-js.md](../16.%20vue-js.md) · **Тема:** Vue.js

---

## Короткий ответ

React и Vue.js разделяют несколько фундаментальных подходов: оба используют Virtual DOM для оптимизации обновлений DOM, строят UI из переиспользуемых компонентов, поддерживают однонаправленный поток данных через props, имеют хуки/composables для инкапсуляции логики, и активно применяются для создания SPA. Оба инструмента — только View-слой, не диктуют архитектуру всего приложения.

---

## Развёрнутый ответ

### Суть и определение

Сходства React и Vue — это общие архитектурные принципы современного компонентного фронтенда. Понимать их важно, чтобы объяснить, почему Vue не является «копией React» и в чём их философские отличия при похожем фундаменте.

### Как это работает

**1. Virtual DOM**
Оба фреймворка поддерживают дерево виртуальных узлов (VNode) в памяти. При изменении данных создаётся новое VNode-дерево, алгоритм diff сравнивает его со старым и патчит только изменившиеся части реального DOM.

**2. Компонентная архитектура**
- Оба строят UI как дерево компонентов
- Компонент = инкапсулированный кусок UI со своим состоянием и логикой
- Компоненты принимают данные (props) и генерируют события/коллбэки вверх

**3. Реактивность и состояние**
- React: `useState`, `useReducer`, `useMemo` — явное управление состоянием
- Vue: `ref`, `reactive`, `computed` — автоматическое отслеживание зависимостей
- Оба: изменение состояния → ре-рендер компонента

**4. Однонаправленный поток данных**
Props передаются от родителя к потомку, изменения поднимаются через события/коллбэки. «Props down, events up».

**5. Composables / Custom Hooks**
- React: кастомные хуки (`useLocalStorage`, `useFetch`)
- Vue: composables (`useLocalStorage`, `useFetch`)
Оба паттерна решают одну задачу: переиспользование stateful-логики между компонентами.

**6. Экосистема для SSR**
- React → Next.js
- Vue → Nuxt.js
Оба предоставляют SSR, SSG, Islands Architecture, серверные компоненты.

**7. Нет встроенного HTTP-клиента и форм**
Оба оставляют выбор инструментов для HTTP (axios, fetch), форм (React Hook Form / Vee-Validate), анимаций на разработчика.

**8. Developer Tools**
Оба имеют браузерные расширения для отладки: Vue DevTools и React DevTools с инспекцией компонентного дерева, состояния, производительности.

**9. Поддержка TypeScript**
Оба обеспечивают первоклассную поддержку TypeScript с полным выводом типов.

**10. Открытый исходный код и MIT-лицензия**

### Практика и применение

Знание общих концепций позволяет быстро переключаться между стеками: разработчик на React освоит Vue за 1–2 недели за счёт общего ментального фундамента.

```typescript
// React custom hook
function useCounter(initial = 0) {
  const [count, setCount] = useState(initial)
  const increment = () => setCount(c => c + 1)
  return { count, increment }
}

// Vue composable — та же идея
function useCounter(initial = 0) {
  const count = ref(initial)
  const increment = () => count.value++
  return { count, increment }
}
```

### Важные нюансы и краеугольные камни

- Virtual DOM в Vue 3 значительно оптимизирован compile-time эвристиками, что даёт преимущество перед React в benchmark-тестах со статическими шаблонами
- В React состояние **явно иммутабельно** (`setState` всегда заменяет значение); в Vue — мутация реактивного объекта напрямую легальна и поощряется

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Если они так похожи, зачем существует Vue?** — разная философия реактивности, шаблонный синтаксис vs JSX, прогрессивность, другой размер экосистемы
- **Чем реактивность Vue отличается от useState React?** — Vue автоматически отслеживает зависимости; React требует явного вызова setter-функции
- **Оба используют Virtual DOM — значит у них одинаковая производительность?** — нет; Vue 3 имеет compile-time оптимизации, которых нет в React без доп. инструментов

### Красные флаги (чего не говорить)

- «Vue — это просто более лёгкий React» — это неточно; у них разная система реактивности и разный синтаксис
- Путать composables Vue с React hooks на уровне реализации — механизм разный, даже если паттерн похож

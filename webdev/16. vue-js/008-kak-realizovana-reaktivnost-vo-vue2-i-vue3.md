# Q008. Как реализована реактивность во Vue2 и Vue3?

> **Источник:** [16. vue-js.md](../16.%20vue-js.md) · **Тема:** Vue.js

---

## Короткий ответ

**Vue 2** реализует реактивность через `Object.defineProperty`: при инициализации компонента Vue обходит все свойства объекта `data` и заменяет их геттером и сеттером. Это ограничивает отслеживание динамически добавляемых свойств и массивов. **Vue 3** использует `Proxy` — перехватывает любые операции с объектом (чтение, запись, удаление, добавление), что устраняет все ограничения Vue 2 и улучшает производительность.

---

## Развёрнутый ответ

### Суть и определение

Реактивность — это механизм, при котором изменение данных автоматически вызывает перерендер всех зависящих от них частей UI. Это «сердце» Vue, и понимание его реализации критично для senior-позиций.

### Как это работает

#### Vue 2 — `Object.defineProperty`

```javascript
function defineReactive(obj, key, value) {
  const dep = new Dep() // список подписчиков

  Object.defineProperty(obj, key, {
    get() {
      if (Dep.target) dep.depend() // запоминаем текущий watcher
      return value
    },
    set(newValue) {
      if (newValue === value) return
      value = newValue
      dep.notify() // уведомляем все зависимости
    }
  })
}
```

**Процесс:**
1. `new Vue({ data() { return { user: { name: 'Иван' } } } })`
2. Vue рекурсивно обходит `data` и вызывает `defineReactive` для каждого свойства
3. При рендере компонента Vue создаёт `Watcher` и устанавливает `Dep.target`
4. Чтение свойства в шаблоне → геттер → `dep.depend()` → зависимость зарегистрирована
5. Изменение свойства → сеттер → `dep.notify()` → ре-рендер

**Ограничения Vue 2:**
- **Нельзя** добавлять новые свойства: `this.user.email = '...'` — не реактивно
- **Нельзя** удалять свойства: `delete this.user.name` — не реактивно
- Массивы: мутирующие методы (`push`, `pop`, `splice` и т.д.) перехвачены, но прямое присваивание по индексу (`arr[0] = val`) — нет
- **Решение в Vue 2**: `Vue.set(obj, 'email', '...')` или `this.$set(this.user, 'email', '...')`

#### Vue 3 — `Proxy`

```javascript
function reactive(target) {
  return new Proxy(target, {
    get(target, key, receiver) {
      track(target, key) // регистрируем зависимость
      return Reflect.get(target, key, receiver)
    },
    set(target, key, value, receiver) {
      const result = Reflect.set(target, key, value, receiver)
      trigger(target, key) // уведомляем эффекты
      return result
    },
    deleteProperty(target, key) {
      const result = Reflect.deleteProperty(target, key)
      trigger(target, key)
      return result
    }
  })
}
```

**Преимущества Vue 3 Proxy:**
- Перехватывает **любые** операции: добавление, удаление, изменение, итерация (`ownKeys`)
- Не нужен `Vue.set` — обычное `user.email = '...'` реактивно
- Ленивая инициализация — вложенные объекты оборачиваются в Proxy только при первом обращении
- Работает с массивами нативно — любой индекс (`arr[5] = val`) реактивен

#### `ref` vs `reactive`

```typescript
import { ref, reactive } from 'vue'

// ref — для примитивов и одиночных значений
const count = ref(0)
count.value++ // доступ через .value

// reactive — для объектов
const state = reactive({ name: 'Иван', age: 25 })
state.name = 'Пётр' // прямая мутация реактивна
```

`ref` внутри использует `reactive({ value: ... })` для объектов или геттер/сеттер для примитивов.

### Практика и применение

- **Деструктуризация ломает реактивность** `reactive`:
```typescript
const state = reactive({ count: 0 })
const { count } = state // count — примитив, реактивность потеряна!

// Решение: toRefs
const { count } = toRefs(state) // count — ref, реактивен
```

- **`shallowRef`/`shallowReactive`** — для оптимизации: не делать реактивными вложенные объекты

### Важные нюансы и краеугольные камни

- `Proxy` работает только в современных браузерах (IE11 не поддерживается) — Vue 3 не поддерживает IE
- **Реактивный массив**: в Vue 3 `arr[0] = val` реактивно, но `arr.length = 0` тоже перехватывается
- **WeakMap для хранения зависимостей**: Vue 3 хранит карту зависимостей в `WeakMap<target, Map<key, Set<effect>>>`, что позволяет GC собирать неиспользуемые объекты
- Функция `watchEffect` аналогична `useEffect(fn)` в React, но зависимости определяются автоматически через трекинг

## Сравнение

| Критерий | Vue 2 (`Object.defineProperty`) | Vue 3 (`Proxy`) |
|----------|----------------------------------|-----------------|
| Добавление свойств | Не реактивно, нужен `Vue.set` | Реактивно нативно |
| Удаление свойств | Не реактивно, нужен `Vue.delete` | Реактивно нативно |
| Массив по индексу (`arr[i] = v`) | Не реактивно | Реактивно |
| Производительность инициализации | Обход всего объекта eager | Ленивая обёртка при обращении |
| Поддержка IE | IE9+ | Нет (Proxy не полифилируется) |
| Отладка | Сложнее | `reactive` — обычный объект в DevTools |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему Proxy нельзя полифилить для IE?** — Proxy — language feature, не API; его поведение нельзя эмулировать через `Object.defineProperty` полностью
- **Что такое `track` и `trigger` в Vue 3?** — внутренние функции системы реактивности: `track` регистрирует зависимость текущего активного эффекта, `trigger` запускает все зависящие эффекты
- **Как работает `computed` в Vue 3?** — ленивый эффект с кешированием; пересчитывается только когда зависимость изменилась и значение запрошено
- **Чем `watchEffect` отличается от `watch`?** — `watchEffect` автоматически определяет зависимости; `watch` явно указывает источник и имеет доступ к старому значению

### Красные флаги (чего не говорить)

- «Во Vue 3 не нужен `.value` для ref» — нужен в `<script>`, только в шаблоне автоматически разворачивается
- «Vue 2 и Vue 3 реактивность одинакова» — принципиально разные механизмы с разными ограничениями
- «`reactive` лучше `ref`» — для примитивов нет выбора кроме `ref`; `reactive` теряет реактивность при деструктуризации

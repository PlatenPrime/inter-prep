# Q005. Разница между React и Vue.js?

> **Источник:** [16. vue-js.md](../16.%20vue-js.md) · **Тема:** Vue.js

---

## Короткий ответ

Главные отличия: React использует JSX и требует явного управления состоянием (useState/useReducer), Vue использует HTML-шаблоны с директивами и имеет встроенную автоматическую реактивность (Proxy). React — библиотека только для View, Vue — более полноценный фреймворк с официальными роутером и стейт-менеджером. Vue имеет более низкий порог вхождения, React — большую экосистему и рынок труда.

---

## Развёрнутый ответ

### Суть и определение

React и Vue решают одну задачу — построение компонентного UI — но с разной философией: React делает ставку на «JavaScript всё», Vue — на «расширенный HTML с реактивностью».

### Как это работает

**Синтаксис шаблонов**

React использует JSX — JavaScript с HTML-синтаксисом внутри:
```jsx
// React — логика и шаблон смешаны в JS
function List({ items }) {
  return (
    <ul>
      {items.filter(Boolean).map((item, i) => (
        <li key={i}>{item}</li>
      ))}
    </ul>
  )
}
```

Vue использует HTML-шаблоны с директивами:
```vue
<!-- Vue — HTML с расширениями -->
<template>
  <ul>
    <li v-for="(item, i) in items" :key="i">{{ item }}</li>
  </ul>
</template>
```

**Система реактивности**

React — **явная, иммутабельная**:
- `useState` возвращает геттер + сеттер
- Нужно явно вызывать `setState`, нельзя мутировать напрямую
- `useMemo`/`useCallback` — ручная оптимизация мемоизации

Vue — **автоматическая, мутабельная**:
- `ref`/`reactive` создают Proxy-обёртку
- Прямая мутация (`user.name = 'Иван'`) триггерит обновление
- `computed` автоматически кэшируется и инвалидируется

**Управление состоянием**

| | React | Vue |
|---|---|---|
| Локальное | `useState`, `useReducer` | `ref`, `reactive` |
| Глобальное | Redux, Zustand, Jotai (сторонние) | Pinia (официальный) |
| Встроено? | Только Context API | Pinia — официальный |

**Двустороннее связывание**

React не поддерживает нативно — нужно писать вручную:
```jsx
// React — явный controlled input
<input value={name} onChange={e => setName(e.target.value)} />
```

Vue предоставляет `v-model`:
```vue
<!-- Vue — сахар над :value + @input -->
<input v-model="name" />
```

### Практика и применение

- **React** предпочтителен при: нужен React Native; команда уже на React; богатая экосистема критична; нужна большая гибкость в выборе паттернов
- **Vue** предпочтителен при: команда с HTML/backend-бэкграундом; Laravel/PHP проект; быстрый старт без конфигурации; предпочтителен один «правильный» способ делать вещи

### Важные нюансы и краеугольные камни

- React Concurrent Mode (React 18) делает рендеринг прерываемым — это уникальная особенность без прямого аналога в Vue
- Vue Composition API синтаксически похож на React Hooks, но механизм принципиально другой: нет правил о порядке вызовов (can be called conditionally), нет проблемы stale closures
- Vue DevTools значительно информативнее для отладки реактивности

## Сравнение

| Критерий | React | Vue.js |
|----------|-------|--------|
| Тип | Библиотека (View) | Прогрессивный фреймворк |
| Шаблоны | JSX | HTML-шаблоны + директивы |
| Реактивность | Явная (useState/setState) | Автоматическая (Proxy) |
| Двустороннее связывание | Нет нативного | `v-model` |
| Стейт-менеджер | Сторонний (Redux/Zustand) | Официальный (Pinia) |
| Роутер | React Router (сторонний) | Vue Router (официальный) |
| SSR | Next.js | Nuxt.js |
| Размер бандла | ~42 KB gzipped | ~22 KB gzipped |
| Рынок труда | Очень большой | Меньше |
| Порог вхождения | Средний | Низкий |
| TypeScript | Хорошо | Отлично (написан на TS) |
| Мобильные приложения | React Native | NativeScript-Vue (ограниченно) |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Когда вы выберете React, а когда Vue?** — ответить через конкретные проектные требования, не «Vue проще»
- **Чем Composition API отличается от React Hooks?** — нет правил порядка вызова, нет stale closures, нет проблемы с замыканиями в useEffect
- **Можно ли использовать JSX в Vue?** — да, Vue поддерживает render functions и JSX, но это не идиоматично
- **Что такое Concurrent Mode и есть ли аналог в Vue?** — нет прямого аналога; Vue использует scheduler для асинхронных обновлений, но без прерываемого рендеринга

### Красные флаги (чего не говорить)

- «React лучше Vue» или «Vue лучше React» без контекста — зрелый ответ всегда через trade-offs
- «В Vue нет TypeScript» — Vue 3 написан на TypeScript
- «Vuex — это официальный стейт-менеджер Vue» — Vuex заменён Pinia начиная с Vue 3

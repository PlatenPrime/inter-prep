# Q001. Что такое Vue.js?

> **Источник:** [16. vue-js.md](../16.%20vue-js.md) · **Тема:** Vue.js

---

## Короткий ответ

**Vue.js** — это прогрессивный JavaScript-фреймворк для построения пользовательских интерфейсов, созданный Эваном Ю в 2014 году. «Прогрессивный» означает, что его можно встроить в существующее приложение как небольшую библиотеку или использовать как полноценный фреймворк для SPA. Vue сочетает реактивную систему данных, компонентную архитектуру и декларативный шаблонный синтаксис.

---

## Развёрнутый ответ

### Суть и определение

Vue.js — это фреймворк для View-слоя, вдохновлённый лучшими идеями Angular (директивы, двустороннее связывание) и React (Virtual DOM, компоненты). Ключевое отличие от обоих — **минимальный порог вхождения**: HTML-шаблоны, понятные разработчикам без глубокого знания JS, и реактивная система, которая отслеживает зависимости автоматически.

Основные концепции:
- **Реактивность** — изменение данных автоматически обновляет DOM
- **Компонентная архитектура** — UI разбивается на изолированные, переиспользуемые компоненты
- **Директивы** — специальные HTML-атрибуты (`v-if`, `v-for`, `v-bind`, `v-model`) для декларативной логики в шаблонах
- **Single File Components (SFC)** — `.vue`-файлы объединяют шаблон, скрипт и стили в одном месте

### Как это работает

1. Vue создаёт **реактивный граф зависимостей**: при обращении к реактивной переменной внутри computed/watch/шаблона Vue запоминает, что этот эффект зависит от данной переменной
2. При изменении переменной Vue уведомляет все зависящие эффекты и планирует ре-рендер
3. Vue строит **Virtual DOM** — дерево VNode, описывающее желаемое состояние UI
4. Алгоритм **патчинга (diffing)** сравнивает новый и старый VNode-деревья и вносит минимальные изменения в реальный DOM
5. В Vue 3 рендеринг оптимизирован **статическим подъёмом (hoisting)** и **patch flags** — статические узлы не сравниваются повторно

### Практика и применение

- **SPA с Vite** — `npm create vue@latest` генерирует проект с Vue 3 + Vite + TypeScript за минуты
- **Nuxt.js** — SSR/SSG фреймворк поверх Vue (аналог Next.js для React), используется для SEO-проектов и контентных сайтов
- **Встраивание в legacy** — можно подключить Vue через CDN-тег и добавить интерактивность к отдельным виджетам без сборщика
- **Progressive enhancement** — миграция с jQuery-приложения на Vue по частям без переписывания всего сразу

### Важные нюансы и краеугольные камни

- Vue 2 использует `Object.defineProperty` для реактивности — не отслеживает добавление/удаление свойств; Vue 3 перешёл на `Proxy` и закрыл эту проблему
- Vue 3 поддерживает **Composition API** (аналог React Hooks) наряду с Options API — логику можно группировать по фичам, а не по типу опций
- **Размер бандла**: Vue 3 runtime ~ 22 KB gzipped — значительно меньше Angular (~60 KB)
- Vue **не является реактивным в стиле RxJS** — он использует pull/push гибрид на основе трекинга зависимостей

### Примеры

```vue
<!-- App.vue — минимальный компонент на Vue 3 Composition API -->
<script setup lang="ts">
import { ref } from 'vue'

const count = ref(0)
const increment = () => count.value++
</script>

<template>
  <button @click="increment">Счётчик: {{ count }}</button>
</template>

<style scoped>
button { padding: 8px 16px; }
</style>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Чем Vue 3 отличается от Vue 2?** — Proxy-реактивность, Composition API, Teleport, Fragments, лучший TypeScript, tree-shaking
- **Что значит «прогрессивный» фреймворк?** — можно использовать только нужные части: только реактивность, только компоненты, или полный стек с роутером и стейт-менеджером
- **Как Vue сравнивается с React?** — Vue имеет официальный роутер и стейт-менеджер, HTML-шаблоны (не JSX), более автоматическую реактивность
- **Что такое Composition API и зачем он появился?** — для решения проблем с переиспользованием логики, которые были у mixins и Options API
- **Что такое Nuxt.js?** — мета-фреймворк для Vue с SSR, SSG, file-based routing, автоматическими импортами

### Красные флаги (чего не говорить)

- «Vue — это просто аналог React» — у Vue принципиально другая система реактивности и шаблонный синтаксис, отличная философия дизайна
- «Vue только для маленьких проектов» — Alibaba, GitLab, Laravel Echo используют Vue в enterprise-масштабе
- «Composition API заменил Options API» — оба поддерживаются в Vue 3, это выбор стиля

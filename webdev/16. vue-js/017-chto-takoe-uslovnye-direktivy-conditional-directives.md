# Q017. Что такое условные директивы (conditional directives)?

> **Источник:** [16. vue-js.md](../16.%20vue-js.md) · **Тема:** Vue.js

---

## Короткий ответ

**Условные директивы** — это встроенные директивы Vue для условного отображения элементов в шаблоне. Основные: `v-if` (полностью добавляет/удаляет элемент из DOM), `v-else-if`, `v-else` (цепочка условий), `v-show` (переключает CSS `display: none` без удаления из DOM). Выбор между ними зависит от частоты переключения и необходимости сохранять DOM-состояние.

---

## Развёрнутый ответ

### Суть и определение

Условные директивы — это декларативный способ управления видимостью элементов. Они обрабатываются компилятором Vue и превращаются в условную логику рендер-функции.

### Как это работает

#### `v-if` / `v-else-if` / `v-else`

```vue
<template>
  <!-- Базовый v-if -->
  <div v-if="isLoggedIn">Добро пожаловать!</div>

  <!-- Цепочка условий -->
  <div v-if="status === 'loading'">Загрузка...</div>
  <div v-else-if="status === 'error'">Ошибка: {{ errorMessage }}</div>
  <div v-else-if="status === 'empty'">Данных нет</div>
  <div v-else>{{ data }}</div>

  <!-- v-if на <template> — для группировки без лишнего wrapper-элемента -->
  <template v-if="showDetails">
    <h3>Детали</h3>
    <p>{{ description }}</p>
    <ul>
      <li v-for="tag in tags" :key="tag">{{ tag }}</li>
    </ul>
  </template>
</template>
```

**Что происходит под капотом:**
- `v-if="false"` — компонент/элемент **размонтируется**, его хуки `onBeforeUnmount`/`onUnmounted` вызываются
- `v-if="true"` — компонент **монтируется** заново, хуки `onMounted` вызываются

#### `v-show`

```vue
<template>
  <!-- DOM-элемент существует всегда, только display меняется -->
  <div v-show="isVisible">Я всегда в DOM</div>
</template>
```

Компилируется примерно в:
```javascript
h('div', {
  style: { display: isVisible ? '' : 'none' }
}, 'Я всегда в DOM')
```

### Практика и применение

**Когда `v-if`, когда `v-show`:**

```vue
<template>
  <!-- v-show: компонент с дорогой инициализацией, часто переключается -->
  <DataChart v-show="activeTab === 'chart'" :data="chartData" />
  <DataTable v-show="activeTab === 'table'" :data="tableData" />

  <!-- v-if: блок, который редко показывается или не нужен сразу -->
  <AdminPanel v-if="user.isAdmin" />

  <!-- v-if: условие по данным с сервера (может быть null) -->
  <UserProfile v-if="user" :user="user" />
</template>
```

**`v-if` с `:key` для сброса состояния:**

```vue
<template>
  <!-- Без :key — при смене userId компонент переиспользуется -->
  <!-- С :key — при смене userId компонент пересоздаётся полностью -->
  <UserForm v-if="userId" :key="userId" :user-id="userId" />
</template>
```

### Важные нюансы и краеугольные камни

- **`v-if` имеет высший приоритет, чем `v-for`** в Vue 3 (в Vue 2 — наоборот). Никогда не использовать на одном элементе — поместить `v-if` на родительский элемент или `<template>`
- **`v-else` и `v-else-if`** должны идти сразу после `v-if` — между ними не может быть других элементов
- **Компоненты внутри `v-if`** полностью монтируются/размонтируются — это стоит производительности при частом переключении
- **`v-show` не работает** с `<template>` — только с реальными элементами
- **`v-show` на компоненте** не скрывает его от lifecycle hooks — `onMounted` вызовется даже если `v-show="false"` при монтировании

## Сравнение

| Критерий | `v-if` | `v-show` |
|----------|--------|----------|
| Наличие в DOM | Удаляет/добавляет | Всегда в DOM |
| Переключение CSS | — | `display: none` |
| Стоимость переключения | Высокая (mount/unmount) | Низкая |
| Начальная стоимость | Низкая (false — не рендерит) | Высокая (рендерит всегда) |
| Lifecycle hooks | Вызываются при каждом mount | Вызываются один раз |
| Работает с `<template>` | Да | Нет |
| Лучше для | Редкое переключение | Частое переключение |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему нельзя использовать `v-if` и `v-for` на одном элементе?** — в Vue 3 `v-if` имеет высший приоритет и выполнится раньше `v-for`, что может вызвать ошибку если `v-if` обращается к переменной цикла
- **Как скрыть группу элементов без обёртки?** — `<template v-if="...">` или `<template v-show="...">` (только v-if для template)
- **Когда `v-show` хуже `v-if`?** — при начальном рендере с `false`: `v-if` не рендерит вообще, `v-show` рендерит, но скрывает; для редко видимых блоков `v-if` лучше

### Красные флаги (чего не говорить)

- «`v-show` и `v-if` одинаковы, только синтаксис разный» — принципиально разные механизмы с разными performance-профилями
- Использовать `v-if` и `v-for` на одном элементе — одна из частых ошибок, ESLint-плагин Vue её запрещает

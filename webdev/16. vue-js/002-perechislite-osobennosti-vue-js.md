# Q002. Перечислите особенности Vue.js?

> **Источник:** [16. vue-js.md](../16.%20vue-js.md) · **Тема:** Vue.js

---

## Короткий ответ

Ключевые особенности Vue.js: реактивная система данных с автоматическим трекингом зависимостей, компонентная архитектура с SFC (`.vue`-файлами), декларативный шаблонный синтаксис с директивами, Virtual DOM с оптимизациями компиляции, двустороннее связывание данных через `v-model`, а также два стиля написания компонентов — Options API и Composition API.

---

## Развёрнутый ответ

### Суть и определение

Особенности Vue.js — это набор архитектурных решений и механизмов, которые определяют философию фреймворка: **простота + мощность + прогрессивность**.

### Как это работает

**1. Реактивная система данных**
Vue 3 использует `Proxy` для создания реактивных объектов. При чтении реактивного свойства внутри эффекта (шаблон, computed, watchEffect) Vue регистрирует зависимость. При записи — уведомляет подписчиков.

**2. Компонентная архитектура**
UI состоит из дерева компонентов. Каждый компонент инкапсулирует шаблон, логику и стили. SFC (Single File Component) объединяет их в одном `.vue`-файле.

**3. Директивы**
Встроенные директивы (`v-if`, `v-for`, `v-bind`, `v-on`, `v-model`, `v-show`, `v-slot`) — декларативные расширения HTML. Можно создавать **пользовательские директивы** для работы с DOM напрямую.

**4. Virtual DOM с compile-time оптимизациями**
Vue 3 компилятор помечает статические узлы (`patch flags`) и поднимает их (`static hoisting`) — при повторных рендерах они пропускаются. Это даёт значительный прирост производительности по сравнению с runtime-only подходом.

**5. Двустороннее связывание данных**
`v-model` — синтаксический сахар над `:value` + `@input`. В Vue 3 компоненты могут иметь несколько `v-model` с разными именами.

**6. Options API и Composition API**
- **Options API**: логика группируется по опциям (`data`, `computed`, `methods`, `watch`, lifecycle hooks)
- **Composition API**: логика группируется по фичам в `setup()` или `<script setup>`; использует `ref`, `reactive`, `computed`, `watch`, lifecycle hooks

**7. Встроенный роутер и стейт-менеджер**
Vue Router и Pinia (Vuex) — официально поддерживаемые решения, разработанные командой Vue.

**8. Transitions и Animations**
Встроенный компонент `<Transition>` и директива для анимации появления/исчезновения элементов без сторонних библиотек.

**9. Teleport и Fragments**
- `<Teleport to="#modal">` — рендер DOM-узла в другом месте дерева
- Компонент может возвращать несколько корневых элементов (fragments)

**10. Прогрессивность**
Можно использовать Vue как замену jQuery через CDN-тег без сборщика, или строить полноценное enterprise-приложение с Vite + TypeScript + Nuxt.

### Практика и применение

- **Реактивность вне компонентов** — `reactive()`, `ref()`, `computed()` работают автономно, что позволяет использовать реактивную систему Vue в Vanilla JS-модулях
- **`<script setup>`** — наиболее лаконичный синтаксис Composition API, компилируется в оптимальный код
- **Composables** — переиспользуемые блоки логики (аналог кастомных хуков в React): `useAuth()`, `useFetch()`, `useLocalStorage()`

### Важные нюансы и краеугольные камни

- Реактивность Vue **не работает с примитивами напрямую** — `ref` оборачивает их в объект; деструктуризация `reactive`-объекта ломает реактивность
- `v-for` требует атрибут `:key` — без него Vue не может корректно сопоставить DOM-узлы при изменении списка
- Компиляция шаблонов происходит **на этапе сборки** (Vite/webpack) — в production попадает только runtime-версия Vue без компилятора

### Примеры

```vue
<script setup lang="ts">
import { ref, computed, watch } from 'vue'

const firstName = ref('Иван')
const lastName = ref('Петров')

// Computed — автоматически обновляется при изменении зависимостей
const fullName = computed(() => `${firstName.value} ${lastName.value}`)

// Watch — реагирует на изменения
watch(fullName, (newVal) => {
  console.log('Имя изменилось:', newVal)
})
</script>

<template>
  <input v-model="firstName" placeholder="Имя" />
  <input v-model="lastName" placeholder="Фамилия" />
  <p>{{ fullName }}</p>
</template>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Что такое compile-time оптимизации Vue 3?** — статический подъём (hoisting), patch flags, tree-shaking — компилятор знает, какие части шаблона статичны
- **Чем Composition API лучше Options API?** — лучшая переиспользуемость логики, нет проблем с `this`, TypeScript-дружественность
- **Зачем `ref` вместо прямых переменных?** — примитивы не являются объектами, Proxy не может их перехватить; `ref` создаёт обёртку
- **Что такое composable?** — функция, использующая Composition API для инкапсуляции и переиспользования stateful-логики

### Красные флаги (чего не говорить)

- «`v-model` — это просто двустороннее связывание с данными» — без понимания, что это сахар над `:value`/`@input` и что его можно кастомизировать
- «Директивы — это как методы» — директивы работают с DOM напрямую, это отдельная абстракция
- «Options API устарел» — в Vue 3 оба API равноценны и официально поддерживаются

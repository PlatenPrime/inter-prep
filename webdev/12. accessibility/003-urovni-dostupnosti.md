# Q003. Уровни доступности?

> **Источник:** [12. accessibility.md](../12.%20accessibility.md) · **Тема:** Accessibility

---

## Короткий ответ

WCAG определяет три уровня соответствия: **A** (минимальный), **AA** (рекомендуемый) и **AAA** (максимальный). Уровень AA является промышленным стандартом — его требуют большинство законодательных актов о доступности. Уровень AAA не ожидается для всего контента: сам стандарт признаёт, что некоторые критерии нереалистичны для отдельных типов контента.

---

## Развёрнутый ответ

### Суть и определение

Уровни (conformance levels) описывают **степень доступности** сайта. Каждый критерий успеха (Success Criterion) в WCAG имеет ровно один уровень: A, AA или AAA. Соответствие уровню означает выполнение всех критериев этого уровня **и всех нижестоящих**.

| Уровень | Критериев в WCAG 2.2 | Смысл |
|---------|----------------------|-------|
| A | 30 | Базовые барьеры устранены |
| AA | 20 дополнительных (итого 50) | Широкая доступность |
| AAA | 28 дополнительных (итого 78) | Максимальная доступность |

### Как это работает

#### Уровень A

Устраняет критические барьеры: без него контент полностью недоступен для отдельных групп.

Примеры критериев:
- 1.1.1 Нетекстовый контент — `alt` для изображений
- 1.3.1 Информация и взаимосвязи — семантическая разметка
- 2.1.1 Клавиатура — весь функционал с клавиатуры
- 4.1.1 Разбор — валидный HTML (в WCAG 2.2 удалён как избыточный)

#### Уровень AA

Устраняет значительные барьеры. Стандарт для коммерческих проектов.

Примеры критериев:
- 1.4.3 Контраст (минимум) — текст ≥ 4.5:1, крупный текст ≥ 3:1
- 1.4.4 Изменение размера текста — до 200% без потери контента
- 2.4.7 Видимый фокус — фокус клавиатуры всегда виден
- 4.1.3 Сообщения о состоянии — `aria-live`/`role="status"` без переноса фокуса

#### Уровень AAA

Максимальная доступность. Не требуется для всего сайта.

Примеры критериев:
- 1.4.6 Контраст (улучшенный) — текст ≥ 7:1
- 2.2.3 Без таймаута — контент без ограничений по времени
- 2.4.9 Цель ссылки только по тексту ссылки

### Практика и применение

- **Государственные сайты ЕС**: WCAG 2.1 AA обязателен с 2019 г. (EU Web Accessibility Directive)
- **США (Section 508)**: WCAG 2.0 AA с 2018 г.
- **Коммерческие проекты**: WCAG 2.1 AA — de facto стандарт при оценке правовых рисков
- **Активистские / специализированные ресурсы**: стремятся к частичному AAA (усиленный контраст, без таймаутов)

При conformance claim обязательно указывают: версию WCAG, уровень, область охвата (весь сайт или конкретные страницы).

### Важные нюансы и краеугольные камни

- Соответствие уровню — **всё или ничего**: нельзя заявить AA, если не выполнен хотя бы один критерий AA или A.
- Принцип «полного соответствия» (full page conformance): вся страница должна соответствовать, не только отдельные компоненты.
- WCAG 2.1 vs 2.2: обратно совместимы. Сайт, соответствующий 2.2 AA, автоматически соответствует 2.1 AA и 2.0 AA.
- AAA не означает «идеально»: доступность — процесс, а не точка финиша.

### Примеры

```html
<!-- Уровень A: alt-текст -->
<img src="promo.jpg" alt="Скидка 50% на все курсы до конца мая" />

<!-- Уровень AA: достаточный контраст + видимый фокус -->
<style>
  /* Текст #333 на #fff = 12.6:1 ✓ AA и AAA */
  body { color: #333; background: #fff; }

  /* Видимый фокус: WCAG 2.4.7 (AA) и 2.4.11 (AA, WCAG 2.2) */
  :focus-visible {
    outline: 2px solid #005fcc;
    outline-offset: 2px;
  }
</style>

<!-- Уровень AAA: расширенный контраст для иконки -->
<!-- Иконка как текстовый символ с 7:1 — AAA 1.4.6 -->
<span aria-hidden="true" style="color: #000;">⚙</span>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Какой уровень требует ваш проект?** — стандартный ответ: AA; упомянуть законодательный контекст (страна, тип организации).
- **Можно ли частично соответствовать уровню AA?** — нет, при formalt conformance claim; на практике используют «частичное соответствие» с оговорками.
- **Чем WCAG 2.2 AA отличается от 2.1 AA?** — 9 новых SC, среди них видимый фокус (2.4.11, 2.4.12), целевой размер 2.5.8.

### Красные флаги (чего не говорить)

- «Мы целимся в AAA» — если без обоснования, это звучит как непонимание реальности проекта.
- «A — это уже достаточно» — A устраняет только критические барьеры; AA требуют законы.
- Путать уровни WCAG с баллами Lighthouse — это совершенно разные метрики.

# Q001. Что такое WCAG?

> **Источник:** [12. accessibility.md](../12.%20accessibility.md) · **Тема:** Accessibility

---

## Короткий ответ

**WCAG (Web Content Accessibility Guidelines)** — международный стандарт доступности веб-контента, разработанный консорциумом W3C в рамках инициативы WAI. Он описывает, как делать веб-контент доступным для людей с ограниченными возможностями: нарушениями зрения, слуха, моторики и когнитивными нарушениями. Стандарт построен на четырёх принципах (POUR) и организован по трём уровням соответствия: A, AA, AAA.

---

## Развёрнутый ответ

### Суть и определение

WCAG — это набор рекомендаций (guidelines), который определяет технические и дизайн-требования к веб-контенту. Поддерживается W3C/WAI (Web Accessibility Initiative). Актуальная версия — **WCAG 2.2** (2023), разрабатывается WCAG 3.0.

Структура документа:
- **Принципы (Principles)** — 4 штуки (POUR)
- **Руководства (Guidelines)** — 13 в WCAG 2.x
- **Критерии успеха (Success Criteria)** — 78 проверяемых условий, каждое с уровнем A/AA/AAA
- **Техники (Techniques)** — конкретные реализации (HTML, CSS, ARIA, скрипты)

### Как это работает

Четыре принципа **POUR**:

| Принцип | Суть |
|---------|------|
| **Perceivable** (Воспринимаемый) | Контент должен быть доступен через разные органы чувств: alt-тексты, субтитры, адаптируемая разметка |
| **Operable** (Управляемый) | Весь функционал — с клавиатуры; достаточно времени; нет контента, вызывающего приступы |
| **Understandable** (Понятный) | Язык, предсказуемость поведения, помощь при ошибках |
| **Robust** (Надёжный) | Совместимость со вспомогательными технологиями (скринридеры, брайль-дисплеи) сейчас и в будущем |

Уровни соответствия:
- **A** — минимальный, устраняет критические барьеры
- **AA** — промышленный стандарт, требуется во большинстве законодательств (EU WCAG 2.1 AA, ADA в США, ГОСТ Р 52872 в РФ)
- **AAA** — максимальный, не всегда достижим для всего контента

### Практика и применение

- В коммерческих проектах целевой уровень — **WCAG 2.1 AA**: требуют European Accessibility Act (2025), Section 508 (США), AODA (Канада).
- Для государственных сайтов в ЕС — обязателен WCAG 2.1 AA с 2019 года.
- Аудит проводится: автоматически (axe, Lighthouse), вручную (клавиатурная навигация, скринридеры), с участием пользователей с ОВЗ.

### Важные нюансы и краеугольные камни

- WCAG — это **рекомендации**, не сам закон; законы ссылаются на WCAG как на стандарт.
- Автоматические инструменты выявляют ~30–40% нарушений; ручное тестирование обязательно.
- WCAG 2.2 добавил критерии 2.4.11–2.4.13 (фокус видимый), 3.2.6, 3.3.7–3.3.8 — особенно важны для форм.
- **AAA не обязателен** для всего сайта — стандарт признаёт, что достичь его для всего контента нереалистично.

### Примеры

```html
<!-- WCAG 1.1.1 (A): нетекстовый контент — alt обязателен -->
<img src="chart.png" alt="Продажи Q1 2024: рост на 23% по сравнению с Q4 2023" />

<!-- Декоративное изображение — alt="" скрывает от скринридера -->
<img src="divider.png" alt="" role="presentation" />

<!-- WCAG 1.4.3 (AA): контрастность текста ≥ 4.5:1 -->
<p style="color: #595959; background: #ffffff;">
  Контраст #595959 на белом = 4.48:1 ✓
</p>

<!-- WCAG 2.1.1 (A): доступность с клавиатуры -->
<button onclick="submitForm()">Отправить</button>
<!-- не <div onclick="..."> — div не фокусируемый по умолчанию -->
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Какой уровень WCAG требуется в вашей стране/проекте?** — AA является стандартом де-факто для коммерческого ПО.
- **Чем WCAG 2.1 отличается от 2.0?** — добавлены критерии для мобильных устройств, слабовидящих и когнитивных нарушений (17 новых SC).
- **Как связаны WCAG и ARIA?** — WCAG требует семантики; ARIA — один из способов её добавить там, где нативного HTML недостаточно.
- **Что такое conformance claim?** — формальное заявление о соответствии WCAG с указанием версии, уровня и области охвата.

### Красные флаги (чего не говорить)

- «WCAG — это про цвет и шрифт» — стандарт охватывает навигацию, семантику, медиа, когнитивную нагрузку и многое другое.
- «AAA — это целевой уровень» — в большинстве проектов целевой AA; AAA избыточен и не всегда достижим.
- «Lighthouse 100 = WCAG AA» — Lighthouse проверяет подмножество критериев автоматически; полное соответствие требует ручного аудита.

# Q017. Какие рекомендации существуют для работы с мультимедиа-контентом для обеспечения лучшей доступности?

> **Источник:** [12. accessibility.md](../12.%20accessibility.md) · **Тема:** Accessibility

---

## Короткий ответ

Для видео и аудио доступность обеспечивается через: **субтитры** (captions) для глухих и слабослышащих, **транскрипты** для аудио/видео, **аудиодескрипцию** для незрячих (описание визуального ряда), **управление с клавиатуры** для плеера, и **отключение автовоспроизведения**. WCAG регулирует это в принципе Perceivable: критерии 1.2.x охватывают все типы медиа.

---

## Развёрнутый ответ

### Суть и определение

Мультимедиа создаёт барьеры для разных групп:
- **Глухие и слабослышащие** — не воспринимают аудиодорожку
- **Незрячие** — не видят видеоряд
- **Когнитивные нарушения** — нужен транскрипт для изучения в своём темпе
- **Вестибулярные нарушения** — автоматически движущееся видео вызывает дискомфорт

### Как это работает

#### Типы медиа по WCAG

| Тип | Определение |
|-----|-------------|
| Только аудио (pre-recorded) | Подкаст, аудиозапись |
| Только видео (pre-recorded) | Немое видео, анимация |
| Аудио + видео (pre-recorded) | Стандартное видео |
| Живое аудио/видео | Стрим, вебинар |

#### Требования WCAG 1.2.x

| Критерий | Что требует | Уровень |
|----------|-------------|---------|
| 1.2.1 | Только аудио/видео: транскрипт или аудиодескрипция | A |
| 1.2.2 | Субтитры для предзаписанного видео с аудио | A |
| 1.2.3 | Аудиодескрипция или полный медиа-альтернатив для видео | A |
| 1.2.4 | Субтитры для живого видео | AA |
| 1.2.5 | Аудиодескрипция для предзаписанного видео | AA |
| 1.2.6 | Сурдоперевод (язык жестов) | AAA |
| 1.2.7 | Расширенная аудиодескрипция | AAA |
| 1.2.8 | Медиа-альтернатив для видео | AAA |
| 1.2.9 | Только живое аудио: транскрипт в реальном времени | AAA |

#### 1. Субтитры (Captions)

Синхронизированный текст речи и значимых звуков (музыка, звуковые эффекты, эмоции).

- Форматы: **WebVTT** (предпочтительно для web), SRT, TTML
- Закрытые (CC) — пользователь включает/выключает (рекомендуется)
- Открытые (burned-in) — встроены в видео, нельзя отключить

```html
<video controls>
  <source src="intro.mp4" type="video/mp4" />
  <!-- Субтитры на русском -->
  <track
    kind="subtitles"
    src="intro.ru.vtt"
    srclang="ru"
    label="Русский"
    default
  />
  <!-- Субтитры на английском -->
  <track
    kind="subtitles"
    src="intro.en.vtt"
    srclang="en"
    label="English"
  />
  <!-- Аудиодескрипция -->
  <track
    kind="descriptions"
    src="intro-desc.vtt"
    srclang="ru"
    label="Аудиодескрипция"
  />
</video>
```

#### Пример WebVTT-файла

```vtt
WEBVTT

00:00:01.000 --> 00:00:04.000
Добро пожаловать на курс по доступности.

00:00:04.500 --> 00:00:08.000
[Вдохновляющая музыка]
Сегодня мы изучим WCAG.

00:00:08.500 --> 00:00:12.000
Докладчик поднимает слайд с диаграммой.
```

#### 2. Транскрипты

Полный текстовый вариант аудио или видео. Позволяет:
- Прочитать в своём темпе
- Использовать поиск
- Перевести через сторонние инструменты

```html
<!-- Ссылка на транскрипт рядом с медиа -->
<figure>
  <video controls src="lecture.mp4">
    <track kind="subtitles" src="lecture.vtt" srclang="ru" default />
  </video>
  <figcaption>
    Лекция о WCAG 2.2 ·
    <a href="lecture-transcript.html">Читать транскрипт</a>
  </figcaption>
</figure>
```

#### 3. Аудиодескрипция

Голосовое описание визуального ряда видео — что происходит на экране в паузах между диалогами.

- Нужна когда видеоряд несёт смысл, не передаваемый аудио
- Простая: дополнительная аудиодорожка
- Расширенная (AAA): видео паузируется для длинных описаний

#### 4. Управление плеером с клавиатуры

```html
<!-- Нативный <video controls> уже поддерживает клавиатуру: -->
<!-- Space — пауза/воспроизведение -->
<!-- Стрелки — перемотка -->
<!-- M — мute -->
<!-- F — полный экран -->

<!-- Кастомный плеер: все кнопки должны быть <button> -->
<div role="group" aria-label="Управление видео">
  <button aria-label="Воспроизвести" aria-pressed="false">▶</button>
  <button aria-label="Выключить звук" aria-pressed="false">🔊</button>
  <input
    type="range"
    aria-label="Громкость"
    min="0" max="100" value="80"
    aria-valuetext="80 процентов"
  />
</div>
```

#### 5. Автовоспроизведение

WCAG 1.4.2 (A): если аудио/видео воспроизводится автоматически дольше 3 секунд, должен быть механизм паузы или отключения звука.

```html
<!-- Разрешено: автовоспроизведение без звука (muted) -->
<video autoplay muted loop playsinline aria-hidden="true">
  <source src="bg-animation.mp4" type="video/mp4" />
</video>

<!-- Если видео несёт контент — controls обязателен -->
<video controls autoplay muted>
  <source src="tutorial.mp4" type="video/mp4" />
  <track kind="subtitles" src="tutorial.vtt" srclang="ru" default />
</video>
```

#### 6. Анимированный контент и GIF

WCAG 2.3.1 (A): ничего не должно мигать чаще 3 раз в секунду (риск эпилептических приступов).

```css
/* Уважаем prefers-reduced-motion для анимаций */
@media (prefers-reduced-motion: reduce) {
  .animated-banner {
    animation: none;
  }
  video[autoplay] {
    /* Опционально: остановить автовоспроизводимое фоновое видео */
  }
}
```

### Практика и применение

Минимальный чеклист для продакшн-видео:

```
□ Субтитры (WebVTT) подключены через <track kind="subtitles">
□ Транскрипт доступен рядом с видео (ссылка или раскрываемый блок)
□ Плеер управляется с клавиатуры
□ Нет автовоспроизведения со звуком > 3 сек без элемента паузы
□ Мигающих элементов < 3 раз/сек
□ prefers-reduced-motion учтён для анимаций
□ Аудиодескрипция для видео с важным визуальным рядом (WCAG AA)
```

### Важные нюансы и краеугольные камни

- Автогенерированные субтитры (YouTube, AWS Transcribe) дают ~80% точности — необходимо редактирование перед публикацией.
- `kind="captions"` (для глухих) vs `kind="subtitles"` (перевод) — разные семантические роли; captions включают звуковые эффекты, subtitles — только речь.
- Живые стримы: вебинар без субтитров в реальном времени нарушает WCAG 1.2.4 (AA) — нужны CART (Computer-Assisted Real-Time Translation) или интегрированные субтитры.

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Чем captions отличаются от subtitles?** — captions предназначены для глухих: включают описание звуков и эмоций, не только речь; subtitles — перевод диалогов.
- **Когда нужна аудиодескрипция?** — когда информация в видеоряде не передаётся аудио (демо интерфейса, диаграммы, выражения лица как смысловые элементы).
- **Как реализовать доступный кастомный плеер?** — все кнопки через `<button>`, слайдер через `<input type="range">` с `aria-label`, `aria-valuetext`; обновление состояния через `aria-live`.

### Красные флаги (чего не говорить)

- «YouTube автоматически генерирует субтитры — этого достаточно» — автосубтитры с ошибками нарушают WCAG; нужно редактировать.
- «Видео у нас фоновое, поэтому без субтитров» — декоративное видео должно быть `muted` и `aria-hidden`, без субтитров; но если несёт контент — субтитры обязательны.
- «Транскрипт — необязательно, субтитров хватит» — транскрипт нужен для поиска, изучения в своём темпе, когнитивных нарушений.

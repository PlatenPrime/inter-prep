# Q012. Разница между `role` и `aria-label` в контексте accessibility?

> **Источник:** [12. accessibility.md](../12.%20accessibility.md) · **Тема:** Accessibility

---

## Короткий ответ

`role` определяет **тип** элемента в Accessibility Tree — что это (кнопка, диалог, навигация). `aria-label` задаёт **имя** элемента — как он называется. Скринридер объявляет оба: «Закрыть _(имя)_, кнопка _(роль)_». Это разные оси одной записи в Accessibility Tree: роль отвечает на вопрос «что это?», имя — «как это называется?».

---

## Развёрнутый ответ

### Суть и определение

Каждый элемент в Accessibility Tree описывается несколькими свойствами:

| Свойство | Атрибуты | Вопрос |
|----------|----------|--------|
| **Role** | `role="button"`, нативный тег | Что это за элемент? |
| **Name** | `aria-label`, `aria-labelledby`, `<label>`, содержимое | Как он называется? |
| **State** | `aria-checked`, `aria-expanded`, `aria-disabled` | В каком состоянии? |
| **Value** | `aria-valuenow`, `aria-valuetext` | Каково текущее значение? |

### Как это работает

#### `role`

Определяет семантический тип элемента. Может быть нативным (из HTML-тега) или явным (через атрибут `role`).

```html
<!-- Нативная роль "button" от тега <button> -->
<button>Сохранить</button>

<!-- Явная роль — кастомный виджет -->
<div role="button" tabindex="0">Сохранить</div>

<!-- Роль "navigation" от тега <nav> -->
<nav>...</nav>
<!-- или явно -->
<div role="navigation" aria-label="Основная навигация">...</div>
```

Роль влияет на поведение AT: для `role="button"` NVDA ожидает реакцию на Enter и Space; для `role="checkbox"` — на Space для переключения состояния.

#### `aria-label`

Задаёт **доступное имя (accessible name)** элемента — строку, которую объявит скринридер. Переопределяет текстовое содержимое элемента.

```html
<!-- Кнопка закрытия без видимого текста -->
<button aria-label="Закрыть диалог">✕</button>
<!-- Скринридер: "Закрыть диалог, кнопка" -->
<!-- Без aria-label: "✕, кнопка" — неинформативно -->

<!-- Ссылка "Читать далее" — неоднозначная без контекста -->
<a href="/article-1" aria-label="Читать далее: Введение в WCAG 2.2">
  Читать далее
</a>
```

#### Алгоритм вычисления accessible name (приоритет)

1. `aria-labelledby` (ссылка на другой элемент)
2. `aria-label` (явная строка)
3. Нативная метка (`<label for>`, `<legend>`, `alt`, `title`, `caption`)
4. Текстовое содержимое элемента (для кнопок, ссылок)

### Практика и применение

Ситуации, когда нужен `aria-label`:

- Иконочные кнопки без видимого текста
- Несколько одинаковых ссылок «Читать далее» / «Купить»
- Элементы управления формой без видимого `<label>`
- Landmark-регионы, если их несколько: `<nav aria-label="Хлебные крошки">`

Предпочтение `aria-labelledby` над `aria-label`:

```html
<!-- Лучше: aria-labelledby использует видимый текст -->
<h2 id="cart-heading">Корзина (3 товара)</h2>
<nav aria-labelledby="cart-heading">
  ...
</nav>

<!-- Хуже: дублирует текст в разметке -->
<nav aria-label="Корзина (3 товара)">
  ...
</nav>
```

### Важные нюансы и краеугольные камни

- `aria-label` **переопределяет** видимый текст кнопки: если кнопка содержит «✕» и имеет `aria-label="Закрыть"`, скринридер скажет «Закрыть», а не «✕».
- `aria-label` не отображается визуально — пользователи без AT его не видят.
- Если `aria-labelledby` ссылается на несуществующий `id`, доступное имя берётся из следующего в цепочке.
- `role` без имени — плохая доступность: скринридер объявит «кнопка» без понимания что это за кнопка.
- `aria-label` на элементе без роли (`<div>`, `<span>`) — бесполезен; AT не объявит его, если нет роли.

### Примеры

```html
<!-- role + aria-label вместе: полная запись в AT -->
<button
  role="button"                    <!-- роль: кнопка (избыточно для <button>, но для наглядности) -->
  aria-label="Добавить в избранное" <!-- имя -->
  aria-pressed="false"             <!-- состояние -->
>
  <svg aria-hidden="true" focusable="false">
    <use href="#icon-heart" />
  </svg>
</button>
<!-- AT объявит: "Добавить в избранное, кнопка-переключатель, нажата: нет" -->

<!-- Несколько nav: роль одна, имена разные -->
<nav aria-label="Основная навигация">
  <ul>...</ul>
</nav>
<nav aria-label="Хлебные крошки">
  <ol>...</ol>
</nav>

<!-- aria-labelledby: имя из видимого заголовка -->
<section aria-labelledby="sec-title">
  <h2 id="sec-title">Рекомендованные товары</h2>
  <ul>...</ul>
</section>
<!-- AT: "Рекомендованные товары, регион" -->
```

## Сравнение

| Критерий | `role` | `aria-label` |
|----------|--------|-------------|
| Что передаёт | Тип элемента | Имя элемента |
| Влияет на поведение AT | Да (клавиатура, навигация по типу) | Нет |
| Переопределяет контент | Нет | Да (переопределяет видимый текст для AT) |
| Отображается визуально | Нет | Нет |
| Когда использовать | Кастомные виджеты; конкретизация нативных ролей | Иконочные кнопки, уточнение неоднозначных имён |
| Приоритет в accessible name | — | Выше нативного текста, ниже `aria-labelledby` |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Когда предпочесть `aria-labelledby` над `aria-label`?** — когда видимый текст уже есть на странице; `aria-labelledby` использует его, избегая дублирования.
- **Что произойдёт, если задать и `aria-label`, и `aria-labelledby`?** — `aria-labelledby` выиграет (он выше в цепочке приоритетов).
- **Может ли `role` без имени вызвать проблемы?** — да; например, `<div role="dialog">` без `aria-labelledby` — модалка без имени, AT объявит просто «диалог».

### Красные флаги (чего не говорить)

- «`aria-label` и `role` — одно и то же» — принципиально разные свойства в Accessibility Tree.
- «`aria-label` отображается как tooltip» — нет, это не видимый текст; не путать с `title`.
- «Добавлю `aria-label` везде для надёжности» — избыточный `aria-label` перезаписывает понятный нативный текст и шумит в AT.

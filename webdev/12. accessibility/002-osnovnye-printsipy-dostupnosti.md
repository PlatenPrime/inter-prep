# Q002. Основные принципы доступности?

> **Источник:** [12. accessibility.md](../12.%20accessibility.md) · **Тема:** Accessibility

---

## Короткий ответ

Основу WCAG составляют четыре принципа, известные по аббревиатуре **POUR**: Perceivable (Воспринимаемый), Operable (Управляемый), Understandable (Понятный), Robust (Надёжный). Каждый принцип отвечает на ключевой вопрос: может ли пользователь воспринять контент, взаимодействовать с ним, понять его и использовать со своими вспомогательными технологиями.

---

## Развёрнутый ответ

### Суть и определение

POUR — фундамент WCAG 2.x. Без выполнения хотя бы одного из принципов контент становится недоступным для определённой группы пользователей. Каждый принцип детализируется в Guidelines, а те — в проверяемых Success Criteria.

### Как это работает

#### 1. Perceivable — Воспринимаемый

Информация и компоненты интерфейса должны быть представлены пользователям так, чтобы они могли их воспринять (хотя бы одним органом чувств).

Ключевые практики:
- Текстовые альтернативы для нетекстового контента (`alt`, `aria-label`)
- Субтитры и аудиодескрипция для медиа
- Контент адаптируется без потери смысла (не зависит от цвета, ориентации, CSS)
- Достаточный цветовой контраст (≥ 4.5:1 для текста)

#### 2. Operable — Управляемый

Компоненты интерфейса и навигация должны быть управляемы.

Ключевые практики:
- Весь функционал доступен с клавиатуры (`Tab`, `Enter`, `Space`, стрелки)
- Достаточно времени для взаимодействия (регулируемые таймауты)
- Нет контента, мигающего чаще 3 раз в секунду (риск эпилептических приступов)
- Skip-links, понятные заголовки страниц, видимый фокус

#### 3. Understandable — Понятный

Информация и управление интерфейсом должны быть понятны.

Ключевые практики:
- Язык страницы задан (`<html lang="ru">`)
- Предсказуемое поведение — смена фокуса/контекста только при явном действии пользователя
- Помощь при ошибках ввода: описание ошибки, подсказки, корректировка

#### 4. Robust — Надёжный

Контент должен быть достаточно надёжным для интерпретации широким спектром вспомогательных технологий.

Ключевые практики:
- Валидная, корректно разобранная HTML-разметка
- Имя, роль и значение элементов доступны программно (через DOM/ARIA)
- Статус-сообщения передаются без перевода фокуса (`aria-live`, `role="status"`)

### Практика и применение

```
Perceivable → alt-тексты, субтитры, контраст
Operable    → клавиатура, достаточно времени, skip-links
Understandable → lang, предсказуемость, обработка ошибок
Robust      → валидный HTML, ARIA, совместимость с AT
```

На код-ревью POUR используется как чеклист: при добавлении нового компонента проверяют все четыре оси.

### Важные нюансы и краеугольные камни

- POUR — принципы, не инструкции. Конкретные требования — в Success Criteria.
- **Robust** часто игнорируют: невалидный HTML ломает скринридеры, даже если визуально всё хорошо.
- Принцип **Operable** нарушается при использовании `outline: none` без альтернативного стиля фокуса — распространённая продакшн-ошибка.
- Принцип **Understandable** охватывает и предсказуемость: автоматический переход при `onchange` в `<select>` без предупреждения нарушает WCAG 3.2.2.

### Примеры

```html
<!-- Perceivable: alt для изображения -->
<img src="logo.svg" alt="Логотип компании Acme" />

<!-- Operable: видимый фокус через CSS -->
:focus-visible {
  outline: 3px solid #005fcc;
  outline-offset: 2px;
}

<!-- Understandable: язык страницы + lang для цитаты на другом языке -->
<html lang="ru">
  <blockquote lang="en">Accessible design is good design.</blockquote>
</html>

<!-- Robust: роль и состояние через ARIA -->
<button aria-pressed="true" aria-label="Включить тёмную тему">
  Тёмная тема
</button>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Можете привести нарушение каждого принципа?** — отсутствие alt (P), нет фокуса с клавиатуры (O), неясные сообщения об ошибках (U), невалидный HTML ломает AT (R).
- **Как принцип Robust связан с семантическим HTML?** — нативные HTML-элементы уже предоставляют имя/роль/значение; кастомные компоненты требуют ARIA.
- **Почему outline: none — это нарушение?** — нарушает Operable: пользователи клавиатуры теряют визуальный ориентир фокуса.

### Красные флаги (чего не говорить)

- «Доступность — это только для слепых» — POUR охватывает слух, моторику, когнитивные нарушения, временные ограничения (сломана рука).
- «Достаточно сделать alt-тексты» — это лишь часть Perceivable, а остальные три принципа не затронуты.
- Перечислять принципы без объяснения смысла каждого — интервьюер услышит, что это заучено, а не понято.

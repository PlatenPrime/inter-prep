# Q016. Какие правила следует соблюдать при работе с цветом для обеспечения лучшей доступности?

> **Источник:** [12. accessibility.md](../12.%20accessibility.md) · **Тема:** Accessibility

---

## Короткий ответ

Главных правил три: достаточный **контраст** текста и фона (≥ 4.5:1 для обычного текста, ≥ 3:1 для крупного и UI-элементов), **не передавать информацию только цветом** (дублировать иконкой, текстом, паттерном), и **поддерживать системные настройки** (`prefers-color-scheme`, `prefers-contrast`, Windows High Contrast). Нарушение хотя бы одного из них делает контент недоступным для людей с цветовой слепотой или слабым зрением.

---

## Развёрнутый ответ

### Суть и определение

Около 8% мужчин и 0.5% женщин имеют ту или иную форму цветовой слепоты (дейтеранопия, протанопия, тританопия). Слабое зрение затрагивает ~246 млн человек. Правила работы с цветом устраняют барьеры для этих групп.

### Как это работает

#### 1. Контрастность (WCAG 1.4.3, 1.4.6, 1.4.11)

Контраст измеряется как соотношение относительной яркости (relative luminance) двух цветов.

| Критерий | Соотношение | Уровень |
|----------|-------------|---------|
| Обычный текст (< 18px / < 14px жирный) | ≥ 4.5:1 | AA |
| Крупный текст (≥ 18px / ≥ 14px жирный) | ≥ 3:1 | AA |
| UI-компоненты и графика (границы полей, иконки) | ≥ 3:1 | AA (1.4.11) |
| Обычный текст (улучшенный) | ≥ 7:1 | AAA |
| Крупный текст (улучшенный) | ≥ 4.5:1 | AAA |

Исключения: логотипы, декоративные изображения, неактивные UI-элементы (`disabled`).

```css
/* Проверка: #595959 на белом = 4.48:1 — проходит для крупного (AA), не для мелкого */
/* #333333 на белом = 12.6:1 — проходит AA и AAA */
color: #333333;
background-color: #ffffff;
```

#### 2. Цвет не единственный носитель информации (WCAG 1.4.1)

Нельзя использовать только цвет для передачи смысла: ошибки, статусы, ссылки, графики.

```html
<!-- ✗ Плохо: ошибка только красным цветом -->
<input style="border-color: red;" />
<p style="color: red;">Поле обязательно</p>

<!-- ✓ Хорошо: иконка + цвет + текст -->
<input
  style="border-color: #cc0000; border-width: 2px;"
  aria-invalid="true"
  aria-describedby="err"
/>
<p id="err" style="color: #cc0000;">
  <span aria-hidden="true">⚠ </span>
  Поле обязательно для заполнения
</p>

<!-- ✗ Плохо: ссылка отличается от текста только цветом -->
<p>Прочитайте <a style="color: blue; text-decoration: none;">политику</a></p>

<!-- ✓ Хорошо: подчёркивание или другой визуальный признак -->
<p>Прочитайте <a style="color: blue; text-decoration: underline;">политику</a></p>
```

#### 3. Поддержка системных медиазапросов

```css
/* Тёмный режим — поддерживать контраст в обеих темах */
@media (prefers-color-scheme: dark) {
  body {
    color: #e0e0e0;       /* контраст с тёмным фоном */
    background: #121212;
  }
}

/* Высококонтрастный режим (Windows High Contrast / Forced Colors) */
@media (forced-colors: active) {
  .button {
    border: 2px solid ButtonText; /* использовать системные цвета */
    background-color: ButtonFace;
    color: ButtonText;
  }
}

/* Снижение насыщенности для чувствительных к контрасту пользователей */
@media (prefers-contrast: more) {
  :root {
    --color-text: #000000;
    --color-bg: #ffffff;
    --color-border: #000000;
  }
}
```

#### 4. Цветовые паттерны для графиков

Для графиков и инфографики нельзя полагаться только на цвет:
- Использовать разные паттерны/штриховку дополнительно к цвету
- Добавлять метки прямо на линии/секторы
- Обеспечивать текстовую таблицу данных как альтернативу

### Практика и применение

Инструменты проверки контраста:

- **Chrome DevTools** → Elements → Styles → клик по цвету → Contrast ratio
- **axe DevTools** (расширение) — автоматическая проверка страницы
- **WebAIM Contrast Checker** (webaim.org/resources/contrastchecker)
- **Figma** — плагины Stark, A11y Annotation Kit

Симуляция цветовой слепоты:

- Chrome DevTools → Rendering → Emulate vision deficiencies
- Figma: плагин Stark

### Важные нюансы и краеугольные камни

- **Placeholder текст** в `<input>` должен иметь контраст ≥ 4.5:1, хотя WCAG формально его не покрывает; низкий контраст — распространённая проблема UX.
- `::focus-visible` — стиль фокуса тоже должен соответствовать 1.4.11 (UI-компоненты): цвет обводки vs фон должен давать ≥ 3:1.
- Анимированные цвета (hover, transition) — проверяй оба состояния.
- Тёмная тема не обязана быть инверсией светлой — достаточно соблюдать требования по контрасту в обоих режимах.
- **APCA** (Advanced Perceptual Contrast Algorithm) — новый стандарт контраста в WCAG 3.0; более точен для современных шрифтов, но пока не обязателен.

### Примеры

```css
/* Токены с проверенными контрастами */
:root {
  --text-primary:    #1a1a1a;   /* на #fff = 17.1:1 ✓ */
  --text-secondary:  #595959;   /* на #fff = 4.48:1 ✓ (только крупный текст AA) */
  --text-muted:      #767676;   /* на #fff = 4.54:1 ✓ AA */
  --accent:          #0057b8;   /* на #fff = 5.9:1 ✓ AA */
  --error:           #cc0000;   /* на #fff = 5.9:1 ✓ */
  --border-input:    #767676;   /* UI-элемент: на #fff = 4.54:1 ✓ 1.4.11 */
}

/* Ссылка с дополнительным признаком (не только цвет) */
a {
  color: var(--accent);
  text-decoration: underline;      /* ✓ дополнительный признак */
  text-underline-offset: 3px;
}

a:hover {
  text-decoration-thickness: 2px;  /* усиление при наведении */
}

/* Статус: не только цветом */
.status-error::before {
  content: "✕ ";  /* иконка + цвет */
  color: var(--error);
}
.status-success::before {
  content: "✓ ";
  color: #007a33;  /* зелёный: #007a33 на #fff = 5.1:1 ✓ */
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Как проверить контраст без сторонних инструментов?** — Chrome DevTools: Elements → кликнуть на цвет в Styles → Contrast ratio с подсказкой AA/AAA.
- **Контраст disabled-кнопки: нужен ли?** — нет, WCAG явно исключает неактивные элементы из 1.4.3/1.4.11.
- **Что такое APCA и чем отличается от WCAG 2.x?** — более точный алгоритм, учитывающий размер шрифта, вес, полярность; планируется в WCAG 3.0.

### Красные флаги (чего не говорить)

- «Достаточно чтобы цвет был „примерно" контрастным» — требование точное: ≥ 4.5:1, проверяемо алгоритмически.
- «Тёмный фон автоматически обеспечивает контраст» — тёмный фон + тёмный текст = плохой контраст.
- «Цветовая слепота — редкость, можно не учитывать» — ~8% мужского населения; для массового продукта это десятки тысяч пользователей.

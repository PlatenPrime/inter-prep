# Q009. Как удалить семантику у элемента?

> **Источник:** [12. accessibility.md](../12.%20accessibility.md) · **Тема:** Accessibility

---

## Короткий ответ

Семантику элемента удаляют через `role="presentation"` или его синоним `role="none"`. Эти значения убирают роль элемента из Accessibility Tree, но сохраняют его потомков. Типичный сценарий — `<table>`, используемый для вёрстки (не данных), или `<ul>` для визуального списка, не несущего семантической нагрузки.

---

## Развёрнутый ответ

### Суть и определение

Когда элемент HTML используется только в визуальных целях (а не для передачи структурного смысла), его роль в Accessibility Tree лишняя — скринридер будет объявлять «таблица», «список», «ссылка» там, где это не нужно. `role="none"` / `role="presentation"` убирают эту роль.

### Как это работает

#### `role="presentation"` / `role="none"`

Два синонима (ARIA 1.1+). `role="none"` — предпочтительный в ARIA 1.2.

**Что делает:**
- Удаляет роль самого элемента из AT
- Удаляет обязательные владельца/потомков (например, у `<table>` удаляет роль `table`, что каскадно удаляет `row`, `cell`)
- **Не** удаляет потомков из AT (они остаются со своими ролями)
- **Не** удаляет интерактивные потомки (фокусируемые элементы игнорируют `role="none"`)

```html
<!-- Таблица-раскладка: "presentation" убирает table/row/cell из AT -->
<table role="presentation">
  <tr>
    <td>Колонка 1</td>
    <td>Колонка 2</td>
  </tr>
</table>

<!-- ul для визуальных целей: убирает роль "list" -->
<ul role="none">
  <li>Элемент</li>
</ul>
```

#### Safari + VoiceOver: list-style: none

Отдельный кейс: в Safari VoiceOver убирает роль `list` у `<ul>`, если CSS-свойство `list-style: none` применено к элементу. Это «фича» WebKit — за счёт неё считается, что список декоративный. Чтобы сохранить семантику при визуальном сбросе стилей:

```html
<ul role="list" style="list-style: none; padding: 0;">
  <li>Пункт навигации</li>
</ul>
```

#### Разница с `aria-hidden`

| | `role="none"` | `aria-hidden="true"` |
|---|---|---|
| Сам элемент в AT | Роль удалена (контент остаётся) | Весь элемент удалён |
| Потомки в AT | Остаются | Удалены |
| Фокусируемые потомки | Не затронуты | Остаются в Tab, но не в AT (баг!) |

### Практика и применение

Сценарии применения `role="none"`:

1. **Layout-таблицы** (устаревшая вёрстка): `<table role="presentation">`
2. **Декоративные списки** (горизонтальное меню без семантики): `<ul role="none">`
3. **Враппер-элемент**: `<div role="none">` — обычно это просто не нужно (`<div>` уже не имеет роли по умолчанию)
4. **Изображения без смысла**: `<img alt="" role="presentation">` (хотя `alt=""` само по себе достаточно)

### Важные нюансы и краеугольные камни

- `role="none"` **игнорируется** на интерактивных элементах (`<button>`, `<a href>`, `<input>`): нельзя «разссемантизировать» кнопку через `role="none"`.
- Не используйте `role="none"` для скрытия контента — это не то же самое, что `aria-hidden`.
- `<div>` и `<span>` изначально не имеют роли в AT (`role="generic"`) — `role="none"` на них бессмысленен.
- ARIA 1.2 рекомендует `role="none"` вместо `role="presentation"` — они синонимы, но `none` семантически понятнее.
- Каскадный эффект в таблицах: `role="none"` на `<table>` автоматически применяется к `<thead>`, `<tbody>`, `<tr>`, `<td>`, `<th>` без явного указания на каждом.

### Примеры

```html
<!-- Layout таблица без данных -->
<table role="presentation" style="width: 100%;">
  <tr>
    <td style="width: 200px;">Сайдбар</td>
    <td>Основной контент</td>
  </tr>
</table>
<!-- AT увидит только текст "Сайдбар" и "Основной контент",
     без объявления "таблица, 1 строка, 2 колонки" -->

<!-- Декоративный ul (список тегов без навигационной семантики) -->
<ul role="none" style="list-style: none; display: flex; gap: 8px;">
  <li>JavaScript</li>
  <li>React</li>
  <li>TypeScript</li>
</ul>

<!-- role="none" на кнопке — ИГНОРИРУЕТСЯ браузером -->
<button role="none">Нажми меня</button>
<!-- Скринридер всё равно объявит как "кнопку" -->

<!-- Изображение декоративное — два равнозначных способа -->
<img src="wave.png" alt="" />
<img src="wave.png" alt="" role="presentation" />
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Чем `role="none"` отличается от `aria-hidden="true"`?** — `none` убирает только роль элемента, потомки остаются в AT; `aria-hidden` убирает весь поддерев.
- **Что произойдёт, если поставить `role="none"` на `<a href>`?** — браузер должен игнорировать, т.к. элемент интерактивен; поведение зависит от AT.
- **Зачем `role="list"` на `<ul>` с `list-style: none`?** — из-за баги VoiceOver/Safari, который автоматически удаляет роль list при CSS-сбросе.

### Красные флаги (чего не говорить)

- «`role="none"` скрывает элемент от скринридера» — нет, только убирает роль; контент всё равно читается.
- «Можно разссемантизировать `<button>` через `role="none"`» — интерактивные элементы игнорируют `role="none"`.
- Путать `role="none"` с `role="generic"` — `generic` — это не явная роль для разработчика, а дефолтная роль `<div>`/`<span>`.

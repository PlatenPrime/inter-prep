# Q011. Для чего используется атрибут `aria-roledescription`?

> **Источник:** [12. accessibility.md](../12.%20accessibility.md) · **Тема:** Accessibility

---

## Короткий ответ

`aria-roledescription` позволяет переопределить человекочитаемое **название роли** элемента, которое озвучивает скринридер. Вместо стандартного «кнопка» или «слайдер» AT произнесёт кастомное описание, например «карточка», «слайд», «шаг мастера». Используется редко и только для уникальных виджетов, у которых нет подходящей нативной ARIA-роли.

---

## Развёрнутый ответ

### Суть и определение

Скринридер при фокусировании объявляет элемент в формате: **имя, роль, состояние**. Например: «Следующий слайд, кнопка». Иногда стандартное название роли не отражает суть виджета. `aria-roledescription` заменяет это название на кастомное.

Спецификация: WAI-ARIA 1.1+. Применяется к большинству ролей, но **не работает** на ролях `generic`, `none`, `presentation`.

### Как это работает

```html
<!-- Без aria-roledescription: "Назад, кнопка" -->
<button aria-label="Назад">←</button>

<!-- С aria-roledescription: "Назад, карусельный элемент управления" -->
<button
  aria-label="Назад"
  aria-roledescription="карусельный элемент управления"
>
  ←
</button>
```

Порядок озвучивания зависит от AT, но обычно: **имя → roledescription → состояние**.

#### Правило применения

`aria-roledescription` не заменяет `role`. Он только переопределяет озвучиваемое название роли. Техническая роль в Accessibility Tree остаётся прежней — браузеры и AT используют `role` для поведения, а `aria-roledescription` — только для озвучивания.

```html
<!-- Правильно: role задан, roledescription — кастомное название -->
<section
  role="group"
  aria-roledescription="слайд"
  aria-label="Слайд 1 из 5: Введение"
>
  ...
</section>

<!-- Неправильно: роль не задана, но roledescription есть — AT может проигнорировать -->
<div aria-roledescription="слайд">...</div>
```

### Практика и применение

Типичные сценарии:

1. **Карусели/слайдеры** — элементы слайда: `role="group" aria-roledescription="слайд"`
2. **Мастеры (wizard)** — шаги: `role="group" aria-roledescription="шаг"`
3. **Кастомные карточки** — `role="article" aria-roledescription="карточка товара"`
4. **Drag-and-drop** — `role="listitem" aria-roledescription="перетаскиваемый элемент"`
5. **Канбан-доска** — `role="group" aria-roledescription="колонка"`

Без `aria-roledescription` скринридер объявит «группа» или «статья» — технически верно, но неинформативно для пользователя.

### Важные нюансы и краеугольные камни

- `aria-roledescription` **не переводится автоматически** — значение должно быть локализовано: если страница на русском, атрибут тоже на русском.
- Не злоупотреблять: если стандартная роль понятна пользователю, `aria-roledescription` только шумит.
- Некоторые AT (старые версии JAWS) игнорируют `aria-roledescription` — не единственный источник информации.
- Нельзя использовать на элементах с `role="none"` / `role="presentation"` — они исключены из AT.
- Не путать с `aria-description` (ARIA 1.3, ещё в разработке) — тот добавляет описание, а не переопределяет название роли.

### Примеры

```html
<!-- Карусель: контейнер слайдов -->
<section aria-label="Акции этой недели" aria-roledescription="карусель">
  <button aria-label="Предыдущий слайд">‹</button>

  <div
    role="group"
    aria-roledescription="слайд"
    aria-label="Слайд 1 из 3: Скидка 30% на ноутбуки"
  >
    <img src="promo1.jpg" alt="Ноутбук MacBook Air" />
    <p>Скидка 30% до 1 июня</p>
  </div>

  <div
    role="group"
    aria-roledescription="слайд"
    aria-label="Слайд 2 из 3: Бесплатная доставка"
    hidden
  >
    ...
  </div>

  <button aria-label="Следующий слайд">›</button>
</section>
<!-- VoiceOver объявит: "Акции этой недели, карусель" -->
<!-- При фокусе на слайде: "Слайд 1 из 3: Скидка 30% на ноутбуки, слайд" -->

<!-- Шаг мастера -->
<div
  role="group"
  aria-roledescription="шаг"
  aria-label="Шаг 2 из 4: Контактная информация"
>
  <h2>Контактная информация</h2>
  <!-- поля формы -->
</div>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Заменяет ли `aria-roledescription` атрибут `role`?** — нет, только переопределяет озвучиваемое название; `role` по-прежнему нужен для поведения AT.
- **Как локализовать `aria-roledescription`?** — значение задаётся в разметке или через JS в зависимости от языка пользователя; нет автоматической локализации.
- **Когда НЕ использовать `aria-roledescription`?** — когда нативная роль понятна; слишком частое применение создаёт шум и снижает понятность.

### Красные флаги (чего не говорить)

- «`aria-roledescription` заменяет `role`» — нет, нужны оба.
- «Можно использовать на любом элементе» — не работает на `role="none"`, `role="presentation"`, `role="generic"`.
- «Это то же самое, что `aria-label`» — `aria-label` — имя элемента, `aria-roledescription` — название его роли.

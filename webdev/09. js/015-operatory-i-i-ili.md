# Q015. Операторы «И» и «ИЛИ» (`&&` и `||`)?

> **Источник:** [09. js.md](../09.%20js.md) · **Тема:** JavaScript

---

## Короткий ответ

`&&` (AND) возвращает **первый falsy операнд** или последний, если все truthy. `||` (OR) возвращает **первый truthy операнд** или последний, если все falsy. Оба оператора «ленивы» (short-circuit): правый операнд не вычисляется, если результат уже определён левым. Они возвращают не обязательно `boolean`, а одно из значений.

---

## Развёрнутый ответ

### Суть и определение

Логические операторы в JS работают не как «строгие булевые операторы» — они возвращают **одно из своих операндных значений**, а не только `true`/`false`. Это ключевое отличие от многих других языков.

**Алгоритм `&&`:**
1. Вычислить левый операнд.
2. Если falsy — вернуть его (не вычислять правый).
3. Если truthy — вернуть правый операнд.

**Алгоритм `||`:**
1. Вычислить левый операнд.
2. Если truthy — вернуть его (не вычислять правый).
3. Если falsy — вернуть правый операнд.

### Как это работает

```javascript
// &&: первый falsy или последний
1 && 2        // 2     (оба truthy → последний)
null && 2     // null  (первый falsy)
0 && "hello"  // 0     (первый falsy)
"a" && "b"    // "b"   (оба truthy → последний)

// ||: первый truthy или последний
1 || 2        // 1     (первый truthy)
null || 2     // 2     (первый truthy)
0 || "hello"  // "hello" (первый truthy)
false || 0    // 0     (оба falsy → последний)

// Short-circuit: правый не вычисляется
null && riskyFunction();  // riskyFunction не вызовется
true || riskyFunction();  // riskyFunction не вызовется
```

### Практика и применение

**`&&` — условное выполнение / защитный доступ:**
```javascript
user && user.login();           // вызов только если user truthy
isAdmin && renderAdminPanel();  // условный рендер
```

**`||` — значения по умолчанию (с оговоркой):**
```javascript
const name = inputName || 'Anonymous'; // если inputName falsy
```

**Цепочки:**
```javascript
const city = user && user.address && user.address.city; // до optional chaining
const city = user?.address?.city; // современный эквивалент
```

**Важное отличие `||` от `??` (ES2020):**
```javascript
const count = 0;
count || 10;  // 10  — 0 falsy, используется дефолт (нежелательно)
count ?? 10;  // 0   — только null/undefined запускают дефолт
```

### Важные нюансы и краеугольные камни

- `&&` и `||` не всегда возвращают `boolean` — они возвращают один из операндов. Для boolean результата нужен `!!` или явный тернарный оператор.
- В React `{count && <Component />}` вернёт `0` если `count === 0` — рендерится как текст `"0"`. Безопаснее: `{count > 0 && <Component />}` или `{!!count && <Component />}`.
- **Приоритет:** `&&` выше `||`. `a || b && c` парсится как `a || (b && c)`.
- **Логические присваивания (ES2021):** `a &&= b` (a = a && b), `a ||= b`.

### Примеры

```javascript
// Паттерн защитного вызова
function getUser(id) {
  const user = db.find(id);
  return user && { id: user.id, name: user.name }; // null если не найден
}

// React: правильный условный рендер
function Counter({ count }) {
  return (
    <div>
      {count > 0 && <span>{count} items</span>}
      {/* НЕ: {count && <span>...} — рендерит "0"! */}
    </div>
  );
}

// Логические присваивания ES2021
const settings = { theme: null, lang: 'en' };
settings.theme ||= 'dark';   // theme было null → 'dark'
settings.lang ||= 'ru';      // lang было 'en' → остаётся 'en'
settings.timeout ??= 5000;   // timeout undefined → 5000
```

## Сравнение

| Критерий | `&&` | `\|\|` | `??` |
|----------|------|--------|------|
| Возвращает | Первый falsy / последний | Первый truthy / последний | Правый только если левый `null`/`undefined` |
| Short-circuit | Если левый falsy | Если левый truthy | Если левый не null/undefined |
| Применение | Условный вызов | Дефолт (осторожно с 0/"") | Безопасный дефолт |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Что вернёт `0 && "hello"`?** — `0` (первый falsy).
- **Почему `{count && <Comp/>}` в React — ловушка?** — При `count=0` рендерит строку `"0"`.
- **Приоритет `&&` и `||`?** — `&&` имеет более высокий приоритет; `a || b && c` = `a || (b && c)`.
- **Когда `||` опасен?** — Когда допустимы falsy значения как `0`, `""`, `false` — тогда нужен `??`.

### Красные флаги (чего не говорить)

- «`&&` всегда возвращает `true` или `false`» — возвращает один из операндов.
- «`||` и `??` взаимозаменяемы» — нет, `||` реагирует на все falsy, `??` только на null/undefined.

# Q032. Что такое чистая функция?

> **Источник:** [09. js.md](../09.%20js.md) · **Тема:** JavaScript

---

## Короткий ответ

**Чистая функция (pure function)** — функция, которая: 1) при одинаковых аргументах **всегда возвращает одинаковый результат** (детерминированность) и 2) **не имеет побочных эффектов** (не изменяет внешнее состояние, не работает с I/O). Чистые функции легко тестировать, кэшировать и комбинировать.

---

## Развёрнутый ответ

### Суть и определение

Два обязательных условия чистоты:

**1. Детерминированность (referential transparency):**
```javascript
// Чистая — всегда одинаковый результат для одних аргументов
const add = (a, b) => a + b;
add(2, 3); // 5 — всегда

// Нечистая — зависит от внешнего состояния
let tax = 0.2;
const addTax = (price) => price * (1 + tax); // зависит от tax!
```

**2. Отсутствие побочных эффектов:**
```javascript
// Чистая — не изменяет внешнее состояние
const double = arr => arr.map(x => x * 2); // возвращает новый массив

// Нечистая — мутирует аргумент (побочный эффект!)
const doubleInPlace = arr => {
  arr.forEach((x, i) => arr[i] = x * 2); // мутация!
  return arr;
};
```

### Что считается побочным эффектом

- Изменение внешних переменных / глобального состояния.
- Мутация аргументов.
- I/O операции: HTTP-запросы, запись в файл, console.log.
- Изменение DOM.
- Генерация случайных чисел (`Math.random()`).
- Работа с текущим временем (`Date.now()`).

### Практика и применение

- **Тестирование:** не нужны моки и стабы — один вызов с аргументами.
- **Мемоизация:** можно кэшировать — одинаковые аргументы → одинаковый результат.
- **Параллелизм:** чистые функции безопасны в многопоточных средах (Web Workers).
- **Отладка:** предсказуемое поведение упрощает дебаг.
- **React:** компонент — «почти чистая» функция `(state, props) => UI`.
- **Redux reducers:** обязаны быть чистыми — `(state, action) => newState`.

### Важные нюансы и краеугольные камни

- Чистоту часто описывают как **referential transparency**: вызов функции можно заменить её результатом без изменения программы.
- Не все программы могут быть чисто функциональными — I/O необходимы; цель — **изолировать** нечистоту на «краях» системы.
- Функция, создающая новый объект внутри себя, может быть чистой, если не изменяет внешнее состояние.
- Читаемость: чистая функция явно объявляет все зависимости через параметры — нет скрытых входов.

### Примеры

```javascript
// Нечистая → рефакторинг в чистую

// До: нечистая (мутация + зависимость от внешнего Date)
function processOrder(order) {
  order.processedAt = new Date(); // мутация + Date.now() = нечистота
  orders.push(order);             // побочный эффект
  return order;
}

// После: чистая функция преобразования
function buildProcessedOrder(order, timestamp) {
  return { ...order, processedAt: timestamp }; // новый объект, нет мутации
}
// Нечистая обёртка — только на краях системы
function processOrder(order) {
  const processed = buildProcessedOrder(order, new Date()); // чистая часть
  orders.push(processed); // нечистая часть изолирована
  return processed;
}

// Чистые Redux-reducers
function cartReducer(state = [], action) {
  switch (action.type) {
    case 'ADD_ITEM':
      return [...state, action.payload]; // новый массив — нет мутации
    case 'REMOVE_ITEM':
      return state.filter(item => item.id !== action.payload);
    default:
      return state;
  }
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Может ли чистая функция выбрасывать исключения?** — Технически нет (это побочный эффект), но на практике разные подходы; в FP используют типы Result/Either.
- **Является ли `console.log` побочным эффектом?** — Да, ввод/вывод — побочный эффект.
- **Почему Redux требует чистых reducers?** — Предсказуемость state, time-travel debugging, reproducibility.

### Красные флаги (чего не говорить)

- «Чистая функция — это просто функция без глобальных переменных» — также не должна мутировать аргументы и иметь другие side effects.
- «Практически невозможно писать только чистые функции» — верно, но архитектурная цель — изолировать нечистые части.

# Q002. Разница между примитивом и объектом?

> **Источник:** [09. js.md](../09.%20js.md) · **Тема:** JavaScript

---

## Короткий ответ

Примитив — неизменяемое (immutable) значение, хранящееся **по значению** и не имеющее собственных методов. Объект — изменяемая структура данных, хранящаяся **по ссылке** в куче; передача и сравнение происходят через ссылку, а не через содержимое.

---

## Развёрнутый ответ

### Суть и определение

**Примитивы** (`undefined`, `null`, `boolean`, `number`, `bigint`, `string`, `symbol`):
- Хранятся в стеке (stack) напрямую в переменной.
- Неизменяемы: строку нельзя изменить «на месте», операции создают новую строку.
- Сравниваются **по значению**: `"hello" === "hello"` → `true`.

**Объекты** (`object`, массивы, функции, `Date`, `Map`, …):
- Создаются в куче (heap), переменная хранит **ссылку** (адрес).
- Изменяемы: можно добавлять/удалять свойства.
- Сравниваются **по ссылке**: два разных объекта с одинаковым содержимым не равны.

### Как это работает

```
Стек                  Куча
┌──────────┐          ┌─────────────────┐
│ a = 42   │          │ obj @ 0x1F3A    │
│ b = 42   │          │ { x: 1, y: 2 } │
│ ref → ───┼─────────▶│                │
└──────────┘          └─────────────────┘
```

При присваивании примитива создаётся **копия**. При присваивании объекта копируется **ссылка**, оба имени указывают на один объект в куче.

### Практика и применение

```javascript
// Примитив — копируется по значению
let a = 10;
let b = a;
b = 20;
console.log(a); // 10 — не изменился

// Объект — копируется ссылка
const obj1 = { x: 1 };
const obj2 = obj1;
obj2.x = 99;
console.log(obj1.x); // 99 — изменился оригинал!

// Сравнение
console.log({ x: 1 } === { x: 1 }); // false — разные ссылки
console.log("hi" === "hi");          // true  — одинаковые значения
```

Понимание разницы критично при передаче аргументов в функции, при работе с React state (иммутабельные обновления), при кэшировании и мемоизации.

### Важные нюансы и краеугольные камни

- **Wrapper-объекты:** при вызове метода на строке (`"hello".toUpperCase()`) движок временно «оборачивает» примитив в объект `String`, выполняет метод и уничтожает обёртку. Это называется **autoboxing**.
- Присвоение свойства примитиву ничего не делает (в strict mode бросает TypeError): `"str".foo = 1; console.log("str".foo) // undefined`.
- `const` для объектов запрещает переприсваивание ссылки, но не мутацию содержимого.

### Примеры

```javascript
// Глубокое сравнение объектов вручную
function isEqual(a, b) {
  return JSON.stringify(a) === JSON.stringify(b); // упрощение, не для всех случаев
}
console.log(isEqual({ x: 1 }, { x: 1 })); // true

// Защита от мутации через spread
function updateUser(user, changes) {
  return { ...user, ...changes }; // новый объект, оригинал не изменён
}
```

## Сравнение

| Критерий | Примитив | Объект |
|----------|----------|--------|
| Хранение | По значению (стек) | По ссылке (куча) |
| Изменяемость | Неизменяем | Изменяем |
| Сравнение `===` | По значению | По ссылке |
| Методы | Через autoboxing | Собственные |
| Передача в функцию | Копия значения | Копия ссылки |
| Примеры | `42`, `"hi"`, `true` | `{}`, `[]`, `new Date()` |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Что произойдёт при передаче объекта в функцию и его мутации внутри?** — Изменится оригинал, т.к. передаётся ссылка.
- **Как сделать «настоящую» копию объекта?** — `structuredClone()`, `JSON.parse(JSON.stringify())` (с ограничениями), рекурсивная копия, lodash `cloneDeep`.
- **Почему `const obj = {}` позволяет изменять obj.x?** — `const` фиксирует ссылку, не значение по ссылке.

### Красные флаги (чего не говорить)

- «Объекты всегда медленнее примитивов» — движки оптимизируют через hidden classes (V8), скаляр vs ссылка не является абсолютным критерием производительности.
- «Строки — изменяемые, ведь я могу `str = str + "!"``» — создаётся новая строка, не мутируется существующая.

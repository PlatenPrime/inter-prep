# Q040. Как передаются параметры в функцию: по ссылке или по значению?

> **Источник:** [09. js.md](../09.%20js.md) · **Тема:** JavaScript

---

## Короткий ответ

В JavaScript **всё передаётся по значению**. Для примитивов — копируется само значение. Для объектов — копируется **значение ссылки** (адрес в памяти), а не объект. Это часто называют «pass-by-sharing» или «call-by-sharing». Переприсваивание параметра-объекта внутри функции не меняет внешнюю переменную, но **мутация свойств** объекта — видна снаружи.

---

## Развёрнутый ответ

### Суть и определение

**Примитивы — чистое pass-by-value:**
```javascript
function double(x) {
  x = x * 2; // изменяем локальную копию
}
let n = 5;
double(n);
console.log(n); // 5 — не изменился
```

**Объекты — передаётся копия ссылки:**
```javascript
function addProperty(obj) {
  obj.newProp = 42;     // мутируем объект по ссылке — видно снаружи
}

function reassign(obj) {
  obj = { replaced: true }; // переприсваиваем локальную переменную — снаружи не видно
}

const myObj = { original: true };
addProperty(myObj);
console.log(myObj.newProp); // 42 — мутация видна!

reassign(myObj);
console.log(myObj.replaced); // undefined — переприсваивание НЕ видно
```

### Как это работает

```
Вызов: fn(myObj)

Стек функции:      Куча
┌──────────────┐   ┌─────────────────┐
│ obj → ───────┼──▶│ { original: true│
└──────────────┘   └─────────────────┘
      ↑
   копия ссылки
   (не сам объект)

┌──────────────┐
│ myObj → ─────┼──▶ (тот же объект в куче)
└──────────────┘
```

`obj` и `myObj` указывают на **один и тот же** объект — мутация видна. Но `obj = нечто_новое` — меняет только локальный указатель, не `myObj`.

### Практика и применение

- **Защита от мутации** — копируйте объект перед изменением:
```javascript
function updateUser(user, changes) {
  return { ...user, ...changes }; // новый объект — оригинал не тронут
}
```

- **Намеренная мутация** — паттерн «output parameter» (реже в JS, чаще в C/C++):
```javascript
function populateResult(result) {
  result.data = fetchData();
  result.status = 'ok';
}
const out = {};
populateResult(out);
```

- **React state**: никогда не мутировать state напрямую — всегда новый объект.

### Важные нюансы и краеугольные камни

- Термин «pass-by-reference» в классическом смысле означает, что переприсваивание параметра изменяет внешнюю переменную (как в C++ с `&`). В JS это **невозможно**.
- Массивы — тоже объекты: `push` внутри функции изменит внешний массив.
- `Object.freeze()` предотвращает мутацию, но только верхнего уровня.

### Примеры

```javascript
// Массив — объект, мутация видна
function addItem(arr, item) {
  arr.push(item); // мутация!
}

const items = [1, 2, 3];
addItem(items, 4);
console.log(items); // [1, 2, 3, 4] — изменился!

// Иммутабельный подход
function addItemPure(arr, item) {
  return [...arr, item]; // новый массив
}
const newItems = addItemPure(items, 5);
console.log(items);    // [1, 2, 3, 4] — не изменился
console.log(newItems); // [1, 2, 3, 4, 5]

// TypeScript: readonly для явного запрета мутации
function processArr(arr: readonly number[]): number[] {
  // arr.push(1); // Error: Property 'push' does not exist on type 'readonly number[]'
  return [...arr, 1];
}
```

## Сравнение

| Тип данных | Что передаётся | Мутация видна снаружи? | Переприсваивание видно? |
|------------|----------------|------------------------|------------------------|
| Примитив | Копия значения | Нет | Нет |
| Объект/массив | Копия ссылки | Да | Нет |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Есть ли в JS настоящий pass-by-reference?** — Нет, переприсваивание параметра не влияет на внешнюю переменную.
- **Как защититься от мутации объекта в функции?** — Spread `{...obj}`, `Object.assign({}, obj)`, `structuredClone(obj)`.
- **Что произойдёт с массивом при `arr.sort()` внутри функции?** — `sort()` мутирует исходный массив — будет виден снаружи.

### Красные флаги (чего не говорить)

- «Объекты передаются по ссылке» — технически не совсем: передаётся копия ссылки; настоящий pass-by-ref позволял бы переприсваивание.
- «Переприсваивание объекта внутри функции изменяет оригинал» — нет.

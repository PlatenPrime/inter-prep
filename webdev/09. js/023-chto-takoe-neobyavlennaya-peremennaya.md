# Q023. Что такое необъявленная переменная?

> **Источник:** [09. js.md](../09.%20js.md) · **Тема:** JavaScript

---

## Короткий ответ

**Необъявленная переменная** — переменная, которой присвоено значение без предварительного объявления через `var`, `let` или `const`. В sloppy mode JS тихо создаёт глобальную переменную; в strict mode — немедленно бросает `ReferenceError`. Использование необъявленных переменных — антипаттерн, загрязняющий глобальный namespace.

---

## Развёрнутый ответ

### Суть и определение

```javascript
// Sloppy mode — тихое создание глобальной переменной
function foo() {
  x = 10; // x не объявлена — становится window.x
}
foo();
console.log(window.x); // 10 — загрязнение глобального scope!

// Strict mode — ошибка
"use strict";
function bar() {
  y = 20; // ReferenceError: y is not defined
}
```

### Как это работает

При присваивании `x = 10` движок ищет переменную `x` по scope chain. Если не находит ни в одном scope (включая глобальный) — в sloppy mode создаёт свойство `globalThis.x`. В strict mode такое поведение запрещено.

**Необъявленная ≠ неинициализированная:**
```javascript
let declared;    // объявлена, значение undefined
undeclared;      // не объявлена — ReferenceError при чтении
```

### Практика и применение

- Необъявленные переменные — источник трудноуловимых багов: опечатка в имени становится новой глобальной переменной.
- Строгий режим и TypeScript устраняют эту проблему на уровне компилятора/runtime.
- ESLint правило `no-undef` сообщает об обращении к необъявленным именам.

### Важные нюансы и краеугольные камни

- `typeof undeclaredVar` → `"undefined"` (не ошибка) — исторический exception для `typeof`.
- После `delete window.x` переменная снова необъявлена.
- В Node.js необъявленная переменная становится свойством `global` (не `window`).

### Примеры

```javascript
// Ловушка: опечатка создаёт глобальную переменную
let usersCount = 0;
function increment() {
  userCount++; // опечатка: userCount (не usersCount) создаст NaN в глобальном scope!
}

// Правильно — всегда объявляйте переменные
function incrementSafe() {
  "use strict"; // или используйте модуль
  usersCount++; // ReferenceError при опечатке — сразу видно баг
}

// typeof — единственная безопасная проверка необъявленной переменной
if (typeof maybeUndeclared !== "undefined") {
  // безопасно
}
```

# Q017. Что такое выражения (expression) и инструкции (statement) в JavaScript?

> **Источник:** [09. js.md](../09.%20js.md) · **Тема:** JavaScript

---

## Короткий ответ

**Выражение (expression)** — любой фрагмент кода, который вычисляется и возвращает **значение**: `5 + 3`, `user.name`, `fn()`, `true`. **Инструкция (statement)** — команда, которая **выполняет действие** но не производит значение: `if (...){}`, `for (...){}`, `return`, объявление переменной. Различие критично: в местах, где ожидается значение (аргумент функции, правая часть присваивания), можно использовать только выражения.

---

## Развёрнутый ответ

### Суть и определение

**Expression (выражение):**
- Всегда производит значение.
- Может быть частью другого выражения.
- Примеры: литералы (`42`, `"hi"`), переменные (`x`), вызовы функций (`Math.max(a, b)`), операторы (`a + b`), тернарный оператор (`a ? b : c`), стрелочные функции (`() => 42`).

**Statement (инструкция/оператор):**
- Выполняет действие.
- Не производит значения (или производит, но оно игнорируется в большинстве контекстов).
- Примеры: `if`, `while`, `for`, `try/catch`, `return`, `throw`, `break`, `continue`, объявления (`let x`, `function foo(){}`).

### Как это работает

```javascript
// Выражения — возвращают значение
42                    // числовой литерал
"hello" + " world"    // строковое выражение
isActive ? "yes" : "no" // тернарное выражение
Math.max(1, 2)        // вызов функции

// Инструкции — выполняют действие
let x = 10;           // объявление (declaration statement)
if (x > 5) { }        // условная инструкция
for (let i = 0; i < 3; i++) { } // цикл
return x;             // инструкция возврата
```

**Expression Statement:** выражение, используемое как инструкция:
```javascript
x++;               // expression (инкремент) использован как statement
console.log("hi"); // вызов функции как statement (значение игнорируется)
```

**Function declaration vs Function expression:**
```javascript
// Инструкция (statement) — поднимается (hoisting)
function greet() { return "hi"; }

// Выражение (expression) — не поднимается
const greet = function() { return "hi"; };
const greet2 = () => "hi";
```

### Практика и применение

- **Тернарный оператор vs `if`:** тернарный — выражение (можно в JSX, в аргументах); `if` — инструкция (нельзя использовать там, где ожидается значение).
- **IIFE (Immediately Invoked Function Expression):** оборачивается в `()` именно потому, что `function` в начале строки — statement; `()` превращает его в expression.
- **JSX:** внутри `{}` ожидаются только **выражения**; нельзя вставить `if`, `for` — нужны тернарный оператор или `&&`.

### Важные нюансы и краеугольные камни

- `const result = if (x) { 1 } else { 2 }` — **синтаксическая ошибка**: `if` — statement, не expression.
- Блок `{}` — statement, но `({})` — object expression; именно поэтому стрелочная функция возвращающая объект требует скобок: `() => ({ key: 'val' })`.
- Comma operator `,` объединяет выражения в одно: `(a++, b++)` — вычисляет оба, возвращает последнее.

### Примеры

```javascript
// JSX: только expressions внутри {}
function App({ isAuth }) {
  return (
    <div>
      {/* Правильно — тернарный (expression) */}
      {isAuth ? <Dashboard /> : <Login />}

      {/* Правильно — && (expression) */}
      {isAuth && <AdminPanel />}

      {/* НЕЛЬЗЯ — if (statement) */}
      {/* {if (isAuth) { <Dashboard /> }} — SyntaxError */}
    </div>
  );
}

// IIFE: превращаем function declaration в expression
(function() {
  console.log('IIFE executed');
})();

// Comma operator (редко, но встречается в for)
for (let i = 0, j = 10; i < 5; i++, j--) {
  console.log(i, j);
}
```

## Сравнение

| Критерий | Expression | Statement |
|----------|------------|-----------|
| Производит значение | Да | Нет (или побочно) |
| В правой части `=` | Да | Нет |
| В аргументе функции | Да | Нет |
| В JSX `{}` | Да | Нет |
| Примеры | `a + b`, `fn()`, `a ? b : c` | `if`, `for`, `let x` |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему IIFE оборачивают в `()`?** — Чтобы превратить function declaration (statement) в function expression.
- **Можно ли использовать `if` в JSX?** — Нет, только expressions; альтернативы: тернарный, `&&`, вынесение в функцию.
- **Что такое expression statement?** — Выражение, используемое как отдельная инструкция: `x++`, `fn()`.

### Красные флаги (чего не говорить)

- «Выражения и инструкции — одно и то же» — принципиально разные концепции.
- «`if` можно использовать внутри JSX напрямую» — нет, синтаксическая ошибка.

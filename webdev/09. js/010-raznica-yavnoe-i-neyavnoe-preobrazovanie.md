# Q010. Разница между явным и неявным преобразованием (Implicit and Explicit Coercion)?

> **Источник:** [09. js.md](../09.%20js.md) · **Тема:** JavaScript

---

## Короткий ответ

**Явное (explicit) преобразование** — программист намеренно вызывает конвертацию типа: `Number("5")`, `String(42)`, `Boolean(0)`. **Неявное (implicit) преобразование** (coercion) — движок автоматически приводит типы в контекстах, где ожидается определённый тип: `"5" - 1`, `if (value)`, `"" + 5`. Неявная коэрция — основной источник непредсказуемого поведения JS.

---

## Развёрнутый ответ

### Суть и определение

JavaScript — слабо типизированный язык: большинство операций ожидают определённый тип, и если аргументы его не имеют, движок пытается привести их автоматически по правилам спецификации.

**Три целевых типа при коэрции:**
1. **ToNumber** — при арифметических операциях (`-`, `*`, `/`), унарном `+`.
2. **ToString** — при конкатенации `+` (когда хотя бы один операнд строка), шаблонных строках.
3. **ToBoolean** — в условиях `if`, логических операторах, `?:`.

### Как это работает

**Явное преобразование:**

```javascript
Number("42")     // 42
Number("")       // 0
Number("abc")    // NaN
Number(true)     // 1
Number(false)    // 0
Number(null)     // 0
Number(undefined)// NaN

String(42)       // "42"
String(null)     // "null"
String(undefined)// "undefined"

Boolean(0)       // false
Boolean("")      // false
Boolean(null)    // false
Boolean(undefined)// false
Boolean(NaN)     // false
Boolean({})      // true  ← объект всегда truthy
Boolean([])      // true  ← пустой массив truthy!
```

**Неявное преобразование:**

```javascript
// ToNumber через арифметику
"5" - 2       // 3    ("5" → 5)
"5" * "3"     // 15
true + 1      // 2
null + 1      // 1
undefined + 1 // NaN

// ToString через +
"5" + 2       // "52"  (число → строка)
5 + {}        // "5[object Object]"
[] + {}       // "[object Object]"
{} + []       // 0 (парсер трактует {} как блок!)

// ToBoolean в if
if ([]) { /* выполнится — пустой массив truthy! */ }
if ("") { /* не выполнится — пустая строка falsy */ }
```

### Практика и применение

- **Явное приведение** — всегда предпочтительно в продакшн-коде: намерения очевидны.
- **`+value`** — идиома для быстрой конвертации в число (используется, но менее читаемо, чем `Number(value)`).
- **Falsy/truthy в условиях** — неявная коэрция в `if` — допустима, если вы осознанно проверяете «наличие» значения.
- Линтер правило `no-implicit-coercion` (ESLint) запрещает `!!x`, `+x`, `""+x`.

### Важные нюансы и краеугольные камни

- `[] + []` → `""` (оба массива toString → `""`), `[] + {}` → `"[object Object]"`, `{} + []` → `0` (если `{}` распознаётся как блок).
- `+[]` → `0` (ToNumber пустой строки), `+{}` → `NaN`.
- Объект с `Symbol.toPrimitive` / `valueOf` / `toString` управляет тем, как он будет конвертирован.

### Примеры

```javascript
// Явное — рекомендуемый стиль
const age = Number(inputValue);
if (Number.isNaN(age)) throw new Error('Invalid age');

const label = String(count) + ' items';

// Неявное — ловушки
console.log(1 + "2" + 3);   // "123"  — слева направо: 1+"2"="12", "12"+3="123"
console.log(1 + 2 + "3");   // "33"   — 1+2=3, 3+"3"="33"

// Объект управляет своей коэрцией
const price = {
  value: 100,
  valueOf() { return this.value; },
  toString() { return `$${this.value}`; }
};

console.log(price + 50);    // 150  (valueOf)
console.log(`${price}`);    // "$100" (toString в шаблонах)
```

## Сравнение

| Критерий | Явное (Explicit) | Неявное (Implicit/Coercion) |
|----------|------------------|-----------------------------|
| Инициатор | Разработчик | Движок JS |
| Читаемость | Высокая | Низкая |
| Предсказуемость | Высокая | Низкая |
| Примеры | `Number(x)`, `String(x)` | `x + ""`, `x - 0`, `!!x` |
| Рекомендация | Всегда предпочтительно | Избегать в критическом коде |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Что вернёт `"" + null`?** — `"null"` — null конвертируется в строку `"null"`.
- **Почему `[] == false` → `true`?** — `[]` → `""` (toString), `""` → `0` (ToNumber), `false` → `0` → `0 == 0`.
- **Как управлять коэрцией для своего класса?** — Реализовать `Symbol.toPrimitive`, `valueOf`, `toString`.

### Красные флаги (чего не говорить)

- «Коэрция предсказуема — просто надо знать правила» — правила слишком сложные для надёжного использования в коде.
- «`Boolean([]) === false`» — нет, пустой массив truthy.
- «Явное и неявное — одно и то же, просто синтаксис разный» — принципиально разные подходы.

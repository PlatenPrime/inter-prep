# Q038. Для чего используется атрибут `pattern`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Атрибут `pattern` задаёт регулярное выражение (regex), которому должно соответствовать значение `<input>` для прохождения валидации. Работает с текстоподобными типами: `text`, `tel`, `email`, `url`, `password`, `search`. Если значение не совпадает с паттерном — браузер показывает встроенное сообщение об ошибке; атрибут `title` используется как подсказка с описанием ожидаемого формата.

---

## Развёрнутый ответ

### Суть и определение

`pattern` принимает строку JavaScript-regex без обрамляющих слэшей `/`. Браузер автоматически оборачивает паттерн в `^(?:...)$` — то есть сопоставление происходит со **всей строкой** целиком, а не с подстрокой.

### Как это работает

```html
<input type="text" pattern="[A-Za-z]{3,}" />
```

Браузер интерпретирует как `/^(?:[A-Za-z]{3,})$/`. Значение «ab» не пройдёт, «abc» — пройдёт.

**Алгоритм проверки:**
1. При submit (или при `reportValidity()`) браузер берёт текущее `value` поля
2. Проверяет через встроенный regex-движок против скомпилированного паттерна
3. Если нет совпадения — устанавливает флаг `validity.patternMismatch = true`
4. Показывает тултип: сначала содержимое `title`, если он есть; иначе дефолтное сообщение браузера

### Практика и применение

- Телефонный номер в конкретном формате: `pattern="\+7\d{10}"`
- Пин-код (только цифры): `pattern="\d{4,6}"`
- Логин (латиница + цифры + `_`): `pattern="[a-zA-Z0-9_]{3,20}"`
- Почтовый индекс: `pattern="\d{6}"`
- Серийный номер: `pattern="[A-Z]{2}-\d{4}"`
- URL конкретного формата: `pattern="https://.*"`

### Важные нюансы и краеугольные камни

- `pattern` **не работает** с типами `number`, `date`, `time`, `checkbox`, `radio` — только с текстоподобными
- Если поле пустое и не имеет `required` — `pattern` не проверяется (пустая строка считается валидной)
- Паттерн неявно оборачивается в `^(?:...)$` — не нужно добавлять якоря `^` и `$` самостоятельно
- `title` — это и тултип при hover, и дополнение к сообщению валидации. Хорошая практика: писать человекочитаемое описание формата, а не сам regex
- Флаги regex не поддерживаются — паттерн регистрозависим по умолчанию
- `pattern` проверяет `value` как строку, поэтому для `type="email"` браузер сначала применяет встроенную валидацию email, потом pattern
- Не злоупотреблять: сложные паттерны катастрофичны для UX — лучше комбинировать с `placeholder` и подсказками

### Примеры

```html
<!-- Телефон в российском формате -->
<label for="phone">Телефон</label>
<input
  type="tel"
  id="phone"
  name="phone"
  pattern="\+7\d{10}"
  title="Формат: +7XXXXXXXXXX (11 цифр)"
  placeholder="+79001234567"
  required
/>

<!-- Логин: латиница, цифры, подчёркивание, 3–20 символов -->
<label for="username">Логин</label>
<input
  type="text"
  id="username"
  name="username"
  pattern="[a-zA-Z0-9_]{3,20}"
  title="От 3 до 20 символов: латинские буквы, цифры, подчёркивание"
  required
/>

<!-- Почтовый индекс -->
<input
  type="text"
  name="zip"
  pattern="\d{6}"
  title="Шестизначный почтовый индекс"
  inputmode="numeric"
  maxlength="6"
/>
```

```js
// Программная проверка patternMismatch
const phoneInput = document.getElementById('phone');
phoneInput.addEventListener('input', () => {
  if (phoneInput.validity.patternMismatch) {
    phoneInput.setCustomValidity('Введите номер в формате +7XXXXXXXXXX');
  } else {
    phoneInput.setCustomValidity('');
  }
});
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Нужно ли добавлять `^` и `$` в паттерн? (Нет — браузер добавляет автоматически)
- Что произойдёт если поле пустое, но есть `pattern` без `required`?
- Как поддерживать флаги regex в `pattern`? (Нельзя — флаги не поддерживаются)
- Чем `pattern` отличается от серверной валидации по regex?
- Как сделать `pattern` регистронезависимым?

### Красные флаги (чего не говорить)

- «`pattern` работает со всеми типами input» — не работает с `number`, `date`, `checkbox` и др.
- «`pattern` заменяет серверную валидацию» — клиентская валидация обходится через DevTools
- «Якоря `^` и `$` обязательны в pattern» — они добавляются автоматически

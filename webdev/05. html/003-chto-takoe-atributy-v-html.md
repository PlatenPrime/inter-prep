# Q003. Что такое атрибуты в HTML? Как они используются в элементах?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Атрибуты — это именованные параметры, которые задаются в открывающем теге элемента и уточняют его поведение, внешний вид или мета-информацию. Они записываются в формате `name="value"` через пробел после имени тега. Атрибуты делятся на глобальные (работают на любом элементе) и специфичные для конкретных тегов. Значение атрибута влияет на DOM-свойства, поведение браузера и доступность.

---

## Развёрнутый ответ

### Суть и определение

Атрибут — это дополнительная информация об элементе, указываемая в разметке. Каждый атрибут имеет имя и, как правило, значение. Браузер при парсинге HTML создаёт DOM-узлы, у которых атрибуты становятся доступны через свойства объекта или через `getAttribute` / `setAttribute`.

Важно разграничивать:
- **HTML-атрибут** — то, что написано в разметке (`<input value="foo">`)
- **DOM-свойство** — объектное свойство JS-объекта (`input.value`)

Они не всегда синхронизированы: атрибут `value` хранит *начальное* значение, свойство `value` — *текущее*.

### Как это работает

**Синтаксис атрибутов:**

```html
<!-- Атрибут со значением в двойных кавычках (рекомендуется) -->
<a href="https://example.com">Ссылка</a>

<!-- В одинарных кавычках (допустимо) -->
<a href='https://example.com'>Ссылка</a>

<!-- Без кавычек (работает, если нет пробелов — не рекомендуется) -->
<a href=https://example.com>Ссылка</a>

<!-- Булевый атрибут (значение = имя атрибута или пустая строка) -->
<input type="checkbox" checked />
<button disabled>Недоступна</button>
```

**Типы атрибутов:**

| Тип | Описание | Пример |
|-----|----------|--------|
| Строковые | Произвольная строка | `href`, `src`, `alt`, `class` |
| Перечисляемые | Ограниченный набор значений | `type`, `method`, `rel` |
| Булевые | Наличие = `true`, отсутствие = `false` | `disabled`, `checked`, `required` |
| Числовые | Целое или дробное число | `width`, `height`, `tabindex` |
| URL | Абсолютный или относительный URL | `href`, `src`, `action` |
| `data-*` | Пользовательские данные | `data-user-id="42"` |

**Атрибут vs DOM-свойство:**

```js
const input = document.querySelector('input');

// Атрибут: начальное значение из HTML (всегда строка или null)
input.getAttribute('value'); // "foo"

// Свойство: текущее значение (тип зависит от свойства)
input.value; // текущий текст в поле
```

### Практика и применение

- `src` / `href` — указывают на ресурс; браузер автоматически загружает их (lazy loading через `loading="lazy"` для `<img>`).
- `aria-*` атрибуты — критичны для доступности, сообщают скринридерам роли и состояния элементов.
- `data-*` — де-факто стандарт хранения пользовательских данных в разметке, доступны через `dataset`.
- `rel` на `<a>` и `<link>` — управляет отношением между документами и влияет на SEO (`nofollow`, `noopener`).

### Важные нюансы и краеугольные камни

- Имена атрибутов нечувствительны к регистру в HTML, но чувствительны в XHTML и XML.
- `class` — это многозначный атрибут (список через пробел); в DOM доступен как `classList`.
- Некоторые атрибуты имеют дефолтные значения: `<input type="text">` — тип по умолчанию.
- `style` как атрибут — встроенные стили с наивысшим приоритетом специфичности (кроме `!important`).
- Нестандартные атрибуты без префикса `data-` — плохая практика; они могут конфликтовать с будущими спецификациями.

### Примеры

```html
<!-- Специфичные атрибуты элемента -->
<img
  src="photo.webp"
  alt="Описание для скринридеров"
  width="800"
  height="600"
  loading="lazy"
  decoding="async"
/>

<!-- data-атрибуты и доступ через JS -->
<button data-product-id="123" data-action="add-to-cart">
  В корзину
</button>

<script>
  const btn = document.querySelector('button');
  console.log(btn.dataset.productId); // "123"
  console.log(btn.dataset.action);    // "add-to-cart"
</script>

<!-- ARIA-атрибуты для доступности -->
<button aria-label="Закрыть модальное окно" aria-expanded="false">
  ✕
</button>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **В чём разница между атрибутом и DOM-свойством?** — атрибут в разметке (строка), свойство в JS-объекте (типизированное, отражает текущее состояние).
- **Почему `<input disabled="false">` всё ещё задизейблен?** — булевый атрибут активен при наличии, значение игнорируется.
- **Как получить все атрибуты элемента?** — через `element.attributes` (NamedNodeMap) или `getAttributeNames()`.
- **Зачем `data-*` если можно использовать произвольные атрибуты?** — нестандартные атрибуты невалидны, могут конфликтовать со спецификацией.
- **Как обновить атрибут через JS?** — `setAttribute('name', 'value')`, `removeAttribute('name')`.

### Красные флаги (чего не говорить)

- «Атрибуты и свойства DOM — одно и то же» — это разные концепции с разной синхронизацией.
- «`disabled="false"` отключает атрибут» — для удаления булевого атрибута нужно его убрать или вызвать `removeAttribute`.
- «Можно использовать любые атрибуты — браузер их проигнорирует» — нестандартные атрибуты делают документ невалидным.

# Q032. Для чего используется элемент `<datalist>`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<datalist>` предоставляет список предложений для автодополнения (autocomplete) к полю `<input>`. В отличие от `<select>`, пользователь может ввести произвольное значение, не ограничиваясь предложенными вариантами. Связывание происходит через атрибут `list` на `<input>`, значение которого совпадает с `id` элемента `<datalist>`.

---

## Развёрнутый ответ

### Суть и определение

`<datalist>` содержит набор элементов `<option>`, которые браузер показывает в виде выпадающего списка подсказок при фокусировке или вводе в связанный `<input>`. Это нативный HTML5-механизм реализации typeahead/autocomplete без JavaScript.

### Как это работает

```html
<input list="browsers" name="browser" id="browser" />
<datalist id="browsers">
  <option value="Chrome"></option>
  <option value="Firefox"></option>
  <option value="Safari"></option>
  <option value="Edge"></option>
</datalist>
```

1. Браузер видит `list="browsers"` на `<input>` и ищет `<datalist id="browsers">`
2. При фокусе или вводе символов показывает фильтрованный список `<option>`
3. Пользователь может выбрать вариант или ввести своё значение — оба сценария валидны
4. В `<option>` можно добавить атрибут `label` для отображения описания рядом со значением

### Практика и применение

- Поле выбора города/страны с возможностью ввода нестандартного значения
- Поле тегов/категорий с частыми предложениями
- Поле поиска с историей или популярными запросами
- Форма ввода валюты или единиц измерения с типичными вариантами
- Поле ввода имени с предложениями из адресной книги

### Важные нюансы и краеугольные камни

- `<datalist>` сам по себе не отображается на странице — это скрытый контейнер данных
- Стилизовать выпадающий список `<datalist>` через CSS крайне ограничено — браузеры рендерят его нативно
- Если нужна точная стилизация или сложная логика фильтрации — используют кастомные компоненты на JS (Combobox pattern)
- Работает со многими типами `<input>`: `text`, `search`, `url`, `tel`, `email`, `date`, `month`, `week`, `time`, `datetime-local`, `number`, `range`, `color`
- Поддержка браузерами хорошая (IE10+), но поведение UI незначительно различается
- `<option>` без атрибута `label` показывает `value`; с `label` — показывает `label`, но в поле подставляет `value`

### Примеры

```html
<!-- Базовый пример с городами -->
<label for="city">Город</label>
<input type="text" id="city" name="city" list="cities-list" placeholder="Введите город..." />
<datalist id="cities-list">
  <option value="Москва"></option>
  <option value="Санкт-Петербург"></option>
  <option value="Новосибирск"></option>
  <option value="Екатеринбург"></option>
</datalist>

<!-- С label для описания значений -->
<input type="text" id="lang" name="lang" list="languages" />
<datalist id="languages">
  <option value="js" label="JavaScript"></option>
  <option value="ts" label="TypeScript"></option>
  <option value="py" label="Python"></option>
</datalist>
```

## Сравнение (datalist vs select)

| Критерий | `<datalist>` | `<select>` |
|----------|-------------|------------|
| Произвольный ввод | Да | Нет |
| Стилизация | Ограничена (нативный UI) | Ограничена, но гибче |
| Обязательный выбор из списка | Нет | Да |
| Фильтрация по вводу | Да (автоматически) | Нет (нативно) |
| Множественный выбор | Нет | Да (`multiple`) |
| Подходит для | Подсказки / typeahead | Строгий выбор из вариантов |

### Красные флаги (чего не говорить)

- «`<datalist>` — это то же самое что `<select>`» — принципиальное отличие в возможности свободного ввода
- «`<datalist>` всегда отображается на странице» — нет, он скрыт и служит источником данных
- «Можно полностью стилизовать через CSS» — нельзя, рендеринг нативный

# Q031. Для чего используется тэг `<label>`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<label>` связывает текстовую метку с элементом управления формы, увеличивая область клика до размера самого лейбла. Связь устанавливается либо через атрибут `for`, значение которого совпадает с `id` инпута, либо путём оборачивания инпута внутрь `<label>`. Тег критически важен для доступности (accessibility): скринридеры озвучивают метку при фокусировке на поле.

---

## Развёрнутый ответ

### Суть и определение

`<label>` — это семантический элемент, который программно связывает текстовое описание с конкретным элементом формы (`<input>`, `<textarea>`, `<select>` и др.). Именно через эту связь браузер и assistive technologies понимают, какое поле что означает.

### Как это работает

Существует два способа связывания:

**Явная связь** — через атрибут `for`, который ссылается на `id` поля:
```html
<label for="email">Email</label>
<input type="email" id="email" name="email" />
```

**Неявная связь** — через оборачивание элемента управления:
```html
<label>
  Email
  <input type="email" name="email" />
</label>
```

При клике на лейбл браузер переводит фокус на связанное поле. Для чекбоксов и радио-кнопок клик по лейблу также меняет их состояние.

### Практика и применение

- Обязательное использование в формах авторизации, регистрации, фильтров, настроек — везде, где есть поля ввода
- Особенно критично для чекбоксов и радио-кнопок: без лейбла кликабельная область крошечная
- При скрытии лейбла визуально для компактного дизайна используют `visually-hidden` CSS-класс, а не `display: none` — иначе скринридеры его не увидят
- В React/Angular компонентах лейбл всегда идёт в паре с `id` поля

### Важные нюансы и краеугольные камни

- `for` связывается именно по `id`, не по `name` — частая путаница
- На одной странице `id` должен быть уникален: дублирование ломает связь
- Если `<label>` обёрнут вокруг инпута, атрибут `for` не обязателен, но его наличие не вредит
- `<label>` нельзя вкладывать друг в друга
- Элементы `<button>` и `type="submit"` не требуют `<label>` — у них есть собственный текстовый контент
- Скрытый лейбл через `aria-label` или `aria-labelledby` является полноценной альтернативой

### Примеры

```html
<!-- Явная связь через for/id -->
<div>
  <label for="username">Имя пользователя</label>
  <input type="text" id="username" name="username" required />
</div>

<!-- Чекбокс с лейблом — большая область клика -->
<label>
  <input type="checkbox" name="agree" value="yes" />
  Я принимаю условия использования
</label>

<!-- Визуально скрытый лейбл для поля поиска -->
<label for="search" class="visually-hidden">Поиск</label>
<input type="search" id="search" placeholder="Поиск..." />
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Чем отличаются `for`/`id` и вложенный вариант — есть ли разница в поведении?
- Что произойдёт, если `for` указывает на несуществующий `id`?
- Как связать один `<label>` с несколькими полями (нельзя — и почему)?
- Как сделать лейбл для `<input type="hidden">`? (Не нужно — скрытое поле не взаимодействует с пользователем)
- Как доступность (a11y) страдает без `<label>`?
- Чем `aria-label` отличается от `<label>`?

### Красные флаги (чего не говорить)

- «`<label>` нужен только для стилизации» — нет, это семантический и a11y-элемент
- «Можно использовать `placeholder` вместо `<label>`» — placeholder исчезает при вводе и не читается скринридерами как метка
- «`for` ссылается на `name` поля» — ссылается на `id`

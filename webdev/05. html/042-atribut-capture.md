# Q047. Для чего используется атрибут `capture`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`capture` — атрибут для `<input type="file">`, указывающий браузеру предпочтительный источник медиафайла при захвате. Значение `user` открывает фронтальную камеру/микрофон, `environment` — заднюю камеру. Работает преимущественно на мобильных устройствах и позволяет сразу снимать фото/видео вместо выбора из галереи.

---

## Развёрнутый ответ

### Суть и определение

Атрибут `capture` является частью спецификации [Media Capture and Streams API](https://www.w3.org/TR/mediacapture-streams/). Он сообщает браузеру, что следует использовать устройство захвата медиа (камеру, микрофон) напрямую, а не файловый менеджер. На десктопе обычно игнорируется.

### Как это работает

```
capture="user"        → фронтальная камера / основной микрофон
capture="environment" → задняя камера
capture=""            → любой доступный источник (поведение по умолчанию)
```

Браузер на мобильном устройстве при наличии атрибута пропускает выбор файла и сразу открывает камеру. Атрибут `accept` уточняет тип медиа:

| `accept`        | Поведение                              |
|-----------------|----------------------------------------|
| `image/*`       | Открывает камеру для фото              |
| `video/*`       | Открывает камеру для видео             |
| `audio/*`       | Открывает микрофон для записи звука    |

### Практика и применение

- **Загрузка аватара**: мгновенное фото фронтальной камерой
- **Сканирование документов**: задняя камера для фото паспорта, чека
- **Голосовые заметки**: захват аудио прямо из формы
- **Видеоотчёты**: запись видео без выхода из браузера

```html
<!-- Фото профиля — фронтальная камера -->
<input type="file" accept="image/*" capture="user" />

<!-- Скан документа — задняя камера -->
<input type="file" accept="image/*" capture="environment" />

<!-- Запись видео -->
<input type="file" accept="video/*" capture="environment" />

<!-- Запись аудио -->
<input type="file" accept="audio/*" capture="user" />
```

### Важные нюансы и краеугольные камни

- На **iOS Safari** значение `capture` часто игнорируется — браузер сам решает, показать ли выбор из галереи или камеру
- На **десктопных браузерах** атрибут не работает — открывается стандартный диалог файлов
- `capture` без `accept` может открыть неожиданный интерфейс — всегда указывайте `accept`
- Наличие атрибута (даже пустого) активирует режим захвата; отсутствие — файловый менеджер с возможностью сделать фото
- Не забывайте про UX: пользователь может предпочесть выбрать фото из галереи — рассмотрите два отдельных button/input

### Примеры

```html
<!-- Полный пример формы с выбором источника -->
<form enctype="multipart/form-data" method="post">
  <label>
    Сфотографировать чек:
    <input
      type="file"
      name="receipt"
      accept="image/*"
      capture="environment"
    />
  </label>

  <label>
    Загрузить из галереи:
    <input type="file" name="gallery-photo" accept="image/*" />
  </label>

  <button type="submit">Отправить</button>
</form>
```

### Красные флаги (чего не говорить)

- «`capture` работает на всех платформах одинаково» — на iOS поведение отличается
- «`capture` заменяет getUserMedia» — это разные API; `capture` только для `<input type="file">`
- «Значение `capture` всегда обязательно» — атрибут можно использовать без значения

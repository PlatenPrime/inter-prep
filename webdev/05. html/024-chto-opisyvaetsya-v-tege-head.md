# Q024. Что описывается в тэге `<head>`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<head>` содержит метаданные документа — информацию о странице, которая не отображается пользователю напрямую. Это кодировка, заголовок вкладки, подключение стилей и скриптов, SEO-данные, иконки и настройки для браузера. Содержимое `<head>` обрабатывается браузером до рендеринга страницы и влияет на производительность, SEO и поведение документа.

---

## Развёрнутый ответ

### Суть и определение

`<head>` — обязательный элемент HTML-документа, дочерний для `<html>`, идущий до `<body>`. Всё его содержимое — инструкции для браузера, роботов, соцсетей. Пользователь видит только результат обработки (заголовок вкладки, иконку, стили).

### Как это работает

Браузер парсит `<head>` линейно сверху вниз до начала рендеринга `<body>`. Порядок элементов важен: `charset` и `viewport` должны быть как можно раньше. Блокирующие ресурсы (CSS, синхронные скрипты) задерживают First Contentful Paint.

### Практика и применение

**Полный состав `<head>` с пояснениями:**

```html
<head>
  <!-- 1. Кодировка — ПЕРВОЙ, до любого текста -->
  <meta charset="UTF-8">

  <!-- 2. Viewport — сразу после charset для мобильных -->
  <meta name="viewport" content="width=device-width, initial-scale=1">

  <!-- 3. Заголовок документа — единственный обязательный элемент head -->
  <title>Название страницы | Сайт</title>

  <!-- 4. Мета-описание для SEO -->
  <meta name="description" content="Описание страницы до 160 символов">
  <meta name="robots" content="index, follow">

  <!-- 5. Иконки -->
  <link rel="icon" href="/favicon.ico">
  <link rel="apple-touch-icon" href="/apple-touch-icon.png">

  <!-- 6. Подключение стилей (не блокирует парсинг HTML) -->
  <link rel="stylesheet" href="/styles/main.css">

  <!-- 7. Preconnect / preload для производительности -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preload" href="/fonts/main.woff2" as="font" crossorigin>

  <!-- 8. Open Graph для соцсетей -->
  <meta property="og:title" content="Название">
  <meta property="og:image" content="https://example.com/og.jpg">

  <!-- 9. Базовый URL для относительных ссылок -->
  <base href="https://example.com/">

  <!-- 10. Инлайновые стили (крайний случай, critical CSS) -->
  <style>body { margin: 0; }</style>

  <!-- 11. Скрипты — желательно внизу body или с defer/async -->
  <script src="/app.js" defer></script>

  <!-- 12. noscript — для браузеров без JS -->
  <noscript><link rel="stylesheet" href="/no-js.css"></noscript>
</head>
```

**Допустимые элементы в `<head>`:**

| Элемент | Назначение |
|---------|------------|
| `<title>` | Заголовок вкладки и результата поиска (обязателен) |
| `<meta>` | Метаданные: кодировка, описание, viewport, og:* |
| `<link>` | Связь с ресурсами: CSS, иконки, preload, canonical |
| `<script>` | JS (лучше с `defer`/`async` или в конце `<body>`) |
| `<style>` | Инлайновые CSS-правила |
| `<base>` | Базовый URL для относительных ссылок (не более одного) |
| `<noscript>` | Фолбэк при отключённом JS |

### Важные нюансы и краеугольные камни

- `<title>` — единственный **обязательный** элемент `<head>` по спецификации
- `<base>` может быть только один на страницу; влияет на все относительные ссылки
- `<script>` без `defer`/`async` блокирует парсинг HTML — антипаттерн в `<head>`
- Порядок `<link rel="stylesheet">` влияет на cascade: поздние правила перекрывают ранние
- Элементы вне разрешённого списка в `<head>` браузер перемещает в `<body>` (неявное исправление ошибок)

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Какой единственный обязательный элемент `<head>`?
- Почему `<script>` лучше не ставить в `<head>` без `defer`?
- Что произойдёт, если поставить `<div>` внутрь `<head>`?
- Чем `<link rel="preload">` отличается от `<link rel="prefetch">`?
- Когда стоит использовать `<base href>`?

### Красные флаги (чего не говорить)

- «В `<head>` можно ставить любые теги» — нет, только определённый набор
- «Порядок элементов в `<head>` не важен» — важен, особенно для charset и viewport
- «`<script>` в `<head>` — нормальная практика» — без defer/async это блокировка рендеринга

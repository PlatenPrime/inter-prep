# Q058. Какие HTML-элементы используются для создания и форматирования таблиц? Какие атрибуты у них есть?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Таблица строится из элементов `<table>`, `<thead>`, `<tbody>`, `<tfoot>`, `<tr>`, `<th>`, `<td>`. Дополнительно: `<caption>` для заголовка, `<colgroup>`/`<col>` для стилизации колонок. Ключевые атрибуты ячеек: `colspan`, `rowspan`, `scope`, `headers`. `<caption>` критичен для accessibility — даёт таблице описание для screen readers.

---

## Развёрнутый ответ

### Суть и определение

HTML-таблица — структурированный способ представления двумерных данных. Каждый элемент имеет чёткую семантическую роль: `<thead>` отделяет заголовок от данных, `<tfoot>` — итоговые строки, `<caption>` описывает таблицу целиком.

### Как это работает

**Структурные элементы (по порядку вложенности):**

| Элемент | Роль |
|---|---|
| `<table>` | Корневой элемент таблицы |
| `<caption>` | Заголовок/описание таблицы (первый дочерний `<table>`) |
| `<colgroup>` | Группа колонок для стилизации |
| `<col>` | Одна колонка внутри `<colgroup>` |
| `<thead>` | Секция заголовков строк |
| `<tbody>` | Секция с данными (может быть несколько) |
| `<tfoot>` | Секция итогов (браузер рендерит после `<tbody>`) |
| `<tr>` | Строка таблицы |
| `<th>` | Заголовочная ячейка |
| `<td>` | Ячейка с данными |

**Атрибуты ячеек `<th>` и `<td>`:**

| Атрибут | Значения | Описание |
|---|---|---|
| `colspan` | число | Ячейка занимает N колонок |
| `rowspan` | число | Ячейка занимает N строк |
| `scope` | `col`, `row`, `colgroup`, `rowgroup` | Область заголовка (только `<th>`) |
| `headers` | ID список | Явная привязка к `<th>` (сложные таблицы) |

**Атрибуты `<col>` и `<colgroup>`:**

| Атрибут | Описание |
|---|---|
| `span` | Количество колонок, к которым применяется стиль |

**Атрибут `<table>`:**
- `summary` — устарел в HTML5, заменён `<caption>` и `aria-describedby`.

### Практика и применение

- `<thead>` / `<tbody>` / `<tfoot>` — браузер позволяет прокручивать только `<tbody>` при `display: block` + overflow; также `<tfoot>` можно печатать как повторяющийся нижний колонтитул при `@media print`.
- `<colgroup>` + `<col>` — единственный способ стилизовать целую колонку (ограниченная поддержка CSS: только `background`, `border`, `visibility`, `width`).
- `<caption>` — позиционируется через `caption-side: top | bottom`.
- Несколько `<tbody>` — позволяют группировать строки логически (например, по категориям).

### Важные нюансы и краеугольные камни

- **Неявное создание `<tbody>`:** если не указан `<tbody>`, браузер создаёт его автоматически — это может ломать JS-код, обходящий `table.children` напрямую.
- **`<tfoot>` в источнике перед `<tbody>`:** HTML5 допускает `<tfoot>` до `<tbody>` в разметке, браузер всё равно рендерит его последним.
- **`<col>` и CSS:** нельзя применить произвольный CSS к `<col>` — только background, border, width, visibility.
- **Доступность без `<caption>`:** добавьте `aria-label` или `aria-labelledby` на `<table>` если `<caption>` скрыт.
- **`display: contents` на `<tbody>`:** «разрушает» семантику таблицы для AT.

### Примеры

```html
<table aria-describedby="sales-note">
  <caption>Продажи по регионам — Q1 2026</caption>

  <!-- Стилизация колонок -->
  <colgroup>
    <col style="width: 40%">
    <col span="3" style="width: 20%">
  </colgroup>

  <thead>
    <tr>
      <th scope="col">Регион</th>
      <th scope="col">Январь</th>
      <th scope="col">Февраль</th>
      <th scope="col">Март</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <th scope="row">Москва</th>
      <td>1 200 000</td>
      <td>980 000</td>
      <td>1 450 000</td>
    </tr>
    <tr>
      <th scope="row">СПб</th>
      <td>800 000</td>
      <td>750 000</td>
      <td>900 000</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <th scope="row">Итого</th>
      <td>2 000 000</td>
      <td>1 730 000</td>
      <td>2 350 000</td>
    </tr>
  </tfoot>
</table>

<p id="sales-note">Данные в рублях, включая НДС.</p>

<!-- Сложная таблица: colspan + rowspan -->
<table>
  <caption>Расписание</caption>
  <thead>
    <tr>
      <th scope="col">Время</th>
      <th scope="col">Понедельник</th>
      <th scope="col">Вторник</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">9:00–10:00</th>
      <td rowspan="2">Стенд-ап (2 часа)</td>
      <td>Планирование</td>
    </tr>
    <tr>
      <th scope="row">10:00–11:00</th>
      <!-- пусто: занято rowspan выше -->
      <td>Код-ревью</td>
    </tr>
  </tbody>
</table>
```

- Как правильно семантически обозначить ячейку-заголовок строки?

### Красные флаги (чего не говорить)

- «`<thead>` и `<tbody>` — необязательные декоративные обёртки» — они несут семантику секций и влияют на поведение при печати.
- «`<caption>` — то же самое что `<p>` перед таблицей» — `<caption>` семантически привязан к таблице и объявляется screen readers.
- «Для стилизации колонок можно использовать любой CSS на `<col>`» — только ограниченный набор свойств.

# Q033. Как можно сгруппировать опции внутри тэга `<select>`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Опции внутри `<select>` группируются с помощью элемента `<optgroup>`, который принимает обязательный атрибут `label` — текст заголовка группы. Сам `<optgroup>` не является выбираемым элементом, он лишь визуально и семантически организует связанные опции. Типичный пример: список городов, сгруппированных по странам или регионам.

---

## Развёрнутый ответ

### Суть и определение

`<optgroup>` — контейнерный элемент внутри `<select>`, создающий визуальную и семантическую группировку `<option>`. Браузер отображает `label` группы как невыбираемый заголовок со специальным отступом и стилем.

### Как это работает

```html
<select name="city" id="city">
  <optgroup label="Россия">
    <option value="msk">Москва</option>
    <option value="spb">Санкт-Петербург</option>
  </optgroup>
  <optgroup label="Германия">
    <option value="ber">Берлин</option>
    <option value="mun">Мюнхен</option>
  </optgroup>
</select>
```

Браузер отрисовывает заголовки групп особым образом — обычно серым жирным текстом с отступом, а опции внутри группы — с дополнительным отступом. Клик на заголовок группы ничего не делает.

### Практика и применение

- Списки географических локаций: города по странам/регионам
- Списки продуктов по категориям в интернет-магазине
- Списки шрифтов по семействам (serif, sans-serif, monospace)
- Списки временных зон, сгруппированных по континентам
- Выпадающий список настроек, разбитый по разделам

### Важные нюансы и краеугольные камни

- `<optgroup>` нельзя выбрать — попытка установить его значение программно не сработает
- Атрибут `label` у `<optgroup>` обязателен (без него поведение не определено стандартом)
- `<optgroup>` нельзя вкладывать друг в друга — поддерживается только один уровень группировки
- Атрибут `disabled` на `<optgroup>` блокирует все дочерние `<option>`
- Стилизация `<optgroup>` и `<option>` через CSS крайне ограничена в большинстве браузеров
- Для сложной группировки с кастомным дизайном используют JavaScript-компоненты (Select, Combobox из библиотек UI)

### Примеры

```html
<!-- Список временных зон -->
<label for="timezone">Часовой пояс</label>
<select id="timezone" name="timezone">
  <option value="">-- Выберите --</option>
  <optgroup label="Европа">
    <option value="Europe/Moscow">Москва (UTC+3)</option>
    <option value="Europe/Berlin">Берлин (UTC+1)</option>
    <option value="Europe/London">Лондон (UTC+0)</option>
  </optgroup>
  <optgroup label="Азия">
    <option value="Asia/Tokyo">Токио (UTC+9)</option>
    <option value="Asia/Shanghai">Шанхай (UTC+8)</option>
  </optgroup>
</select>

<!-- Отключённая группа -->
<select name="plan">
  <optgroup label="Доступные тарифы">
    <option value="free">Бесплатный</option>
    <option value="pro">Pro</option>
  </optgroup>
  <optgroup label="Недоступно" disabled>
    <option value="enterprise">Enterprise</option>
  </optgroup>
</select>
```

### Каверзные follow-up

- Можно ли вложить `<optgroup>` внутрь `<optgroup>`? (Нет, HTML не поддерживает)

### Красные флаги (чего не говорить)

- «`<optgroup>` можно выбрать как значение» — нельзя, это только визуальный заголовок
- «Можно вложить `<optgroup>` для многоуровневого меню» — HTML-стандарт это не поддерживает
- «Атрибут `label` необязателен» — он обязателен по спецификации

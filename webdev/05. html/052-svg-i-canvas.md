# Q052. Что такое `<svg>` и `<canvas>`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<svg>` (Scalable Vector Graphics) — это XML-формат векторной графики, встроенный в DOM: каждая фигура является DOM-узлом, стилизуется через CSS и реагирует на события. `<canvas>` — это пиксельный растровый буфер, которым управляет JavaScript: нарисованные объекты не хранятся в DOM и не масштабируются без потерь. Оба элемента решают задачи графики, но принципиально по-разному — SVG декларативен, canvas — процедурен.

---

## Развёрнутый ответ

### Суть и определение

**SVG** (Scalable Vector Graphics) — это XML-словарь для описания двумерной векторной графики. Браузер разбирает SVG-разметку и добавляет её в дерево рендеринга так же, как обычный HTML. Фигуры (`<rect>`, `<circle>`, `<path>`, ...) существуют как DOM-узлы.

**Canvas** — HTML-элемент, предоставляющий низкоуровневый растровый буфер (bitmap). Содержимое формируется императивными вызовами JavaScript API (`fillRect`, `drawImage`, `arc`). После растеризации информация о «сцене» теряется — canvas не знает, какие объекты на нём нарисованы.

### Как это работает

**SVG:**
1. Браузер парсит XML → строит SVG DOM (параллельно с HTML DOM).
2. Layout-движок вычисляет размеры и позиции фигур.
3. Рендерер рисует векторные примитивы — при масштабировании пересчитывает кривые.
4. Изменения через JS/CSS автоматически вызывают repaint нужного узла.

**Canvas:**
1. Браузер резервирует пиксельный буфер заданного размера.
2. JS вызывает команды API → пиксели записываются немедленно.
3. Для анимации нужен явный цикл (`requestAnimationFrame` + `clearRect` + перерисовка).
4. Масштабирование через CSS растягивает уже растеризованный буфер.

### Практика и применение

**SVG используют для:**
- Иконок и логотипов (масштабируются под любой DPI).
- Интерактивных диаграмм (D3.js в SVG-режиме).
- Инфографики и иллюстраций с hover-эффектами.
- Анимаций интерфейса (CSS-анимации на SVG-элементах).

**Canvas используют для:**
- Браузерных игр и интерактивных симуляций.
- Визуализации больших объёмов данных в реальном времени.
- Обработки изображений (фильтры, кроп, watermark).
- WebGL-рендеринга (3D-графика).

### Важные нюансы и краеугольные камни

- **Доступность:** SVG-элементы можно снабдить `<title>`, `<desc>`, `role` — screen readers их видят. Canvas требует внешней a11y-разметки.
- **Производительность:** При сотнях DOM-узлов SVG замедляется из-за layout/repaint; canvas справляется с тысячами объектов, так как не обновляет DOM.
- **Стилизация:** SVG наследует CSS переменные и поддерживает `:hover` на отдельных фигурах. На canvas нельзя навесить CSS-псевдоклассы.
- **Сериализация:** SVG можно сохранить как текст/XML, встроить в HTML или использовать как `src` для `<img>`. Canvas экспортируется только в растровый формат (`toDataURL`, `toBlob`).

### Примеры

```html
<!-- SVG: векторный круг, реагирует на hover через CSS -->
<svg width="100" height="100" viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <title>Логотип компании</title>
  <circle cx="50" cy="50" r="40" fill="#4f46e5" class="logo-circle" />
</svg>

<style>
  .logo-circle:hover { fill: #7c3aed; }
</style>
```

```html
<!-- Canvas: то же самое, но через JS -->
<canvas id="c" width="100" height="100" aria-label="Логотип компании" role="img"></canvas>

<script>
  const ctx = document.getElementById('c').getContext('2d');
  ctx.beginPath();
  ctx.arc(50, 50, 40, 0, Math.PI * 2);
  ctx.fillStyle = '#4f46e5';
  ctx.fill();
</script>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Можно ли встроить `<canvas>` внутрь `<svg>` и наоборот? (SVG в canvas — да, через `drawImage`; canvas в SVG — через `<foreignObject>`)
- Что такое `viewBox` в SVG и как он связан с масштабированием?
- Почему у canvas два разных понятия размера (`width`/`height` атрибуты vs CSS)?
- Как реализовать клик по конкретному объекту на canvas без DOM? (математический hit-test или offscreen canvas)
- Можно ли анимировать SVG без JS? (да, через CSS-анимации и SMIL)

### Красные флаги (чего не говорить)

- «SVG и canvas — одно и то же, просто разный синтаксис» — принципиально разные модели рендеринга.
- «Canvas лучше SVG во всех случаях» — зависит от задачи; для иконок и UI-графики SVG предпочтительнее.
- «SVG не поддерживает JavaScript-события» — поддерживает; можно навешивать listeners на любой SVG-элемент.

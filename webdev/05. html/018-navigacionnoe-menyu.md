# Q018. Как семантически верно сверстать навигационное меню?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Семантически корректное навигационное меню — это `<nav>` с внутренним `<ul>/<li>/<a>`. Элемент `<nav>` создаёт landmark-регион с ARIA-ролью `navigation`, что позволяет скринридерам мгновенно перейти к навигации. Атрибут `aria-label` на `<nav>` различает несколько навигаций на одной странице. Атрибут `aria-current="page"` на активной ссылке сообщает AT о текущей странице.

---

## Развёрнутый ответ

### Суть и определение

`<nav>` — HTML5-семантический элемент для блоков навигационных ссылок. Согласно спецификации, его следует использовать для **основных навигационных блоков**, а не для всех групп ссылок. Например, набор ссылок в подвале не обязательно оборачивать в `<nav>` — только если это полноценный навигационный блок.

### Как это работает

`<nav>` имеет неявную ARIA-роль `navigation` и является **landmark-регионом**. Пользователи скринридеров могут перечислять все landmark-регионы страницы и переходить к нужному нажатием одной клавиши.

Структура `<ul>/<li>/<a>` выбрана намеренно:
- `<ul>` сообщает AT, что это **список** (количество пунктов объявляется автоматически).
- `<li>` — элемент списка.
- `<a>` — интерактивная ссылка с фокусом и семантикой навигации.

### Практика и применение

**Несколько `<nav>` на странице**: допустимо и правильно. Различать их нужно через `aria-label`:
- Основная навигация: `<nav aria-label="Основная навигация">`
- Хлебные крошки: `<nav aria-label="Навигационная цепочка">`
- Постраничная навигация: `<nav aria-label="Постраничная навигация">`

**`aria-current="page"`**: атрибут указывает скринридеру, что ссылка ведёт на текущую страницу. Значения: `page`, `step`, `location`, `date`, `time`, `true`.

**Мобильное меню (hamburger)**: кнопка открытия должна быть `<button>` с `aria-expanded` и `aria-controls`. При закрытом меню `<nav>` скрывается через `hidden` или CSS, но остаётся в DOM.

### Важные нюансы и краеугольные камни

- **Не используй `role="navigation"` на `<div>`** если можно использовать `<nav>` — нативный элемент надёжнее.
- **Не оборачивай все ссылки в `<nav>`**: ссылки «Войти» / «Регистрация» в шапке могут быть просто `<a>` без `<nav>`.
- **Активный пункт не делать ссылкой на себя**: текущую страницу можно оставить как `<a aria-current="page">` (ссылка на текущую страницу) или заменить на `<span>`, если клик бессмысленен.
- **`tabindex` не нужен** для `<a>` и `<button>` — они фокусируемы по умолчанию.
- **CSS-классы vs ARIA**: `class="active"` влияет только на стиль; `aria-current="page"` — на дерево доступности. Нужны оба.

### Примеры

```html
<!-- Основная навигация -->
<header>
  <nav aria-label="Основная навигация">
    <ul>
      <li><a href="/" aria-current="page">Главная</a></li>
      <li><a href="/about">О нас</a></li>
      <li><a href="/services">Услуги</a></li>
      <li><a href="/contact">Контакты</a></li>
    </ul>
  </nav>
</header>

<!-- Хлебные крошки -->
<nav aria-label="Навигационная цепочка">
  <ol>
    <li><a href="/">Главная</a></li>
    <li><a href="/blog">Блог</a></li>
    <li><span aria-current="page">Семантика HTML</span></li>
  </ol>
</nav>

<!-- Мобильное меню с hamburger-кнопкой -->
<header>
  <button
    type="button"
    aria-expanded="false"
    aria-controls="main-nav"
    aria-label="Открыть меню"
  >
    <span aria-hidden="true">☰</span>
  </button>

  <nav id="main-nav" aria-label="Основная навигация" hidden>
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/about">О нас</a></li>
    </ul>
  </nav>
</header>

<!-- Постраничная навигация -->
<nav aria-label="Постраничная навигация">
  <ul>
    <li><a href="?page=1" aria-label="Страница 1">1</a></li>
    <li><a href="?page=2" aria-current="page" aria-label="Страница 2, текущая">2</a></li>
    <li><a href="?page=3" aria-label="Страница 3">3</a></li>
  </ul>
</nav>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Почему `<ul>/<li>` предпочтительнее `<div>` внутри `<nav>`? (скринридер объявляет количество пунктов списка)
- Что означает `aria-current="page"` и чем отличается от CSS-класса `active`? (aria-current — для AT, class — для стилей)
- Как различить несколько `<nav>` для скринридера? (aria-label или aria-labelledby)
- Должен ли активный пункт меню быть ссылкой? (можно `<a aria-current="page">` или заменить `<span>`)
- Как реализовать доступное hamburger-меню? (button с aria-expanded + aria-controls, nav с id)

### Красные флаги (чего не говорить)

- «Для навигации достаточно `<div>` со ссылками» — теряется landmark-навигация для AT.
- «Нужно добавить `role="navigation"` к `<nav>`» — это избыточно, роль уже встроена.
- «`<nav>` нужен вокруг любой группы ссылок» — только для основных навигационных блоков.

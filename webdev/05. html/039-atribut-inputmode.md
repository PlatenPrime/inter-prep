# Q039. Для чего используют атрибут `inputmode`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`inputmode` подсказывает браузеру, какую виртуальную клавиатуру показать на мобильном устройстве при фокусировке на поле. Он не влияет на тип данных поля, валидацию или поведение — это исключительно UX-атрибут для оптимизации ввода. Основные значения: `text`, `numeric`, `decimal`, `tel`, `search`, `email`, `url`, `none`.

---

## Развёрнутый ответ

### Суть и определение

На мобильных устройствах виртуальная клавиатура конфигурируется под разные контексты: числовой пад, клавиатура с `@` и `.com`, телефонный набор и т.д. `inputmode` позволяет разработчику явно управлять этим, независимо от типа `<input>`.

### Как это работает

```html
<input type="text" inputmode="numeric" />
```

Несмотря на `type="text"` (то есть любое текстовое значение), браузер покажет **цифровую клавиатуру** на iOS и Android. Это особенно важно когда нужны только цифры, но нельзя использовать `type="number"` (из-за стрелок, особого поведения при вводе и проблем с leading zeros).

### Значения `inputmode`

| Значение | Клавиатура | Когда использовать |
|----------|------------|-------------------|
| `text` | Стандартная текстовая | По умолчанию |
| `numeric` | Цифровой пад (0–9) | PIN-коды, количества, индексы |
| `decimal` | Цифры с десятичным разделителем | Цены, измерения |
| `tel` | Телефонный пад (+, *, #) | Телефонные номера |
| `search` | Текстовая с кнопкой «Поиск» | Строки поиска |
| `email` | Текстовая с `@` и `.com` | Email-адреса |
| `url` | Текстовая с `/`, `.`, `.com` | URL-адреса |
| `none` | Клавиатура не показывается | Кастомный экранный ввод (пин-пад, рисование) |

### Практика и применение

- Поле кода верификации/OTP: `type="text" inputmode="numeric"` — чистые цифры без `type="number"` и его побочных эффектов
- Поле цены: `type="text" inputmode="decimal"` — числа с запятой/точкой
- Поле поиска без `type="search"` (чтобы избежать крестика): `type="text" inputmode="search"`
- Кастомный PIN-пад: `inputmode="none"` — предотвращает системную клавиатуру, рендерится своя
- Поле номера карты: `type="text" inputmode="numeric" autocomplete="cc-number"`

### Важные нюансы и краеугольные камни

- `inputmode` **не заменяет** `type` и не влияет на валидацию — это только подсказка браузеру
- `type="number"` имеет нежелательные эффекты: убирает leading zeros, реагирует на скролл колесом мыши, имеет локализованные форматы — поэтому `type="text" inputmode="numeric"` часто предпочтительнее
- На десктопе `inputmode` игнорируется — виртуальной клавиатуры нет
- Поддержка: iOS Safari с 12.1+, Chrome Android с 66+ — покрытие хорошее
- `inputmode="decimal"` показывает разделитель согласно локали устройства: в русской локали — запятая, в английской — точка
- Применяется не только к `<input>`, но и к `<textarea>` и любому элементу с `contenteditable`

### Примеры

```html
<!-- OTP код — цифровая клавиатура без type="number" -->
<label for="otp">Код из SMS</label>
<input
  type="text"
  id="otp"
  name="otp"
  inputmode="numeric"
  autocomplete="one-time-code"
  pattern="\d{6}"
  maxlength="6"
  placeholder="------"
/>

<!-- Цена с десятичной точкой -->
<label for="price">Цена</label>
<input
  type="text"
  id="price"
  name="price"
  inputmode="decimal"
  placeholder="0.00"
/>

<!-- Кастомный PIN-пад — системная клавиатура не нужна -->
<input
  type="password"
  inputmode="none"
  id="pin"
  name="pin"
  readonly
/>
<!-- Ниже рендерится кастомный экранный пад -->

<!-- Email с удобной клавиатурой -->
<input type="text" inputmode="email" name="contact-email" />
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Почему используют `type="text" inputmode="numeric"` вместо `type="number"`?
- Влияет ли `inputmode` на валидацию?
- Как `inputmode="none"` используется на практике?
- Что покажет `inputmode="decimal"` на устройстве с русской локалью?
- Работает ли `inputmode` на `contenteditable` элементах?

### Красные флаги (чего не говорить)

- «`inputmode` влияет на валидацию данных» — не влияет совсем
- «`type="number"` и `inputmode="numeric"` взаимозаменяемы» — `type="number"` имеет побочные эффекты
- «`inputmode` работает на десктопе» — десктоп игнорирует атрибут

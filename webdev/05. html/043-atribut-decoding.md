# Q048. Для чего используется атрибут `decoding`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`decoding` — атрибут элемента `<img>`, управляющий тем, как браузер декодирует изображение. Значение `async` позволяет декодировать картинку в фоновом потоке, не блокируя main thread и не задерживая отрисовку остального контента. `sync` блокирует рендеринг до завершения декодирования, `auto` оставляет решение браузеру.

---

## Развёрнутый ответ

### Суть и определение

Декодирование изображения — это процесс преобразования сжатых данных (JPEG, PNG, WebP) в пиксельный буфер, который GPU может отрендерить. По умолчанию этот процесс синхронный и выполняется в main thread, что может вызывать задержки отрисовки. Атрибут `decoding` даёт разработчику явный контроль над этим поведением.

### Как это работает

```
sync  → декодирование блокирует парсинг и рендеринг DOM (поведение до HTML5.1)
async → декодирование в параллельном потоке; изображение появится позже,
        но страница рисуется без задержки
auto  → браузер выбирает стратегию самостоятельно (значение по умолчанию)
```

Браузеры также предоставляют JS API `HTMLImageElement.decode()` — промис, который резолвится, когда изображение готово к отрисовке.

### Практика и применение

- **Hero-изображения above the fold**: `sync` или `auto` чтобы картинка была готова до первой отрисовки
- **Галереи и ленты**: `async` для тяжёлых изображений ниже fold — не блокируют LCP
- **Динамически добавляемые изображения (JS)**: используйте `img.decode().then(...)` для гарантии готовности

```html
<!-- Фоновое декодирование для изображений в ленте -->
<img src="product.jpg" alt="Продукт" decoding="async" />

<!-- Критичное изображение — синхронно -->
<img src="hero.jpg" alt="Главный баннер" decoding="sync" />

<!-- Оставить решение браузеру -->
<img src="thumbnail.webp" alt="Превью" decoding="auto" />
```

```js
// JS API — добавить изображение только когда оно декодировано
const img = new Image();
img.src = 'large-photo.jpg';
img.decoding = 'async';

img.decode().then(() => {
  document.body.appendChild(img);
});
```

### Важные нюансы и краеугольные камни

- `decoding="async"` **не влияет на загрузку** — сеть и fetch работают независимо. Атрибут управляет только фазой декодирования после загрузки
- Не путать с `loading="lazy"` — тот откладывает **загрузку**, `decoding` откладывает **декодирование**
- Реальный выигрыш заметен для **больших изображений** (> 1 МБ в сжатом виде); для мелких иконок разница незначительна
- `decoding="async"` может вызвать FOUI (Flash of Undecoded Image) — кратковременный blank до появления картинки
- Совместим со всеми современными браузерами (Chrome 65+, Firefox 63+, Safari 11.1+)
- При использовании совместно с `fetchpriority="high"` и `loading="eager"` можно точно управлять приоритетом критичных изображений

### Примеры

```html
<!-- Оптимальная стратегия для разных сценариев -->

<!-- LCP-изображение: загрузить и декодировать как можно раньше -->
<img
  src="hero.webp"
  alt="Главный баннер"
  fetchpriority="high"
  loading="eager"
  decoding="sync"
/>

<!-- Изображения в карточках ниже fold -->
<img
  src="card-image.webp"
  alt="Карточка товара"
  loading="lazy"
  decoding="async"
/>
```

## Сравнение

| Критерий           | `decoding="sync"`                  | `decoding="async"`                   |
|--------------------|------------------------------------|--------------------------------------|
| Main thread        | Блокирует до декодирования         | Не блокирует                         |
| Момент отображения | Сразу после декодирования          | Позже, после фоновой обработки       |
| FOUI риск          | Нет                                | Возможен                             |
| Применение         | Hero, LCP-изображения              | Галереи, ленты, lazy-контент         |
| Связанный API      | —                                  | `img.decode()` промис                |

### Красные флаги (чего не говорить)

- «`decoding="async"` ускоряет загрузку изображений» — нет, только декодирование
- «`decoding` и `loading` делают одно и то же» — разные фазы жизненного цикла изображения
- «`async` всегда лучше» — для LCP-изображений может навредить показателю

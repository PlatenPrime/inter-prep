# Q054. Что такое `srcset`? Как работает `srcset`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`srcset` — атрибут элементов `<img>` и `<source>`, позволяющий задать набор изображений с дескрипторами разрешения (`x`) или ширины (`w`). Браузер самостоятельно выбирает оптимальный вариант исходя из размера экрана, плотности пикселей (DPR) и условий сети. Атрибут `sizes` уточняет, какой размер займёт изображение в layout, помогая браузеру сделать правильный выбор при `w`-дескрипторах.

---

## Развёрнутый ответ

### Суть и определение

До `srcset` для Retina-дисплеев использовали CSS media queries или JS-хаки. `srcset` — декларативное, встроенное в HTML решение: разработчик описывает **что доступно**, браузер решает **что загрузить**. Это критично для производительности — браузер может учитывать скорость соединения и кэш.

### Как это работает

**Два режима работы:**

#### 1. Дескриптор плотности пикселей (`x`)

Выбирает изображение по Device Pixel Ratio (DPR). Простой и предсказуемый.

```html
<img
  src="logo.png"
  srcset="logo.png 1x, logo@2x.png 2x, logo@3x.png 3x"
  alt="Логотип"
/>
```

| DPR устройства | Загрузится         |
|----------------|--------------------|
| 1 (обычный)    | `logo.png`         |
| 2 (Retina)     | `logo@2x.png`      |
| 3 (QHD)        | `logo@3x.png`      |

#### 2. Дескриптор ширины (`w`) + `sizes`

Браузер знает физическую ширину каждого изображения (`w`) и через `sizes` узнаёт, сколько пикселей займёт изображение в layout. Вычисляет: `нужно = sizes × DPR`, берёт ближайшее подходящее.

```html
<img
  src="photo-800.jpg"
  srcset="
    photo-400.jpg   400w,
    photo-800.jpg   800w,
    photo-1200.jpg 1200w,
    photo-1600.jpg 1600w
  "
  sizes="
    (max-width: 480px)  100vw,
    (max-width: 960px)  50vw,
    33vw
  "
  alt="Фото товара"
/>
```

**Разбор `sizes`:**

```
(max-width: 480px) 100vw  → на экранах ≤480px изображение занимает 100% ширины viewport
(max-width: 960px)  50vw  → на экранах ≤960px — 50%
33vw                      → иначе — треть viewport
```

Браузер на Retina (DPR=2) при viewport 480px: `100vw × 480px × 2 = 960px` → загрузит `photo-1200.jpg`.

### Практика и применение

- **Responsive изображения в сетках**: карточки товаров, галереи
- **Hero-секции**: разные разрешения одного изображения
- **Иконки и логотипы**: `1x`/`2x`/`3x` для Retina
- **Авторские фото**: полноширинные изображения в статьях

```html
<!-- Иконка с поддержкой Retina -->
<img src="icon.png" srcset="icon.png 1x, icon@2x.png 2x" alt="Иконка" />

<!-- Карточка в сетке (4 колонки на десктопе, 2 на планшете, 1 на мобильном) -->
<img
  src="card.jpg"
  srcset="card-400.webp 400w, card-800.webp 800w, card-1200.webp 1200w"
  sizes="(max-width: 480px) 100vw, (max-width: 1024px) 50vw, 25vw"
  alt="Карточка продукта"
  loading="lazy"
/>
```

### Важные нюансы и краеугольные камни

- `src` обязателен как fallback для браузеров без поддержки `srcset` (старые браузеры)
- Браузер **не обязан** выбрать то изображение, которое «правильно» по математике — он может оставить закэшированную большую версию
- `w`-дескрипторы нельзя смешивать с `x`-дескрипторами в одном `srcset`
- `sizes` без `srcset` с `w`-дескрипторами не имеет смысла
- **`<picture>` vs `srcset`**: `srcset` — для одного контента разных размеров (resolution switching); `<picture>` — когда нужны разные кадры или разные форматы (art direction)
- При использовании `w`-дескрипторов без `sizes` браузер считает, что изображение занимает `100vw`

### Примеры

```html
<!-- Полный пример: responsive hero с WebP и JPEG fallback -->
<picture>
  <source
    type="image/webp"
    srcset="
      hero-480.webp   480w,
      hero-768.webp   768w,
      hero-1280.webp 1280w,
      hero-1920.webp 1920w
    "
    sizes="100vw"
  />
  <img
    src="hero-1280.jpg"
    srcset="
      hero-480.jpg   480w,
      hero-768.jpg   768w,
      hero-1280.jpg 1280w,
      hero-1920.jpg 1920w
    "
    sizes="100vw"
    alt="Команда за работой"
    width="1280"
    height="720"
    fetchpriority="high"
  />
</picture>
```

## Сравнение

| Критерий              | `x`-дескрипторы                  | `w`-дескрипторы + `sizes`                  |
|-----------------------|----------------------------------|--------------------------------------------|
| Простота              | Высокая                          | Средняя                                    |
| Учёт размера в layout | Нет                              | Да                                         |
| Подходит для          | Фиксированный размер (иконки)    | Fluid / responsive изображения             |
| Учёт DPR              | Явно                             | Косвенно (браузер считает сам)             |
| Гибкость              | Низкая                           | Высокая                                    |

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Чем `w`-дескрипторы отличаются от `x`-дескрипторов?
- Что браузер делает с `sizes` при выборе изображения?
- Почему браузер может загрузить изображение больше, чем «нужно»?
- Когда достаточно `srcset` на `<img>`, а когда нужен `<picture>`?
- Как `srcset` влияет на LCP и производительность?

### Красные флаги (чего не говорить)

- «Браузер всегда загружает наименьшее подходящее изображение» — он может использовать кэш
- «`sizes` обязателен для любого `srcset`» — обязателен только для `w`-дескрипторов
- «`srcset` и `<picture>` делают одно и то же» — разные задачи

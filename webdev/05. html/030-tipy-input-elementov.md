# Q030. Типы `<input>` элементов в HTML?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<input>` поддерживает более 20 типов, управляемых атрибутом `type`. Тип определяет визуальное представление, встроенную валидацию, отображаемую клавиатуру на мобильных и поведение при отправке формы. Браузеры с неизвестным типом откатываются к `type="text"` — это обеспечивает обратную совместимость. Правильный выбор типа улучшает UX, доступность и уменьшает количество JS-кода для валидации.

---

## Развёрнутый ответ

### Суть и определение

`<input type="...">` — самозакрывающийся inline-элемент формы. Атрибут `type` кардинально меняет его поведение. Неверный или устаревший тип вместо ошибки даёт деградацию к `type="text"`.

### Как это работает

Браузер использует тип для:
1. Выбора нативного UI-виджета
2. Показа подходящей клавиатуры на мобильных (virtual keyboard)
3. Встроенной валидации (email, url, number, required)
4. Форматирования данных перед отправкой формы

### Практика и применение

**Текстовые типы:**

```html
<!-- Однострочный текст (default) -->
<input type="text" placeholder="Имя" autocomplete="name">

<!-- Пароль — скрывает символы -->
<input type="password" autocomplete="current-password">

<!-- Email — валидирует формат, мобильная клавиатура с @ -->
<input type="email" autocomplete="email">

<!-- Телефон — мобильная цифровая клавиатура, без валидации формата -->
<input type="tel" autocomplete="tel">

<!-- URL — валидирует протокол, мобильная клавиатура с / и .com -->
<input type="url" placeholder="https://">

<!-- Поиск — стилизуется как поле поиска, кнопка очистки в Chrome -->
<input type="search" placeholder="Поиск...">
```

**Числовые типы:**

```html
<!-- Число со стрелками вверх/вниз -->
<input type="number" min="0" max="100" step="1">

<!-- Слайдер -->
<input type="range" min="0" max="100" value="50" step="5">
```

**Дата и время:**

```html
<!-- Дата: YYYY-MM-DD -->
<input type="date" min="2020-01-01" max="2030-12-31">

<!-- Время: HH:MM -->
<input type="time">

<!-- Дата и время (локальное, без timezone) -->
<input type="datetime-local">

<!-- Месяц: YYYY-MM -->
<input type="month">

<!-- Неделя: YYYY-Wnn -->
<input type="week">
```

**Выбор и переключение:**

```html
<!-- Чекбокс — булевое значение -->
<input type="checkbox" id="agree" name="agree" value="yes">
<label for="agree">Я согласен</label>

<!-- Радиокнопка — одно значение из группы -->
<input type="radio" id="yes" name="answer" value="yes">
<input type="radio" id="no" name="answer" value="no">

<!-- Цвет — color picker -->
<input type="color" value="#ff0000">
```

**Файлы:**

```html
<!-- Загрузка файла -->
<input type="file" accept=".pdf,.docx" multiple>

<!-- Камера на мобильных -->
<input type="file" accept="image/*" capture="environment">
```

**Кнопки (предпочти `<button>` для гибкости):**

```html
<!-- Отправка формы -->
<input type="submit" value="Отправить">

<!-- Сброс формы -->
<input type="reset" value="Очистить">

<!-- Кнопка без action -->
<input type="button" value="Нажать">

<!-- Кнопка-изображение (submit с координатами клика) -->
<input type="image" src="/submit.png" alt="Отправить">
```

**Скрытые данные:**

```html
<!-- Скрытое поле — отправляется в форме, не видно пользователю -->
<input type="hidden" name="csrf_token" value="abc123">
```

### Важные нюансы и краеугольные камни

- `type="number"` не подходит для номеров карт, телефонов, ZIP-кодов — используй `type="text"` с `inputmode="numeric"` и `pattern`
- `type="email"` валидирует только базовый формат — не полную RFC-совместимость. Серверная валидация обязательна
- `type="date"` не поддерживался в Safari до 2020 года — при необходимости нужен полифил или кастомный picker
- `type="range"` не отправляет значение если пользователь не трогал слайдер — всегда задавай `value` по умолчанию
- `type="hidden"` не является защитой от манипуляций — пользователь может изменить значение через DevTools
- `type="password"` с `autocomplete="new-password"` указывает менеджеру паролей предложить генерацию

### Красные флаги (чего не говорить)

- «`type="number"` лучше всегда для цифр» — нет, для телефонов/карт используй `text` + `inputmode`
- «`type="hidden"` защищает данные от пользователя» — нет, DevTools всё покажет и позволит изменить
- «Валидации в `type="email"` достаточно для production» — нет, серверная валидация обязательна

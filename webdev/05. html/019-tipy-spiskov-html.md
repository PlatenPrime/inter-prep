# Q019. Типы списков в HTML?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

В HTML есть три типа списков: `<ul>` — неупорядоченный (маркированный), `<ol>` — упорядоченный (нумерованный) и `<dl>` — список определений с парами `<dt>` (термин) / `<dd>` (описание). Выбор типа — семантический: `<ul>` когда порядок не важен, `<ol>` когда порядок имеет значение, `<dl>` для пар «термин — описание».

---

## Развёрнутый ответ

### Суть и определение

Каждый тип списка несёт семантическую нагрузку:
- **`<ul>` (Unordered List)** — набор элементов без подразумеваемого порядка. Браузер рендерит их с маркерами.
- **`<ol>` (Ordered List)** — последовательность, где порядок важен. Браузер нумерует элементы.
- **`<dl>` (Description List)** — ассоциативный список пар «термин → описание». Используется для глоссариев, метаданных, списков FAQ.

### Как это работает

Скринридер объявляет количество элементов в `<ul>` и `<ol>`. Для `<dl>` — группы термин/описание. Пользователь может навигировать по элементам списка клавишами.

**Атрибуты `<ol>`:**
- `start` — начальное значение нумерации (`<ol start="5">` начнёт с 5).
- `reversed` — обратная нумерация (от n до 1).
- `type` — стиль нумерации: `1` (числа), `a` (буквы строчные), `A` (буквы заглавные), `i` (римские строчные), `I` (римские заглавные). Предпочтительно управлять через CSS `list-style-type`.

**Атрибут `value` у `<li>` в `<ol>`:** переопределяет номер конкретного элемента.

### Практика и применение

- **`<ul>`**: навигационное меню, список функций, теги статьи, элементы корзины.
- **`<ol>`**: шаги инструкции, рецепт, рейтинг, нумерованные требования.
- **`<dl>`**: глоссарий, FAQ (вопрос — ответ), метаданные статьи (автор, дата), список настроек.
- **Вложенные списки**: `<ul>` может содержать вложенный `<ul>` или `<ol>` — но вложение идёт внутрь `<li>`, а не напрямую в `<ul>`.

### Важные нюансы и краеугольные камни

- **Прямые дочерние элементы `<ul>` и `<ol>` — только `<li>`**: вставка других элементов напрямую невалидна.
- **`<dl>` допускает несколько `<dd>` на один `<dt>`** и несколько `<dt>` на один `<dd>`.
- **CSS `list-style: none` убирает семантику у скринридеров Safari/VoiceOver**: при сбросе маркеров список перестаёт объявляться как list. Фикс: добавить `role="list"` явно.
- **`<menu>` в HTML5**: переопределён как семантический аналог `<ul>` для интерактивных элементов (кнопки, действия). Не путать с `<nav>`.
- **`type` у `<ol>` vs CSS**: атрибут `type` влияет и на отображение, и на содержимое при печати/копировании. CSS `list-style-type` — только визуальный.

### Примеры

```html
<!-- Неупорядоченный список: порядок не важен -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Упорядоченный список: шаги инструкции -->
<ol>
  <li>Установите Node.js</li>
  <li>Клонируйте репозиторий</li>
  <li>Выполните <code>npm install</code></li>
</ol>

<!-- Атрибуты ol: start и reversed -->
<ol start="3" reversed>
  <li>Третье место</li>  <!-- отображается как 3 -->
  <li>Второе место</li>  <!-- отображается как 2 -->
  <li>Первое место</li>  <!-- отображается как 1 -->
</ol>

<!-- Список определений: глоссарий -->
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language — язык разметки гипертекста</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets — каскадные таблицы стилей</dd>
</dl>

<!-- dl для FAQ -->
<dl>
  <dt>Сколько стоит доставка?</dt>
  <dd>Доставка бесплатна при заказе от 1000 рублей.</dd>

  <dt>Как оформить возврат?</dt>
  <dd>Заполните форму на сайте в течение 14 дней.</dd>
</dl>

<!-- Вложенный список: вложение строго внутри li -->
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>

<!-- Фикс для VoiceOver при list-style: none -->
<ul role="list" style="list-style: none; padding: 0;">
  <li>Элемент 1</li>
  <li>Элемент 2</li>
</ul>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Что произойдёт с семантикой `<ul>` если применить `list-style: none`? (VoiceOver/Safari перестаёт объявлять как list — нужен `role="list"`)
- Можно ли вложить `<ol>` в `<ul>`? (да, внутрь `<li>`)
- Какие атрибуты есть у `<ol>`? (`start`, `reversed`, `type`)
- Когда использовать `<dl>` вместо `<ul>`? (для пар термин-описание: глоссарий, FAQ, метаданные)
- Какие прямые дочерние элементы допустимы в `<ul>`? (только `<li>`, в `<dl>` — `<dt>` и `<dd>`)

### Красные флаги (чего не говорить)

- «`<ul>` и `<ol>` одинаковы, разница только в маркерах» — принципиальная семантическая разница в порядке.
- «`<dl>` — это словарь и нигде больше не используется» — используется для метаданных, FAQ, любых пар ключ-значение.
- «Можно вставить `<p>` прямо в `<ul>`» — невалидно, только `<li>` как прямой дочерний элемент.

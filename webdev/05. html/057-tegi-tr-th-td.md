# Q057. Для чего используются тэги `<tr>`, `<th>`, `<td>`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<tr>` (table row) — определяет строку таблицы и является контейнером для ячеек. `<th>` (table header) — заголовочная ячейка: по умолчанию bold и выровнена по центру, семантически обозначает заголовок колонки или строки. `<td>` (table data) — обычная ячейка с данными. Атрибуты `colspan` и `rowspan` позволяют ячейкам занимать несколько колонок или строк.

---

## Развёрнутый ответ

### Суть и определение

Таблица в HTML строится из строк и ячеек. `<tr>` — это строка, `<th>`/`<td>` — ячейки внутри строки. Разделение на `<th>` и `<td>` — не косметика, а семантика: `<th>` создаёт связь между заголовком и данными, что критично для accessibility и screen readers.

### Как это работает

**`<tr>` (Table Row):**
- Контейнер для ячеек (`<th>` и `<td>`).
- Размещается внутри `<thead>`, `<tbody>` или `<tfoot>`.
- CSS-свойства `height`, `background-color`, `border` применяются к строке.

**`<th>` (Table Header Cell):**
- Семантически обозначает заголовок колонки (`scope="col"`) или строки (`scope="row"`).
- User-agent stylesheet: `font-weight: bold; text-align: center`.
- Атрибут `scope` — ключевой для a11y: указывает screen reader, к чему относится заголовок.
- Атрибут `abbr` — сокращение для длинных заголовков (screen reader произносит abbr вместо полного текста).

**`<td>` (Table Data Cell):**
- Содержит данные таблицы.
- User-agent stylesheet: `text-align: start; vertical-align: inherit`.
- Атрибут `headers` — список ID `<th>`-элементов, к которым относится ячейка (для сложных таблиц).

**Общие атрибуты `<th>` и `<td>`:**
- `colspan="N"` — ячейка занимает N колонок.
- `rowspan="N"` — ячейка занимает N строк.

### Практика и применение

- **Простые таблицы:** `<th scope="col">` в заголовке, `<td>` в теле — достаточно для a11y.
- **Таблицы с заголовками строк:** `<th scope="row">` в первой ячейке каждой строки.
- **Сложные таблицы** с merged cells: `<th id="...">` + `<td headers="...">` для явных ассоциаций.
- **Не используйте `<table>` для layout** — только для табличных данных (нарушение семантики и a11y).

### Важные нюансы и краеугольные камни

- **`scope` vs `headers`:** `scope` — простой и достаточный для обычных таблиц; `headers` нужен только для сложных merged-cell таблиц.
- **`colspan="0"`** — теоретически «до конца строки», но реализация нестабильна в браузерах; лучше явное число.
- **Стилизация `<tr>`:** нельзя применить `display: flex` к `<tr>` без нарушения tabular layout.
- **`rowspan` и `colspan` на `<th>`:** работает так же, как на `<td>`.
- Браузеры игнорируют `<td>`/`<th>` вне `<tr>`, а `<tr>` вне `<table>` — автоматически создаётся anonymous table.

### Примеры

```html
<!-- Базовая таблица с правильной семантикой -->
<table>
  <caption>Квартальные продажи 2026</caption>
  <thead>
    <tr>
      <th scope="col">Продукт</th>
      <th scope="col">Q1</th>
      <th scope="col">Q2</th>
      <th scope="col">Итого</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Widget A</th>
      <td>150</td>
      <td>200</td>
      <td>350</td>
    </tr>
    <tr>
      <th scope="row">Widget B</th>
      <td>300</td>
      <td>280</td>
      <td>580</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th scope="row">Всего</th>
      <td>450</td>
      <td>480</td>
      <td>930</td>
    </tr>
  </tfoot>
</table>

<!-- colspan и rowspan -->
<table>
  <tr>
    <th colspan="2">Объединённый заголовок</th>
  </tr>
  <tr>
    <td rowspan="2">Ячейка на 2 строки</td>
    <td>Строка 1</td>
  </tr>
  <tr>
    <td>Строка 2</td>
  </tr>
</table>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Чем семантически отличается `<th>` от `<td>` с `style="font-weight: bold"`? (смысловая роль заголовка для AT, SEO)
- Что произойдёт, если `colspan` суммарно превысит число колонок таблицы?
- Как screen reader обрабатывает `<th scope="col">` vs `<th>` без scope?
- Можно ли использовать `display: flex` или `grid` на `<tr>`? (нарушает tabular layout)
- Для чего нужен атрибут `headers` на `<td>` и когда он необходим?

### Красные флаги (чего не говорить)

- «`<th>` и `<td>` — одно и то же, просто разный дефолтный стиль» — `<th>` несёт семантическую роль заголовка.
- «`<table>` можно использовать для вёрстки layout» — это антипаттерн, нарушает a11y и семантику.
- «`scope` атрибут необязателен» — без него screen readers не могут ассоциировать данные с заголовками.

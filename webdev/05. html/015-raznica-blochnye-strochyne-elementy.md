# Q015. Разница между блочными и строчными элементами?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Блочные элементы (block-level) занимают всю доступную ширину контейнера и начинаются с новой строки: `<div>`, `<p>`, `<h1>`–`<h6>`, `<ul>`, `<ol>`, `<section>`, `<article>` и другие. Строчные элементы (inline) занимают только пространство своего содержимого и не разрывают поток текста: `<span>`, `<a>`, `<strong>`, `<em>`, `<img>`, `<input>`. Это различие — на уровне CSS-свойства `display`, а не фиксированного типа элемента: его можно переопределить в стилях.

---

## Развёрнутый ответ

### Суть и определение

Деление на блочные и строчные — это концепция CSS Box Model, отражённая в HTML через дефолтные значения свойства `display`. В HTML5 на уровне спецификации это деление заменено системой **Content Categories**, но на практике браузерные дефолтные стили (User Agent Stylesheet) по-прежнему делают большинство элементов либо `block`, либо `inline`.

### Как это работает

Браузер применяет встроенную таблицу стилей (User Agent Stylesheet), которая задаёт `display: block` или `display: inline` элементам. Это можно переопределить через CSS.

**Поведение блочных элементов:**
- Начинаются с новой строки.
- По умолчанию занимают 100% ширины родителя.
- Уважают `width`, `height`, `margin` (все стороны), `padding` (все стороны).
- Содержат другие блочные и строчные элементы.

**Поведение строчных элементов:**
- Располагаются в потоке текста, не разрывая его.
- Ширина и высота определяются содержимым.
- Игнорируют `width` и `height`.
- `margin-top` / `margin-bottom` не работают (только left/right).
- `padding-top` / `padding-bottom` применяются визуально, но не влияют на layout окружающих элементов.

### Практика и применение

- `display: inline-block` — гибрид: элемент располагается в строке, но уважает `width`, `height` и все `margin`/`padding`.
- Современная верстка активно использует `display: flex` и `display: grid`, которые переопределяют блочное/строчное поведение дочерних элементов.
- `<img>` — технически inline, но обладает собственными размерами (replaced element), что делает его поведение нестандартным: он уважает `width`/`height`.

### Важные нюансы и краеугольные камни

- **Replaced elements**: `<img>`, `<video>`, `<iframe>`, `<input>` — строчные, но уважают размеры, так как их содержимое заменяется внешним ресурсом.
- **`<img>` и whitespace gap**: так как `<img>` inline, под ним появляется пространство для «хвостика» букв (descender). Убирается через `display: block` или `vertical-align: bottom`.
- **Вложенность**: строчный элемент не должен содержать блочный. `<a>` — исключение (transparent content model) и при определённых условиях может содержать блочный контент.
- **`<button>` и `<input>`**: технически `display: inline-block` по умолчанию.

### Примеры

```html
<!-- Блочные элементы — каждый на новой строке, полная ширина -->
<div>Блок 1</div>
<p>Параграф (блочный)</p>
<h2>Заголовок (блочный)</h2>

<!-- Строчные элементы — в потоке текста -->
<p>
  Текст с <span>выделением</span>,
  <a href="#">ссылкой</a> и
  <strong>важным словом</strong>.
</p>

<!-- Replaced inline element: img имеет размеры, но стоит в строке -->
<p>Иконка: <img src="icon.svg" alt="" width="16" height="16" /> в тексте</p>

<!-- Переопределение через CSS -->
<style>
  span { display: block; }       /* строчный → блочный */
  div  { display: inline; }      /* блочный → строчный */
  a    { display: inline-block; } /* inline + box model */
</style>
```

## Сравнение

| Критерий | Блочные (block) | Строчные (inline) |
|----------|-----------------|-------------------|
| Начало | С новой строки | В потоке текста |
| Ширина по умолчанию | 100% родителя | По содержимому |
| `width` / `height` | Работают | Игнорируются* |
| `margin` (все стороны) | Работают | Только left/right |
| `padding` (все стороны) | Работают | Визуально, но не влияют на layout |
| Содержат | Block + inline | Только inline (phrasing content) |
| Примеры | `div`, `p`, `h1`–`h6`, `ul`, `ol`, `li`, `table`, `header`, `footer`, `section`, `article`, `main`, `aside`, `nav` | `span`, `a`, `strong`, `em`, `abbr`, `code`, `label` |
| Исключения | — | `img`, `input`, `button` — inline, но уважают размеры |

*кроме replaced elements (`img`, `video`, `input`)

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Что такое replaced element и почему `<img>` уважает `width`/`height` будучи inline? (внешний ресурс заменяет содержимое)
- Почему под `<img>` бывает лишний зазор и как его убрать? (`display: block` или `vertical-align: bottom/middle`)
- Чем `inline-block` отличается от `inline`? (уважает все стороны margin/padding и width/height)
- Можно ли поместить `<div>` в `<span>`? (технически браузер «починит» DOM, но это невалидно)
- Как flex/grid влияют на display дочерних элементов? (flex/grid items становятся block-level независимо от исходного display)

### Красные флаги (чего не говорить)

- «Блочные/строчные — это фиксированное свойство HTML-элемента» — это дефолтный CSS, его можно переопределить.
- «`<img>` — блочный, потому что имеет размеры» — нет, это inline replaced element.
- «`padding-top` не работает для inline-элементов» — работает визуально, но не влияет на layout потока.

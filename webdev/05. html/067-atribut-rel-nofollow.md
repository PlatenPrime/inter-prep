# Q067. Что такое и зачем нужен атрибут `rel="nofollow"` в HTML?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`rel="nofollow"` — значение атрибута `rel` тега `<a>`, которое сигнализирует поисковым роботам не передавать «вес» (PageRank) по данной ссылке. Используется для платных ссылок, пользовательского контента (UGC) и ненадёжных источников. В 2019 году Google расширил систему: появились `rel="ugc"` и `rel="sponsored"` как более семантические варианты.

---

## Развёрнутый ответ

### Суть и определение

PageRank — алгоритм Google, оценивающий авторитетность страниц на основе входящих ссылок. Ссылка без `nofollow` передаёт часть «ссылочного веса» целевой странице и влияет на её позицию в поиске.

`rel="nofollow"` появился в 2005 году как рекомендация Google для борьбы со спамом комментариев. Ссылка с этим атрибутом остаётся кликабельной для пользователей, но поисковики (по рекомендации) не учитывают её при расчёте PageRank.

### Как это работает

```html
<a href="https://example.com" rel="nofollow">Ссылка</a>
```

Атрибут `rel` принимает пространство-разделённый список link types. Значения можно комбинировать:

```html
<a href="https://example.com" rel="nofollow noopener noreferrer">Ссылка</a>
```

**С 2019 года Google ввёл три варианта:**

| Значение | Смысл | Когда использовать |
|----------|-------|--------------------|
| `nofollow` | Общий «не доверяю» | Любые ссылки, которым не хотите передавать PageRank |
| `ugc` | User Generated Content | Комментарии, форумы, отзывы пользователей |
| `sponsored` | Платная/рекламная ссылка | Реклама, партнёрские программы, спонсорский контент |

Google обрабатывает все три как **hints** (подсказки), а не директивы — может всё равно передать вес, если сочтёт ссылку авторитетной.

### Практика и применение

- **Рекламные ссылки** — обязательно `rel="sponsored"` (нарушение → мануальные санкции от Google).
- **Блоги с комментариями** — `rel="ugc"` на все ссылки из пользовательских полей.
- **Платные партнёрские ссылки** (affiliate) — `rel="sponsored nofollow"`.
- **Недоверенные источники** — `nofollow` если нужно сослаться, но не хотите «одобрять» сайт.
- Внутренние ссылки сайта — `nofollow` **не нужен** и может навредить.

### Важные нюансы и краеугольные камни

- Атрибут **не делает ссылку невидимой** — она всё равно краулится и индексируется, просто вес не передаётся.
- **Bing, Yandex, другие ПС** тоже поддерживают `nofollow`, но интерпретируют по-своему.
- Злоупотребление `nofollow` на внутренних ссылках (PageRank sculpting) не даёт преимуществ с 2009 года — Google нормализует вес.
- `nofollow` **не защищает от краулинга** — для этого нужен `robots.txt` или `<meta name="robots" content="noindex">`.
- Атрибут `rel` на `<a>` влияет только на исходящие ссылки; для входящих — мета-тег robots или HTTP-заголовок.

### Примеры

```html
<!-- Партнёрская (affiliate) ссылка -->
<a href="https://shop.example.com?ref=mysite" rel="sponsored nofollow">
  Купить на Example Shop
</a>

<!-- Ссылка из комментария пользователя -->
<a href="https://user-site.example.com" rel="ugc nofollow">
  Сайт пользователя
</a>

<!-- Ссылка на ненадёжный источник -->
<a href="https://questionable-source.com" rel="nofollow">
  Источник (мнение автора)
</a>

<!-- Правильная ссылка с безопасностью и SEO -->
<a
  href="https://external.example.com"
  rel="nofollow noopener noreferrer"
  target="_blank"
>
  Внешний ресурс
</a>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Чем `rel="ugc"` отличается от `rel="nofollow"` с точки зрения Google? (semantic hint — Google лучше понимает контекст)
- Блокирует ли `nofollow` краулинг страницы? (нет — только передачу PageRank)
- Что будет если поставить `nofollow` на все внутренние ссылки? (не навредит напрямую, но и не поможет — устаревшая тактика)
- Как `nofollow` соотносится с `X-Robots-Tag: nofollow`? (HTTP-заголовок — серверная альтернатива)
- Обязателен ли `nofollow` для рекламных ссылок? (да, по правилам Google — иначе мануальные санкции)

### Красные флаги (чего не говорить)

- «`nofollow` скрывает ссылку от поисковиков» — краулинг происходит, только вес не передаётся.
- «Нужно ставить `nofollow` на все внешние ссылки» — только на платные, UGC и ненадёжные.
- «`nofollow` — директива, Google обязан её выполнять» — нет, с 2019 это hint.

# Q029. Разница между кнопкой и ссылкой в HTML?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<a href>` — для навигации: переход к URL, якорю на странице или смене маршрута. `<button>` — для действий: отправка формы, открытие модального окна, запуск JS-логики. Выбор между ними влияет на семантику, доступность, поведение клавиатуры и контекстное меню браузера. Правило простое: если нажатие переводит пользователя куда-то — `<a>`, если что-то происходит на текущей странице — `<button>`.

---

## Развёрнутый ответ

### Суть и определение

- **`<a>` (anchor)** — гиперссылка. Обязательно должна иметь `href` для реальной навигации. Без `href` становится placeholder-ссылкой, но теряет поведение навигации.
- **`<button>`** — интерактивный элемент управления. Выполняет действие в текущем контексте.

### Как это работает

| Аспект | `<a href>` | `<button>` |
|--------|-----------|------------|
| Клавиша активации | `Enter` | `Enter` и `Space` |
| Контекстное меню | «Открыть в новой вкладке», «Копировать ссылку» | Нет навигационных опций |
| `href` | Обязателен для навигации | Нет |
| Индикатор загрузки | Браузер показывает URL в статусбаре | Нет |
| История браузера | Добавляет в историю | Нет (если не через JS) |
| Cursor по умолчанию | `pointer` + URL preview | `default` (или `pointer` через CSS) |
| Drag & Drop | Можно перетащить ссылку | Нет по умолчанию |

### Практика и применение

```html
<!-- ССЫЛКИ — навигация -->
<a href="/about">О нас</a>
<a href="#section-2">К разделу 2</a>
<a href="mailto:hello@example.com">Написать письмо</a>
<a href="/report.pdf" download>Скачать отчёт</a>
<a href="https://external.com" target="_blank" rel="noopener noreferrer">Внешний сайт</a>

<!-- КНОПКИ — действия -->
<button type="button" onclick="openModal()">Открыть диалог</button>
<button type="submit">Отправить форму</button>
<button type="button" onclick="addToCart(id)">Добавить в корзину</button>
<button type="button" aria-expanded="false" onclick="toggleMenu()">Меню</button>
```

**Проблемные антипаттерны:**

```html
<!-- ПЛОХО: ссылка без href как кнопка -->
<a onclick="doAction()">Действие</a>
<!-- Нет keyboard support для Space, контекстное меню не то, нет role -->

<!-- ПЛОХО: кнопка для навигации -->
<button onclick="window.location.href='/page'">Перейти</button>
<!-- Нет URL в статусбаре, нельзя открыть в новой вкладке, нет истории браузера -->

<!-- ХОРОШО в SPA: ссылка с перехватом -->
<a href="/page" onclick="navigate(event)">Перейти</a>
<!-- Работает как ссылка И как SPA-навигация с preventDefault() -->
```

**В контексте SPA (React Router, Vue Router):**

```html
<!-- React: <Link> рендерит <a href>, перехватывает click -->
<Link to="/about">О нас</Link>

<!-- Действие vs навигация в SPA -->
<Link to="/product/42">Открыть товар</Link>    <!-- ссылка -->
<button onClick={addToCart}>В корзину</button>  <!-- действие -->
```

### Важные нюансы и краеугольные камни

- `<a>` без `href` — валидный HTML, но не ссылка. Теряет навигационную семантику и keyboard behaviour
- `target="_blank"` требует `rel="noopener noreferrer"` для безопасности (предотвращает tabnabbing)
- Стилизовать `<a>` как кнопку (`.btn`) — нормальная практика для CTA-ссылок, но семантика остаётся навигационной
- Screen readers сообщают о `<a>` как «ссылка» (нажать Enter), о `<button>` как «кнопка» (нажать Enter или Space)

---

## Сравнение

| Критерий | `<a href>` (ссылка) | `<button>` (кнопка) |
|----------|---------------------|---------------------|
| Назначение | Навигация к URL | Выполнение действия |
| Активация клавиатурой | `Enter` | `Enter` + `Space` |
| ARIA роль по умолчанию | `link` | `button` |
| Контекстное меню браузера | «Открыть в новой вкладке» и др. | Стандартное |
| URL preview в статусбаре | Да | Нет |
| Открытие в новой вкладке | Ctrl/Cmd + Click | Нет по умолчанию |
| История браузера | Добавляет запись | Нет |
| Внутри `<form>` | Не отправляет форму | Отправляет (`type="submit"`) |
| Cursor по умолчанию | `pointer` + подсказка | `default` |

- «Разницы нет — можно использовать любой, если добавить `onclick`» — разная семантика и поведение
- «`<button>` можно использовать для перехода на другую страницу» — теряется нативная браузерная функциональность
- «`<a>` без `href` — то же самое, что с `href`» — теряется keyboard accessibility и навигационная семантика

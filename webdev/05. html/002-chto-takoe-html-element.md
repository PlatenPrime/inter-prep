# Q002. Что такое HTML-элемент? Какова его структура?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

HTML-элемент — это единица разметки, состоящая из открывающего тега, содержимого и закрывающего тега. Открывающий тег может содержать атрибуты, которые уточняют поведение или характеристики элемента. Некоторые элементы являются «пустыми» (void elements) — у них нет закрывающего тега и содержимого (`<img>`, `<input>`, `<br>`). Элементы вкладываются друг в друга, образуя дерево DOM.

---

## Развёрнутый ответ

### Суть и определение

HTML-элемент — строительный блок HTML-документа. В отличие от «тега» (который является лишь синтаксической конструкцией `<...>`), элемент включает в себя открывающий тег, содержимое и закрывающий тег вместе. Разговорно их часто путают, но на собеседовании важно понимать разницу.

```
┌─────────────────────────────────────────────┐
│             HTML-элемент                    │
│                                             │
│  <a href="/page" class="link">Текст</a>     │
│  └─────────────────────────┘└────┘└──┘     │
│  открывающий тег с атрибутами  содержимое  │
│                                    закрывающий тег
└─────────────────────────────────────────────┘
```

### Как это работает

**Анатомия элемента:**

```
<tagname attribute="value">content</tagname>
```

- **Тег** — имя элемента в угловых скобках. Регистронезависимо, но принято в нижнем регистре.
- **Атрибут** — пара `имя="значение"` внутри открывающего тега. Значение можно опускать у булевых атрибутов (`disabled`, `checked`).
- **Содержимое** — текст, другие элементы или их смесь.
- **Закрывающий тег** — `</tagname>`.

**Виды элементов:**

| Тип | Описание | Примеры |
|-----|----------|---------|
| Нормальные | Имеют открывающий и закрывающий тег | `<p>`, `<div>`, `<span>` |
| Пустые (void) | Только открывающий тег, нет содержимого | `<img>`, `<input>`, `<br>`, `<hr>`, `<meta>`, `<link>` |
| Сырого текста | Содержимое не парсится как HTML | `<script>`, `<style>` |
| Шаблонные | Инертное содержимое, не рендерится | `<template>` |

**Вложенность элементов** описывает модель содержимого (content model). Например, `<p>` не может содержать блочные элементы — это нарушение спецификации, которое браузер «исправит» неожиданным образом.

### Практика и применение

- Правильная вложенность элементов напрямую влияет на доступность: скринридеры обходят DOM-дерево.
- Понимание void-элементов важно для JSX — в React `<img />` и `<img>` работают по-разному в HTML и JSX.
- В шаблонизаторах (Jinja, Blade, Handlebars) генерируется именно HTML-разметка с теми же правилами.

### Важные нюансы и краеугольные камни

- Незакрытый тег у нормального элемента — ошибка, которую парсер исправит сам, но результат непредсказуем.
- В HTML5 теги `<html>`, `<head>`, `<body>` можно опустить — браузер добавит их в DOM автоматически.
- Булевые атрибуты: `<input disabled>` и `<input disabled="disabled">` эквивалентны; `<input disabled="false">` не отключает атрибут — он всё равно будет активен.
- Имена тегов нечувствительны к регистру в HTML (но чувствительны в XHTML/SVG).

### Примеры

```html
<!-- Нормальный элемент с атрибутами -->
<a href="https://example.com" target="_blank" rel="noopener">Ссылка</a>

<!-- Пустой элемент -->
<img src="photo.jpg" alt="Описание фото" width="800" height="600" />

<!-- Вложенные элементы -->
<article>
  <h2>Заголовок</h2>
  <p>Абзац с <strong>важным</strong> словом.</p>
</article>

<!-- Булевый атрибут -->
<input type="checkbox" checked />
<button disabled>Недоступна</button>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **В чём разница между тегом и элементом?** — тег — синтаксис (`<p>`), элемент — концепция (открывающий тег + содержимое + закрывающий тег).
- **Что такое void-элементы и почему у них нет закрывающего тега?** — по спецификации они не могут иметь дочерних узлов.
- **Что произойдёт, если вложить `<p>` внутрь `<p>`?** — браузер автоматически закроет первый `<p>` перед открытием второго.
- **Как работают булевые атрибуты?** — наличие атрибута означает `true`, отсутствие — `false`; значение игнорируется.
- **Как получить доступ к элементу через JavaScript?** — через `document.querySelector`, `getElementById` и другие методы DOM API.

### Красные флаги (чего не говорить)

- «Тег и элемент — это одно и то же» — тег это лишь синтаксическая часть, элемент — полная структура.
- «Void-элементы нужно закрывать слешем `<br/>`» — в HTML5 слеш опционален и не обязателен (в отличие от XHTML).
- «Атрибут `disabled="false"` отключает атрибут disabled» — нет, для отключения нужно убрать атрибут полностью.

# Q025. Расскажите о meta-тэге с `name="viewport"`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<meta name="viewport">` управляет тем, как браузер масштабирует и отображает страницу на мобильных устройствах. Без него мобильный браузер рендерит страницу в виртуальный viewport шириной ~980px и масштабирует до экрана, что делает текст нечитаемым. Стандартное значение `content="width=device-width, initial-scale=1"` говорит браузеру использовать реальную ширину устройства и не масштабировать страницу при загрузке.

---

## Развёрнутый ответ

### Суть и определение

**Viewport** — видимая область браузера. На десктопе она совпадает с окном браузера. На мобильных устройствах различают:
- **Layout viewport** — виртуальная область, в которой браузер рендерит HTML
- **Visual viewport** — то, что пользователь видит на экране
- **Ideal viewport** — ширина, при которой контент выглядит наилучшим образом (обычно = ширина устройства в CSS-пикселях)

Мета-тег viewport был введён Apple в Safari для iPhone и стал стандартом де-факто.

### Как это работает

Без viewport-мета браузер использует layout viewport ≈ 980px, рендерит страницу для "десктопа" и уменьшает до размера экрана. Пользователь видит крошечный текст.

С `width=device-width` layout viewport = ширина устройства в CSS-пикселях (например, 375px для iPhone). CSS media queries начинают работать корректно.

### Практика и применение

**Стандартная строка для адаптивного дизайна:**
```html
<meta name="viewport" content="width=device-width, initial-scale=1">
```

**Параметры `content`:**

| Параметр | Значения | Описание |
|----------|----------|---------|
| `width` | `device-width` или число в px | Ширина layout viewport |
| `initial-scale` | `0.1` – `10` (обычно `1`) | Начальный масштаб при загрузке |
| `minimum-scale` | число | Минимальный масштаб при зуме |
| `maximum-scale` | число | Максимальный масштаб при зуме |
| `user-scalable` | `yes` / `no` | Разрешить ли пользователю зумировать |
| `viewport-fit` | `cover` / `contain` | Работа с notch (вырезом камеры) |

**Для устройств с notch (iPhone X+):**
```html
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">
```

**Пример неправильного использования:**
```html
<!-- НЕ делать так — блокирует zoom, проблема с доступностью -->
<meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=no, maximum-scale=1">
```

### Важные нюансы и краеугольные камни

- `user-scalable=no` и `maximum-scale=1` — нарушение доступности (WCAG 1.4.4). Пользователи со слабым зрением не смогут приблизить контент. iOS Safari с версии 10 игнорирует `user-scalable=no` для пользователей
- `width=device-width` нужен для корректной работы CSS media queries
- `initial-scale=1` предотвращает автоматическое масштабирование iOS при смене ориентации
- На десктопе viewport мета игнорируется — это только мобильная директива
- `viewport-fit=cover` + CSS `env(safe-area-inset-*)` позволяет контенту заходить за notch с корректными отступами

### Красные флаги (чего не говорить)

- «`user-scalable=no` — норма для мобильных приложений» — нарушение доступности
- «Viewport мета влияет на десктоп тоже» — нет, только мобильные
- «Без viewport мета страница не откроется» — откроется, просто отобразится некорректно

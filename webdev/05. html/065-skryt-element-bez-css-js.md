# Q065. Как можно скрыть элемент разметки не используя CSS и JS?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

В HTML есть несколько способов скрыть элемент без явного CSS и JS: атрибут `hidden`, `type="hidden"` для `<input>`, элемент `<details>` без атрибута `open`, атрибут `disabled` на форм-элементах и HTML-комментарии `<!-- -->`. Важный нюанс: атрибут `hidden` реализован через `display: none` в UA (User Agent) stylesheet — технически это CSS, просто встроенный в браузер.

---

## Развёрнутый ответ

### Суть и определение

«Скрыть без CSS и JS» означает использовать семантические HTML-механизмы, которые браузер обрабатывает нативно. На практике все они в итоге отражаются в стилях или DOM-состоянии, но декларируются через HTML-атрибуты.

### Как это работает

**1. Атрибут `hidden`**

Boolean-атрибут. Браузер применяет `display: none` через UA stylesheet (`[hidden] { display: none !important }` в некоторых браузерах). Элемент остаётся в DOM, но не рендерится и не занимает место.

```html
<p hidden>Этот текст не виден</p>
```

Важно: CSS с более высокой специфичностью может переопределить это поведение. `display: flex` на элементе с `hidden` сделает его видимым — потенциальная ловушка.

**2. `type="hidden"` для `<input>`**

Единственный тип input, который изначально невидим и не занимает место в layout. Значение отправляется вместе с формой.

```html
<input type="hidden" name="csrf_token" value="abc123" />
```

Не является «скрытием» визуального элемента — это отдельный тип данных без представления.

**3. `<details>` без атрибута `open`**

`<details>` по умолчанию показывает только `<summary>` и скрывает остальное содержимое. Без `open` контент свёрнут.

```html
<details>
  <summary>Показать детали</summary>
  <p>Скрытое содержимое (нет open)</p>
</details>
```

**4. Атрибут `disabled`**

`disabled` на форм-элементах (`<button>`, `<input>`, `<select>`, `<fieldset>`) делает их неинтерактивными и визуально приглушёнными, но не скрывает — элемент остаётся виден. Строго говоря, это не «скрытие».

**5. HTML-комментарий `<!-- -->`**

Контент внутри комментария не рендерится, но остаётся в источнике страницы (виден в DevTools). Подходит только для временного комментирования кода — не для сокрытия пользовательского контента.

```html
<!-- <p>Этот параграф закомментирован</p> -->
```

**6. `<template>`**

Содержимое `<template>` инертно — не рендерится, не загружает ресурсы. Семантически предназначен для хранения переиспользуемых фрагментов, а не для скрытия.

### Практика и применение

- `hidden` — временное скрытие элемента до момента активации (табы, аккордеоны без JS, постепенное раскрытие).
- `type="hidden"` — передача мета-данных в форме (CSRF-токены, ID записей).
- `<details>` — нативный аккордеон без единой строки JS.
- Комментарии — отладка разметки в dev-окружении.

### Важные нюансы и краеугольные камни

- `hidden` не является семантическим скрытием для скрин-ридеров в полной мере: элементы с `hidden` недоступны AT (Assistive Technology), что может быть нежелательно (например, для offscreen-контента). Используйте `aria-hidden="true"` если нужно скрыть только визуально.
- `display: none` (через `hidden`) удаляет элемент из accessibility tree.
- `visibility: hidden` и `opacity: 0` — CSS, а не HTML (элемент занимает место).
- `type="hidden"` input не скрывается от пользователя в смысле безопасности — значение видно в DevTools и HTML-источнике.
- Контент в `<!-- -->` комментариях индексируется некоторыми поисковиками — не используйте для конфиденциальной информации.

### Примеры

```html
<!-- 1. Атрибут hidden -->
<section id="step-2" hidden>
  <h2>Шаг 2</h2>
  <p>Заполните дополнительные данные.</p>
</section>

<!-- 2. input type="hidden" -->
<form action="/submit" method="post">
  <input type="hidden" name="user_id" value="42" />
  <input type="hidden" name="csrf" value="xK9p2mN..." />
  <button type="submit">Отправить</button>
</form>

<!-- 3. details без open — нативный аккордеон -->
<details>
  <summary>Подробнее о доставке</summary>
  <p>Доставка осуществляется в течение 3–5 рабочих дней.</p>
</details>

<!-- 4. Комментарий — только для разработки -->
<!-- TODO: временно скрыто до A/B теста
<div class="promo-banner">...</div>
-->
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Является ли `hidden` «настоящим» CSS-независимым скрытием? (нет — UA stylesheet)
- Как `hidden` взаимодействует с `display: flex`? (flex переопределит hidden — баг на практике)
- Чем `hidden` отличается от `aria-hidden`? (hidden убирает из дерева доступности, aria-hidden — только из AT)
- Почему `type="hidden"` нельзя считать «безопасным» скрытием? (видно в источнике и DevTools)
- Что происходит с содержимым `<details>` без open для SEO?

### Красные флаги (чего не говорить)

- «`hidden` — это чистый HTML без CSS» — технически реализуется через UA stylesheet.
- «Данные в `type="hidden"` скрыты от пользователя» — только визуально, не технически.
- «Комментарии скрывают контент от поисковиков» — не всегда.

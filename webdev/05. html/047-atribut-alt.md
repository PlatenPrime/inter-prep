# Q052. Для чего используется атрибут `alt` и зачем он нужен?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`alt` — обязательный атрибут элемента `<img>`, задающий альтернативный текст для изображения. Используется скринридерами для озвучивания контента незрячим пользователям, отображается при ошибке загрузки и учитывается поисковыми системами при индексации. Для декоративных изображений задаётся пустым (`alt=""`), что сигнализирует скринридерам о пропуске элемента.

---

## Развёрнутый ответ

### Суть и определение

`alt` (alternative text) решает три задачи одновременно:

1. **Доступность (Accessibility)**: скринридеры (NVDA, VoiceOver, JAWS) озвучивают `alt` вместо изображения для незрячих пользователей
2. **Деградация (Fallback)**: текст отображается когда изображение не загрузилось (медленный интернет, неверный URL, блокировщик изображений)
3. **SEO**: поисковые роботы не «видят» изображения — `alt` описывает их содержимое для индексации

**Требования WCAG 2.1 (Web Content Accessibility Guidelines):** все содержательные изображения обязаны иметь текстовую альтернативу (уровень A).

### Как это работает

Браузер использует `alt` в трёх сценариях:
- Рендеринг текста на месте изображения при ошибке загрузки
- Передача текста в Accessibility Tree (дерево доступности) → скринридер
- Предоставление метаданных для поисковых роботов

```
<img alt="">          → скринридер: пропустить элемент (декоративный)
<img>                 → скринридер: озвучит URL/имя файла (плохо!)
<img alt="описание">  → скринридер: "описание"
```

### Практика и применение

**Классификация изображений по типу alt:**

| Тип изображения                    | Рекомендация                             |
|-------------------------------------|------------------------------------------|
| Информационное (содержит контент)   | Описательный `alt`                       |
| Функциональное (кнопка/ссылка)      | `alt` описывает действие, а не внешний вид |
| Декоративное (фон, разделитель)     | `alt=""`                                 |
| Текстовое изображение               | `alt` дублирует текст на картинке        |
| Сложное (график, диаграмма)         | Краткий `alt` + развёрнутое описание рядом |

```html
<!-- Информационное изображение -->
<img src="product.jpg" alt="Беспроводные наушники Sony WH-1000XM5, чёрные" />

<!-- Функциональная ссылка-изображение -->
<a href="/">
  <img src="logo.svg" alt="На главную страницу" />
</a>

<!-- Декоративное изображение -->
<img src="divider.png" alt="" role="presentation" />

<!-- Текстовое изображение -->
<img src="sale-banner.png" alt="Скидка 50% до 31 декабря" />

<!-- Иконка кнопки -->
<button>
  <img src="search-icon.svg" alt="" />
  Поиск
</button>
```

### Важные нюансы и краеугольные камни

- **`alt` без значения vs без атрибута**: `alt=""` — сознательное решение «декоративное»; отсутствие `alt` — ошибка валидации HTML и плохой UX для скринридеров
- **Избегайте «картинка», «изображение» в `alt`** — скринридеры и так говорят «изображение» перед `alt`
- **Длина alt**: рекомендуется до 150 символов; для сложного контента используйте `longdesc` (устарел) или `aria-describedby` со скрытым описанием
- **CSS background-image** не имеет `alt` — только для декоративных изображений; контентные изображения должны быть в `<img>`
- **`alt` влияет на CLS (Cumulative Layout Shift)**: всегда задавайте `width` и `height` атрибуты для предотвращения сдвига контента
- **SVG inline**: используйте `<title>` и `aria-labelledby` вместо `alt`

### Примеры

```html
<!-- Карточка товара с правильной доступностью -->
<article class="product-card">
  <img
    src="/images/headphones.webp"
    alt="Sony WH-1000XM5 — накладные наушники с шумоподавлением"
    width="300"
    height="300"
    loading="lazy"
  />
  <h3>Sony WH-1000XM5</h3>
  <p>₽ 29 990</p>
  <button>
    <img src="/icons/cart.svg" alt="" aria-hidden="true" />
    Добавить в корзину
  </button>
</article>

<!-- Сложная диаграмма с развёрнутым описанием -->
<figure>
  <img
    src="sales-chart.png"
    alt="График продаж за Q1 2026"
    aria-describedby="chart-desc"
  />
  <figcaption id="chart-desc">
    Продажи выросли с 1 200 единиц в январе до 3 400 в марте.
    Пик — февральская акция (2 800 единиц).
  </figcaption>
</figure>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Чем отличается `alt=""` от отсутствия атрибута `alt`?
- Когда `alt` описывает действие, а не содержимое?
- Как обеспечить доступность для сложных изображений (графиков)?
- Почему CSS `background-image` не используют для контентных изображений?
- Как `alt` влияет на Core Web Vitals (CLS)?

### Красные флаги (чего не говорить)

- «`alt` нужен только для SEO» — в первую очередь для доступности
- «Для декоративных изображений `alt` не нужен» — нужен, но пустой (`alt=""`)
- «В `alt` надо писать "Изображение..."» — скринридер и так объявляет тип элемента

# Q053. Разница между `<canvas>` и `<svg>`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<canvas>` — растровый пиксельный буфер: JS рисует пиксели напрямую, DOM не знает о нарисованных объектах. `<svg>` — векторная графика в DOM: каждая фигура является узлом с атрибутами, стилями и событиями. Canvas масштабируется с потерей качества, SVG — нет. Canvas эффективнее при тысячах динамических объектов; SVG — при небольшом количестве сложных интерактивных фигур.

---

## Развёрнутый ответ

### Суть и определение

Оба элемента предназначены для графики, но реализуют принципиально разные подходы:

- **Canvas** — это *immediate mode rendering*: команды выполняются немедленно, состояние сцены нигде не хранится.
- **SVG** — это *retained mode rendering*: браузер хранит «сцену» как дерево DOM-узлов и сам управляет перерисовкой при изменениях.

### Как это работает

**Canvas (immediate mode):**
- Пиксели записываются в буфер при каждом вызове API.
- Нет понятия «объект» — только пиксели.
- Анимация требует явного цикла: `clearRect` → перерисовка каждого кадра.
- Hit-testing реализуется вручную (математически или через offscreen canvas).

**SVG (retained mode):**
- Каждый `<circle>`, `<path>`, `<rect>` — узел DOM.
- Браузер самостоятельно перерисовывает только изменённые узлы.
- Можно навешивать CSS, `:hover`, JavaScript listeners на отдельные фигуры.
- Браузер сам масштабирует векторные кривые при любом разрешении.

### Практика и применение

**Canvas подходит когда:**
- Большое количество объектов (>1000) обновляется каждый кадр (игры, симуляции частиц).
- Нужна пиксельная обработка (фильтры, видео-эффекты, watermark).
- Используется WebGL для 3D.

**SVG подходит когда:**
- Небольшое число фигур с CSS-анимацией или интерактивностью.
- Требуется масштабируемость (иконки, логотипы, print).
- Важна доступность (screen readers читают `<title>`, `<desc>`).
- Контент должен индексироваться или сериализоваться как текст.

### Важные нюансы и краеугольные камни

- **Масштабирование:** SVG масштабируется без потерь через `viewBox`; canvas при масштабировании через CSS растягивает растр.
- **Доступность:** SVG доступен для AT (assistive technology) из коробки; canvas — «чёрный ящик» для screen readers.
- **Производительность DOM:** Сотни SVG-узлов создают нагрузку на layout engine; canvas — один DOM-узел независимо от сложности сцены.
- **Сериализация:** SVG — текстовый XML, можно сохранить/индексировать; canvas экспортируется только в bitmap (`toDataURL`, `toBlob`).
- **CORS:** canvas «отравляется» cross-origin изображениями без CORS-заголовков.

### Примеры

```html
<!-- SVG: интерактивный круг, hover через CSS -->
<svg width="200" height="200" viewBox="0 0 200 200">
  <circle cx="100" cy="100" r="80" fill="#4f46e5" style="cursor:pointer"
          onmouseenter="this.setAttribute('fill','#7c3aed')"
          onmouseleave="this.setAttribute('fill','#4f46e5')" />
</svg>

<!-- Canvas: тот же круг, hover только через JS hit-test -->
<canvas id="c" width="200" height="200"></canvas>
<script>
  const canvas = document.getElementById('c');
  const ctx = canvas.getContext('2d');

  function draw(color) {
    ctx.clearRect(0, 0, 200, 200);
    ctx.beginPath();
    ctx.arc(100, 100, 80, 0, Math.PI * 2);
    ctx.fillStyle = color;
    ctx.fill();
  }

  draw('#4f46e5');

  canvas.addEventListener('mousemove', (e) => {
    const rect = canvas.getBoundingClientRect();
    const x = e.clientX - rect.left - 100;
    const y = e.clientY - rect.top - 100;
    draw(Math.sqrt(x * x + y * y) < 80 ? '#7c3aed' : '#4f46e5');
  });
</script>
```

## Сравнение

| Критерий | `<canvas>` | `<svg>` |
|---|---|---|
| Модель рендеринга | Immediate mode (пиксели) | Retained mode (DOM) |
| Тип графики | Растровая (bitmap) | Векторная |
| Масштабирование | С потерей качества | Без потерь (`viewBox`) |
| DOM-узлы | Один элемент | Узел на каждую фигуру |
| CSS-стилизация | Нет (только JS) | Полная поддержка |
| События на объектах | Ручной hit-test | Нативные DOM events |
| Доступность (a11y) | Ручная реализация | Встроенная (`<title>`, `<desc>`) |
| Производительность | Лучше при >1000 объектов | Лучше при <100 объектов |
| Сериализация | Только bitmap (`toDataURL`) | Текст/XML |
| 3D-поддержка | WebGL / WebGL2 | Нет (только 2D) |
| Анимация | `requestAnimationFrame` вручную | CSS, SMIL, JS |
| Индексация поисковиком | Нет | Текстовый контент индексируется |

## На что обратить внимание на собеседовании

### Каверзные follow-up

- При каком количестве объектов SVG начинает «тормозить» и почему? (сотни узлов → нагрузка на layout и repaint)
- Как реализовать tooltip при hover на конкретную «фигуру» в canvas? (математический hit-test, offscreen canvas с цветовыми метками)
- Можно ли комбинировать SVG и canvas? (да — `drawImage(svgImage, ...)` или `<foreignObject>`)
- Что такое `viewBox` и чем отличается от `width`/`height`? (viewport vs coordinate system)
- Как экспортировать SVG в PNG? (serializer → blob → canvas → toDataURL)

### Красные флаги (чего не говорить)

- «Canvas можно стилизовать через CSS как обычный элемент» — только сам элемент, не его содержимое.
- «SVG работает медленнее canvas в любом сценарии» — при малом числе объектов SVG эффективнее.
- «Canvas поддерживает векторное масштабирование» — нет, только растровое.

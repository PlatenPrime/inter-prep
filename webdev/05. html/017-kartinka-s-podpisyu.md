# Q017. Как семантически правильно сверстать картинку с подписью?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Для картинки с подписью используется пара `<figure>` + `<figcaption>`. `<figure>` — самодостаточный блок иллюстративного контента (изображение, диаграмма, код, видео), `<figcaption>` — его подпись. Это семантически связывает визуальный контент с описанием, что понимают и скринридеры, и поисковые роботы. Альтернатива через `<div>` + `<p>` лишена этой связи.

---

## Развёрнутый ответ

### Суть и определение

`<figure>` — HTML5-элемент для самодостаточного иллюстративного контента, на который ссылается основной поток документа, но который можно переместить в приложение, боковую колонку или другое место без нарушения смысла документа.

`<figcaption>` — необязательный элемент, первый или последний дочерний элемент `<figure>`, предоставляющий подпись или легенду к содержимому `<figure>`.

### Как это работает

`<figure>` имеет неявную ARIA-роль `figure`. Если внутри есть `<figcaption>`, браузер автоматически связывает их через accessible name — скринридер объявит «figure» и прочитает подпись как его метку.

Если `<figcaption>` отсутствует, но нужно описание для AT (Assistive Technologies), используют `aria-label` или `aria-labelledby` на `<figure>`.

### Практика и применение

- **Изображения с подписью**: фотографии в статьях, диаграммы в документации.
- **Блоки кода с описанием**: `<figure>` может содержать `<pre><code>...</code></pre>` с `<figcaption>`.
- **Видео и аудио**: `<figure>` вокруг `<video>` с подписью.
- **Цитаты**: `<figure>` + `<blockquote>` + `<figcaption>` с указанием автора — семантически корректный способ оформить цитату с атрибуцией.
- **`alt` vs `figcaption`**: `alt` — это текстовая замена изображения (для случаев, когда оно не загрузилось), `figcaption` — видимая подпись для всех пользователей. Оба могут присутствовать одновременно.

### Важные нюансы и краеугольные камни

- **`figcaption` должен быть первым или последним дочерним**: нельзя размещать его в середине `<figure>`.
- **`alt` и `figcaption` не должны дублировать друг друга**: если `figcaption` полностью описывает изображение, `alt` можно сделать пустым (`alt=""`), чтобы скринридер не читал описание дважды.
- **`<figure>` ≠ обёртка для любого изображения**: не каждое `<img>` нужно оборачивать в `<figure>`. Используй только если изображение является самостоятельной иллюстрацией с подписью или без, на которую ссылается текст.
- **Декоративные изображения**: иконки и декоративные фоны не нужны в `<figure>` — используй `alt=""` и не оборачивай.
- **Цитаты**: спецификация явно рекомендует `<figure>` для цитат с атрибуцией.

### Примеры

```html
<!-- Базовый вариант: изображение с подписью -->
<figure>
  <img
    src="architecture-diagram.webp"
    alt="Схема микросервисной архитектуры с API Gateway и тремя сервисами"
    width="800"
    height="450"
  />
  <figcaption>Рис. 1. Микросервисная архитектура с централизованным API Gateway</figcaption>
</figure>

<!-- Figcaption полностью описывает изображение → alt пустой -->
<figure>
  <img src="chart.webp" alt="" width="600" height="400" />
  <figcaption>
    График роста продаж за 2025–2026 год: январь — 100 ед., декабрь — 340 ед.
  </figcaption>
</figure>

<!-- Блок кода с подписью -->
<figure>
  <pre><code>const greet = (name) => `Hello, ${name}!`;</code></pre>
  <figcaption>Пример стрелочной функции в JavaScript</figcaption>
</figure>

<!-- Цитата с атрибуцией -->
<figure>
  <blockquote>
    <p>Любая достаточно сложная технология неотличима от магии.</p>
  </blockquote>
  <figcaption>— Артур Кларк, <cite>Profiles of the Future</cite></figcaption>
</figure>

<!-- НЕ нужна figure: декоративная иконка в кнопке -->
<button type="button">
  <img src="icon-close.svg" alt="" aria-hidden="true" />
  <span>Закрыть</span>
</button>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- В чём разница между `alt` и `figcaption`? (`alt` — текстовая замена изображения, `figcaption` — видимая подпись для всех)
- Когда `alt` следует оставить пустым при наличии `figcaption`? (когда figcaption полностью описывает изображение — дублирование вредит accessibility)
- Какую ARIA-роль имеет `<figure>`? (`figure`)
- Можно ли использовать `<figure>` без `<figcaption>`? (да, если изображение самодостаточно и подписи нет)
- Где должен располагаться `<figcaption>` внутри `<figure>`? (первым или последним дочерним элементом)

### Красные флаги (чего не говорить)

- «Для картинки с подписью достаточно `<div>` + `<p>`» — это лишает семантической связи между изображением и подписью.
- «`alt` и `figcaption` — одно и то же» — они разные по природе и назначению.
- «`<figure>` нужна вокруг каждого `<img>`» — только для иллюстративного контента с контекстуальной ролью.

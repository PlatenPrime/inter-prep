# Q012. Какие проблемы могут возникать при использовании `<div>` вместо семантических элементов?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<div>` — нейтральный контейнер без семантической нагрузки и без встроенной ARIA-роли. Массовое использование `<div>` вместо семантических тегов приводит к «div soup» — разметке, непонятной для скринридеров, поисковиков и разработчиков. Пользователи с нарушениями зрения теряют возможность навигации по ориентирам страницы. SEO-роботы не могут определить иерархию и важность контента.

---

## Развёрнутый ответ

### Суть и определение

«Div soup» (дословно «суп из дивов») — антипаттерн верстки, при котором структура страницы строится исключительно из вложенных `<div>` и `<span>`, а смысл блоков передаётся только через классы CSS. Элемент `<div>` имеет нейтральную роль `generic` в дереве доступности и не несёт никакой семантической информации.

### Как это работает

Браузер строит два дерева: **DOM** и **Accessibility Tree**. `<div>` не создаёт запись в дереве доступности как ориентир (landmark). Скринридер, перемещаясь по странице, не видит разницы между шапкой, основным контентом и сайдбаром, если все они — `<div>`.

Поисковые роботы (Google Crawler) анализируют семантическую структуру для определения веса контента. `<h1>` внутри `<main>` воспринимается как ключевой заголовок страницы; `<h1>` внутри `<div class="main">` — просто заголовок в блоке.

### Практика и применение

- **Скринридеры**: пользователи NVDA/JAWS/VoiceOver не могут использовать landmark-навигацию (быстрый переход между `<nav>`, `<main>`, `<aside>`). Им приходится последовательно прослушивать весь контент.
- **SEO**: без `<article>`, `<main>`, `<header>` поисковик сложнее определяет основной контент и его важность. Это влияет на ранжирование.
- **Поддержка кода**: `<div class="navbar">` может быть переименован или описан неточно — нет гарантии соответствия CSS-класса реальной роли блока.
- **Браузерные стили**: семантические элементы имеют полезные стили по умолчанию (например, `<ul>` + `<li>` с отступами), которые теряются при использовании `<div>`.

### Важные нюансы и краеугольные камни

- **`<div>` с `role` как компромисс**: если по каким-то причинам нельзя изменить тег, можно добавить ARIA-роль (`<div role="navigation">`) — но это хуже семантического тега, так как не даёт всех преимуществ нативного элемента (фокусируемость, стили по умолчанию и т.д.).
- **Первое правило ARIA**: не используй ARIA там, где можно использовать нативный HTML-элемент.
- **Проблемы с клавиатурной навигацией**: `<div>` не фокусируемый элемент по умолчанию. `<button>` и `<a>` фокусируются нативно; кликабельный `<div>` требует `tabindex="0"` и обработчиков `keydown` для Enter/Space.
- **Интерактивный `<div>` — почти всегда ошибка**: вместо `<div onclick="...">` использовать `<button>` или `<a>` — они семантичны, фокусируемы и доступны из коробки.

### Примеры

```html
<!-- Проблема: div soup — скринридер не видит структуру -->
<div class="header">
  <div class="logo">...</div>
  <div class="nav">
    <div class="nav-item"><div onclick="go('/home')">Главная</div></div>
  </div>
</div>
<div class="content">
  <div class="article">
    <div class="title">Заголовок</div>
  </div>
</div>

<!-- Решение: семантическая разметка -->
<header>
  <a href="/" aria-label="На главную">
    <img src="logo.svg" alt="Название компании" />
  </a>
  <nav aria-label="Основная навигация">
    <ul>
      <li><a href="/home" aria-current="page">Главная</a></li>
    </ul>
  </nav>
</header>
<main>
  <article>
    <h1>Заголовок</h1>
  </article>
</main>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Что такое первое правило ARIA? (используй нативные HTML-элементы вместо ARIA, где возможно)
- Как сделать `<div>` доступным для клавиатурных пользователей? (`tabindex="0"` + обработчики `keydown`)
- Влияет ли div soup на производительность рендеринга? (незначительно, но глубокое DOM-дерево увеличивает время layout)
- Какой инструмент поможет проверить дерево доступности? (DevTools → Accessibility панель, axe, Lighthouse)

### Красные флаги (чего не говорить)

- «`<div>` с правильным классом работает так же, как семантический тег» — это неверно с точки зрения доступности.
- «SEO не зависит от HTML-тегов» — зависит, семантика влияет на понимание структуры контента поисковиком.
- «Добавлю `role` к `<div>` и будет то же самое» — нативные элементы дают больше преимуществ.

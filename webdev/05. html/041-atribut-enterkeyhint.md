# Q046. Для чего используется атрибут `enterkeyhint`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`enterkeyhint` — атрибут для текстовых полей ввода, который подсказывает браузеру, какой текст или иконку отобразить на кнопке Enter виртуальной клавиатуры. Поддерживает значения: `enter`, `done`, `go`, `next`, `previous`, `search`, `send`. Не влияет на поведение — только на внешний вид клавиши. Улучшает мобильный UX, давая пользователю визуальный контекст действия.

## Развёрнутый ответ

### Суть и определение

`enterkeyhint` — глобальный HTML-атрибут (применим ко всем `contenteditable` элементам и полям ввода), который является подсказкой для пользовательского агента об отображении кнопки подтверждения на виртуальной клавиатуре. Стандартизирован в HTML Living Standard.

### Как это работает

Браузер читает значение атрибута и заменяет стандартный символ Enter на соответствующий текст или иконку:

| Значение   | Типичное отображение          |
|------------|-------------------------------|
| `enter`    | ↵ (возврат каретки)           |
| `done`     | «Готово» / «Done»             |
| `go`       | «Перейти» / «Go»              |
| `next`     | «Далее» / «Next»              |
| `previous` | «Назад» / «Previous»          |
| `search`   | 🔍 / «Найти» / «Search»       |
| `send`     | «Отправить» / «Send»          |

Реализация зависит от ОС и браузера — iOS Safari, Android Chrome показывают разные варианты.

### Практика и применение

- **Многошаговые формы**: на промежуточных полях ставят `next`, на последнем — `done` или `send`
- **Поиск**: поле поиска получает `search` → клавиатура показывает иконку лупы
- **Чат/мессенджер**: поле сообщения с `send` сигнализирует о действии отправки
- **Логин-форма**: поле пароля с `go` или `done`

```html
<!-- Многошаговая форма -->
<input type="text" name="firstName" enterkeyhint="next" placeholder="Имя" />
<input type="text" name="lastName" enterkeyhint="next" placeholder="Фамилия" />
<input type="email" name="email" enterkeyhint="done" placeholder="Email" />

<!-- Поле поиска -->
<input type="search" enterkeyhint="search" placeholder="Поиск..." />

<!-- Чат -->
<textarea enterkeyhint="send"></textarea>
```

### Важные нюансы и краеугольные камни

- Атрибут **не меняет поведение** клавиши — только внешний вид. Если нужно перехватить нажатие Enter, используйте JS-обработчики
- Работает только на **виртуальных клавиатурах** (мобильные устройства, планшеты); на десктопе игнорируется
- Применим не только к `<input>`, но и к `<textarea>` и элементам с `contenteditable="true"`
- Поддержка: все современные мобильные браузеры; Chrome 77+, Safari 13.1+, Firefox 94+
- Не путать с `inputmode` — тот управляет **типом** клавиатуры, `enterkeyhint` — только кнопкой Enter

### Примеры

```html
<!-- Форма авторизации -->
<form>
  <input
    type="email"
    name="email"
    enterkeyhint="next"
    inputmode="email"
    placeholder="Email"
  />
  <input
    type="password"
    name="password"
    enterkeyhint="go"
    placeholder="Пароль"
  />
</form>

<!-- contenteditable -->
<div contenteditable="true" enterkeyhint="send" role="textbox"></div>
```

- «Атрибут меняет действие при нажатии Enter» — нет, только отображение

- «Это нестандартный / экспериментальный атрибут» — стандартизирован в HTML Living Standard

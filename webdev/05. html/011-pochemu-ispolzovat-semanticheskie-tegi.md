# Q011. Почему стоит использовать семантические тэги в верстке?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Семантические теги (`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`) явно описывают роль блока в документе. Это улучшает SEO — поисковые движки лучше понимают структуру страницы. Скринридеры используют семантику для навигации по странице, что критично для accessibility. Код становится самодокументируемым и легче сопровождается командой.

---

## Развёрнутый ответ

### Суть и определение

Семантический HTML (Semantic HTML) — подход, при котором выбор тега отражает **смысловую роль** контента, а не только его внешний вид. HTML5 ввёл набор новых семантических элементов, заменяющих универсальные `<div>` и `<span>` в смысловых контекстах.

### Как это работает

Браузер строит **дерево доступности (Accessibility Tree)** параллельно с DOM. Семантические элементы автоматически получают встроенные ARIA-роли:

| Элемент | Неявная ARIA-роль |
|---------|------------------|
| `<header>` | `banner` (внутри `<body>`) |
| `<nav>` | `navigation` |
| `<main>` | `main` |
| `<article>` | `article` |
| `<section>` | `region` (при наличии accessible name) |
| `<aside>` | `complementary` |
| `<footer>` | `contentinfo` (внутри `<body>`) |

Эти роли передаются вспомогательным технологиям (AT — Assistive Technologies) без дополнительных атрибутов.

### Практика и применение

- **SEO**: Google, Bing и другие поисковики используют семантическую структуру для понимания иерархии контента. `<main>` сигнализирует о главном контенте страницы, `<article>` — о самостоятельной единице контента.
- **Accessibility**: скринридеры (NVDA, JAWS, VoiceOver) предоставляют пользователям навигацию по ориентирам (landmark navigation) — прыжки между `<nav>`, `<main>`, `<aside>` нажатием одной клавиши.
- **Читаемость кода**: `<footer>` однозначнее, чем `<div class="footer">` — нет риска расхождения имени класса и реального назначения.
- **Стили по умолчанию**: браузеры применяют базовые стили к семантическим элементам, что помогает при прогрессивном улучшении (Progressive Enhancement).

### Важные нюансы и краеугольные камни

- **Не все семантические теги создают landmark-роли**: `<section>` получает роль `region` только при наличии `aria-label` или `aria-labelledby`, иначе остаётся без роли в дереве доступности.
- **Вложенность `<header>` и `<footer>`**: внутри `<article>` или `<section>` они не имеют ролей `banner`/`contentinfo` — это локальные заголовок/подвал секции.
- **Семантика ≠ внешний вид**: семантический тег не определяет CSS-отображение. `<article>` это не блочный элемент сам по себе в смысле бизнес-логики — он лишь указывает на самодостаточность контента.
- **Переиспользование ARIA**: не нужно дублировать роль атрибутом `role`, если тег уже её несёт (`<nav role="navigation">` — избыточно).

### Примеры

```html
<!-- Плохо: div soup -->
<div class="header">
  <div class="nav">...</div>
</div>
<div class="main">
  <div class="article">...</div>
  <div class="sidebar">...</div>
</div>
<div class="footer">...</div>

<!-- Хорошо: семантическая разметка -->
<header>
  <nav aria-label="Основная навигация">...</nav>
</header>
<main>
  <article>...</article>
  <aside aria-label="Связанные материалы">...</aside>
</main>
<footer>...</footer>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Какую ARIA-роль имеет `<section>` без `aria-label`? (никакой — не становится landmark)
- Может ли быть несколько `<main>` на странице? (нет, должен быть один видимый `<main>`)
- В чём разница между `<header>` как дочерним элементом `<body>` и как дочерним элементом `<article>`?
- Как семантика влияет на позицию в поисковой выдаче?
- Зачем нужен `<aside>`, если можно использовать `<div class="sidebar">`?

### Красные флаги (чего не говорить)

- «Семантика нужна только для SEO» — это неполный ответ, accessibility важнее.
- «`<div>` и семантические теги работают одинаково» — они имеют разные ARIA-роли.
- «Нужно добавить `role="navigation"` к `<nav>`» — это избыточно.

# Q040. Для чего используется атрибут `autofocus`? Где он применяется?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`autofocus` автоматически переводит фокус на элемент при загрузке страницы (или при открытии диалога). Применяется на интерактивных элементах: `<input>`, `<textarea>`, `<select>`, `<button>`. На странице должен быть только один элемент с `autofocus`; при множественном использовании фокус получает последний по порядку в DOM. При неосторожном применении создаёт серьёзные проблемы с доступностью (accessibility).

---

## Развёрнутый ответ

### Суть и определение

`autofocus` — булев атрибут (присутствие = `true`). Он является декларативным способом указать браузеру, какой элемент должен получить фокус сразу после завершения разбора DOM. Это эквивалентно программному вызову `element.focus()` после загрузки страницы, но более семантично и работает без JavaScript.

### Как это работает

```html
<input type="search" autofocus placeholder="Поиск..." />
```

1. Браузер парсит HTML-документ
2. После разбора обнаруживает элемент с `autofocus`
3. Устанавливает фокус на этот элемент
4. На мобильных устройствах это также **открывает виртуальную клавиатуру**

**Применяется на:**
- `<input>` (все типы кроме `hidden`)
- `<textarea>`
- `<select>`
- `<button>`
- `<a>` (не часто, но технически допустимо)
- `<dialog>` — фокусируется первый focusable элемент внутри

### Практика и применение

- Страница логина / поиска: фокус на главном поле без клика
- Модальные окна (`<dialog>`): автофокус на первом действии (кнопке подтверждения или поле ввода)
- Одностраничные формы: пользователь сразу может начинать вводить данные
- Страница 404 или результатов поиска: фокус на поле поиска

### Важные нюансы и краеугольные камни

- На странице допускается **только один** элемент с `autofocus`. Если их несколько — фокус получит последний в DOM-порядке (поведение не специфицировано строго, зависит от браузера)
- На мобильных устройствах `autofocus` вызывает появление виртуальной клавиатуры — это **нежелательно** на большинстве страниц, так как сдвигает viewport и скрывает контент
- **Accessibility-проблема:** скринридеры начинают озвучивание с элемента, получившего фокус. Если autofocus стоит на поле в середине страницы — пользователь скринридера потеряет весь контент до него. Безопасно только если элемент с `autofocus` — первый значимый элемент страницы
- В `<dialog>`: `autofocus` на кнопке «Закрыть» или деструктивном действии — плохая практика. Лучше на первом поле ввода или кнопке «Подтвердить»
- Переопределить через JS: если `autofocus` нежелателен в конкретном сценарии — `element.blur()` после загрузки или использование `dialog.showModal()` который сам управляет фокусом
- Атрибут поддерживается всеми современными браузерами

### Примеры

```html
<!-- Страница поиска — фокус на поле поиска -->
<header>
  <h1>Поиск товаров</h1>
  <form role="search">
    <label for="search-query">Что ищете?</label>
    <input type="search" id="search-query" name="q" autofocus placeholder="Введите запрос..." />
    <button type="submit">Найти</button>
  </form>
</header>

<!-- Модальный диалог с autofocus на поле ввода -->
<dialog id="rename-dialog" aria-labelledby="dialog-title">
  <h2 id="dialog-title">Переименовать файл</h2>
  <form method="dialog">
    <label for="new-name">Новое название</label>
    <input type="text" id="new-name" name="name" autofocus required />
    <div>
      <button type="submit">Переименовать</button>
      <button type="button" onclick="this.closest('dialog').close()">Отмена</button>
    </div>
  </form>
</dialog>

<!-- Неправильно: autofocus на кнопке деструктивного действия -->
<!-- <button autofocus>Удалить аккаунт</button> — так делать не нужно -->
```

```js
// Управление фокусом в SPA при открытии модального окна
function openModal(dialog) {
  dialog.showModal();
  // showModal() автоматически фокусирует первый focusable элемент
  // или элемент с autofocus внутри dialog
}
```

### Красные флаги (чего не говорить)

- «`autofocus` всегда улучшает UX» — на мобильных вызывает клавиатуру и сдвигает контент
- «Можно поставить `autofocus` на скрытый input» — `type="hidden"` не focusable
- «Несколько `autofocus` на странице — нормально» — нестандартное поведение, зависит от браузера

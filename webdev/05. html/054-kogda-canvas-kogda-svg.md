# Q054. В каких случаях лучше использовать `<canvas>`, а в каких `<svg>`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Используйте canvas, когда нужно рендерить тысячи динамически обновляемых объектов, обрабатывать пиксели или использовать WebGL. Используйте SVG, когда важны масштабируемость, доступность, CSS-стилизация и интерактивность небольшого числа фигур. Главный критерий выбора — количество объектов на сцене и необходимость их независимой интерактивности.

## Развёрнутый ответ

### Суть и определение

Выбор между canvas и SVG определяется несколькими ключевыми факторами: количество объектов, частота обновления, необходимость интерактивности каждого объекта, требования к масштабируемости и доступности.

### Как это работает

**Canvas** — immediate mode рендеринг. Один DOM-узел, вся сложность в пикселях. Браузер не знает о «форме» нарисованных объектов — это просто пиксели. Эффективен, когда нужно рисовать много объектов быстро.

**SVG** — retained mode рендеринг. Каждая фигура — DOM-узел. Браузер управляет перерисовкой. Эффективен, когда сцена относительно статична или изменяется точечно.

### Практика и применение

**Используйте `<canvas>` для:**
- **Игр** (Phaser, PixiJS) — сотни/тысячи спрайтов, обновление 60fps.
- **Real-time data visualization** — live-графики с большим потоком данных.
- **Image processing** — фильтры, кроп, color grading на стороне клиента.
- **Particle systems** — эффекты частиц (снег, огонь, дым).
- **Video effects** — попиксельная обработка кадров из `<video>`.
- **WebGL / 3D** — Three.js, Babylon.js.
- **Heatmaps** с тысячами точек.

**Используйте `<svg>` для:**
- **Иконок и логотипов** — векторное масштабирование, Retina без артефактов.
- **Интерактивных диаграмм** — D3.js, Recharts в SVG-режиме.
- **Инфографики** с hover-эффектами на отдельных элементах.
- **Карт** с интерактивными регионами.
- **Анимированных иллюстраций** (Lottie, CSS-анимации).
- **Документов/принта** — SVG масштабируется для любого DPI.
- **Когда важна доступность** — screen readers читают SVG-контент.

### Важные нюансы и краеугольные камни

- **Порог производительности:** при >~500 независимо обновляемых DOM-узлах SVG начинает ощутимо тормозить, canvas сохраняет стабильный FPS.
- **Гибридный подход:** можно использовать оба — например, статичный SVG для UI, canvas для анимированного фона.
- **D3.js** поддерживает оба режима (SVG-default и canvas-mode) — выбирайте canvas-режим для больших датасетов.
- **Three.js, Babylon.js** используют canvas с WebGL для 3D.
- **Lottie** работает поверх SVG или canvas (зависит от рендерера).

### Примеры

```html
<!-- Canvas: particle system — 5000 частиц, 60fps -->
<canvas id="particles" width="800" height="600"></canvas>
<script>
  const canvas = document.getElementById('particles');
  const ctx = canvas.getContext('2d');
  const particles = Array.from({ length: 5000 }, () => ({
    x: Math.random() * 800,
    y: Math.random() * 600,
    vx: (Math.random() - 0.5) * 2,
    vy: (Math.random() - 0.5) * 2,
  }));

  function animate() {
    ctx.clearRect(0, 0, 800, 600);
    ctx.fillStyle = 'rgba(79, 70, 229, 0.8)';
    for (const p of particles) {
      p.x += p.vx;
      p.y += p.vy;
      if (p.x < 0 || p.x > 800) p.vx *= -1;
      if (p.y < 0 || p.y > 600) p.vy *= -1;
      ctx.beginPath();
      ctx.arc(p.x, p.y, 1.5, 0, Math.PI * 2);
      ctx.fill();
    }
    requestAnimationFrame(animate);
  }
  animate();
</script>

<!-- SVG: интерактивная карта с hover на регионах -->
<svg viewBox="0 0 800 600" xmlns="http://www.w3.org/2000/svg">
  <style>
    .region { fill: #e0e7ff; transition: fill 0.2s; cursor: pointer; }
    .region:hover { fill: #4f46e5; }
  </style>
  <path class="region" d="M100,100 L300,100 L300,250 L100,250 Z">
    <title>Регион Север</title>
  </path>
  <path class="region" d="M320,100 L500,100 L500,250 L320,250 Z">
    <title>Регион Юг</title>
  </path>
</svg>
```

## Сравнение

| Сценарий | Лучший выбор | Причина |
|---|---|---|
| Браузерная игра (60fps) | Canvas | Много объектов, нет DOM-оверхеда |
| Иконки и логотипы | SVG | Масштабируемость, Retina |
| Real-time график (>1000 точек) | Canvas | Скорость рендеринга |
| Интерактивная диаграмма (D3) | SVG | Hover на отдельных элементах |
| Обработка изображений | Canvas | Доступ к пикселям |
| Карта с регионами | SVG | Нативные click/hover события |
| 3D-рендеринг | Canvas (WebGL) | WebGL API только в canvas |
| Анимированная инфографика | SVG | CSS-анимации, доступность |
| Видео-эффекты | Canvas | `drawImage` из `<video>` |
| Печатная графика / PDF | SVG | Векторное масштабирование |

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Как бы вы реализовали real-time chart с обновлением 30 раз в секунду — SVG или canvas? Почему?
- Начиная с какого количества DOM-узлов SVG теряет производительность? (~300–500 обновляемых узлов)
- Можно ли использовать D3.js с canvas? (да, D3 поддерживает canvas-рендеринг)
- Как реализовать tooltip при hover на конкретный объект в canvas? (hit-test по координатам)
- Как Lottie выбирает между SVG и canvas рендерером?

### Красные флаги (чего не говорить)

- «Canvas всегда быстрее SVG» — при малом числе статичных объектов SVG не уступает или даже быстрее.
- «SVG нельзя анимировать» — можно через CSS, SMIL и JS.
- «Canvas автоматически поддерживает разные разрешения экранов» — нет, нужна ручная работа с `devicePixelRatio`.

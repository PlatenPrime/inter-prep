# Q001. Что такое HTML и для чего он используется?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

HTML (HyperText Markup Language) — язык разметки, с помощью которого браузер понимает структуру и смысл содержимого веб-страницы. Он описывает, что является заголовком, абзацем, ссылкой или изображением — не как это выглядит, а что это означает. Браузер парсит HTML-документ, строит DOM-дерево и на его основе отображает страницу. Без HTML нет структуры — CSS и JS работают поверх него.

---

## Развёрнутый ответ

### Суть и определение

HTML — декларативный язык разметки, стандарт которого поддерживает WHATWG (Living Standard). Он не является языком программирования: в нём нет переменных, условий или циклов. Задача HTML — описать структуру документа через набор элементов (тегов), каждый из которых несёт определённый смысл (семантику).

Место в экосистеме:
- **HTML** — структура и смысл контента
- **CSS** — визуальное представление
- **JavaScript** — поведение и взаимодействие

### Как это работает

Браузер загружает HTML-документ и запускает HTML-парсер:

1. **Tokenization** — поток байт разбивается на токены (теги, атрибуты, текст).
2. **Tree construction** — токены превращаются в узлы DOM-дерева.
3. **Render tree** — DOM объединяется с CSSOM для построения дерева рендеринга.
4. **Layout → Paint → Composite** — пиксели отображаются на экране.

Парсер HTML5 толерантен к ошибкам: при встрече невалидного HTML он пытается «починить» документ по алгоритму, описанному в спецификации WHATWG. Это означает, что браузер всегда показывает что-то, даже при сломанной разметке.

### Практика и применение

- Основа любого веб-приложения — SPA-фреймворки (React, Vue, Angular) генерируют HTML динамически, но итоговый вывод всегда HTML.
- SSR (Server-Side Rendering) и SSG (Static Site Generation) возвращают HTML с сервера для ускорения первой отрисовки и SEO.
- Email-рассылки используют HTML-разметку с ограниченным набором тегов.
- WebView в мобильных приложениях отображает HTML-контент нативно.

### Важные нюансы и краеугольные камни

- HTML — **Living Standard**, версии (HTML4, XHTML, HTML5) уже неактуальны как концепция, стандарт обновляется непрерывно.
- Парсер исправляет ошибки по-своему в каждом браузере — невалидный HTML может рендериться по-разному в Chrome и Firefox.
- `<html>`, `<head>`, `<body>` теги **не обязательны** синтаксически — браузер добавит их в DOM автоматически, но их принято писать явно.
- HTML чувствителен к порядку загрузки ресурсов: `<script>` без `async`/`defer` блокирует парсер.

### Примеры

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8" />
    <title>Моя страница</title>
  </head>
  <body>
    <h1>Привет, мир</h1>
    <p>HTML описывает <strong>структуру</strong> документа.</p>
  </body>
</html>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Чем HTML отличается от XHTML?** — XHTML требует строгого соответствия XML-синтаксису (закрытые теги, кавычки вокруг атрибутов), HTML5 более толерантен.
- **Что такое DOM и как он связан с HTML?** — DOM — объектное представление HTML-документа в памяти браузера, построенное парсером.
- **Почему HTML не язык программирования?** — нет вычислений, управления потоком или состояния; только декларативное описание структуры.
- **Как браузер обрабатывает ошибки в HTML?** — по алгоритму error recovery из спецификации WHATWG, результат может различаться между браузерами.
- **Что будет если не написать `<!DOCTYPE html>`?** — браузер перейдёт в Quirks Mode, имитируя поведение старых браузеров IE5.

### Красные флаги (чего не говорить)

- «HTML5 — это новая версия HTML» — это Living Standard, не версия с фиксированной датой.
- «HTML и XML — одно и то же» — HTML5 не является XML, у него другой парсер и правила обработки ошибок.
- «HTML отвечает за стили» — разделение ответственности: HTML — структура, CSS — стили.

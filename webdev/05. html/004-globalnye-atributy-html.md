# Q004. Какие глобальные атрибуты есть в HTML?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Глобальные атрибуты — это атрибуты, которые можно применить к любому HTML-элементу, независимо от его типа. К наиболее важным относятся: `id`, `class`, `style`, `title`, `lang`, `dir`, `hidden`, `tabindex`, `contenteditable`, `draggable`, `data-*` и группа `aria-*`. Они обеспечивают базовую идентификацию, доступность, интернационализацию и расширяемость элементов.

---

## Развёрнутый ответ

### Суть и определение

Глобальные атрибуты определены спецификацией HTML и применимы к любому элементу, включая пользовательские (custom elements). Часть из них влияет на поведение браузера, часть — на доступность, часть предназначена для хранения данных или управления фокусом.

### Как это работает

**Идентификация и стилизация:**

| Атрибут | Описание |
|---------|----------|
| `id` | Уникальный идентификатор в документе. Используется для якорей, связи `<label for>`, CSS и JS-выборки. |
| `class` | Список CSS-классов через пробел. Основной механизм стилизации. |
| `style` | Инлайновые CSS-стили. Наивысший приоритет специфичности среди стилей. |

**Мета-информация и локализация:**

| Атрибут | Описание |
|---------|----------|
| `title` | Подсказка (tooltip) при наведении. Для `<abbr>` — расшифровка аббревиатуры. |
| `lang` | Язык содержимого (`ru`, `en`, `de`). Влияет на произношение в скринридерах и правила переноса. |
| `dir` | Направление текста: `ltr`, `rtl`, `auto`. Критично для арабского, иврита. |

**Видимость и взаимодействие:**

| Атрибут | Описание |
|---------|----------|
| `hidden` | Скрывает элемент (аналог `display: none`). Семантически сообщает, что элемент неактуален. |
| `tabindex` | Управляет порядком фокуса клавишей Tab. `0` — в естественном порядке, `-1` — вне потока, `>0` — явный порядок (не рекомендуется). |
| `contenteditable` | `true`/`false` — позволяет редактировать содержимое элемента прямо на странице. |
| `draggable` | `true`/`false` — включает Drag and Drop API для элемента. |

**Пользовательские данные:**

| Атрибут | Описание |
|---------|----------|
| `data-*` | Любые пользовательские атрибуты с префиксом `data-`. Хранят данные без нарушения валидности. |

**Доступность:**

| Атрибут | Описание |
|---------|----------|
| `aria-*` | Атрибуты WAI-ARIA. Описывают роли, состояния и свойства для assistive technology. |

**Прочие глобальные атрибуты:**

| Атрибут | Описание |
|---------|----------|
| `accesskey` | Клавиша быстрого доступа через Alt/Ctrl+Alt. |
| `spellcheck` | Включает/отключает проверку орфографии. |
| `translate` | `yes`/`no` — подсказка переводчикам о необходимости перевода. |
| `nonce` | Одноразовый токен для CSP (Content Security Policy). |
| `slot` | Назначает элемент в именованный слот Web Component. |
| `part` | Открывает элемент shadow DOM для стилизации снаружи через `::part()`. |
| `exportparts` | Экспортирует части shadow DOM выше по иерархии. |
| `enterkeyhint` | Подсказывает мобильной клавиатуре, что написать на клавише Enter. |
| `inputmode` | Тип виртуальной клавиатуры для ввода. |
| `autocapitalize` | Управляет автокапитализацией на мобильных устройствах. |
| `is` | Используется с customized built-in elements. |
| `itemscope`, `itemprop`, `itemtype` | Микроразметка Schema.org (Microdata). |

### Практика и применение

- `id` используется для якорных ссылок (`#section`) и как цель `for` у `<label>` — без этого форма не работает правильно.
- `lang` на `<html>` — обязателен для правильной работы скринридеров и SEO.
- `tabindex="-1"` позволяет программно фокусировать элемент через JS (`.focus()`), не включая его в Tab-обход.
- `data-*` — стандартный способ передавать данные из HTML в JS без засорения пространства имён.
- `aria-label` и `aria-describedby` — критичны для доступности иконочных кнопок без текста.

### Важные нюансы и краеугольные камни

- Одинаковый `id` у нескольких элементов — нарушение спецификации; `querySelector('#id')` вернёт первый, но поведение браузеров может различаться.
- `hidden` не работает, если элемент явно задан через `display: flex` или другое значение в CSS — CSS выигрывает.
- `tabindex` со значением `>0` — антипаттерн: ломает естественный порядок фокуса и создаёт проблемы доступности.
- `contenteditable` без обработки событий и сохранения — бесполезен в продакшене.
- `title` как основной механизм описания — плохая практика для доступности: скринридеры могут его игнорировать, на touch-устройствах tooltip недоступен.

### Примеры

```html
<!-- id, class, lang, dir -->
<section id="about" class="section section--about" lang="en" dir="ltr">
  <h2>About us</h2>
</section>

<!-- tabindex и aria для доступности -->
<div
  role="button"
  tabindex="0"
  aria-pressed="false"
  aria-label="Добавить в избранное"
  class="btn-icon"
>
  ♥
</div>

<!-- data-атрибуты -->
<ul>
  <li data-id="1" data-status="active">Элемент 1</li>
  <li data-id="2" data-status="inactive">Элемент 2</li>
</ul>

<script>
  document.querySelectorAll('[data-status="active"]').forEach(el => {
    console.log(el.dataset.id); // "1"
  });
</script>

<!-- hidden -->
<div hidden>Это не отображается</div>

<!-- contenteditable -->
<p contenteditable="true">Этот текст можно редактировать</p>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Чем `hidden` отличается от `display: none` в CSS?** — функционально одинаково, но `hidden` несёт семантику «элемент неактуален», а CSS — только визуальное скрытие.
- **Почему `tabindex > 0` считается антипаттерном?** — нарушает естественный порядок фокуса, усложняет поддержку.
- **Как `data-*` атрибуты доступны в JS?** — через `element.dataset` как объект с camelCase ключами.
- **Зачем `lang` на `<html>` теге?** — скринридеры используют его для выбора голосового движка; поисковики — для языковой классификации.
- **В чём разница `aria-label` и `title`?** — `aria-label` надёжно читается скринридерами, `title` — ненадёжен на touch-устройствах.

### Красные флаги (чего не говорить)

- «`id` не обязательно должен быть уникальным» — должен быть уникальным в пределах документа.
- «`tabindex="1"` улучшает навигацию» — нарушает естественный порядок, создаёт проблемы доступности.
- «`style` лучше `class` для быстрой стилизации» — инлайн-стили затрудняют поддержку и переопределение.

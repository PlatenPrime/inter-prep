# Q026. Что такое инлайновый стиль? Можно ли его переопределить?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Инлайновый стиль — это CSS, заданный непосредственно через атрибут `style` на HTML-элементе. Он имеет наивысшую специфичность среди обычных стилей (1-0-0-0 в нотации специфичности), поэтому переопределяется только через `!important` в таблицах стилей или через JavaScript. Использовать инлайновые стили следует в исключительных случаях — динамические значения из JS, email-вёрстка, critical inline CSS.

---

## Развёрнутый ответ

### Суть и определение

Инлайновый стиль задаётся атрибутом `style` прямо на элементе:

```html
<div style="color: red; font-size: 16px;">Текст</div>
```

В каскаде CSS источники стилей имеют следующий приоритет (от низшего к высшему):
1. User-agent stylesheet (браузерные стили по умолчанию)
2. Пользовательские стили (настройки браузера)
3. Авторские стили (CSS файлы, `<style>`)
4. **Инлайновые стили** (`style=""`)
5. `!important` в авторских стилях
6. `!important` в пользовательских стилях

### Как это работает

Специфичность инлайнового стиля — `1-0-0-0` (первая цифра в четырёхкомпонентной нотации). Это выше любого CSS-селектора:
- ID-селектор: `0-1-0-0`
- Класс: `0-0-1-0`
- Тег: `0-0-0-1`

Единственный CSS-способ переопределить инлайновый стиль — `!important`. JS-способ — изменить `element.style` или `element.removeAttribute('style')`.

### Практика и применение

**Когда инлайновый стиль уместен:**

```html
<!-- Динамические значения из JS (позиция, размер) -->
<div style="top: {{ position.y }}px; left: {{ position.x }}px;">Drag me</div>

<!-- CSS Custom Properties через JS -->
<div style="--progress: 75%;">
  <div class="progress-bar"></div>
</div>

<!-- Email-вёрстка — CSS-файлы не поддерживаются клиентами -->
<td style="padding: 16px; background-color: #ffffff;">...</td>
```

**Переопределение через `!important`:**

```css
/* Переопределяет inline style="color: red" */
.override {
  color: blue !important;
}
```

**Переопределение через JavaScript:**

```js
// Прямое изменение
element.style.color = 'blue';

// Удаление конкретного свойства
element.style.removeProperty('color');

// Полная очистка инлайнового стиля
element.removeAttribute('style');
```

### Важные нюансы и краеугольные камни

- Инлайновые стили **не поддерживают псевдоклассы и псевдоэлементы** (`::before`, `:hover`) — только реальные элементы
- Инлайновые стили **не поддерживают медиазапросы** — адаптивность только через JS
- Они **смешивают структуру и представление** — нарушение принципа separation of concerns
- В React/Vue инлайновые стили применяются для динамических значений, но не как основной подход
- CSS Custom Properties (`--var`) через `style=""` — легитимный паттерн для передачи динамических значений в CSS

### Красные флаги (чего не говорить)

- «Инлайновые стили всегда плохая практика» — они уместны для динамических значений
- «`!important` в стилях всегда переопределит инлайн» — `!important` в инлайне ещё выше
- «Инлайновые стили поддерживают медиазапросы» — не поддерживают

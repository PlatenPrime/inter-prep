# Q006. Опишите базовую структуру HTML-страницы?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Базовая HTML-страница состоит из четырёх обязательных частей: `<!DOCTYPE html>` — переключатель режима рендеринга, `<html>` — корневой элемент документа, `<head>` — мета-информация (не отображается), `<body>` — видимое содержимое. Внутри `<head>` минимально необходимы `<meta charset>` и `<title>`. Этот скелет является отправной точкой любого веб-документа.

---

## Развёрнутый ответ

### Суть и определение

Структура HTML-страницы — это иерархия узлов DOM, которую браузер строит при парсинге. Каждый уровень имеет чёткую роль: DOCTYPE управляет парсером, `<html>` является корнем, `<head>` содержит метаданные, `<body>` — контент. Все остальные элементы размещаются внутри `<body>`.

### Как это работает

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <!-- Метаданные — не видны пользователю -->
  </head>
  <body>
    <!-- Контент — виден пользователю -->
  </body>
</html>
```

**Разбор каждой части:**

**`<!DOCTYPE html>`**
- Инструкция парсеру (не тег).
- Включает Standards Mode рендеринга.
- Должна быть первой строкой документа, до любых пробелов.

**`<html lang="...">`**
- Корневой элемент, содержит все остальные.
- `lang` — обязателен для скринридеров, поисковиков и браузерного переводчика.
- Может содержать атрибуты `xmlns` (для XHTML), `dir`.

**`<head>` — секция метаданных:**

| Элемент | Роль |
|---------|------|
| `<meta charset="UTF-8">` | Кодировка документа — первое в `<head>`, до `<title>` |
| `<meta name="viewport" ...>` | Настройка viewport для мобильных устройств |
| `<title>` | Заголовок вкладки, используется в поиске и при шаринге |
| `<meta name="description">` | Описание для поисковиков и Open Graph |
| `<link rel="canonical">` | Канонический URL для SEO |
| `<link rel="icon">` | Фавиконка |
| `<link rel="stylesheet">` | Подключение CSS |
| `<script>` | JS (с `defer`/`async` для не-блокирующей загрузки) |
| `<base>` | Базовый URL для относительных ссылок |

**`<body>` — основной контент:**
- Всё видимое содержимое страницы.
- Семантические секции: `<header>`, `<main>`, `<footer>`, `<nav>`, `<aside>`.
- Скрипты в конце `<body>` (или в `<head>` с `defer`) — паттерн для производительности.

### Практика и применение

Минимально необходимый шаблон для продакшена:

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta name="description" content="Описание страницы" />
    <title>Заголовок страницы</title>
    <link rel="icon" href="/favicon.ico" />
    <link rel="stylesheet" href="/styles.css" />
  </head>
  <body>
    <header>...</header>
    <main>...</main>
    <footer>...</footer>
    <script src="/app.js" defer></script>
  </body>
</html>
```

### Важные нюансы и краеугольные камни

- `<meta charset="UTF-8">` должен быть в первых 1024 байтах документа — браузер читает кодировку до полной загрузки.
- `<meta name="viewport">` без него мобильные браузеры масштабируют страницу как десктопную.
- `<title>` — единственный обязательный элемент внутри `<head>` по спецификации; без него браузер создаст пустой title.
- `<script>` в `<head>` без `defer`/`async` блокирует парсинг и рендеринг — критично для производительности.
- `<link rel="stylesheet">` блокирует рендеринг (render-blocking) — загрузка CSS должна быть оптимизирована.
- Тег `<html>`, `<head>`, `<body>` технически не обязателен — браузер их добавит сам, но явное написание — обязательная практика.

### Примеры

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Моя страница</title>
  </head>
  <body>
    <header>
      <nav>
        <a href="/">Главная</a>
      </nav>
    </header>

    <main>
      <h1>Добро пожаловать</h1>
      <p>Основной контент страницы.</p>
    </main>

    <footer>
      <p>© 2026 Моя компания</p>
    </footer>
  </body>
</html>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему `<meta charset>` должен быть первым в `<head>`?** — браузер читает кодировку до обработки остального контента, чтобы правильно декодировать байты.
- **Что произойдёт без `<meta name="viewport">`?** — мобильный браузер отрендерит страницу шириной ~980px и масштабирует её, ломая адаптивный дизайн.
- **Зачем `defer` у `<script>` в `<head>`?** — скрипт загружается параллельно, выполняется после парсинга DOM, не блокируя отрисовку.
- **Можно ли размещать `<link>` в `<body>`?** — технически можно, но это нарушение спецификации и может вызвать FOUC (Flash of Unstyled Content).

### Красные флаги (чего не говорить)

- «`<body>` и `<head>` можно не писать — браузер сам добавит» — технически верно, но на практике всегда пишем явно.
- «Скрипты всегда ставим в конец `<body>`» — современный подход: `<script defer>` в `<head>` лучше для производительности.
- «`<title>` необязателен» — обязателен по спецификации; его отсутствие ухудшает SEO и доступность.

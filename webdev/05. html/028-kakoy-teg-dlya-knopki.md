# Q028. Какой тэг использовать для того, чтобы сверстать кнопку?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Для кнопки нужно использовать нативный тег `<button>`. Он автоматически получает фокус с клавиатуры, поддерживает события `click` при нажатии `Enter`/`Space`, имеет ARIA-роль `button` и корректно работает с screen readers. Использование `<div>` или `<span>` вместо `<button>` требует ручной реализации всего этого поведения через атрибуты и JavaScript, что увеличивает риск ошибок доступности.

---

## Развёрнутый ответ

### Суть и определение

`<button>` — интерактивный элемент формы с тремя типами поведения, управляемыми атрибутом `type`:

| `type` | Поведение |
|--------|-----------|
| `button` | Ничего не делает по умолчанию — для JS-обработчиков |
| `submit` | Отправляет форму (значение по умолчанию внутри `<form>`) |
| `reset` | Сбрасывает форму к начальным значениям |

### Как это работает

Браузер предоставляет `<button>` встроенное поведение:
- **Focusable:** автоматически добавляется в tab order (tabindex=0)
- **Keyboard events:** `Enter` и `Space` генерируют `click`
- **ARIA role:** неявная роль `button` — screen reader объявляет элемент как кнопку
- **Form integration:** знает о родительской `<form>` без явной связи

### Практика и применение

```html
<!-- Кнопка действия (JS) — всегда указывай type="button" явно -->
<button type="button" onclick="handleClick()">Нажми меня</button>

<!-- Кнопка отправки формы -->
<form action="/submit" method="POST">
  <input type="text" name="name">
  <button type="submit">Отправить</button>
</form>

<!-- Кнопка сброса формы -->
<button type="reset">Очистить форму</button>

<!-- Кнопка с иконкой — обязателен aria-label -->
<button type="button" aria-label="Закрыть диалог">
  <svg aria-hidden="true">...</svg>
</button>

<!-- Disabled кнопка -->
<button type="button" disabled>Недоступно</button>
```

**Антипаттерны и почему они плохи:**

```html
<!-- ПЛОХО: div не focusable, нет role, нет keyboard events -->
<div onclick="handleClick()">Кнопка</div>

<!-- ПЛОХО: нужно добавить tabindex, role, onkeydown — много ручной работы -->
<div role="button" tabindex="0" onclick="handleClick()" onkeydown="handleKey(event)">
  Кнопка
</div>

<!-- ПЛОХО: <a> без href — для навигации, не для действий -->
<a onclick="handleClick()">Действие</a>
```

**Когда `<input type="button">` вместо `<button>`:**
```html
<!-- input не поддерживает HTML внутри — только text через value -->
<input type="submit" value="Отправить">

<!-- button позволяет HTML-контент внутри -->
<button type="submit">
  <svg>...</svg>
  Отправить
</button>
```

### Важные нюансы и краеугольные камни

- `type="button"` следует указывать **явно** для кнопок вне форм: дефолтный тип — `submit`, что может случайно отправить ближайшую форму
- `<button>` может содержать HTML-контент (иконки, теги), `<input type="button">` — только текст через `value`
- `disabled` не только блокирует взаимодействие, но и убирает кнопку из tab order
- `aria-disabled="true"` блокирует ARIA, но оставляет в tab order (кнопка фокусируется, но не срабатывает) — лучше для UX в некоторых случаях
- Кнопки не должны использоваться для навигации — для этого `<a href>`

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Почему важно указывать `type="button"` явно для кнопок не в форме?
- В чём разница между `disabled` и `aria-disabled="true"`?
- Когда использовать `<a>`, а когда `<button>`?
- Как правильно сделать кнопку с иконкой без текста доступной?
- Чем `<button>` лучше `<input type="button">`?

### Красные флаги (чего не говорить)

- «`<div>` с `role="button"` так же хорош, как `<button>`» — нет, пропускаются клавиатурные события по умолчанию
- «Тип кнопки не важен, если нет формы» — дефолт `submit` может сломать неочевидную форму-предка
- «`<a>` подходит для кнопок» — `<a>` для навигации, `<button>` для действий

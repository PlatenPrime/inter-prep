# Q010. Что такое семантика? Какие семантические тэги вы знаете?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Семантика (semantics) в HTML — это использование тегов в соответствии с их смысловым назначением, а не внешним видом. Семантические теги сообщают браузеру, поисковику и скринридеру, что означает данный блок контента, а не просто как он выглядит. Примеры: `<header>`, `<main>`, `<nav>`, `<article>`, `<section>`, `<aside>`, `<footer>`, `<h1>`–`<h6>`, `<figure>`, `<time>`, `<mark>`. Без семантики — только `<div>` и `<span>`, которые не несут смысла.

---

## Развёрнутый ответ

### Суть и определение

Семантическая HTML-разметка — это практика выбора тегов, которые точно описывают роль и смысл содержимого документа. Появилась как альтернатива «дивной вёрстке» (div soup), где вся структура строилась на `<div>` и `<span>` с классами.

Семантика решает три задачи:
1. **Доступность** — скринридеры понимают роль элемента и навигируют по нему.
2. **SEO** — поисковые роботы лучше понимают структуру и важность контента.
3. **Поддерживаемость** — разработчики понимают назначение блоков без чтения CSS.

### Как это работает

Браузеры сопоставляют HTML-элементы с [ARIA Accessibility Tree](https://www.w3.org/TR/accname-1.1/) — параллельным деревом, которое читают вспомогательные технологии. Семантические теги автоматически получают нативную ARIA-роль:

| Тег | Неявная ARIA-роль |
|-----|------------------|
| `<header>` (в `<body>`) | `banner` |
| `<nav>` | `navigation` |
| `<main>` | `main` |
| `<aside>` | `complementary` |
| `<footer>` (в `<body>`) | `contentinfo` |
| `<article>` | `article` |
| `<section>` (с именем) | `region` |
| `<button>` | `button` |
| `<a href>` | `link` |
| `<h1>`–`<h6>` | `heading` |

### Практика и применение

**Структурные теги (document outline):**

```html
<body>
  <header>         <!-- Шапка сайта или секции -->
    <nav>          <!-- Навигационный блок -->
      <ul>...</ul>
    </nav>
  </header>

  <main>           <!-- Основное содержимое (один на страницу!) -->
    <article>      <!-- Самодостаточный контент (пост, карточка) -->
      <header>     <!-- Шапка статьи (заголовок, мета) -->
        <h1>...</h1>
      </header>
      <section>    <!-- Смысловой раздел внутри статьи -->
        ...
      </section>
    </article>

    <aside>        <!-- Боковой контент, связанный с основным -->
      ...
    </aside>
  </main>

  <footer>         <!-- Подвал сайта или секции -->
    ...
  </footer>
</body>
```

**Текстовые семантические теги:**

| Тег | Назначение |
|-----|-----------|
| `<h1>`–`<h6>` | Иерархия заголовков; `<h1>` — один на страницу |
| `<p>` | Абзац текста |
| `<strong>` | Важное (семантически), отображается жирным |
| `<em>` | Акцент (семантически), отображается курсивом |
| `<blockquote>` | Цитата из внешнего источника |
| `<cite>` | Название произведения или автора цитаты |
| `<time datetime="...">` | Дата/время в машиночитаемом формате |
| `<mark>` | Выделение как в маркере — по контексту поиска |
| `<abbr title="...">` | Аббревиатура с расшифровкой |
| `<code>` | Код (inline) |
| `<pre>` | Преформатированный текст |
| `<kbd>` | Клавиша клавиатуры |
| `<samp>` | Вывод программы |
| `<var>` | Переменная |
| `<dfn>` | Определение термина |
| `<address>` | Контактная информация |

**Медиа и группировка:**

| Тег | Назначение |
|-----|-----------|
| `<figure>` | Самодостаточный контент с подписью (фото, диаграмма, код) |
| `<figcaption>` | Подпись к `<figure>` |
| `<picture>` | Адаптивное изображение с условиями выбора источника |
| `<details>` | Раскрываемый блок |
| `<summary>` | Заголовок раскрываемого блока `<details>` |
| `<dialog>` | Нативное модальное окно |

### Важные нюансы и краеугольные камни

- `<article>` vs `<section>`: `<article>` самодостаточен (можно скопировать в другой контекст); `<section>` — смысловой раздел, зависящий от контекста документа.
- `<header>` и `<footer>` могут быть внутри `<article>` или `<section>` — тогда они теряют ARIA-роль `banner`/`contentinfo`.
- `<main>` — только один на страницу; второй `<main>` должен иметь `hidden`.
- `<section>` без `<h2>`–`<h6>` внутри не получает ARIA-роль `region` — скринридеры будут игнорировать.
- `<strong>` ≠ `<b>`: `<strong>` — семантическая важность; `<b>` — стилистическое выделение без значимости.
- `<em>` ≠ `<i>`: `<em>` — интонационный акцент; `<i>` — альтернативный голос, технический термин, мысль.
- `<time datetime>` должен содержать машиночитаемый формат ISO 8601 (`2026-05-26T11:00`), даже если отображается текст.

### Примеры

```html
<!-- Семантическая структура страницы блога -->
<header>
  <a href="/" class="logo">My Blog</a>
  <nav aria-label="Основная навигация">
    <ul>
      <li><a href="/posts">Статьи</a></li>
      <li><a href="/about">О нас</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <header>
      <h1>Заголовок статьи</h1>
      <p>
        Опубликовано:
        <time datetime="2026-05-26">26 мая 2026</time>
        автором <address><a href="/author">Иван</a></address>
      </p>
    </header>

    <section>
      <h2>Первый раздел</h2>
      <p>
        Это <strong>важная информация</strong>,
        а это <em>акцент</em> в тексте.
      </p>

      <figure>
        <img src="diagram.png" alt="Схема архитектуры" />
        <figcaption>Рис. 1 — Архитектура системы</figcaption>
      </figure>
    </section>

    <section>
      <h2>Пример кода</h2>
      <pre><code>const x = 42;</code></pre>
    </section>
  </article>

  <aside aria-label="Похожие статьи">
    <h2>Читайте также</h2>
    <!-- ... -->
  </aside>
</main>

<footer>
  <p><small>© 2026 My Blog. Все права защищены.</small></p>
</footer>
```

```html
<!-- details/summary — нативный аккордеон без JS -->
<details>
  <summary>Показать подробности</summary>
  <p>Скрытый контент, который раскрывается по клику.</p>
</details>

<!-- mark — выделение релевантного текста (поиск) -->
<p>Результаты поиска по запросу «<mark>HTML семантика</mark>»</p>

<!-- abbr — аббревиатура -->
<p><abbr title="HyperText Markup Language">HTML</abbr> — язык разметки.</p>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **В чём разница между `<article>` и `<section>`?** — `<article>` самодостаточен (можно взять и вставить в другой контекст); `<section>` — тематический раздел в рамках документа.
- **Почему стоит использовать семантику вместо `<div>`?** — доступность (скринридеры), SEO, поддерживаемость, нативные ARIA-роли.
- **Можно ли добавить `role="main"` к `<div>` вместо `<main>`?** — технически да, но нативный `<main>` предпочтительнее: меньше кода, надёжнее поддержка.
- **Сколько `<h1>` должно быть на странице?** — спецификация допускает несколько (при использовании sectioning elements), но SEO-практика рекомендует один.
- **Что такое outline algorithm?** — алгоритм определения структуры документа по заголовкам и секционным элементам; реализован в браузерах неполно, поэтому важна явная иерархия `<h1>`–`<h6>`.

### Красные флаги (чего не говорить)

- «Семантика нужна только для SEO» — в первую очередь для доступности; SEO — вторично.
- «`<div>` и `<section>` — это одно и то же, просто `<section>` для секций» — `<section>` несёт семантику региона и получает ARIA-роль, `<div>` — нет.
- «`<strong>` — это просто жирный текст» — это семантика важности; для стилизации есть CSS `font-weight: bold`.
- «Можно использовать `<h3>` сразу после `<h1>` если так выглядит лучше» — нарушение иерархии заголовков ломает навигацию скринридеров.

# Q071. Что такое поток документа?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

Поток документа (normal flow) — алгоритм размещения элементов по умолчанию, при котором браузер расставляет блочные элементы вертикально (один под другим), а инлайновые — горизонтально в строках. Элементы «вырываются» из потока через `float`, `position: absolute` или `position: fixed`. Flexbox и Grid создают новые контексты форматирования, не нарушая поток, но переопределяя его правила внутри контейнера.

---

## Развёрнутый ответ

### Суть и определение

Когда браузер строит страницу, он последовательно обходит DOM и вычисляет, где разместить каждый элемент. Этот процесс по умолчанию называется **normal flow** (нормальный поток). Каждый элемент занимает место в соответствии со своей display-моделью:

- **Block-level elements** (`<div>`, `<p>`, `<h1>`): занимают всю доступную ширину, располагаются вертикально.
- **Inline-level elements** (`<span>`, `<a>`, `<strong>`): занимают ровно столько ширины, сколько нужно, располагаются горизонтально в строке.

### Как это работает

**Блочный контекст форматирования (BFC — Block Formatting Context)**

Изолированный регион, в котором блочные элементы размещаются вертикально. Внутри BFC элементы не взаимодействуют со floats снаружи. BFC создаётся при:
- `overflow` ≠ `visible` (например, `overflow: hidden`)
- `display: flow-root` (явное создание BFC)
- `float` ≠ `none`
- `position: absolute` / `fixed`
- `display: flex` / `grid` / `inline-flex` / `inline-grid`
- `contain: layout` / `content`

**Инлайновый контекст форматирования (IFC — Inline Formatting Context)**

Область строк, в которой инлайновые элементы размещаются горизонтально. Высота строки определяется самым высоким элементом. Вертикальное выравнивание через `vertical-align`.

**Выход из потока (out-of-flow):**

| Свойство | Поведение |
|----------|-----------|
| `float: left/right` | Элемент «всплывает» влево/вправо, обтекается инлайновым контентом; место в потоке освобождается |
| `position: absolute` | Позиционируется относительно ближайшего positioned-предка; место в потоке освобождается |
| `position: fixed` | Позиционируется относительно viewport; место в потоке освобождается |
| `position: sticky` | Остаётся в потоке до достижения порога, затем «прилипает» (гибрид) |

**Flexbox и Grid не вырывают из потока**, но создают новые контексты форматирования для дочерних элементов, полностью переопределяя правила их размещения.

### Практика и применение

- Понимание потока необходимо для отладки «почему элемент занимает не то место».
- `overflow: hidden` на родителе — классический хак для «обёртки float-элементов» (clearfix через BFC).
- `display: flow-root` — современная замена clearfix без побочных эффектов overflow.
- Margin collapsing (схлопывание отступов) — происходит только в нормальном потоке между блочными элементами; в BFC, flex и grid контекстах не происходит.

### Важные нюансы и краеугольные камни

- **Margin collapsing:** вертикальные margins соседних блочных элементов схлопываются в больший из двух. Это поведение нормального потока — в flex/grid не происходит.
- `position: sticky` остаётся **в потоке** — место элемента сохраняется (в отличие от `fixed`).
- Float исторически использовался для многоколоночной верстки — сегодня заменён Flexbox/Grid.
- Создание BFC решает классические проблемы: содержит floats, предотвращает margin collapsing через границы родителя.
- Абсолютно позиционированный элемент ищет ближайшего предка с `position` ≠ `static` — если не находит, позиционируется по initial containing block (viewport).
- `writing-mode` меняет направление потока: при `writing-mode: vertical-rl` блочный поток идёт горизонтально.

### Примеры

```html
<!-- Нормальный поток: блочные элементы -->
<div style="background: lightblue; padding: 8px;">Блок 1 (занимает всю ширину)</div>
<div style="background: lightgreen; padding: 8px;">Блок 2 (под первым)</div>
<span style="background: lightyellow;">Инлайн 1</span>
<span style="background: lightsalmon;">Инлайн 2 (рядом)</span>

<!-- Float: выход из потока + BFC через overflow:hidden -->
<div style="overflow: hidden; border: 2px solid navy;">
  <img src="photo.jpg" alt="" style="float: left; margin-right: 16px;" width="150" />
  <p>Текст обтекает изображение слева. Родитель содержит float благодаря overflow:hidden (BFC).</p>
</div>

<!-- display:flow-root — современный clearfix -->
<div style="display: flow-root; border: 2px solid green;">
  <div style="float: left; width: 100px; height: 80px; background: coral;"></div>
  <p>Контейнер растягивается под float без overflow хаков.</p>
</div>

<!-- Абсолютное позиционирование — выход из потока -->
<div style="position: relative; height: 100px; border: 1px dashed gray;">
  <p>Этот текст в потоке.</p>
  <div style="
    position: absolute;
    top: 10px;
    right: 10px;
    background: tomato;
    padding: 4px;
  ">
    Вне потока (absolute)
  </div>
</div>

<!-- Margin collapsing: итоговый отступ = max(16px, 24px) = 24px -->
<p style="margin-bottom: 16px;">Первый параграф</p>
<p style="margin-top: 24px;">Второй параграф (отступ схлопнется)</p>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Что такое margin collapsing и когда оно не происходит? (только блочные элементы в нормальном потоке; не в flex, grid, BFC)
- Как создать BFC и зачем это нужно? (overflow, flow-root, flex, grid — изоляция floats, предотвращение схлопывания)
- Чем `position: sticky` отличается от `fixed`? (sticky остаётся в потоке; fixed — нет)
- Почему float-элемент не влияет на высоту родителя? (float вне потока; родитель не «видит» его)
- Что такое containing block для `position: absolute`?

### Красные флаги (чего не говорить)

- «Flexbox и Grid вырывают элементы из потока» — нет, они создают новый контекст форматирования для дочерних элементов.
- «`position: sticky` работает как `fixed`» — sticky остаётся в потоке; fixed — нет.
- «Margin collapsing — это баг» — это намеренное поведение нормального потока для типографики.
- «Float устарел и не нужно знать» — float до сих пор используется и влияет на поток; понимать нужно.

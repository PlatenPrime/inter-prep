# Q066. Разница между `<meter>` и `<progress>`?

> **Источник:** [05. html.md](../05.%20html.md) · **Тема:** HTML

---

## Короткий ответ

`<progress>` отображает ход выполнения задачи (от 0 до max), аналог progress bar в UI. `<meter>` отображает скалярное измерение в известном диапазоне (заряд батареи, загрузка диска) и поддерживает атрибуты `low`, `high`, `optimum` для цветовой индикации. Оба элемента семантичны и улучшают доступность (accessibility), передавая смысл скрин-ридерам.

---

## Развёрнутый ответ

### Суть и определение

**`<progress>`** — индикатор прогресса незавершённой задачи. Отвечает на вопрос «сколько уже сделано?». Может быть определённым (с `value`) или неопределённым (без `value` — анимированный спиннер/полоса).

**`<meter>`** — числовое измерение в пределах известного диапазона. Отвечает на вопрос «каково текущее значение относительно нормы?». Всегда имеет конкретное значение, поддерживает оценку уровня (хорошо/плохо/нормально).

### Как это работает

**`<progress>` атрибуты:**
- `value` — текущее значение (от 0 до `max`); без него — indeterminate state.
- `max` — максимальное значение (по умолчанию 1).

**`<meter>` атрибуты:**
- `value` — текущее значение (обязателен).
- `min` / `max` — диапазон (по умолчанию 0 и 1).
- `low` — порог «низкого» значения.
- `high` — порог «высокого» значения.
- `optimum` — оптимальное значение; определяет, какой диапазон считается «хорошим».

**Логика цвета `<meter>`:**
- Браузер окрашивает полосу в зелёный (оптимально), жёлтый (субоптимально) или красный (плохо) в зависимости от соотношения `value`, `low`, `high` и `optimum`.

### Практика и применение

- `<progress>`: загрузка файла, выполнение установки, шаги мастера, время воспроизведения видео.
- `<meter>`: уровень заряда батареи, использование дискового пространства, рейтинг товара, оценка сложности пароля, SEO-оценка страницы.

### Важные нюансы и краеугольные камни

- `<meter>` **не подходит** для прогресса задачи — семантически неверно использовать его для «загрузки 50%».
- Стилизация обоих элементов ограничена и браузерно-зависима. Для кастомного дизайна часто строят UI на `<div>` с `role="progressbar"` / `role="meter"` и ARIA-атрибутами.
- `<progress>` без `value` показывает анимацию «в процессе» — используйте для операций с неизвестным временем завершения.
- Добавляйте `<label>` или `aria-label` / `aria-labelledby` — скрин-ридеры озвучивают значение, но нужен контекст.
- `<meter>` не имеет `role="meter"` в ARIA — браузеры сами передают семантику; дублировать `role` не нужно.

### Примеры

```html
<!-- progress: загрузка файла -->
<label for="file-upload">Загрузка файла:</label>
<progress id="file-upload" value="65" max="100">65%</progress>

<!-- progress: indeterminate (неизвестное время) -->
<label for="processing">Обработка...</label>
<progress id="processing">Идёт обработка</progress>

<!-- meter: использование диска -->
<label for="disk-usage">Диск:</label>
<meter
  id="disk-usage"
  value="75"
  min="0"
  max="100"
  low="70"
  high="90"
  optimum="30"
>75 ГБ из 100 ГБ</meter>
<!-- value=75 > high=90? Нет, 75 < 90, но > low=70 → жёлтый -->
<!-- optimum=30 → оптимальная зона 0-70, value вне неё → субоптимально -->

<!-- meter: рейтинг -->
<label for="rating">Рейтинг товара:</label>
<meter id="rating" value="4.2" min="0" max="5" optimum="5">4.2 из 5</meter>
```

## Сравнение

| Критерий | `<progress>` | `<meter>` |
|----------|-------------|-----------|
| Назначение | Ход выполнения задачи | Скалярное измерение в диапазоне |
| Indeterminate состояние | Да (без `value`) | Нет |
| Атрибуты диапазона | `value`, `max` | `value`, `min`, `max`, `low`, `high`, `optimum` |
| Цветовая индикация | Нет | Да (зелёный/жёлтый/красный) |
| Семантика для AT | «прогресс задачи» | «измеренное значение» |
| Типичные сценарии | Загрузка, установка, шаги | Заряд, диск, рейтинг, пароль |
| ARIA role | `progressbar` (встроен) | нет явного role="meter" в ARIA |

## На что обратить внимание на собеседовании

### Каверзные follow-up

- Когда `<progress>` без `value` полезен? (indeterminate — операции с неизвестной длительностью)
- Как работает `optimum` в `<meter>` и влияет ли он на цвет? (да, определяет «хорошую» зону)
- Как кастомизировать `<progress>` и `<meter>` под дизайн-систему? (`-webkit-progress-bar`, псевдоэлементы, или div+ARIA)
- Почему не использовать `<div>` с шириной вместо `<progress>`? (семантика, доступность, меньше JS)

### Красные флаги (чего не говорить)

- «`<meter>` и `<progress>` взаимозаменяемы» — принципиально разная семантика.
- «`<meter>` можно использовать для отображения прогресса загрузки» — нет, это `<progress>`.
- «Эти элементы нельзя стилизовать» — можно, но с ограничениями и браузерными хаками.

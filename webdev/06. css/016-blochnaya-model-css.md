# Q016. Блочная модель CSS?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Блочная модель (box model)** — способ расчёта размеров и пространства каждого CSS-элемента. Элемент состоит из четырёх областей: **content** → **padding** → **border** → **margin**. По умолчанию `width`/`height` задают размер только content-области (`content-box`). `box-sizing: border-box` включает padding и border в заданные размеры — это стандарт в современной разработке.

---

## Развёрнутый ответ

### Суть и определение

```
┌──────────────── margin ─────────────────┐
│  ┌─────────────── border ─────────────┐  │
│  │  ┌──────────── padding ──────────┐  │  │
│  │  │  ┌──── content ────┐          │  │  │
│  │  │  │  width × height │          │  │  │
│  │  │  └─────────────────┘          │  │  │
│  │  └───────────────────────────────┘  │  │
│  └─────────────────────────────────────┘  │
└───────────────────────────────────────────┘
```

**Две модели расчёта:**

| Модель | `box-sizing` | `width` включает |
|--------|-------------|-----------------|
| Content-box | `content-box` (дефолт) | Только content |
| Border-box | `border-box` | content + padding + border |

Пример: `width: 200px; padding: 20px; border: 2px solid`:
- `content-box` → фактическая ширина: 200 + 40 + 4 = **244px**
- `border-box` → фактическая ширина: **200px** (content = 200 - 40 - 4 = 156px)

### Как это работает

**Content area** — область для содержимого (текст, изображение, дочерние элементы). Размер задаётся `width`/`height`, `min-*`, `max-*`.

**Padding** — прозрачная область между content и border. Наследует `background` родительского элемента. Нет отрицательных значений.

**Border** — рамка вокруг padding. Имеет `width`, `style`, `color`. Может быть скруглён через `border-radius`.

**Margin** — прозрачная область вне border, не имеет фона. Может быть отрицательным. Подвержен схлопыванию (margin collapse).

**`outline`** — не является частью box model! Не занимает место в layout, рисуется поверх.

### Практика и применение

Глобальный сброс для `border-box`:
```css
*, *::before, *::after {
  box-sizing: border-box;
}
```

Это стандартная практика (React, Tailwind, Bootstrap). Позволяет задавать `width: 100%` с padding без выхода за пределы контейнера.

**Инспекция в DevTools**: вкладка Computed → блочная модель визуализируется как цветная диаграмма.

### Важные нюансы и краеугольные камни

- `margin: auto` горизонтально — центрирует блочный элемент с заданной шириной.
- `padding` влияет на кликабельную/осязаемую область элемента; `margin` — нет.
- Replaced elements (`<img>`) имеют внутренний размер по умолчанию (intrinsic size).
- `outline` не участвует в box model, не влияет на layout — поэтому он не сдвигает соседние элементы при `:focus`.
- `box-shadow` также не участвует в box model — не влияет на layout.
- `width: 100%` + `padding` при `content-box` → переполнение контейнера.

### Примеры

```css
/* Глобальный сброс */
*, *::before, *::after {
  box-sizing: border-box;
}

/* Стандартная карточка */
.card {
  width: 320px;
  padding: 24px;        /* padding включён в 320px при border-box */
  border: 1px solid #ddd;
  border-radius: 8px;
  margin: 16px;         /* внешний отступ, не влияет на размер карточки */
}

/* Центрирование блока */
.container {
  width: min(100%, 1200px);
  margin-inline: auto;  /* современный синтаксис вместо margin: 0 auto */
}

/* Изображение полной ширины */
img {
  display: block;
  width: 100%;
  height: auto;          /* сохраняем пропорции */
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Зачем глобально задавать `box-sizing: border-box`?** — Предсказуемость: `width: 100%` + padding не вызывает переполнения.
- **Влияет ли `outline` на box model?** — Нет, `outline` рисуется поверх layout, не занимает место.
- **Что такое схлопывание margin?** — Вертикальные margin соседних/родитель-потомок блоков объединяются в один (большее значение).
- **Чем `padding` от `margin` отличается визуально?** — `padding` — часть элемента (фон, клик), `margin` — прозрачный отступ снаружи.

### Красные флаги (чего не говорить)

- «`width: 100%` всегда безопасно» — при `content-box` + padding = overflow.
- «`outline` — это то же самое, что `border`» — outline вне box model, не занимает место в layout.
- «Margin — часть элемента» — margin прозрачен и вне видимой границы элемента.

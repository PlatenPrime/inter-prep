# Q031. Какое CSS-свойство используется для изменения порядка отображения элементов без изменения их физического расположения в HTML-коде?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

Свойство **`order`** (в контексте Flexbox и Grid) изменяет визуальный порядок flex/grid-items без изменения DOM. По умолчанию все items имеют `order: 0`; items с меньшим значением отображаются первыми. Также `flex-direction: row-reverse` / `column-reverse` и `grid-template-areas` могут переупорядочивать визуально.

---

## Развёрнутый ответ

### Суть и определение

**`order: <integer>`** — применяется к flex-items и grid-items:
- Значение по умолчанию: `0`.
- Можно задать отрицательное: `order: -1` выводит item перед всеми с `order: 0`.
- Одинаковые значения → порядок по DOM.

**Другие механизмы:**

| Метод | Контекст | Описание |
|-------|---------|---------|
| `order` | Flex/Grid | Произвольный порядок |
| `flex-direction: row-reverse` | Flex | Разворот всех items |
| `flex-direction: column-reverse` | Flex | Разворот по вертикали |
| `grid-template-areas` | Grid | Явное позиционирование |
| `grid-row` / `grid-column` | Grid | Явная ячейка |

### Как это работает

Браузер сортирует flex/grid items по `order` перед layout-расчётом. DOM остаётся неизменным — это чисто презентационное изменение.

**Важно:** `order` меняет визуальный порядок, но **не меняет порядок фокуса** при табуляции и порядок чтения для screen readers — они следуют DOM.

### Практика и применение

- Мобильная адаптация: в mobile layout изображение карточки выводится выше текста без изменения HTML.
- «Важный» элемент первым при медиазапросе: `.sidebar { order: -1 }` при `@media (max-width: 768px)`.
- Реализация паттерна «Last child first» для акцента.

### Важные нюансы и краеугольные камни

- **A11y-проблема**: визуальный порядок расходится с DOM-порядком → пользователи клавиатуры и screen readers получают другой порядок. WCAG 1.3.2 требует соответствия порядков.
- `order` не работает на элементах вне flex/grid контейнера.
- Лучшая практика: менять DOM-порядок в JS при необходимости кардинального изменения, а `order` использовать для небольших адаптивных изменений.

### Примеры

```css
/* Мобильная адаптация: изображение выше текста */
.card {
  display: flex;
  flex-direction: column;
}
.card__image { order: -1; } /* всегда первым */
.card__body  { order: 0; }

/* Grid: sidebar перед main на десктопе */
@media (min-width: 768px) {
  .layout {
    display: grid;
    grid-template-columns: 280px 1fr;
  }
  .sidebar { order: -1; } /* слева */
  .main    { order: 0;  }
}

/* flex-direction: row-reverse */
.actions {
  display: flex;
  flex-direction: row-reverse; /* кнопки отображаются справа налево */
  gap: 8px;
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Влияет ли `order` на tab order?** — Нет, только на визуальный порядок; навигация клавиатуры следует DOM-порядку.
- **Как изменить порядок focus без изменения DOM?** — `tabindex` для изменения порядка фокуса (но осторожно с a11y).
- **Когда лучше изменить DOM вместо `order`?** — Когда визуальный и DOM-порядок расходятся значительно (нарушение WCAG 1.3.2).

### Красные флаги (чего не говорить)

- «`order` работает на любом элементе» — только на flex/grid items.
- «`order: -1` ставит элемент в конец» — наоборот: меньшее значение = раньше.
- «`order` безопасен для a11y» — создаёт конфликт визуального/DOM-порядка.

# Q046. Расскажите о свойстве `text-rendering`?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**`text-rendering`** — CSS-свойство (на самом деле SVG-атрибут, принятый в CSS), управляющее тем, что браузер оптимизирует при отрисовке текста: скорость рендеринга, читаемость или геометрическую точность. Значение `optimizeLegibility` включает кернинг и лигатуры, но может снижать производительность на больших объёмах текста.

---

## Развёрнутый ответ

### Суть и определение

| Значение | Приоритет | Описание |
|---------|----------|---------|
| `auto` | — | Браузер выбирает сам (дефолт) |
| `optimizeSpeed` | Скорость | Отключает кернинг и лигатуры для быстрого рендеринга |
| `optimizeLegibility` | Читаемость | Включает кернинг, лигатуры; медленнее |
| `geometricPrecision` | Точность | Масштабирование сохраняет геометрическую форму глифов |

### Как это работает

- **Кернинг** — корректировка расстояния между конкретными парами букв (AV, To и т.д.) для визуальной равномерности.
- **Лигатуры** — объединение нескольких символов в один глиф (fi, fl, ff) для красивой типографики.
- **Hinting** — алгоритм выравнивания глифов по пикселям; при высоком DPI менее важен.

`optimizeLegibility` включает эти функции через шрифтовой движок операционной системы, что может быть дорогостоящим при большом объёме текста.

### Практика и применение

- Заголовки и короткие строки: `text-rendering: optimizeLegibility` улучшает визуальное качество.
- Основной текст больших статей: лучше использовать CSS `font-kerning` и `font-variant-ligatures` — они дают контроль без нагрузки `text-rendering`.
- `geometricPrecision` — для SVG-текста при масштабировании.

### Важные нюансы и краеугольные камни

- **Производительность**: `optimizeLegibility` на длинных блоках текста (тысячи слов) может значительно замедлить рендеринг — особенно на мобильных.
- Это **не стандартное CSS-свойство** (SVG-атрибут), поэтому браузеры реализуют его по-разному.
- Современная альтернатива: `font-kerning: auto | normal | none` и `font-variant-ligatures: normal | common-ligatures | none` — стандартные CSS-свойства с лучшей поддержкой и контролем.
- На экранах с высоким DPI (Retina) разница между режимами менее заметна визуально.
- Сильно зависит от конкретного шрифта и платформы.

### Примеры

```css
/* Заголовки: качество важнее скорости */
h1, h2, h3 {
  text-rendering: optimizeLegibility;
}

/* Основной текст: без накладных расходов */
body {
  text-rendering: auto; /* или optimizeSpeed для длинных статей */
}

/* Современная альтернатива */
h1 {
  font-kerning: normal;            /* включить кернинг */
  font-variant-ligatures: common-ligatures discretionary-ligatures;
}

/* SVG-точность */
.svg-text {
  text-rendering: geometricPrecision;
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему `optimizeLegibility` может замедлить страницу?** — Включает кернинг и лигатуры через шрифтовой движок; при большом объёме текста — CPU-нагрузка.
- **Какие современные альтернативы?** — `font-kerning`, `font-variant-ligatures`, `font-feature-settings` — стандартные CSS-свойства с более тонким контролем.
- **Когда использовать `geometricPrecision`?** — Для SVG-текста при масштабировании (анимации, трансформации).

### Красные флаги (чего не говорить)

- «`text-rendering: optimizeLegibility` везде улучшает производительность» — ухудшает при больших объёмах текста.
- «Это стандартное CSS-свойство» — SVG-атрибут, принятый в CSS; поведение может различаться в браузерах.

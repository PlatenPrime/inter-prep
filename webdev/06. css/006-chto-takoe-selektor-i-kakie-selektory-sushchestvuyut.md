# Q006. Что такое селектор? И какие селекторы существуют?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Селектор** — паттерн, описывающий, к каким элементам DOM применить CSS-правило. Браузер проверяет каждый элемент документа на соответствие селектору и применяет декларации совпавшим. Селекторы различаются по типу (тег, класс, id, атрибут, псевдокласс, псевдоэлемент) и способу комбинирования (потомок, дочерний, соседний и т.д.).

---

## Развёрнутый ответ

### Суть и определение

**Простые селекторы:**

| Тип | Синтаксис | Пример |
|-----|-----------|--------|
| Универсальный | `*` | `* { box-sizing: border-box }` |
| Тег (type) | `tag` | `p { margin: 0 }` |
| Класс | `.class` | `.card { ... }` |
| ID | `#id` | `#app { ... }` |
| Атрибут | `[attr]`, `[attr=val]`, `[attr^=val]`, `[attr$=val]`, `[attr*=val]` | `[type="text"]` |
| Псевдокласс | `:state` | `:hover`, `:focus`, `:nth-child()` |
| Псевдоэлемент | `::part` | `::before`, `::placeholder` |

**Комбинаторы:**

| Тип | Синтаксис | Значение |
|-----|-----------|----------|
| Потомок | `A B` | B является потомком A (любой глубины) |
| Дочерний | `A > B` | B является прямым дочерним элементом A |
| Соседний | `A + B` | B — первый сосед после A |
| Общий соседний | `A ~ B` | B — любой сосед после A |
| Колонка | `A || B` | B в колонке A таблицы |

**Псевдоклассы состояния:** `:hover`, `:focus`, `:focus-visible`, `:active`, `:checked`, `:disabled`, `:enabled`, `:required`, `:valid`, `:invalid`

**Псевдоклассы структуры:** `:first-child`, `:last-child`, `:nth-child(n)`, `:nth-of-type(n)`, `:only-child`, `:root`, `:empty`, `:not()`, `:is()`, `:where()`, `:has()`

### Как это работает

Браузер матчит селекторы **справа налево** для производительности: начинает с самого правого простого селектора (key selector), затем проверяет контекст влево. Поэтому `div .item` эффективнее, чем `.item div` (меньше кандидатов для проверки).

`:is()` и `:where()` принимают список селекторов. Разница: `:is()` берёт специфичность наиболее специфичного аргумента, `:where()` всегда 0 специфичность.

### Практика и применение

- Атрибутные селекторы для стилизации форм: `input[type="checkbox"]`, `[aria-expanded="true"]`.
- `:focus-visible` вместо `:focus` — показывает outline только при клавиатурной навигации.
- `:has()` позволяет стилизовать родителя по состоянию потомка: `.form:has(:invalid) .submit { opacity: 0.5 }`.
- `:nth-child(odd)` / `:nth-child(even)` для зебра-таблиц без добавления классов.

### Важные нюансы и краеугольные камни

- Высокая специфичность ID-селекторов делает их трудно переопределяемыми — в современных проектах почти не используются для стилизации.
- `*` (универсальный) имеет специфичность 0, но всё равно перекрывает наследуемые значения.
- `:nth-child(n)` считает **все** дочерние элементы, `:nth-of-type(n)` — только одного типа.
- Вложенные селекторы (CSS Nesting, CSS 2023) позволяют писать `& .child` внутри родительского правила.

### Примеры

```css
/* Атрибутный селектор */
[data-theme="dark"] {
  --bg: #1a1a2e;
  --text: #eee;
}

/* :has() — стилизация родителя */
.field:has(input:invalid) label {
  color: red;
}

/* :is() — группировка без дублирования специфичности */
:is(h1, h2, h3) > a {
  text-decoration: none;
}

/* CSS Nesting (современный) */
.card {
  padding: 16px;

  & .title {
    font-size: 1.25rem;
  }

  &:hover {
    box-shadow: 0 4px 16px rgb(0 0 0 / 0.15);
  }
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему браузер матчит селекторы справа налево?** — Оптимизация: key selector (правый) отсеивает большинство кандидатов сразу.
- **Чем `:is()` отличается от `:where()`?** — Специфичность: `:is()` наследует max специфичность аргументов, `:where()` — всегда 0.
- **Как работает `:has()`?** — Родительский селектор: стилизует элемент, если в нём есть совпадающий потомок.
- **Что такое CSS Nesting?** — Стандартизированная вложенность, нативно без препроцессора, синтаксис `& .child`.

### Красные флаги (чего не говорить)

- «ID-селекторы — лучший выбор для уникальных элементов» — слишком высокая специфичность ломает каскад.
- «Чем сложнее селектор, тем он точнее» — сложные селекторы медленнее и труднее поддерживать.
- Путать `:nth-child` и `:nth-of-type` — разная логика подсчёта.

# Q009. Разница между классом и идентификатором в CSS?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Класс** (`.foo`) предназначен для многократного использования на множестве элементов и имеет специфичность (0,1,0). **ID** (`#foo`) должен быть уникальным на странице и имеет специфичность (1,0,0). В современной CSS-архитектуре ID практически не используется для стилизации из-за высокой специфичности, которая ломает каскад.

---

## Развёрнутый ответ

### Суть и определение

| Критерий | Класс `.foo` | ID `#foo` |
|----------|-------------|-----------|
| Синтаксис HTML | `class="foo"` | `id="foo"` |
| Повторяемость | Любое кол-во элементов | Уникален на странице (HTML-правило) |
| Специфичность | (0,1,0) | (1,0,0) |
| CSS-использование | Стилизация | Якорные ссылки, JS-хуки |
| JavaScript | `getElementsByClassName`, `querySelectorAll` | `getElementById`, `querySelector` |

### Как это работает

В HTML `id` должен быть уникальным в пределах документа — это требование спецификации (браузеры не «ломаются» при дублировании, но поведение непредсказуемо: DOM API вернёт первый).

Специфичность (1,0,0) у `#id` означает, что его невозможно переопределить классами без повышения специфичности или `!important`. Это создаёт «войны специфичности» при поддержке большого CSS.

```css
#header { color: red; }          /* (1,0,0) */
.header.special.override { color: blue; } /* (0,3,0) — проигрывает! */
```

### Практика и применение

- **ID в CSS** — почти никогда: только в особых случаях (стили страниц с уникальными секциями, print-стили конкретного блока).
- **ID в HTML** — для якорных ссылок (`href="#section"`), для `for`/`aria-labelledby` связи элементов формы, для JS-хуков.
- **Классы** — основной инструмент стилизации в любой методологии (BEM, OOCSS, utility-first).

### Важные нюансы и краеугольные камни

- Несколько ID на одном элементе невозможно — атрибут `id` единственный. Несколько классов — обычная практика.
- `querySelector('#foo')` работает даже при дублированном id (вернёт первый), `getElementById('foo')` — аналогично.
- Атрибутный селектор `[id="foo"]` эквивалентен `#foo` по поведению, но имеет специфичность класса (0,1,0) — трюк для снижения специфичности ID.
- В CSS Houdini и Shadow DOM ID не «утекают» за пределы компонента — там их использование безопаснее.

### Примеры

```css
/* Плохо: ID в CSS создаёт проблемы */
#submit-button {
  background: blue;
}
/* Трудно переопределить в компоненте без #id или !important */

/* Хорошо: классы */
.btn--submit {
  background: blue;
}
.btn--submit:hover {
  background: darkblue;
}

/* Трюк: снизить специфичность ID */
[id="submit-button"] { /* (0,1,0) вместо (1,0,0) */
  background: blue;
}

/* ID для a11y, не для стилей */
```

```html
<label for="email">Email</label>
<input id="email" type="email" aria-describedby="email-hint">
<p id="email-hint">Введите корпоративный email</p>
```

## Сравнение

| Критерий | Класс | ID |
|----------|-------|-----|
| Специфичность | (0,1,0) | (1,0,0) |
| Повторяемость | Да | Нет (HTML-правило) |
| Стилизация | Рекомендуется | Не рекомендуется |
| Якоря / a11y | Не подходит | Идеально |
| Переопределение | Легко | Сложно |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему ID плохо использовать для стилизации?** — Высокая специфичность ломает каскад; трудно переопределить в компонентах и темизации.
- **Как переопределить стиль с `#id` без `!important`?** — Использовать `[id="foo"]` (специфичность класса) или `@layer`.
- **Что произойдёт, если два элемента имеют одинаковый `id`?** — Невалидный HTML; DOM API вернёт первый, CSS применится к обоим (браузер не валидирует уникальность).

### Красные флаги (чего не говорить)

- «ID быстрее для браузера» — разница в производительности матчинга пренебрежимо мала в современных браузерах.
- «ID удобнее для стилей, потому что точнее» — высокая специфичность = плохая поддерживаемость.

# Q017. Разница между `margin` и `padding`?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**`padding`** — внутренний отступ между содержимым и границей элемента; является частью элемента (наследует фон, кликабелен). **`margin`** — внешний отступ между элементом и его соседями; прозрачен и не кликабелен. Ключевая практическая разница: вертикальные margin схлопываются, padding — никогда.

---

## Развёрнутый ответ

### Суть и определение

| Критерий | `padding` | `margin` |
|----------|-----------|----------|
| Расположение | Внутри border | Снаружи border |
| Фон | Наследует фон элемента | Прозрачный |
| Кликабельность | Да | Нет |
| Отрицательные значения | Нет | Да |
| Схлопывание | Нет | Да (только вертикальный) |
| `auto` | Нет эффекта | Горизонтально — центрирование |
| `box-sizing` влияет | Да (`border-box`) | Нет |

### Как это работает

**padding:**
- Увеличивает кликабельную область кнопок и ссылок.
- При `background-color` или `background-image` — фон распространяется на padding.
- `background-clip: content-box` ограничивает фон только content-area.
- Задаётся через longhand: `padding-top`, `padding-right`, `padding-bottom`, `padding-left` или шортхенд `padding: top right bottom left`.

**margin:**
- `margin: auto` по горизонтали центрирует блочный элемент с явной шириной.
- Отрицательный margin «притягивает» элемент, перекрывая соседние.
- Margin collapse: вертикальные margins двух соседних блоков объединяются в один (бóльший из двух).
- `margin-inline: auto` — современный CSS для горизонтального центрирования.

### Практика и применение

- **Внутреннее пространство компонента** — padding: `button { padding: 8px 16px }`.
- **Пространство между компонентами** — margin или `gap` во Flexbox/Grid.
- **Центрирование** — `margin: 0 auto` для блоков с заданной шириной.
- Увеличение **hit area** кнопки без изменения видимого размера: `padding` увеличивает, `margin` — нет.
- В дизайн-системах spacing-токены применяются и к padding, и к margin исходя из семантики.

### Важные нюансы и краеугольные камни

- `gap` во Flexbox/Grid заменяет margin между элементами без проблем схлопывания — предпочтительный подход.
- `margin: auto` во Flexbox работает по-другому: поглощает всё свободное пространство (используется для центрирования flex-item или выравнивания последнего элемента вправо).
- При `overflow: hidden`/`overflow: auto` на родителе схлопывание с дочерними margin не происходит.
- `padding` нельзя задать на `<tr>`, `<col>`, `<colgroup>` — используется `border-spacing` или `border-collapse`.

### Примеры

```css
/* padding — внутреннее пространство */
.btn {
  padding: 12px 24px;        /* кликабельная область больше текста */
  background: blue;           /* синий фон включает padding */
  border: none;
  border-radius: 6px;
}

/* margin — внешнее расстояние */
.section + .section {
  margin-top: 48px;           /* отступ между секциями */
}

/* Центрирование блока */
.container {
  width: min(100%, 1200px);
  margin-inline: auto;        /* горизонтальное центрирование */
}

/* Отрицательный margin — перекрытие */
.card__image {
  margin: -24px -24px 16px;  /* «выход» изображения за пределы padding карточки */
}

/* gap вместо margin между flex-элементами */
.tags {
  display: flex;
  flex-wrap: wrap;
  gap: 8px;                   /* не схлопывается, чище чем margin */
}
```

## Сравнение

| Критерий | `padding` | `margin` |
|----------|-----------|----------|
| Где находится | Внутри элемента | Снаружи элемента |
| Фон | Наследует | Прозрачный |
| Кликабельность | Да | Нет |
| Схлопывание | Нет | Вертикально — да |
| `auto` | Не работает | Центрирует горизонтально |
| Отрицательные | Нет | Да |

---

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Когда использовать `gap` вместо `margin`?** — В Flexbox/Grid: `gap` не создаёт схлопывания, не применяется к первому/последнему элементу, семантически точнее.
- **Как работает `margin: auto` в Flexbox?** — Поглощает всё оставшееся свободное пространство по оси; `margin-left: auto` прижимает элемент вправо.
- **Как увеличить hit area кнопки без изменения её внешнего размера?** — `padding` (увеличивает видимую область) или `::before`/`::after` с `position: absolute` (скрытая hit area).

### Красные флаги (чего не говорить)

- «`margin` и `padding` взаимозаменяемы» — семантически и визуально разные: фон, кликабельность, схлопывание.
- «Отступы между компонентами лучше через `padding`» — padding — внутренний отступ компонента; расстояние между компонентами — margin или gap.

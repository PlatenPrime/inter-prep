# Q007. Что такое специфичность селектора? Как считать вес селектора?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Специфичность (specificity)** — числовой вес селектора, определяющий, чьё объявление победит при конфликте нескольких правил на одно свойство одного элемента. Считается по трём колонкам: `(A, B, C)`, где A — количество ID, B — классов/атрибутов/псевдоклассов, C — тегов/псевдоэлементов. Сравнение идёт слева направо.

---

## Развёрнутый ответ

### Суть и определение

Специфичность представляется тремя числами **(A, B, C)**:

| Тип | Колонка | Примеры |
|-----|---------|---------|
| Инлайновый стиль | выше A=1,0,0 (отдельный уровень) | `style=""` |
| ID | A | `#header` → (1,0,0) |
| Класс, атрибут, псевдокласс | B | `.btn`, `[type]`, `:hover` → (0,1,0) |
| Тег, псевдоэлемент | C | `div`, `::before` → (0,0,1) |
| `*`, `:where()`, комбинаторы | 0 | `*`, `+`, `~`, `>` → (0,0,0) |

Инлайновые стили имеют специфичность **(1,0,0,0)** — выше любого CSS-правила (но ниже `!important`).

### Как это работает

Сравнение идёт слева направо: сначала A, при равенстве — B, при равенстве — C, при равенстве — порядок в источнике (позднее = сильнее).

```
#nav .item a        → (1, 1, 1) = 111
.menu > li.active   → (0, 2, 1) = 021
div ul li           → (0, 0, 3) = 003
```

`(1,0,0)` vs `(0,9,9)` — побеждает `(1,0,0)`: A сравнивается первым.

**`:is()` и `:not()`** берут специфичность наиболее специфичного аргумента:
```css
:is(#id, .class) → (1,0,0)  /* берёт специфичность #id */
:not(.foo)       → (0,1,0)
:where(.foo)     → (0,0,0)  /* всегда 0 */
```

### Практика и применение

- BEM держит специфичность на уровне `(0,1,0)` (один класс) — это упрощает переопределение.
- Utility-first (Tailwind) работает с `(0,1,0)` на каждый класс + порядком в файле.
- Для управления приоритетом без повышения специфичности используется `@layer`.
- DevTools показывают перечёркнутые объявления, проигравшие в каскаде — первый шаг отладки.

### Важные нюансы и краеугольные камни

- `!important` не является частью специфичности — оно изменяет **origin** декларации, поднимая её выше author-слоя.
- `:nth-child()` и `:nth-of-type()` — псевдоклассы, дают +1 к колонке B.
- Дублирование класса повышает специфичность: `.btn.btn` → (0,2,0).
- Отрицание `:not(.foo)` увеличивает специфичность, но само `:not()` не добавляет ничего — только аргумент.

### Примеры

```css
/* Специфичность (0,0,1) */
p { color: blue; }

/* (0,1,0) — побеждает */
.text { color: red; }

/* (0,1,1) */
p.text { color: green; }

/* (1,0,0) — побеждает всех выше */
#content { color: purple; }

/* Трюк: повышение специфичности без ID */
.button.button { /* (0,2,0) вместо (0,1,0) */
  background: orange;
}

/* Лучше: используй @layer */
@layer overrides {
  .button { background: orange; } /* побеждает base-слой */
}
```

## Сравнение

| Критерий | Повышение специфичности | `@layer` | `!important` |
|----------|------------------------|----------|--------------|
| Побочные эффекты | Трудно переопределить | Нет | Войны important |
| Читаемость | Снижается | Высокая | Снижается |
| Применение | Legacy-код | Новые проекты | Крайний случай |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Что победит: `#id` или 10 классов?** — `#id` (1,0,0) > 10 классов (0,10,0): A всегда важнее B.
- **Как `:is()` влияет на специфичность?** — Берёт специфичность наиболее специфичного аргумента из списка.
- **Как сравнить `(0,2,0)` и `(0,1,5)`?** — (0,2,0) > (0,1,5): сравниваем B (2 > 1).
- **Как `@layer` соотносится со специфичностью?** — Layer-priority проверяется до specificity; более высокий слой побеждает даже с меньшей специфичностью.

### Красные флаги (чего не говорить)

- «11 классов победят один ID» — колонки не суммируются в одно число; (1,0,0) > (0,100,0).
- «`!important` — это высокая специфичность» — `!important` изменяет origin, а не specificity.
- «Универсальный селектор `*` имеет специфичность 0,0,1» — у него (0,0,0).

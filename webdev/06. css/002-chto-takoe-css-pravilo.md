# Q002. Что такое CSS-правило?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**CSS-правило (rule)** — основная синтаксическая единица CSS, состоящая из **селектора** и **блока деклараций**. Селектор указывает, к каким элементам применяются стили, а блок деклараций содержит одну или несколько пар «свойство: значение», разделённых точкой с запятой.

---

## Развёрнутый ответ

### Суть и определение

```
selector { property: value; property: value; }
↑          ↑               ↑
селектор   свойство        значение
           ←—— декларация ———→
           ←——— блок деклараций ——→
```

Помимо **qualified rules** (обычных правил с селектором), в CSS существуют **at-rules** — специальные директивы, начинающиеся с `@`:

- `@media` — условные правила
- `@keyframes` — описание анимации
- `@import` — подключение другого файла
- `@layer` — управление слоями каскада
- `@supports` — feature detection
- `@font-face` — подключение шрифта
- `@property` — регистрация Custom Property с типизацией

### Как это работает

Браузер парсит CSS-файл и строит список правил. При вычислении стилей для конкретного DOM-элемента движок:

1. Находит все правила, чьи селекторы совпадают с элементом.
2. Сортирует их по **origin**, **specificity**, **order**.
3. Применяет победившее значение к каждому свойству (каскад).

Объявление `!important` меняет «origin» декларации, поднимая её выше в каскаде.

### Практика и применение

- Каждое правило — атомарная единица переопределения стилей: переопределяете только нужное, не дублируя всё остальное.
- В CSS Modules каждый файл компилируется в набор правил с уникальными именами классов — одно правило на класс.
- DevTools отображают applied rules в порядке убывания приоритета — понимание структуры правила помогает быстро найти источник конфликта.

### Важные нюансы и краеугольные камни

- **Неверный синтаксис декларации** приводит к игнорированию только этой декларации, а не всего правила (error recovery).
- **Неверный селектор** в группе `selector1, invalid-selector, selector3 {}` делает недействительным всё правило в старых браузерах (`:is()` решает эту проблему).
- Незакрытый `{` приводит к игнорированию всех последующих правил до конца файла.
- At-rules могут содержать вложенные qualified rules (`@media { .foo { ... } }`).

### Примеры

```css
/* Qualified rule */
.button--primary {
  background-color: var(--color-primary);
  color: #fff;
  border: none;
  border-radius: 4px;
  padding: 8px 16px;
  cursor: pointer;
}

/* At-rule с вложенным qualified rule */
@media (max-width: 768px) {
  .button--primary {
    width: 100%;
  }
}

/* At-rule без блока */
@import url('variables.css');

/* Регистрация типизированного custom property */
@property --hue {
  syntax: '<number>';
  inherits: false;
  initial-value: 0;
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Какой приоритет у `!important`?** — Меняет origin декларации; `!important` в user-agent stylesheet перекрывается `!important` в author.
- **Что произойдёт, если селектор в группе невалиден?** — Всё правило игнорируется; `:is()` / `:where()` изолирует невалидные селекторы.
- **Чем at-rule отличается от qualified rule?** — At-rule начинается с `@`, может не иметь селектора и содержать условия или метаданные.
- **Как браузер обрабатывает CSS с ошибками?** — Error recovery: игнорирует проблемную декларацию или правило, продолжает парсинг.

### Красные флаги (чего не говорить)

- «CSS-правило — это просто класс» — путать концепцию правила (синтаксис) с селектором (частью правила).
- «`!important` — это часть значения» — `!important` изменяет приоритет декларации, не является частью значения свойства.

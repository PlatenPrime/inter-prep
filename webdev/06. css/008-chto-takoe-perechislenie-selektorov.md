# Q008. Что такое перечисление селекторов?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Перечисление селекторов (selector list)** — синтаксис `,` между несколькими селекторами, позволяющий применить один и тот же блок деклараций к разным элементам. Это DRY-подход: вместо дублирования одних и тех же стилей в нескольких правилах — объединить в одно.

---

## Развёрнутый ответ

### Суть и определение

```css
h1, h2, h3 {
  font-family: 'Georgia', serif;
  line-height: 1.2;
}
```

Эквивалентно трём отдельным правилам, но при изменении стилей меняется только в одном месте.

Перечисление бывает двух видов:
- **Regular selector list** — `A, B { ... }`: если один из селекторов невалиден, **всё правило** игнорируется браузером.
- **Forgiving selector list** — внутри `:is()`, `:where()`, `:has()`: невалидный селектор просто пропускается, остальные работают.

### Как это работает

Браузер при парсинге правила с `A, B, C { ... }` создаёт три отдельных матчера — то есть фактически три независимых правила с одинаковым блоком деклараций. Специфичность каждого матчера определяется индивидуально.

**Проблема невалидных селекторов:**

```css
/* CSS3 псевдокласс :focus-visible неизвестен старым браузерам */
/* Всё правило игнорируется в IE! */
:focus,
:focus-visible {
  outline: 2px solid blue;
}

/* Решение: разделить на два правила */
:focus { outline: 2px solid blue; }
:focus-visible { outline: 2px solid blue; }
```

**Forgiving list через `:is()`:**

```css
/* Невалидный :focus-visible просто игнорируется, :focus работает */
:is(:focus, :focus-visible) {
  outline: 2px solid blue;
}
```

### Практика и применение

- Сброс стилей по умолчанию для нескольких тегов: `h1, h2, h3, h4, h5, h6 { margin: 0 }`.
- Стилизация нескольких состояний: `input:focus, textarea:focus { border-color: blue }`.
- В дизайн-системах — объединение вариантов компонента.

### Важные нюансы и краеугольные камни

- Пустой селектор в списке (trailing comma `h1, {}`) делает правило невалидным в строгих парсерах.
- В современных браузерах Forgiving Selector List (через `:is()`) решает проблему совместимости.
- Специфичность каждого селектора в списке остаётся своей — `h1, .title { ... }` применяется с разной специфичностью к `<h1>` и `.title`.
- PostCSS-плагины могут автоматически разбивать перечисления для кроссбраузерности.

### Примеры

```css
/* Базовый сброс типографики */
h1, h2, h3, h4, h5, h6 {
  margin: 0;
  font-weight: 700;
  line-height: 1.2;
}

/* Состояния интерактивных элементов */
a:hover,
a:focus-visible,
button:hover,
button:focus-visible {
  outline: 2px solid var(--color-focus);
  outline-offset: 2px;
}

/* Forgiving selector list — безопасно для новых псевдоклассов */
:is(article, section, aside) > :is(h1, h2, h3) {
  color: var(--color-heading);
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему один невалидный селектор в списке ломает всё правило?** — CSS error recovery отбрасывает правило целиком при синтаксической ошибке.
- **Как решить проблему невалидных селекторов в списке?** — Либо разделить на отдельные правила, либо использовать `:is()` / `:where()`.
- **В чём разница между `A, B { }` и `A { } B { }` с одинаковыми стилями?** — Функционально одинаково, но перечисление менее многословно; при дублировании — два отдельных события каскада.

### Красные флаги (чего не говорить)

- «Перечисление повышает специфичность» — каждый селектор имеет свою независимую специфичность.
- «`:is(A, B)` это то же самое, что `A, B`» — `:is()` использует forgiving parsing, а обычный список — нет.

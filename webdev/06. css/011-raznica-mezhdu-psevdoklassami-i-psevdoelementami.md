# Q011. Разница между псевдоклассами и псевдоэлементами?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Псевдоклассы** (`:hover`, `:focus`) описывают **состояние или контекст** существующего элемента — они не создают новых узлов. **Псевдоэлементы** (`::before`, `::first-line`) создают **виртуальные части** элемента или вставляют новое содержимое, которого нет в HTML. Синтаксически отличаются: псевдоклассы — одно двоеточие `:`, псевдоэлементы — два `::`.

---

## Развёрнутый ответ

### Суть и определение

| Критерий | Псевдоклассы | Псевдоэлементы |
|----------|-------------|----------------|
| Синтаксис | `:name` | `::name` |
| Что делает | Выбирает элемент по состоянию/позиции | Создаёт виртуальный суб-элемент |
| DOM | Существующий элемент | Вне DOM, не доступен через JS |
| Примеры | `:hover`, `:focus`, `:nth-child()` | `::before`, `::after`, `::placeholder` |
| Специфичность | (0,1,0) | (0,0,1) |

**Псевдоклассы** — фильтр по динамическому состоянию (`:hover`, `:focus`, `:checked`, `:disabled`) или структурной позиции (`:first-child`, `:nth-of-type()`, `:not()`).

**Псевдоэлементы** — создают новую «точку стилизации», которой нет в разметке: `::before`/`::after` вставляют содержимое, `::first-line` стилизует только первую строку, `::selection` — выделение.

### Как это работает

Спецификация CSS определяет их принципиально по-разному:

- **Псевдокласс** расширяет условие матчинга элемента: «выбери `<a>`, но только когда он в состоянии hover».
- **Псевдоэлемент** создаёт абстракцию над частью элемента: «создай виртуальный первый дочерний элемент».

Два двоеточия `::` были введены в CSS3 именно для визуального разграничения (CSS2 использовал `:before`, `:after` и `:first-line` с одним двоеточием — эти старые формы браузеры всё ещё поддерживают).

На один элемент можно навесить несколько псевдоклассов: `.btn:hover:focus-visible`. Псевдоэлементов `::before` и `::after` — по одному на элемент.

### Практика и применение

- Псевдокласс `:focus-visible` — показывать outline только при клавиатурной навигации.
- Псевдокласс `:has()` — стилизовать родителя по состоянию потомка (`.form:has(:invalid)`).
- Псевдоэлемент `::before` — декоративные элементы (кавычки, иконки, оверлеи).
- Псевдоэлемент `::placeholder` — кастомные стили плейсхолдеров форм.
- Псевдоэлемент `::marker` — кастомные маркеры `<li>` без `list-style`.

### Важные нюансы и краеугольные камни

- Специфичность: псевдоклассы дают (0,1,0) как обычный класс; псевдоэлементы — (0,0,1) как тег.
- Псевдоклассы-функции (`:nth-child()`, `:is()`, `:not()`) принимают аргументы; их специфичность зависит от аргументов.
- Нельзя вложить псевдоэлемент в псевдоэлемент: `::before::after` — невалидно.
- `::before` нельзя использовать на replaced элементах (`<img>`, `<input>`), но `:hover` — можно.

### Примеры

```css
/* Псевдоклассы — состояние */
.btn:hover {
  background: darkblue;
}

.input:focus-visible {
  outline: 2px solid var(--color-focus);
}

.list li:last-child {
  border-bottom: none;
}

/* Псевдоэлементы — виртуальные части */
.quote::before {
  content: '«';
  color: var(--color-primary);
}

.quote::after {
  content: '»';
  color: var(--color-primary);
}

/* Комбинация: псевдокласс + псевдоэлемент */
.link:hover::after {
  content: ' →';
}
```

## Сравнение

| Критерий | Псевдокласс | Псевдоэлемент |
|----------|------------|---------------|
| Назначение | Состояние/позиция элемента | Виртуальная часть элемента |
| Синтаксис | `:` | `::` |
| Специфичность | (0,1,0) | (0,0,1) |
| Доступен в DOM | Да | Нет |
| Применим к replaced elements | Да | Нет (`::before`/`::after`) |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему `::` вместо `:`?** — CSS3 разграничил псевдоклассы (`:`) и псевдоэлементы (`::`); `:before` работает как устаревший синтаксис.
- **Сколько `::before` может быть на элементе?** — Ровно один; два `::before` на одном элементе невозможно.
- **Можно ли комбинировать псевдокласс и псевдоэлемент?** — Да: `.link:hover::after { content: '→' }` — работает.
- **Какой псевдоэлемент использовать для стилизации scrollbar?** — `::-webkit-scrollbar` (нестандартный); стандарт — `scrollbar-color` и `scrollbar-width`.

### Красные флаги (чего не говорить)

- «`:before` — это псевдокласс» — это псевдоэлемент с устаревшим синтаксисом.
- «Псевдоэлементы существуют в DOM» — нет, они не доступны через `querySelector`.
- «Псевдоклассы создают новые элементы» — они только описывают состояние существующих.

# Q001. Что такое CSS? И для чего он используется?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**CSS (Cascading Style Sheets)** — язык описания внешнего вида HTML-документа: цвета, шрифты, отступы, позиционирование, анимации. CSS отделяет представление от структуры (HTML) и поведения (JS), следуя принципу разделения ответственностей. Без CSS браузер отображает «голый» HTML с дефолтными стилями, что непригодно для реальных интерфейсов.

## Развёрнутый ответ

### Суть и определение

CSS — декларативный язык стилей, стандартизированный W3C. Версия CSS3 — фактически набор независимых модулей (Selectors, Flexbox, Grid, Animations и др.), каждый из которых развивается самостоятельно. CSS работает поверх DOM: браузер строит CSSOM (CSS Object Model) и затем объединяет его с DOM в Render Tree.

### Как это работает

1. Браузер загружает HTML и начинает строить DOM.
2. При встрече `<link rel="stylesheet">` или `<style>` загружает и парсит CSS → строит CSSOM.
3. DOM + CSSOM → **Render Tree** (только видимые узлы с применёнными стилями).
4. **Layout (Reflow)** — вычисление размеров и позиций.
5. **Paint** — отрисовка пикселей.
6. **Composite** — склейка слоёв (GPU).

Каскад определяет, какое правило «победит», если несколько правил претендуют на одно свойство элемента.

### Практика и применение

- **Визуальный дизайн** — типографика, цветовая схема, отступы, иконки.
- **Адаптивность** — `@media`, `@container`, `clamp()` для разных экранов.
- **Анимации и переходы** — `transition`, `@keyframes` без JS.
- **Темизация** — CSS Custom Properties (`--color-primary`) позволяют менять тему через один DOM-атрибут.
- **Производительность** — правильный CSS уменьшает количество JS-манипуляций DOM.

Без CSS каждый элемент потребовал бы inline-атрибутов, что привело бы к дублированию и катастрофической поддерживаемости.

### Важные нюансы и краеугольные камни

- CSS — **render-blocking ресурс**: браузер приостанавливает рендеринг до полной загрузки таблиц стилей, поэтому критический CSS выносят инлайн.
- CSSOM строится **после** полной загрузки файла — один большой CSS-файл блокирует дольше нескольких маленьких.
- CSS не имеет области видимости по умолчанию — стили «утекают» глобально (проблема решается BEM, CSS Modules, Shadow DOM).
- Специфичность, порядок источников и `!important` — три кита каскада, которые легко путают.

### Примеры

```css
/* Базовое правило: селектор + декларации */
.card {
  display: flex;
  flex-direction: column;
  gap: 16px;
  padding: 24px;
  border-radius: 8px;
  background-color: var(--color-surface);
  box-shadow: 0 2px 8px rgb(0 0 0 / 0.1);
}

/* Адаптивность через Custom Properties + clamp */
:root {
  --font-size-body: clamp(14px, 1.5vw, 18px);
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему CSS — render-blocking?** — CSSOM нужен до построения Render Tree; без него браузер не знает, какие узлы видимы.
- **Как работает каскад?** — Порядок: origin + importance → specificity → order of appearance.
- **Что такое CSSOM и как он связан с DOM?** — Два отдельных дерева, объединяются в Render Tree.
- **Чем CSS-in-JS отличается от обычного CSS?** — Генерация классов в runtime vs статический файл; компромисс DX vs производительность.
- **Как избежать глобального загрязнения стилей?** — BEM, CSS Modules, Scoped CSS, Shadow DOM.

### Красные флаги (чего не говорить)

- «CSS — это просто раскраска» — CSS управляет layout, анимациями, доступностью (focus-visible), производительностью рендеринга.
- «`!important` решает все проблемы специфичности» — создаёт неуправляемый каскад, правильное решение — пересмотреть архитектуру селекторов.

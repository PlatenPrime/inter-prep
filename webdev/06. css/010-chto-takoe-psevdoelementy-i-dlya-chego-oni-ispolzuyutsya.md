# Q010. Что такое псевдоэлементы? И для чего они используются?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Псевдоэлементы** — CSS-конструкции с синтаксисом `::name`, позволяющие стилизовать или создавать части элемента, не существующие в HTML-разметке. Наиболее популярны `::before` и `::after` — они вставляют виртуальный дочерний элемент в начало или конец содержимого. Псевдоэлементы используются для декоративных эффектов, типографики, иконок и форм без лишних DOM-узлов.

---

## Развёрнутый ответ

### Суть и определение

**Полный список стандартных псевдоэлементов:**

| Псевдоэлемент | Назначение |
|---------------|-----------|
| `::before` | Вставить содержимое перед содержимым элемента |
| `::after` | Вставить содержимое после содержимого элемента |
| `::first-line` | Стилизовать первую строку текстового блока |
| `::first-letter` | Стилизовать первую букву (drop cap) |
| `::selection` | Стили для выделенного текста |
| `::placeholder` | Стили плейсхолдера `<input>` |
| `::marker` | Стили маркера списка (`<li>`) |
| `::backdrop` | Фон за `<dialog>` или fullscreen-элементом |
| `::spelling-error` | Подчёркивание орфографических ошибок |
| `::grammar-error` | Подчёркивание грамматических ошибок |
| `::file-selector-button` | Кнопка `<input type="file">` |
| `::cue` | Стили субтитров WebVTT |
| `::part()` | Части Shadow DOM, открытые через `part` |
| `::slotted()` | Элементы, попавшие в `<slot>` Shadow DOM |

### Как это работает

`::before` и `::after` генерируют **псевдоэлемент** — виртуальный inline-элемент (по умолчанию), который является дочерним относительно своего хоста. Для их отображения обязательно свойство `content` (может быть пустой строкой `""`).

Псевдоэлементы:
- Не существуют в DOM (нет в `querySelectorAll`, не доступны из JS).
- Подчиняются box-модели CSS: можно задать `display: block`, `width`, `height`, `position`, `z-index`.
- Наследуют стили родителя.

### Практика и применение

- **Декоративные элементы** — разделители, стрелки, уголки без лишних HTML-тегов.
- **Иконки** — вставка иконок через `content: url(...)` или `content: "\f000"` (иконочные шрифты).
- **Счётчики** — `counter(item)` в `content` для автонумерации.
- **Оверлеи** — `::before` с `position: absolute; inset: 0` поверх изображения без дополнительного div.
- **Clearfix** — `::after { content: ""; display: block; clear: both }` для флоатов (устарело с Flexbox/Grid).
- **Кастомный маркер** — `::marker { color: red; font-size: 1.2em }` для `<li>`.

### Важные нюансы и краеугольные камни

- `content` обязателен для `::before`/`::after`; без него псевдоэлемент не отображается.
- Псевдоэлементы не работают на replaced elements (`<img>`, `<input>`, `<br>`) и void-элементах — нет «содержимого» для вставки.
- `::before`/`::after` по умолчанию `display: inline` — для layout задайте `display: block` или `display: flex`.
- Двоеточие `::` — CSS3-синтаксис (отличает от псевдоклассов `:`) ; одиночное `:before` — устаревший CSS2-синтаксис, но всё ещё работает.
- `content` с текстом влияет на accessibility tree: скринридеры могут читать содержимое `content`.

### Примеры

```css
/* Оверлей поверх изображения */
.card__image {
  position: relative;
}
.card__image::before {
  content: '';
  position: absolute;
  inset: 0;
  background: linear-gradient(to bottom, transparent 50%, rgb(0 0 0 / 0.7));
}

/* Drop cap */
.article__body::first-letter {
  font-size: 3.5em;
  float: left;
  line-height: 0.8;
  margin-right: 0.1em;
}

/* Кастомный маркер списка */
ul.checklist li::marker {
  content: '✓ ';
  color: green;
  font-weight: bold;
}

/* Иконка-стрелка через ::after без HTML */
.link--external::after {
  content: ' ↗';
  font-size: 0.8em;
  vertical-align: super;
}

/* Выделение текста */
::selection {
  background-color: var(--color-primary);
  color: white;
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему псевдоэлементы не работают на `<input>`?** — `<input>` — replaced element, не имеет «внутреннего содержимого» для `::before`/`::after`; но `::placeholder` и `::file-selector-button` работают.
- **Видны ли псевдоэлементы скринридерам?** — Зависит от `content`: текст читается, `content: ""` — нет; следите за a11y.
- **Сколько псевдоэлементов можно на одном элементе?** — Максимум один `::before` и один `::after`, но `::first-line` и `::first-letter` можно комбинировать.
- **Чем `::before` отличается от `:before`?** — Синтаксис: `::` — CSS3 (псевдоэлементы), `:` — CSS2 (псевдоклассы). Функционально идентичны для `::before`/`::after`.

### Красные флаги (чего не говорить)

- «Псевдоэлементы — это псевдоклассы» — принципиально разные концепции: псевдоклассы описывают состояние, псевдоэлементы — части.
- «`::before` можно использовать на любом теге» — не работает на replaced/void элементах.
- «Псевдоэлементы есть в DOM» — их нет, JS не видит их через `querySelector`.

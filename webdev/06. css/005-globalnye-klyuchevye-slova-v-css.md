# Q005. Глобальные ключевые слова в CSS?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Глобальные ключевые слова** в CSS — специальные значения, применимые к **любому** CSS-свойству: `inherit`, `initial`, `unset`, `revert` и `revert-layer`. Они управляют тем, откуда берётся итоговое значение свойства — от родителя, из спецификации или из таблицы стилей браузера.

---

## Развёрнутый ответ

### Суть и определение

| Ключевое слово | Поведение |
|----------------|-----------|
| `inherit` | Берёт computed value от непосредственного родителя |
| `initial` | Сбрасывает к спецификационному дефолту (не к браузерному!) |
| `unset` | `inherit`, если свойство наследуемое; `initial` — если нет |
| `revert` | Откатывается к значению из таблицы стилей более низкого origin (обычно User-Agent) |
| `revert-layer` | Откатывается к значению из предыдущего `@layer` |

Все пять можно задать через `all: unset` / `all: revert` — это сбрасывает/откатывает **все** свойства разом.

### Как это работает

**`inherit`:** копирует computed value родителя вне зависимости от того, является ли свойство изначально наследуемым.

```css
/* SVG не наследует color, но принуждаем к этому */
svg { color: inherit; }
```

**`initial`:** возвращает к значению из спецификации CSS. Например, `display: initial` → `inline`, а не к значению браузерного UA stylesheet (`block` для `<div>`).

**`unset`:** «умный» сброс — для наследуемых ведёт себя как `inherit`, для ненаследуемых как `initial`. Используется с `all: unset` для полного обнуления компонента.

**`revert`:** в отличие от `initial`, возвращает к UA stylesheet. `display: revert` на `<div>` даст `block`, а `display: initial` — `inline`.

**`revert-layer`:** полезен при работе с `@layer` — откатывает к значению из предыдущего объявленного слоя.

### Практика и применение

- `all: unset` — обнуление стилей web-компонента или кнопки перед кастомизацией.
- `all: revert` — возврат к «нормальному» браузерному отображению в контексте тяжёлых CSS-сбросов (Reset CSS).
- `color: inherit` — наследование цвета для иконок, SVG, псевдоэлементов.
- `revert-layer` — управление перекрытием слоёв `@layer` без повышения специфичности.

### Важные нюансы и краеугольные камни

- `initial` ≠ «браузерный дефолт» — для `display` на `<div>` `initial` → `inline`, а не `block`.
- `revert` не доступен в старых браузерах (IE, старый Edge); `all: revert` особенно осторожно.
- `inherit` работает только на непосредственного родителя, а не на ближайшего предка с нужным значением.
- Применение `all: unset` сбрасывает в том числе `display`, `visibility`, `pointer-events` — нужно явно восстанавливать нужные значения.

### Примеры

```css
/* Полный сброс компонента */
.unstyled-button {
  all: unset;
  cursor: pointer; /* восстанавливаем нужное */
}

/* revert: возврат к UA-стилям после глобального сброса */
.prose h1 {
  all: revert; /* заголовок снова выглядит как браузерный H1 */
}

/* inherit для SVG-иконок */
.icon {
  width: 1em;
  height: 1em;
  fill: currentColor; /* currentColor берёт color от родителя */
}

/* revert-layer: перекрытие слоёв */
@layer base {
  .btn { background: grey; }
}
@layer theme {
  .btn { background: revert-layer; } /* откатится к grey из base */
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Чем `initial` отличается от `revert`?** — `initial` возвращает к CSS-спецификации, `revert` — к UA stylesheet (браузерным дефолтам).
- **Что делает `all: unset`?** — Сбрасывает все свойства: наследуемые к `inherit`, ненаследуемые к `initial`.
- **Когда использовать `revert-layer`?** — При архитектуре `@layer`, когда нужно «пропустить» текущий слой и взять значение из предыдущего.

### Красные флаги (чего не говорить)

- «`initial` — это браузерный дефолт» — это спецификационный дефолт, который может отличаться от UA stylesheet.
- «`inherit` копирует значение от любого предка с нужным свойством» — только от непосредственного родителя.

# Q026. Что такое плавающие элементы (floats)? Как они работают?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Float** — CSS-свойство (`float: left | right | none`), изначально созданное для обтекания изображений текстом. Float-элемент «всплывает» к одному из краёв контейнера, частично выпадая из нормального потока — блочные элементы его игнорируют, но inline-содержимое обтекает. В современной разработке Flexbox и Grid заменяют большинство layout-применений float.

---

## Развёрнутый ответ

### Суть и определение

Float-элемент:
- Смещается к `left` или `right` краю своего containing block.
- Выходит из нормального потока для **блочных** элементов (они ведут себя, как будто float не существует).
- Остаётся в потоке для **inline-содержимого** (текст и inline-элементы обтекают float).
- Генерирует BFC (block formatting context) неявно.

### Как это работает

```
┌─────────────────────────────┐
│ Float left:                 │
│ ┌───────┐  Текст обтекает   │
│ │ img   │  float-элемент    │
│ │ float │  по правой        │
│ └───────┘  стороне.         │
│ Блочный <p> начинается      │
│ под float (если clearfix).  │
└─────────────────────────────┘
```

**Проблема «схлопывания» родителя:**
Если родитель содержит только float-элементы, его высота = 0 (float выпали из потока).

**Решения (clearfix):**

1. `::after { content: ''; display: block; clear: both }` на родителе (классический clearfix).
2. `overflow: hidden` / `overflow: auto` на родителе (создаёт BFC).
3. `display: flow-root` на родителе (современный способ, явный BFC).

**`clear: left | right | both`** — заставляет элемент начинаться ниже всех предшествующих float.

### Практика и применение

- **Изображение с обтеканием текстом** — единственный современный use-case для float.
- **CSS Shapes** — `shape-outside` используется совместно с float для обтекания нестандартных форм (кругов, многоугольников).
- **Legacy layout** — float-сетки (Bootstrap 3 и старше) — устарели, заменены Flexbox/Grid.

### Важные нюансы и краеугольные камни

- Float автоматически делает элемент `display: block` (независимо от исходного display).
- Нельзя использовать float на `position: absolute`/`fixed` (игнорируется).
- `clear` применяется к следующему **блочному** элементу — на inline не работает.
- `float` создаёт неявный BFC — float-элемент не схлопывает margin с соседями и не «утекает» в scroll-контейнер.
- В Flexbox/Grid `float` игнорируется: flex/grid items не могут быть плавающими.
- `shape-outside` работает только с float-элементами.

### Примеры

```css
/* Обтекание изображения */
.article__image {
  float: left;
  margin: 0 16px 8px 0;
  shape-outside: circle(50%); /* текст обтекает круг */
  border-radius: 50%;
}

/* Clearfix (классический) */
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}

/* Современный clearfix */
.container {
  display: flow-root; /* BFC содержит floats */
}

/* clear: both на следующем элементе */
.section {
  clear: both; /* начинается ниже float */
}
```

```html
<div class="article">
  <img class="article__image" src="photo.jpg" alt="...">
  <p>Текст обтекает изображение слева...</p>
</div>
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему float-родитель схлопывается?** — Float выпадают из нормального потока; высота родителя вычисляется без учёта float-детей.
- **Три способа clearfix?** — `::after clear: both`; `overflow: hidden`; `display: flow-root`.
- **Что такое `shape-outside`?** — CSS-свойство, задающее форму обтекания текстом вокруг float-элемента (круг, полигон, изображение).
- **Работает ли float внутри Flexbox?** — Нет, flex-items не могут быть плавающими.

### Красные флаги (чего не говорить)

- «Float — основной инструмент layout» — устарело; Flexbox/Grid решают эти задачи лучше.
- «`clear: both` на самом float» — `clear` применяется к следующему блочному элементу, не к float.

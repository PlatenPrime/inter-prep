# Q014. Расскажите о псевдоклассе `:has()`?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**`:has()`** — «родительский» (relational) псевдокласс, который выбирает элемент, если внутри него есть потомок, совпадающий с аргументом. До его появления стилизация родителя на основе состояния потомка была невозможна без JavaScript. Поддерживается во всех современных браузерах с 2023 года.

---

## Развёрнутый ответ

### Суть и определение

```css
A:has(B) { ... }
/* выбирает A, если внутри A есть элемент, совпадающий с B */
```

`:has()` — это **forgiving selector list** (невалидные аргументы игнорируются), что делает его безопасным для прогрессивного улучшения.

Семантически близок к «существует ли совпадение среди потомков» — по сути реализует SQL-подобный `EXISTS` в CSS.

### Как это работает

Браузер для каждого элемента-кандидата (A) проверяет, существует ли хотя бы один потомок (B). Это требует обратного обхода DOM — именно поэтому реализация долго откладывалась (производительность). Современные браузеры оптимизировали матчинг через инкрементальный пересчёт стилей.

**Поддержка:**
- Chrome/Edge: 105+ (август 2022)
- Safari: 15.4+ (март 2022)
- Firefox: 121+ (декабрь 2023)

`:has()` является **not-forgiving только на верхнем уровне** — в самом `:has()` список аргументов forgiving.

### Практика и применение

**Layout-адаптация по наличию элемента:**
```css
/* Сетка с сайдбаром — если сайдбар есть */
.layout:has(.sidebar) {
  grid-template-columns: 1fr 300px;
}
```

**Стилизация родителя по состоянию потомка:**
```css
/* Подсветить .field если input внутри невалиден */
.field:has(input:invalid:not(:placeholder-shown)) {
  --field-border: var(--color-error);
}
```

**Навигация и активные состояния:**
```css
nav:has(> .active) { border-bottom: 2px solid blue; }
```

**Изображения:**
```css
figure:has(figcaption) img {
  border-radius: 0 0 0 0; /* без скруглений если есть подпись */
}
```

### Важные нюансы и краеугольные камни

- `:has()` нельзя вложить в `:has()` (спецификация запрещает из соображений производительности).
- `:has()` нельзя использовать внутри `:not()` в некоторых браузерах (исторически).
- Специфичность `:has()` определяется наиболее специфичным аргументом, как у `:is()`.
- При `:has(:hover)` перерасчёт стилей происходит при каждом движении мыши — следите за производительностью на сложных страницах.
- `:has()` не работает с `::before`/`::after` — псевдоэлементы не являются «настоящими» дочерними элементами для матчинга.

### Примеры

```css
/* 1. Родительский селектор */
.card:has(> img) {
  padding: 0; /* убрать padding если карточка начинается с изображения */
}

/* 2. Состояние формы */
form:has(:invalid):has(:not(:placeholder-shown)) .form__submit {
  opacity: 0.5;
  pointer-events: none;
}

/* 3. Адаптивный layout без медиазапроса */
.grid:has(> :nth-child(4)) {
  grid-template-columns: repeat(2, 1fr); /* 2 колонки при 4+ элементах */
}

/* 4. Тёмная тема через data-атрибут на body */
body:has([data-theme="dark"]) {
  background: #1a1a2e;
  color: #eee;
}

/* 5. Ссылка с открытым dropdown */
.nav-item:has(.dropdown[open]) > .nav-link {
  color: var(--color-primary);
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему `:has()` так долго не появлялся в браузерах?** — Проблема производительности: требует обратного обхода DOM для матчинга.
- **Можно ли вложить `:has()` в `:has()`?** — Нет, спецификация запрещает.
- **Какая специфичность у `:has(.foo #bar)`?** — Берётся наивысшая из аргументов: (1,1,0) — ID + класс.
- **Как `:has()` сравнивается с CSS Houdini для похожих задач?** — `:has()` декларативен и оптимизирован браузером; Houdini через JS API менее производительный.

### Красные флаги (чего не говорить)

- «`:has()` всё ещё не поддерживается» — поддержка 96%+ с декабря 2023.
- «`:has()` можно вложить» — это невалидно по спецификации.
- «`:has()` заменяет все JS-манипуляции классами» — подходит для CSS-реакций на DOM-состояние, не для бизнес-логики.

# Q018. Что такое схлопывание границ (margin collapsing)?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Схлопывание margin (margin collapsing)** — механизм CSS, при котором вертикальные `margin` двух соседних блочных элементов объединяются в один — равный большему из двух (или меньшему при отрицательных). Происходит в трёх ситуациях: между соседними блоками, между родителем и первым/последним потомком, и в случае пустого блока.

---

## Развёрнутый ответ

### Суть и определение

Схлопывание применяется **только к вертикальным** (block-axis) margin в **нормальном потоке документа**. Горизонтальные margin не схлопываются никогда.

**Три случая схлопывания:**

1. **Соседние элементы (siblings)**: `margin-bottom` первого + `margin-top` второго → max(a, b).

2. **Родитель и первый/последний потомок**: если между `margin-top` родителя и `margin-top` первого потомка нет border, padding, inline-content или BFC — они схлопываются с внешним окружением родителя.

3. **Пустой блок**: если блок не имеет border, padding, inline-content, height — его `margin-top` и `margin-bottom` схлопываются между собой.

**Правила вычисления:**
- Все положительные → берётся максимальный.
- Один отрицательный → max(положительные) + min(отрицательные).
- Все отрицательные → берётся минимальный (наибольший по модулю).

### Как это работает

```
h1 { margin-bottom: 20px; }
p  { margin-top: 30px; }
/* Расстояние между h1 и p = 30px, а не 50px */
```

```
.parent { margin-top: 0; }
.child  { margin-top: 20px; }
/* Если нет BFC на .parent → margin-top потомка "утекает" наружу */
```

### Что предотвращает схлопывание

- `padding` или `border` на родителе (отделяет от потомка)
- `overflow: hidden / auto / scroll` (создаёт BFC)
- `display: flex` / `display: grid` (контейнер не схлопывает дочерние)
- `float` / `position: absolute` / `position: fixed`
- `display: flow-root` (явное создание BFC)

### Практика и применение

- Классический случай «прилипания» `margin-top` дочернего элемента к родителю — решается `overflow: hidden` или `padding-top: 1px` на родителе.
- В дизайн-системах используют `gap` вместо margin в Flexbox/Grid — схлопывание не применяется.
- Выравнивание заголовков с параграфами: `h2 { margin-bottom: 8px } p { margin-top: 16px }` — между ними 16px (не 24px).

### Важные нюансы и краеугольные камни

- В Flexbox и Grid контекстах схлопывание не происходит — это популярный вопрос.
- `display: flow-root` — чистый способ создать BFC без побочных эффектов `overflow: hidden`.
- Схлопывание не происходит для `position: absolute/fixed/sticky` и `float`-элементов.
- Схлопывание транзитивно: если три блока стоят рядом и все схлопываются, берётся максимум из всех трёх.
- Отрицательный + положительный margin: итог = 30 + (-20) = 10px (сумма, но отрицательный уменьшает).

### Примеры

```css
/* Схлопывание соседних элементов */
h2 { margin-bottom: 24px; }
p  { margin-top: 16px; }
/* Расстояние: 24px (max) */

/* Утечка margin через родителя */
.wrapper { /* без padding/border/overflow */ }
.content  { margin-top: 32px; }
/* 32px утекает из .wrapper во внешний поток */

/* Решение 1: overflow */
.wrapper { overflow: hidden; }

/* Решение 2: display: flow-root (чистый BFC) */
.wrapper { display: flow-root; }

/* Решение 3: padding-top */
.wrapper { padding-top: 1px; }

/* Решение 4: Flexbox (схлопывание не применяется) */
.wrapper { display: flex; flex-direction: column; }

/* Предотвращение через gap */
.stack {
  display: flex;
  flex-direction: column;
  gap: 24px; /* не схлопывается */
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Происходит ли схлопывание во Flexbox?** — Нет: flex-контейнер создаёт BFC для дочерних элементов.
- **Как исправить «утечку» margin через родителя?** — `overflow: hidden`, `display: flow-root`, `padding-top: 1px`, Flexbox/Grid.
- **Что такое BFC и как он связан со схлопыванием?** — Block Formatting Context: блоки внутри BFC не схлопывают margin с внешним контекстом.
- **Происходит ли схлопывание горизонтальных margin?** — Нет, только вертикальных в нормальном потоке.

### Красные флаги (чего не говорить)

- «Схлопывание происходит для всех margin» — только вертикальные, только в нормальном потоке.
- «Flexbox тоже схлопывает margin» — нет.
- «`margin: 0` на потомке решит утечку» — решает симптом; правильно — создать BFC на родителе.

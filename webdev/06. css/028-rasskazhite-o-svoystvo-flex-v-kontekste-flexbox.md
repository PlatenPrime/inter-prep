# Q028. Расскажите о свойстве `flex` в контексте Flexbox?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**`flex`** — шортхенд-свойство flex-item, объединяющее три компонента: `flex-grow` (насколько item может расти), `flex-shrink` (насколько сжиматься) и `flex-basis` (базовый размер по главной оси). Наиболее распространённое значение — `flex: 1`, которое означает `flex: 1 1 0` и позволяет items равномерно заполнить доступное пространство.

---

## Развёрнутый ответ

### Суть и определение

```css
flex: <flex-grow> <flex-shrink> <flex-basis>
/* или */
flex: <flex-grow>                 /* flex: 1 = flex: 1 1 0 */
flex: <flex-grow> <flex-basis>    /* flex: 1 200px = flex: 1 1 200px */
flex: auto                        /* = flex: 1 1 auto */
flex: none                        /* = flex: 0 0 auto */
flex: initial                     /* = flex: 0 1 auto (дефолт) */
```

**`flex-grow: <number>`** — безразмерный коэффициент. Если суммарный размер items < контейнера, свободное пространство распределяется пропорционально `flex-grow`.

```
Item A: flex-grow: 1  → получает 1/3 свободного пространства
Item B: flex-grow: 2  → получает 2/3 свободного пространства
```

**`flex-shrink: <number>`** — коэффициент сжатия. Если суммарный размер > контейнера, items сжимаются пропорционально `flex-shrink * flex-basis`.

**`flex-basis: <length> | auto | content`** — размер item до применения grow/shrink:
- `auto` — берёт `width`/`height` или размер по содержимому (intrinsic size).
- `0` — игнорирует собственный размер; всё пространство только через `flex-grow`.
- `<length>` — конкретный размер (px, %, em...).

### Как это работает

Алгоритм гибкого размера:

1. Вычисляется `flex-basis` каждого item (гипотетический main size).
2. Суммируется. Сравнивается с размером контейнера.
3. Если **положительное** свободное пространство → распределяется через `flex-grow`.
4. Если **отрицательное** (overflow) → items сжимаются через `flex-shrink * flex-basis`.
5. Применяются `min-width`/`max-width` как ограничители.

**Ключевая разница `flex-basis: 0` vs `auto`:**

```css
/* flex: 1 1 0 — все items равного размера */
.item { flex: 1; }

/* flex: 1 1 auto — items растут от своего содержимого */
.item { flex: auto; }
```

При `flex-basis: 0` длинный текст в item не даёт ему «преимущества» — все items начинают с 0 и получают равные доли. При `auto` — item с большим содержимым начинает с большего размера.

### Практика и применение

- `flex: 1` — равномерное заполнение строки (табы, колонки дашборда).
- `flex: none` — item с фиксированным размером (сайдбар, аватар).
- `flex: 0 0 200px` — item строго 200px, не растёт и не сжимается.
- `flex: 1 0 auto` — item растёт, но не сжимается ниже своего содержимого.

### Важные нюансы и краеугольные камни

- **`flex: 1` ≠ `flex: 1 1 auto`**: это `flex: 1 1 0` — `flex-basis` равен 0, а не `auto`.
- `flex-shrink: 0` защищает item от сжатия (например, иконка фиксированного размера в навигации).
- `min-width: 0` на flex-item может потребоваться для текста с `overflow: hidden` — по умолчанию `min-width: auto` запрещает сжатие ниже `min-content`.
- `flex-basis` в % вычисляется от **main size** родительского flex-контейнера.
- При `flex-direction: column` `flex-basis` задаёт высоту, а не ширину.

### Примеры

```css
/* Равные колонки */
.cols {
  display: flex;
  gap: 16px;
}
.col { flex: 1; } /* flex: 1 1 0 */

/* Фиксированная боковая колонка + растягивающийся контент */
.layout {
  display: flex;
}
.sidebar { flex: 0 0 280px; } /* не растёт, не сжимается */
.main    { flex: 1; }          /* занимает всё остальное */

/* Защита от сжатия и переполнения текстом */
.tag {
  flex: none;
  white-space: nowrap;
}

/* min-width: 0 для truncated текста */
.card__title {
  flex: 1;
  min-width: 0; /* разрешает сжатие ниже min-content */
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему `flex: 1` не делает равные колонки при разном содержимом?** — `flex-basis: 0` делает — все стартуют с 0; при `auto` стартуют с содержимого.
- **Зачем `min-width: 0` на flex-item?** — По умолчанию `min-width: auto` = `min-content`; item не сжимается ниже содержимого. `min-width: 0` снимает ограничение для `text-overflow: ellipsis`.
- **Как `flex-shrink` вычисляется при переполнении?** — Пропорционально `flex-shrink * flex-basis`, а не только `flex-shrink`.
- **Что произойдёт, если все items имеют `flex-grow: 0`?** — Свободное пространство не распределяется; `justify-content` управляет его положением.

### Красные флаги (чего не говорить)

- «`flex: 1` — это `flex: 1 1 auto`» — нет, это `flex: 1 1 0`.
- «Overflow во flex всегда обрезается» — нет, по умолчанию flex-items могут выходить за контейнер при `flex-shrink: 0` или `min-width`.

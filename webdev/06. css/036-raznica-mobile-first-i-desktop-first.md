# Q036. Разница между mobile-first и desktop-first подходами?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Mobile-first** — начинаем с базовых стилей для мобильного, расширяем через `min-width` медиазапросы. **Desktop-first** — начинаем с широкого layout, переопределяем через `max-width` для мобильного. Mobile-first является современным стандартом: производительнее (меньше CSS на мобильных), соответствует принципу прогрессивного улучшения и удобнее для контентного мышления.

---

## Развёрнутый ответ

### Суть и определение

**Mobile-first:**
```css
/* Базовые стили — мобильный */
.nav { flex-direction: column; }

/* Расширяем для планшета */
@media (min-width: 768px) {
  .nav { flex-direction: row; }
}

/* Расширяем для десктопа */
@media (min-width: 1024px) {
  .nav { gap: 24px; }
}
```

**Desktop-first:**
```css
/* Базовые стили — десктоп */
.nav { flex-direction: row; gap: 24px; }

/* Переопределяем для планшета */
@media (max-width: 1023px) {
  .nav { gap: 16px; }
}

/* Переопределяем для мобильного */
@media (max-width: 767px) {
  .nav { flex-direction: column; }
}
```

### Как это работает

**Алгоритм CSS:**
- `min-width`: условие выполнено при ширине ≥ значения → стили «добавляются» с ростом экрана.
- `max-width`: условие выполнено при ширине ≤ значения → стили «убираются» с ростом экрана.

**Производительность:**
- Mobile-first: мобильные браузеры загружают базовый CSS без медиазапросов + опциональные `min-width` блоки. Если медиазапрос не совпадает — блок парсируется, но не применяется (всё равно загружается).
- Разница в производительности незначительна для CSS, но mobile-first + `media="print"` / lazy CSS существенно.

### Практика и применение

- **Mobile-first** — правильный выбор для новых проектов: большинство трафика мобильный, проще добавлять сложность, чем убирать.
- **Desktop-first** — legacy проекты, сложные дашборды/admin panels, где мобильная версия вторична.
- Bootstrap 5 — mobile-first: `.col-12 col-md-6 col-lg-4`.
- Tailwind CSS — mobile-first: `sm:flex md:grid`.

### Важные нюансы и краеугольные камни

- «Mobile-first» — это не только размеры, но и мышление: приоритизировать контент, убирать неважное для мобильных.
- Смешивать `min-width` и `max-width` в одном проекте — сложно поддерживать; выбрать один подход.
- CSS Media Queries Level 4 добавил **Range syntax**: `@media (width >= 768px)` — более читаемо.
- `prefers-color-scheme`, `prefers-reduced-motion` — тоже медиазапросы, не зависят от mobile/desktop.
- Container queries могут заменить часть breakpoint-логики для компонентов.

### Примеры

```css
/* Mobile-first с Range syntax (CSS4) */
.container {
  padding: 16px;
}

@media (width >= 768px) {
  .container {
    padding: 24px;
    max-width: 720px;
    margin-inline: auto;
  }
}

@media (width >= 1200px) {
  .container {
    max-width: 1140px;
  }
}

/* Типичная мобильная типография → десктопная */
h1 {
  font-size: clamp(1.5rem, 5vw, 3rem); /* плавное масштабирование без breakpoints */
}

/* Desktop-first: переопределяем для мобильного */
.sidebar {
  width: 280px;
  display: block;
}

@media (max-width: 767px) {
  .sidebar {
    width: 100%;
    display: none; /* скрыт на мобильном */
  }
  .sidebar.is-open {
    display: block;
  }
}
```

## Сравнение

| Критерий | Mobile-first | Desktop-first |
|----------|-------------|--------------|
| Базовые стили | Мобильные | Десктопные |
| Медиазапросы | `min-width` | `max-width` |
| Прогрессивное улучшение | Да | Нет |
| Подход | Добавляем сложность | Убираем сложность |
| Современный стандарт | Да | Legacy |

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему mobile-first лучше для производительности?** — Основные стили применяются без условий; сложные desktop-стили условны. Меньше CSS-кода на парсинг при первой загрузке.
- **Как `clamp()` уменьшает число breakpoints?** — `clamp(min, preferred, max)` плавно масштабирует значение между viewports без медиазапросов.
- **Что такое container queries в контексте адаптивности?** — Компонентный уровень адаптивности, не зависящий от viewport.

### Красные флаги (чего не говорить)

- «Desktop-first — плохо» — для ряда проектов (admin panel) это обоснованный выбор.
- «Mobile-first — только о размерах экрана» — это подход к дизайну: приоритизация контента.

# Q004. Как работают каскадность и наследование в CSS?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

**Каскад** — алгоритм, определяющий, какое объявление «победит», если несколько CSS-правил претендуют на одно свойство одного элемента. Приоритет определяется по цепочке: origin + важность → specificity → порядок в источнике. **Наследование** — механизм, при котором часть свойств (в основном типографические) автоматически передаётся от родителя к потомкам, что позволяет не дублировать стили.

---

## Развёрнутый ответ

### Суть и определение

**Каскад (Cascade):**

Браузер собирает все декларации, претендующие на свойство, и сортирует их по критериям:

1. **Origin + Importance** (от низшего к высшему):
   - User-Agent stylesheet (браузерные дефолты)
   - User stylesheet (пользовательские настройки)
   - Author stylesheet (CSS разработчика)
   - Author `!important`
   - User `!important`
   - User-Agent `!important`
   - CSS Transitions (перекрывают всё временно)
   - CSS Animations
2. **Cascade Layer** (слои `@layer` внутри одного origin): слои позже объявленные имеют приоритет; стили вне слоёв всегда побеждают слои.
3. **Specificity** (специфичность селектора)
4. **Order of appearance** (порядок в источнике — последнее выигрывает)

**Наследование (Inheritance):**

Свойства делятся на **наследуемые** (inherited) и **ненаследуемые**. Наследуемые: `color`, `font-*`, `line-height`, `visibility`, `cursor`, `direction`, `word-spacing` и др. Ненаследуемые: `margin`, `padding`, `border`, `background`, `display`, `width`, `height`.

### Как это работает

```
Computed value
    ↑
Inherited value (если свойство inherited и нет явного значения)
    ↑
Cascaded value (победитель каскада)
    ↑
Specified value (явно указанное или initial/inherit/unset)
    ↑
Initial value (дефолт из спецификации)
```

Ключевые этапы:
- **Cascaded value** — результат алгоритма каскада среди всех деклараций.
- **Specified value** — if нет cascaded, то `inherit` для наследуемых или `initial` для остальных.
- **Computed value** — абсолютное значение (например, `em` → `px`), используется в наследовании.
- **Used value** — окончательное значение после layout (например, `auto` → конкретное px).

### Практика и применение

- Задание `font-family` на `:root` или `body` наследуется всем документом — не нужно дублировать на каждом элементе.
- `color: inherit` явно включает наследование для ненаследуемого свойства.
- `all: unset` сбрасывает все свойства к унаследованным значениям — полезно для стилизации компонентов с нуля.
- `@layer` позволяет управлять приоритетом библиотечных стилей vs авторских без войны специфичностей.

### Важные нюансы и краеугольные камни

- `inherit` — явно наследовать от родителя; `initial` — сбросить к спецификационному дефолту; `unset` — `inherit` если наследуемое, иначе `initial`; `revert` — откатиться к user-agent стилям.
- `currentColor` — специальное значение, наследующее `color` родителя.
- Наследование работает по **computed value**, а не по указанному — `font-size: 1.5em` на родителе наследуется как абсолютное px-значение.
- `opacity: 0` наследуется; `visibility: hidden` — наследуется, но потомок может сделать `visibility: visible`.

### Примеры

```css
/* Наследование типографики через :root */
:root {
  font-family: 'Inter', system-ui, sans-serif;
  font-size: 16px;
  line-height: 1.5;
  color: #1a1a2e;
}

/* Все потомки наследуют font-family, color, line-height */

/* Явное управление наследованием */
.icon {
  color: inherit;        /* наследует color родителя (ненаследуемое по умолчанию для SVG) */
  border: initial;       /* сбрасывает border к none */
}

/* @layer: библиотека vs авторские стили */
@layer base, components, utilities;

@layer base {
  button { background: grey; }
}

/* Стили вне @layer перекрывают любой слой */
button { background: blue; }
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему `!important` плохо?** — Создаёт «войны important», сломанный каскад; правильное решение — рефакторинг специфичности.
- **Чем `inherit` отличается от `unset`?** — `inherit` всегда наследует; `unset` наследует для inherited-свойств и ставит `initial` для остальных.
- **Как `@layer` влияет на специфичность?** — Слои не меняют специфичность внутри себя, но слой с меньшим приоритетом всегда проигрывает более приоритетному слою.
- **Что такое computed value и почему оно важно для наследования?** — Наследуется уже вычисленное значение (px), а не relative (em), что предотвращает накопление.

### Красные флаги (чего не говорить)

- «Каскад = специфичность» — специфичность лишь третий шаг алгоритма, после origin и importance.
- «Все CSS-свойства наследуются» — большинство layout-свойств не наследуются.
- «`!important` всегда побеждает» — user/UA `!important` может победить author `!important`.

# Q043. Способы задания цвета в CSS?

> **Источник:** [06. css.md](../06.%20css.md) · **Тема:** CSS

---

## Короткий ответ

CSS поддерживает несколько синтаксисов: **именованные цвета** (`red`, `transparent`), **HEX** (`#ff0000`, `#f00`), **RGB/RGBA** (`rgb(255 0 0)`, `rgba(255 0 0 / 0.5)`), **HSL/HSLA** (`hsl(0 100% 50%)`), **HWB**, **LAB/LCH**, **OKLCH/OKLAB** — перцептивно однородные пространства, **`color()`** для широких gamut. Современный стандарт — OKLCH для дизайн-систем.

---

## Развёрнутый ответ

### Суть и определение

**Синтаксисы и форматы:**

| Формат | Пример | Примечание |
|--------|--------|-----------|
| Именованный | `red`, `transparent`, `currentColor` | 148 именованных цветов |
| HEX | `#ff0000`, `#f00`, `#ff0000ff` (с alpha) | Компактно |
| RGB (legacy) | `rgb(255, 0, 0)`, `rgba(255, 0, 0, 0.5)` | CSS2 синтаксис |
| RGB (modern) | `rgb(255 0 0)`, `rgb(255 0 0 / 50%)` | CSS4, без запятых |
| HSL | `hsl(0 100% 50% / 0.5)` | Интуитивно для дизайна |
| HWB | `hwb(0 0% 0%)` | Hue + White + Black |
| Lab | `lab(50% 50 0)` | CIE L*a*b*, широкий gamut |
| LCH | `lch(50% 75 0)` | Lab в полярных координатах |
| Oklab | `oklab(0.5 0.1 0)` | Перцептивно однородный |
| Oklch | `oklch(0.5 0.2 0deg)` | Лучший для палитр |
| `color()` | `color(display-p3 1 0 0)` | Произвольное цветовое пространство |
| `color-mix()` | `color-mix(in oklch, blue, red)` | Смешивание цветов |

### Как это работает

**sRGB (традиционное):** HEX, RGB, HSL, HWB — все работают в sRGB цветовом пространстве. sRGB охватывает ~35% видимых цветов.

**Wide gamut (широкий охват):** Display-P3, Rec. 2020 — охватывают больший диапазон цветов. Доступны через `color(display-p3 r g b)`. Поддерживаются на современных мониторах (OLED, Pro Display).

**Перцептивные пространства (OKLCH/OKLAB):**
- Человеческое восприятие нелинейно — одинаковое изменение L в Lab ≠ одинаково воспринимаемое изменение яркости в sRGB.
- OKLCH решает это: `L` (lightness 0-1), `C` (chroma), `H` (hue 0-360).
- Идеален для генерации цветовых палитр с равномерным контрастом.

### Практика и применение

- **HEX** — для статичных цветов из дизайн-макета.
- **HSL** — для программного управления цветом (светлее/темнее: изменить L).
- **OKLCH** — для дизайн-токенов, палитр с предсказуемым контрастом.
- **`color-mix()`** — создание промежуточных цветов: `color-mix(in oklch, var(--color-primary) 20%, white)`.
- **`currentColor`** — наследовать текущий `color` (для SVG, border, outline, text-decoration).

```css
/* Цветовые токены на OKLCH */
:root {
  --color-primary: oklch(0.55 0.2 260);  /* синий */
  --color-primary-light: oklch(0.75 0.15 260);
  --color-primary-dark:  oklch(0.35 0.25 260);
}
```

### Важные нюансы и краеугольные камни

- `rgba(255, 0, 0, 0.5)` = `rgb(255 0 0 / 0.5)` = `#ff000080` — все эквивалентны.
- `transparent` = `rgb(0 0 0 / 0)` (прозрачный чёрный) — важно при `transition: background`, может привести к «серому» артефакту.
- `color-mix()` — Chrome 111+, Firefox 113+, Safari 16.2+.
- Wide-gamut цвета отображаются как sRGB на мониторах без поддержки P3.
- OKLCH `C` (chroma) для одного и того же hue может быть разной максимальной — разные цвета имеют разный maximum chroma.

### Примеры

```css
/* Разные синтаксисы одного цвета */
.element {
  color: #ff0000;
  color: rgb(255 0 0);
  color: hsl(0 100% 50%);
  color: oklch(0.63 0.26 29);
}

/* color-mix — создание оттенков */
:root {
  --primary: oklch(0.55 0.2 260);
  --primary-tint: color-mix(in oklch, var(--primary) 20%, white);
  --primary-shade: color-mix(in oklch, var(--primary) 80%, black);
}

/* Transparent → цвет без артефактов */
.btn {
  background: rgb(0 112 243 / 0); /* прозрачный нужного цвета */
  transition: background 0.2s;
}
.btn:hover {
  background: rgb(0 112 243 / 1);
}

/* Wide gamut */
@supports (color: color(display-p3 0 0 0)) {
  .vivid-red {
    color: color(display-p3 1 0 0); /* насыщеннее sRGB red */
  }
}
```

## На что обратить внимание на собеседовании

### Каверзные follow-up (что спросят дальше)

- **Почему OKLCH лучше HSL для дизайн-систем?** — Перцептивно однородный: одинаковое изменение L даёт одинаково воспринимаемое изменение яркости; HSL этого не гарантирует.
- **Что такое wide gamut и когда использовать?** — Цветовые пространства с охватом шире sRGB (P3, Rec.2020); использовать на поддерживающих мониторах с `@supports`.
- **Почему `transparent` может вызвать артефакт при transition?** — `transparent` = прозрачный чёрный; при переходе от `rgba(0,0,255,1)` к `transparent` промежуточный цвет становится серым.

### Красные флаги (чего не говорить)

- «HEX — единственный правильный способ» — разные форматы для разных задач; OKLCH для систем, HEX для статичного.
- «RGB и HEX — разные цветовые пространства» — оба sRGB, только разный синтаксис.
